Fuente canónica de curl de Firecrawl para agentes. Alineada con `api-reference/v2-openapi.json` (URL base del servidor `https://api.firecrawl.dev/v2`).

<div id="authenticate">
  ## Autenticación
</div>

```bash
export FIRECRAWL_API_KEY="fc-your-api-key"
```

<div id="when-to-use-what">
  ## Cuándo usar cada uno
</div>

* `search`: úsalo cuando empieces con una consulta y necesites descubrir resultados.
* `scrape`: úsalo cuando ya tengas una URL y quieras el contenido de la página.
* `interact`: úsalo cuando la página requiera clics, formularios o acciones del navegador posteriores al scraping.
* `support/ask`: úsalo cuando una llamada a la API de Firecrawl falle o devuelva resultados inesperados y necesites un diagnóstico.
* `support/docs-search`: úsalo cuando necesites consultar la documentación de Firecrawl.

<div id="search">
  ## Search
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa search para descubrir páginas relevantes a partir de una consulta y luego elegir URLs para hacer scraping o interactuar con ellas. Puedes restringir los resultados a un sitio con `site:`, por ejemplo `site:docs.firecrawl.dev crawl webhooks`.

<div id="endpoint">
  ### Endpoint
</div>

`POST /search`

<div id="simple-example">
  ### Ejemplo simple
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/search" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "site:docs.firecrawl.dev webhook retries"
  }'
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/search" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "site:docs.firecrawl.dev crawl webhooks",
    "sources": [{"type": "web"}, {"type": "news"}],
    "categories": [{"type": "research"}],
    "limit": 10,
    "tbs": "qdr:m",
    "location": "San Francisco,California,United States",
    "country": "US",
    "ignoreInvalidURLs": true,
    "timeout": 60000,
    "enterprise": ["anon"],
    "scrapeOptions": {
      "formats": [
        "markdown",
        "links",
        {"type": "json", "prompt": "Extract title and key endpoints."}
      ],
      "onlyMainContent": true,
      "includeTags": ["main", "article"],
      "excludeTags": ["nav", "footer"],
      "waitFor": 1000
    }
  }'
```

<div id="response">
  ### Respuesta
</div>

Las respuestas exitosas incluyen `success`, `data`, `warning` opcional, `id` y `creditsUsed`.

* `data.web`, `data.images`, `data.news`: arrays de resultados; las claves que aparecen dependen de `sources` (de manera predeterminada, solo `data.web` contiene datos).
* Los resultados web y de noticias incluyen campos como `title`, `url` y, cuando `scrapeOptions` / los formatos lo solicitan, `markdown`, `html`, `rawHtml`, `links`, `screenshot`, `audio`, `video` y `metadata`.
* Los resultados de imagen incluyen campos como `imageUrl`, `url` y las dimensiones, cuando están disponibles.
* `warning`: aviso opcional legible por humanos.
* `id`: cadena con el ID de trabajo del trabajo de búsqueda.
* `creditsUsed`: número entero de créditos cobrados por la llamada.

<div id="parameters">
  ### Parámetros
</div>

* `query`
  * Tipo: string (obligatorio, longitud máxima 500)
  * Úsalo cuando: necesites una consulta de búsqueda.
  * Notas: usa `site:example.com` para limitar los resultados a un dominio.

* `sources`
  * Tipo: array de objetos de fuente con tipo
  * Úsalo cuando: quieras controlar en qué fuentes se busca.
  * Estructuras de objeto confirmadas:
    * `{ "type": "web" }` con `tbs` y `location` opcionales por fuente
    * `{ "type": "news" }`
    * `{ "type": "images" }`

* `categories`
  * Tipo: array de objetos de categoría con tipo
  * Úsalo cuando: quieras filtrar los resultados por categoría.
  * Estructuras de objeto confirmadas:
    * `{ "type": "github" }`
    * `{ "type": "research" }`
    * `{ "type": "pdf" }`

* `limit`
  * Tipo: integer (mínimo 1, máximo 100, predeterminado 5)
  * Úsalo cuando: quieras limitar la cantidad de resultados.

* `tbs`
  * Tipo: string
  * Úsalo cuando: necesites un filtro basado en el tiempo (por ejemplo, `qdr:d`, `qdr:w`, `sbd:1,qdr:m`).

* `location`
  * Tipo: string
  * Úsalo cuando: quieras resultados localizados.

* `country`
  * Tipo: string (predeterminado `"US"`)
  * Úsalo cuando: quieras segmentar por ISO 3166-1 alpha-2 (por ejemplo, `"US"`).

* `ignoreInvalidURLs`
  * Tipo: boolean (predeterminado false)
  * Úsalo cuando: quieras descartar URLs que no puedan procesarse mediante scraping con otros endpoints.

* `timeout`
  * Tipo: integer (milisegundos, predeterminado 60000)
  * Úsalo cuando: necesites un tiempo de espera de la solicitud en milisegundos.

* `enterprise`
  * Tipo: array de strings (`"anon"` o `"zdr"` por elemento)
  * Úsalo cuando: necesites controles de búsqueda empresariales.
  * Valores:
    * `"zdr"`: retención de datos cero de extremo a extremo
    * `"anon"`: retención de datos cero anonimizada

* `scrapeOptions`
  * Tipo: object
  * Úsalo cuando: quieras extraer cada resultado de búsqueda (consulta los parámetros de scraping para ver los campos).

<div id="scrape">
  ## Scraping
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa scrape cuando ya tengas una URL y quieras contenido estructurado en uno o varios formatos.

<div id="endpoint">
  ### Endpoint
</div>

`POST /scrape`

<div id="simple-example">
  ### Ejemplo simple
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://docs.firecrawl.dev",
    "formats": ["markdown"]
  }'
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/pricing",
    "formats": [
      "markdown",
      "links",
      {"type": "json", "prompt": "Extract plan names and prices."},
      {"type": "screenshot", "fullPage": true, "quality": 80, "viewport": {"width": 1280, "height": 720}},
      {"type": "changeTracking", "modes": ["git-diff"], "tag": "pricing"}
    ],
    "headers": {"User-Agent": "FirecrawlDocsBot/1.0"},
    "onlyMainContent": true,
    "waitFor": 1000,
    "parsers": [{"type": "pdf", "mode": "auto", "maxPages": 5}],
    "actions": [
      {"type": "click", "selector": "#accept"},
      {"type": "wait", "milliseconds": 750},
      {"type": "scrape"}
    ],
    "location": {"country": "US", "languages": ["en-US"]},
    "removeBase64Images": true,
    "blockAds": true,
    "proxy": "auto",
    "maxAge": 86400000,
    "minAge": 1,
    "storeInCache": true,
    "profile": {"name": "docs-session", "saveChanges": true},
    "zeroDataRetention": false
  }'
```

<div id="response">
  ### Respuesta
</div>

Las respuestas exitosas incluyen `success` y `data`. Campos comunes de `data` (según `formats` y las opciones):

* `markdown`, `summary`, `html`, `rawHtml`, `screenshot`, `audio`, `video`, `links`
* `actions`: cuando la solicitud incluyó `actions` durante el scraping, contiene resultados ordenados como `screenshots`, `scrapes`, `javascriptReturns` y `pdfs`
* `metadata`: metadatos de la página (`title`, `sourceURL`, `url`, `statusCode`, `error` y otros campos extraídos)
* `warning`: aviso opcional sobre la extracción o el formato
* `changeTracking`: presente cuando se solicita el formato `changeTracking`

<div id="parameters">
  ### Parámetros
</div>

* `url`
  * Tipo: string
  * Usar cuando: quieres hacer scraping de una página específica.

* `formats`
  * Tipo: array de cadenas de formato u objetos de formato
  * Usar cuando: quieres varios formatos de salida.
  * Cadenas de formato confirmadas:
    * `"markdown"`: contenido en markdown
    * `"html"`: HTML limpio
    * `"rawHtml"`: HTML sin procesar
    * `"links"`: enlaces de la página
    * `"images"`: URL de imágenes
    * `"screenshot"`: salida de captura de pantalla
    * `"summary"`: salida de resumen
    * `"changeTracking"`: salida de seguimiento de cambios
    * `"json"`: extracción JSON
    * `"branding"`: salida del perfil de marca
    * `"audio"`: extracción de audio
    * `"video"`: extracción de video
  * Campos del objeto de formato:
    * `type`: una de las cadenas de formato anteriores
    * `prompt`, `schema`: opciones de extracción JSON para `type: "json"`
    * `modes`, `schema`, `prompt`, `tag`: opciones de seguimiento de cambios para `type: "changeTracking"`
    * `fullPage`, `quality`, `viewport`: opciones de captura de pantalla para `type: "screenshot"`

* `headers`
  * Tipo: object
  * Usar cuando: necesitas encabezados de solicitud personalizados.

* `includeTags`
  * Tipo: array de strings
  * Usar cuando: quieres incluir solo etiquetas HTML específicas.

* `excludeTags`
  * Tipo: array de strings
  * Usar cuando: quieres excluir etiquetas HTML específicas.

* `onlyMainContent`
  * Tipo: boolean
  * Usar cuando: quieres eliminar la navegación, el pie de página y otros elementos repetitivos.

* `timeout`
  * Tipo: number
  * Usar cuando: necesitas un tiempo de espera en milisegundos.

* `waitFor`
  * Tipo: number
  * Usar cuando: necesitas esperar a que la página se renderice (milisegundos).

* `mobile`
  * Tipo: boolean
  * Usar cuando: quieres una viewport móvil.

* `parsers`
  * Tipo: array de objetos
  * Usar cuando: necesitas controles de análisis de archivos.
  * Forma confirmada: `{ "type": "pdf", "mode": "fast" | "auto" | "ocr", "maxPages": number }` (`type` obligatorio; los demás campos son opcionales con valores predeterminados según la especificación)

* `actions`
  * Tipo: array de objetos de acción
  * Usar cuando: necesitas acciones ligeras previas al scraping.
  * Tipos de acción confirmados:
    * `wait`: `milliseconds` o `selector` obligatorios
    * `screenshot`: `fullPage`, `quality`, `viewport` opcionales
    * `click`: `selector` obligatorio, `all` opcional
    * `write`: `text` obligatorio (haz clic primero para enfocar el campo de entrada)
    * `press`: `key` obligatoria
    * `scroll`: `type` obligatoria; `direction` (`up` o `down`, predeterminado `down`); `selector` opcional
    * `scrape`: sin campos adicionales
    * `executeJavascript`: `script` obligatorio
    * `pdf`: `format` (A0, A1, A2, A3, A4, A5, A6, Letter, Legal, Tabloid, Ledger), `landscape`, `scale` opcionales

* `location`
  * Tipo: objeto con `country` y `languages`
  * Usar cuando: necesitas scraping según la ubicación o el idioma.

* `skipTlsVerification`
  * Tipo: boolean
  * Usar cuando: necesitas omitir la verificación de TLS.

* `removeBase64Images`
  * Tipo: boolean
  * Usar cuando: quieres eliminar imágenes en base64 de la salida markdown.

* `blockAds`
  * Tipo: boolean
  * Usar cuando: quieres bloquear anuncios y ventanas emergentes de cookies.

* `proxy`
  * Tipo: string
  * Usar cuando: necesitas controlar el proxy.
  * Valores confirmados: `"basic"`, `"enhanced"`, `"auto"`

* `maxAge`
  * Tipo: number
  * Usar cuando: quieres datos en caché con una antigüedad máxima determinada (milisegundos).

* `minAge`
  * Tipo: number
  * Usar cuando: quieres datos en caché solo si tienen al menos esta antigüedad (milisegundos).

* `storeInCache`
  * Tipo: boolean
  * Usar cuando: quieres que Firecrawl almacene el resultado en caché.

* `profile`
  * Tipo: objeto con `name` y `saveChanges` opcional
  * Usar cuando: quieres un perfil de navegador persistente compartido entre scrapes e interacciones.

* `zeroDataRetention`
  * Tipo: boolean
  * Usar cuando: quieres retención de datos cero para este scraping.

<div id="interact">
  ## Interact
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa Interact cuando una página requiera acciones del navegador o la ejecución de código una vez iniciado el scraping.

<div id="endpoint">
  ### Endpoint
</div>

`POST /scrape/{jobId}/interact`

<div id="simple-example">
  ### Ejemplo simple
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape/<jobId>/interact" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "code": "console.log(await page.title());"
  }'
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape/<jobId>/interact" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "code": "const title = await page.title();\nconst h1 = await page.locator(\"h1\").first().textContent().catch(() => null);\nconsole.log(JSON.stringify({ title, h1 }));",
    "language": "node",
    "timeout": 120
  }'
```

<div id="parameters">
  ### Parámetros
</div>

* `jobId` (path)
  * Tipo: string (UUID)
  * Usar cuando: tienes el ID de trabajo de scraping para la sesión activa del navegador.

* `code` (JSON body)
  * Tipo: string (obligatorio en OpenAPI; longitud mín. 1, longitud máx. 100000)
  * Usar cuando: quieres ejecutar código en el sandbox del navegador asociado al scraping.

* `language` (JSON body)
  * Tipo: string
  * Usar cuando: necesitas un entorno de ejecución específico.
  * Valores admitidos: `"python"`, `"node"`, `"bash"` (predeterminado: `"node"`)

* `timeout` (JSON body)
  * Tipo: integer (segundos; mínimo 1, máximo 300, predeterminado 30)
  * Usar cuando: necesitas un timeout de ejecución.

<div id="response">
  ### Respuesta
</div>

Las respuestas exitosas incluyen `success`, además de campos de ejecución como `stdout`, `result` (alias de `stdout`), `stderr`, `exitCode`, `killed` y `error` (admite valores nulos).

### DELETE /scrape/{jobId}/interact

<div id="example">
  ### Ejemplo
</div>

```bash
curl -X DELETE "https://api.firecrawl.dev/v2/scrape/<jobId>/interact" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY"
```

Sin cuerpo JSON. Una respuesta exitosa incluye `success`.

<div id="ask-agentic-debugging">
  ## Ask (Depuración con agentes)
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa ask cuando una llamada a la API de Firecrawl falla o devuelve resultados inesperados. El agente de soporte con IA diagnostica el problema, propone parámetros para corregirlo y, opcionalmente, valida la solución contra la API real. Latencia típica: 15-30 segundos.

<div id="endpoint">
  ### Endpoint
</div>

`POST /support/ask`

### Ejemplo básico

```bash
curl -X POST "https://api.firecrawl.dev/v2/support/ask" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "my scrape returned empty markdown for https://example.com"
  }'
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/support/ask" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "crawl returned 3 pages but I expected 50",
    "rationale": "user is on their third failed crawl attempt today",
    "context": {"userPlan": "standard", "retryCount": 3}
  }'
```

<div id="response">
  ### Respuesta
</div>

Las respuestas exitosas incluyen `requestId`, `answer`, `confidence`, `fixParameters`, `validation`, `usage` y `durationMs`.

* `answer`: texto de 2 a 4 frases que describe el diagnóstico y la corrección.
* `confidence`: `high`, `medium` o `low`.
* `fixParameters`: parámetros de API accionables por máquina para aplicar la corrección (`null` si no aplica ninguna corrección).
* `validation.tested`: si el agente probó la corrección con la API real.
* `validation.result`: `success`, `failure` o `skipped`.
* `feedback`: presente cuando el agente se atasca; `null` en caso de éxito.

<div id="parameters">
  ### Parámetros
</div>

* `question`
  * Tipo: string (obligatorio, 1-8000 caracteres)
  * Úsalo cuando: necesites describir el problema.

* `rationale`
  * Tipo: string (1-2000 caracteres)
  * Úsalo cuando: seas un agente de IA que llama en nombre de un usuario. Describe lo que el usuario intenta lograr.

* `context`
  * Tipo: objeto (formato libre)
  * Úsalo cuando: quieras pasar metadatos de tu agente al prompt de depuración.

<div id="docs-search">
  ## Búsqueda en la documentación
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa docs-search para buscar en la documentación de Firecrawl.

<div id="endpoint">
  ### Endpoint
</div>

`POST /support/docs-search`

<div id="example">
  ### Ejemplo
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/support/docs-search" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "how do I verify webhook signatures?"
  }'
```

<div id="parameters">
  ### Parámetros
</div>

* `question`
  * Tipo: string (obligatorio, 1-8000 caracteres)
  * Úsalo cuando: necesites una respuesta basada en la documentación.

<div id="notes">
  ## Notas
</div>

* Las filas de los resultados de búsqueda están anidadas en `data.web`, `data.images` o `data.news`, no en un array `data` plano.
* Usa `POST /scrape/{jobId}/interact` para flujos de trabajo de navegador de varios pasos; las `actions` de scraping son más adecuadas para pasos previos al scraping pequeños.
* El esquema de OpenAPI publicado requiere `code` para interact. Algunos SDKs y clientes también aceptan un campo `prompt` para instrucciones en lenguaje natural; ese alias no está en `v2-openapi.json`.
