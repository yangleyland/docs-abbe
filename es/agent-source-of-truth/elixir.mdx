Fuente canónica de Firecrawl para Elixir dirigida a agentes. Generada a partir del código fuente del SDK y de la especificación OpenAPI v2.

<div id="install">
  ## Instalación
</div>

Agrega a `mix.exs`:

```elixir
{:firecrawl, "~> 1.0.0"}
```

<div id="authenticate">
  ## Autenticación
</div>

```elixir
# config/runtime.exs o config.exs
config :firecrawl, api_key: System.get_env("FIRECRAWL_API_KEY")

# o pasa api_key en cada llamada
{:ok, res} = Firecrawl.search_and_scrape([query: "site:docs.firecrawl.dev webhook retries"], api_key: "fc-your-api-key")
```

<div id="when-to-use-what">
  ## Cuándo usar cada opción
</div>

* `search`: úsalo cuando empiezas con una consulta y necesitas descubrir resultados.
* `scrape`: úsalo cuando ya tienes una URL y quieres el contenido de la página.
* `interact`: úsalo cuando la página requiere clics, formularios o acciones del navegador posteriores al scraping.

<div id="search">
  ## Búsqueda
</div>

### Por qué usarlo

Usa search para descubrir páginas relevantes a partir de una consulta y luego elegir las URL que quieras para hacer scraping o interactuar con ellas. Puedes limitar los resultados a un sitio con `site:`, por ejemplo `site:docs.firecrawl.dev crawl webhooks`.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`Firecrawl.search_and_scrape(params \\ [], opts \\ [])`

<div id="simple-example">
  ### Ejemplo simple
</div>

```elixir
{:ok, res} = Firecrawl.search_and_scrape(query: "site:docs.firecrawl.dev webhook retries")
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```elixir
{:ok, res} = Firecrawl.search_and_scrape(
  query: "site:docs.firecrawl.dev crawl webhooks",
  sources: [:web, :news],
  categories: [:research],
  limit: 10,
  tbs: "qdr:m",
  location: "San Francisco,California,United States",
  country: "US",
  ignore_invalid_urls: true,
  timeout: 60000,
  scrape_options: [
    formats: [
      "markdown",
      "links",
      %{type: "json", prompt: "Extract title and key endpoints."}
    ],
    only_main_content: true,
    include_tags: ["main", "article"],
    exclude_tags: ["nav", "footer"],
    wait_for: 1000
  ]
)
```

<div id="parameters">
  ### Parámetros
</div>

* `query`
  * Tipo: string
  * Úsalo cuando: necesites una consulta de búsqueda.
  * Notas: usa `site:example.com` para limitar los resultados a un dominio.

* `sources`
  * Tipo: lista de átomos, strings o mapas
  * Úsalo cuando: quieras controlar qué fuentes se consultan.
  * Valores confirmados:
    * `"web"` o `:web`
    * `"news"` o `:news`
    * `"images"` o `:images`
    * `%{type: "web" | "news" | "images"}`

* `categories`
  * Tipo: lista de átomos, strings o mapas
  * Úsalo cuando: quieras filtrar los resultados por categoría.
  * Valores confirmados:
    * `"github"` o `:github`
    * `"research"` o `:research`
    * `"pdf"` o `:pdf`
    * `%{type: "github" | "research" | "pdf"}`

* `limit`
  * Tipo: integer
  * Úsalo cuando: quieras establecer un límite de resultados.

* `tbs`
  * Tipo: string
  * Úsalo cuando: necesites un filtro basado en tiempo (por ejemplo, `qdr:d`, `qdr:w`, `sbd:1,qdr:m`).

* `location`
  * Tipo: string
  * Úsalo cuando: quieras resultados localizados.

* `country`
  * Tipo: string
  * Úsalo cuando: quieras segmentación ISO 3166-1 alpha-2 (por ejemplo, `"US"`).

* `ignore_invalid_urls`
  * Tipo: boolean
  * Úsalo cuando: quieras descartar las URL que otros endpoints no puedan extraer.

* `timeout`
  * Tipo: integer
  * Úsalo cuando: necesites un tiempo de espera de la solicitud en milisegundos.

* `enterprise`
  * Tipo: lista de strings
  * Úsalo cuando: necesites controles de búsqueda empresariales.
  * Valores confirmados:
    * `"zdr"`: retención de datos cero de extremo a extremo
    * `"anon"`: retención de datos cero anonimizada

* `scrape_options`
  * Tipo: lista de keywords
  * Úsalo cuando: quieras extraer cada resultado de búsqueda (consulta los parámetros de scraping para ver los campos).

<div id="scrape">
  ## Scraping
</div>

### Por qué usarlo

Usa `scrape` si ya tienes una URL y quieres contenido estructurado en uno o más formatos.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`Firecrawl.scrape_and_extract_from_url(params \\ [], opts \\ [])`

<div id="simple-example">
  ### Ejemplo simple
</div>

```elixir
{:ok, res} = Firecrawl.scrape_and_extract_from_url(
  url: "https://docs.firecrawl.dev",
  formats: ["markdown"]
)
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```elixir
{:ok, res} = Firecrawl.scrape_and_extract_from_url(
  url: "https://example.com/pricing",
  formats: [
    "markdown",
    "links",
    %{type: "json", prompt: "Extract plan names and prices."},
    %{type: "screenshot", fullPage: true, quality: 80, viewport: %{width: 1280, height: 720}},
    %{type: "changeTracking", modes: ["git-diff"], tag: "pricing"}
  ],
  headers: %{"User-Agent" => "FirecrawlDocsBot/1.0"},
  only_main_content: true,
  wait_for: 1000,
  parsers: [%{type: "pdf", mode: "auto", maxPages: 5}],
  actions: [
    %{type: "click", selector: "#accept"},
    %{type: "wait", milliseconds: 750},
    %{type: "scrape"}
  ],
  location: [country: "US", languages: ["en-US"]],
  remove_base64_images: true,
  block_ads: true,
  proxy: :auto,
  max_age: 86400000,
  min_age: 1,
  store_in_cache: true,
  profile: [name: "docs-session", save_changes: true],
  zero_data_retention: false
)
```

<div id="parameters">
  ### Parámetros
</div>

* `url`
  * Tipo: string
  * Úsalo cuando: quieras hacer scraping de una página específica.

* `formats`
  * Tipo: lista de cadenas de formato o mapas de formato
  * Úsalo cuando: quieras varios formatos de salida.
  * Cadenas de formato confirmadas:
    * `"markdown"`: contenido en markdown
    * `"html"`: HTML limpio
    * `"rawHtml"`: HTML sin procesar
    * `"links"`: enlaces de la página
    * `"images"`: URL de imágenes
    * `"screenshot"`: salida de captura de pantalla
    * `"summary"`: salida de resumen
    * `"changeTracking"`: salida de seguimiento de cambios
    * `"json"`: extracción JSON
    * `"branding"`: salida de perfil de marca
    * `"audio"`: extracción de audio
    * `"video"`: extracción de video
  * Campos del mapa de formato:
    * `type`: una de las cadenas de formato anteriores
    * `prompt`, `schema`: opciones de extracción JSON para `type: "json"`
    * `modes`, `schema`, `prompt`, `tag`: opciones de seguimiento de cambios para `type: "changeTracking"`
    * `fullPage`, `quality`, `viewport`: opciones de captura de pantalla para `type: "screenshot"`

* `headers`
  * Tipo: mapa
  * Úsalo cuando: necesites encabezados de solicitud personalizados.

* `include_tags`
  * Tipo: lista de strings
  * Úsalo cuando: quieras incluir solo etiquetas HTML específicas.

* `exclude_tags`
  * Tipo: lista de strings
  * Úsalo cuando: quieras excluir etiquetas HTML específicas.

* `only_main_content`
  * Tipo: boolean
  * Úsalo cuando: quieras eliminar la navegación, el pie de página y otros elementos repetitivos.

* `timeout`
  * Tipo: integer
  * Úsalo cuando: necesites un tiempo de espera en milisegundos.

* `wait_for`
  * Tipo: integer
  * Úsalo cuando: necesites esperar a que la página se renderice (milisegundos).

* `mobile`
  * Tipo: boolean
  * Úsalo cuando: quieras una ventana gráfica móvil.

* `parsers`
  * Tipo: lista de cadenas de parser o mapas
  * Úsalo cuando: necesites controles de análisis de archivos.
  * Valores confirmados:
    * `"pdf"`
    * `%{type: "pdf", mode: "fast" | "auto" | "ocr", maxPages: integer}`

* `actions`
  * Tipo: lista de mapas de acciones
  * Úsalo cuando: necesites acciones ligeras previas al scraping.
  * Tipos de acción confirmados:
    * `wait`: `milliseconds` o `selector` obligatorios
    * `screenshot`: `fullPage`, `quality`, `viewport` opcionales
    * `click`: `selector` obligatorio, `all` opcional
    * `write`: `text` obligatorio (haz clic primero para enfocar el campo de entrada)
    * `press`: `key` obligatorio
    * `scroll`: `direction` (`up` o `down`) obligatorio, `selector` opcional
    * `scrape`: sin campos adicionales
    * `executeJavascript`: `script` obligatorio
    * `pdf`: `format` (A0, A1, A2, A3, A4, A5, A6, Letter, Legal, Tabloid, Ledger), `landscape`, `scale` opcionales

* `location`
  * Tipo: lista de keywords (por ejemplo, `country:` y `languages:`)
  * Úsalo cuando: necesites scraping según la ubicación o el idioma.

* `skip_tls_verification`
  * Tipo: boolean
  * Úsalo cuando: necesites omitir la verificación de TLS.

* `remove_base64_images`
  * Tipo: boolean
  * Úsalo cuando: quieras eliminar las imágenes en base64 de la salida en markdown.

* `block_ads`
  * Tipo: boolean
  * Úsalo cuando: quieras bloquear anuncios y ventanas emergentes de cookies.

* `proxy`
  * Tipo: atom o string
  * Úsalo cuando: necesites controlar el proxy.
  * Valores confirmados: `:basic`, `:enhanced`, `:auto`

* `max_age`
  * Tipo: integer
  * Úsalo cuando: quieras datos en caché hasta una antigüedad máxima (milisegundos).

* `min_age`
  * Tipo: integer
  * Úsalo cuando: quieras datos en caché solo si tienen al menos esta antigüedad (milisegundos).

* `store_in_cache`
  * Tipo: boolean
  * Úsalo cuando: quieras que Firecrawl almacene el resultado en caché.

* `profile`
  * Tipo: lista de keywords con `name:` y `save_changes:` opcional (o `saveChanges:`)
  * Úsalo cuando: quieras un perfil persistente de navegador compartido entre scrapes e interacciones.

* `zero_data_retention`
  * Tipo: boolean
  * Úsalo cuando: quieras retención de datos cero para este scraping.

<div id="interact">
  ## Interact
</div>

<div id="why-use-it">
  ### Por qué usar interact
</div>

Usa interact cuando una página requiera acciones del navegador o la ejecución de código después de que se inicie el scraping.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`Firecrawl.interact_with_scrape_browser_session(job_id, params \\ [], opts \\ [])`

<div id="simple-example">
  ### Ejemplo simple
</div>

```elixir
{:ok, res} = Firecrawl.interact_with_scrape_browser_session(
  "<scrapeJobId>",
  code: "console.log(await page.title());",
  language: :node,
  timeout: 60
)
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```elixir
{:ok, res} = Firecrawl.interact_with_scrape_browser_session(
  "<scrapeJobId>",
  code: "// Usa los métodos de página de Playwright aquí",
  language: :node,
  timeout: 120
)
```

<div id="parameters">
  ### Parámetros
</div>

* `job_id`
  * Tipo: string
  * Úsalo cuando: tengas un ID de trabajo de scraping.

* `code`
  * Tipo: string
  * Úsalo cuando: quieras ejecutar código en la sesión del navegador.

* `language`
  * Tipo: átomo o string
  * Úsalo cuando: necesites un entorno de ejecución específico.
  * Valores confirmados: `:python`, `:node`, `:bash` (o sus equivalentes como string)

* `timeout`
  * Tipo: integer
  * Úsalo cuando: necesites un tiempo de espera de ejecución en segundos.

* `origin`
  * Tipo: string
  * Úsalo cuando: quieras una etiqueta de origen opcional para la telemetría de ejecución.

<div id="stop-interactive-session">
  ### Detener la sesión interactiva
</div>

`Firecrawl.stop_interactive_scrape_browser_session(job_id, opts \\ [])` envía `DELETE /scrape/{jobId}/interact` y cierra la sesión del navegador de ese trabajo de scraping. También está disponible la variante con bang `stop_interactive_scrape_browser_session!/2`.

```elixir
{:ok, res} = Firecrawl.stop_interactive_scrape_browser_session("<scrapeJobId>")
```

<div id="notes">
  ## Notas
</div>

* El cliente de Elixir sigue la estructura de OpenAPI; los nombres de las funciones y las claves de los parámetros se generan a partir de la especificación.
* Cada función pública tiene una variante bang (`!`) que lanza una excepción en caso de error en lugar de devolver `{:error, _}`.
* Este SDK solo expone interacciones mediante código (sin el parámetro `prompt` en `interact_with_scrape_browser_session`).
