Fuente canónica de Firecrawl en Java para agentes. Generada a partir del código fuente del SDK y de la especificación OpenAPI v2.

<div id="install">
  ## Instalación
</div>

Maven:

```xml
<dependency>
  <groupId>com.firecrawl</groupId>
  <artifactId>firecrawl-java</artifactId>
  <version>1.2.0</version>
</dependency>
```

Gradle:

```gradle
implementation("com.firecrawl:firecrawl-java:1.2.0")
```

<div id="authenticate">
  ## Autenticación
</div>

```java
import com.firecrawl.client.FirecrawlClient;

FirecrawlClient client = FirecrawlClient.builder()
    .apiKey(System.getenv("FIRECRAWL_API_KEY"))
    .build();
```

<div id="when-to-use-what">
  ## Cuándo usar cada opción
</div>

* `search`: úsalo cuando partes de una consulta y necesitas descubrir resultados.
* `scraping`: úsalo cuando ya tienes una URL y quieres el contenido de la página.
* `interact`: úsalo cuando la página requiere clics, formularios o acciones del navegador después del scraping.
* `support/ask`: úsalo cuando una llamada a la API de Firecrawl falla o devuelve resultados inesperados y necesitas un diagnóstico.
* `support/docs-search`: úsalo cuando necesites consultar la documentación de Firecrawl.

<div id="search">
  ## Búsqueda
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa search para descubrir páginas relevantes a partir de una consulta y luego elegir las URL para hacer scraping o interactuar con ellas. Puedes limitar los resultados a un sitio con `site:`, por ejemplo `site:docs.firecrawl.dev crawl webhooks`.

<div id="preferred-sdk-methods">
  ### Métodos recomendados del SDK
</div>

* `client.search(query)`
* `client.search(query, options)`

<div id="return-value">
  ### Valor devuelto
</div>

`search` devuelve `SearchData`. Accede a los grupos de resultados con `getWeb()`, `getNews()` y `getImages()` (cada uno es `List<Map<String, Object>>` y puede ser null). No trates `SearchData` como una lista de resultados directamente iterable.

<div id="simple-example">
  ### Ejemplo simple
</div>

```java
import com.firecrawl.models.SearchData;
import java.util.List;
import java.util.Map;

SearchData results = client.search("site:docs.firecrawl.dev webhook retries");
List<Map<String, Object>> web = results.getWeb();
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```java
import com.firecrawl.models.SearchOptions;
import com.firecrawl.models.ScrapeOptions;
import com.firecrawl.models.JsonFormat;
import com.firecrawl.models.LocationConfig;

SearchOptions options = SearchOptions.builder()
    .sources(List.of("web", "news"))
    .categories(List.of("research"))
    .limit(10)
    .tbs("qdr:m")
    .location("San Francisco,California,United States")
    .ignoreInvalidURLs(true)
    .timeout(60000)
    .scrapeOptions(
        ScrapeOptions.builder()
            .formats(List.of(
                "markdown",
                "links",
                JsonFormat.builder().prompt("Extract title and key endpoints.").build()
            ))
            .onlyMainContent(true)
            .waitFor(1000)
            .build()
    )
    .build();

SearchData results = client.search("site:docs.firecrawl.dev crawl webhooks", options);
```

<div id="parameters">
  ### Parámetros
</div>

* `query`
  * Tipo: String
  * Úsalo cuando: necesites una consulta de búsqueda.
  * Notas: usa `site:example.com` para limitar los resultados a un dominio.

* `options.sources`
  * Tipo: Lista de cadenas de fuentes o mapas
  * Úsalo cuando: quieras controlar qué fuentes se consultan.
  * Valores confirmados:
    * `"web"`: resultados del índice web
    * `"news"`: resultados de noticias
    * `"images"`: resultados de imágenes
    * `{type: "web" | "news" | "images"}`: formato de mapa de fuente tipado

* `options.categories`
  * Tipo: Lista de cadenas de categorías o mapas
  * Úsalo cuando: quieras filtrar los resultados por categoría.
  * Valores confirmados:
    * `"github"`: resultados centrados en GitHub
    * `"research"`: resultados de investigación y académicos
    * `"pdf"`: resultados centrados en PDF
    * `{type: "github" | "research" | "pdf"}`: formato de mapa de categoría tipado

* `options.limit`
  * Tipo: Integer
  * Úsalo cuando: quieras limitar la cantidad de resultados.

* `options.tbs`
  * Tipo: String
  * Úsalo cuando: necesites un filtro basado en el tiempo (por ejemplo, `qdr:d`, `qdr:w`, `sbd:1,qdr:m`).

* `options.location`
  * Tipo: String
  * Úsalo cuando: quieras obtener resultados localizados.

* `options.ignoreInvalidURLs`
  * Tipo: Boolean
  * Úsalo cuando: quieras omitir las URL que otros endpoints no puedan extraer.

* `options.timeout`
  * Tipo: Integer
  * Úsalo cuando: necesites un timeout de la solicitud en milisegundos.

* `options.scrapeOptions`
  * Tipo: `ScrapeOptions`
  * Úsalo cuando: quieras extraer cada resultado de búsqueda (consulta los parámetros de Scrape para ver los campos).

* `options.integration`
  * Tipo: String
  * Úsalo cuando: la API espere un identificador de integración en la solicitud.

<div id="scrape">
  ## Scraping
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa `scrape` si ya tienes una URL y quieres contenido estructurado en uno o más formatos.

<div id="preferred-sdk-methods">
  ### Métodos recomendados del SDK
</div>

* `client.scrape(url)`
* `client.scrape(url, options)`

<div id="return-value">
  ### Valor devuelto
</div>

`scraping` devuelve `Document`. Los métodos de acceso habituales incluyen `getMarkdown()`, `getHtml()`, `getRawHtml()`, `getJson()`, `getMetadata()`, `getLinks()`, `getAudio()`, `getVideo()` y campos adicionales cuando se solicitan los formatos correspondientes.

<div id="simple-example">
  ### Ejemplo simple
</div>

```java
Document doc = client.scrape(
    "https://docs.firecrawl.dev",
    ScrapeOptions.builder().formats(List.of("markdown")).build()
);
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```java
import com.firecrawl.models.ScrapeOptions;
import com.firecrawl.models.JsonFormat;

List<Map<String, Object>> actions = List.of(
    Map.of("type", "click", "selector", "#accept"),
    Map.of("type", "wait", "milliseconds", 750),
    Map.of("type", "scrape")
);

LocationConfig location = LocationConfig.builder()
    .country("US")
    .languages(List.of("en-US"))
    .build();

ScrapeOptions options = ScrapeOptions.builder()
    .formats(List.of(
        "markdown",
        "links",
        JsonFormat.builder().prompt("Extract plan names and prices.").build(),
        Map.of("type", "screenshot", "fullPage", true, "quality", 80)
    ))
    .onlyMainContent(true)
    .waitFor(1000)
    .parsers(List.of(Map.of("type", "pdf", "maxPages", 5)))
    .actions(actions)
    .location(location)
    .removeBase64Images(true)
    .blockAds(true)
    .proxy("auto")
    .maxAge(86400000L)
    .storeInCache(true)
    .build();

Document doc = client.scrape("https://example.com/pricing", options);
```

<div id="parameters">
  ### Parámetros
</div>

* `url`
  * Tipo: String
  * Usar cuando: quieras hacer scraping de una página específica.

* `options.formats`
  * Tipo: Lista de cadenas de formato u objetos de formato
  * Usar cuando: quieras varios formatos de salida.
  * Cadenas de formato confirmadas:
    * `"markdown"`: contenido en markdown
    * `"html"`: HTML limpio
    * `"rawHtml"`: HTML sin procesar
    * `"links"`: enlaces de la página
    * `"images"`: URL de imágenes
    * `"screenshot"`: salida de captura de pantalla
    * `"summary"`: salida de resumen
    * `"changeTracking"`: salida de seguimiento de cambios
    * `"json"`: extracción JSON
    * `"attributes"`: extracción de atributos
    * `"branding"`: salida del perfil de marca
    * `"audio"`: extracción de audio
    * `"video"`: extracción de video
  * Campos del objeto de formato:
    * `type`: una de las cadenas de formato anteriores
    * `prompt`, `schema`: opciones de extracción JSON para `type: "json"`
    * `modes`, `schema`, `prompt`, `tag`: opciones de seguimiento de cambios para `type: "changeTracking"`
    * `fullPage`, `quality`, `viewport`: opciones de captura de pantalla para `type: "screenshot"`
    * `selectors`: array de `{selector, attribute}` para `type: "attributes"`

* `options.headers`
  * Tipo: `Map<String, String>`
  * Usar cuando: necesites encabezados personalizados para la solicitud.

* `options.includeTags`
  * Tipo: `List<String>`
  * Usar cuando: quieras incluir solo etiquetas HTML específicas.

* `options.excludeTags`
  * Tipo: `List<String>`
  * Usar cuando: quieras excluir etiquetas HTML específicas.

* `options.onlyMainContent`
  * Tipo: Boolean
  * Usar cuando: quieras quitar la navegación, el pie de página y otros elementos repetitivos.

* `options.timeout`
  * Tipo: Integer
  * Usar cuando: necesites un timeout en milisegundos.

* `options.waitFor`
  * Tipo: Integer
  * Usar cuando: necesites esperar a que la página se renderice (milisegundos).

* `options.mobile`
  * Tipo: Boolean
  * Usar cuando: quieras un viewport móvil.

* `options.parsers`
  * Tipo: `List<Object>`
  * Usar cuando: necesites controles de parsing de archivos.
  * Valores confirmados:
    * `"pdf"`
    * `{type: "pdf", maxPages: number}`

* `options.actions`
  * Tipo: `List<Map<String, Object>>`
  * Usar cuando: necesites acciones ligeras antes del scraping.
  * Tipos de acciones confirmados:
    * `wait`: `milliseconds` o `selector` obligatorios
    * `screenshot`: `fullPage`, `quality`, `viewport` opcionales
    * `click`: `selector` obligatorio
    * `write`: `text` obligatorio (haz clic primero para enfocar el campo de entrada)
    * `press`: `key` obligatorio
    * `scroll`: `direction` (`up` o `down`) obligatorio, `selector` opcional
    * `scrape`: sin campos adicionales
    * `executeJavascript`: `script` obligatorio
    * `pdf`: `format` (A0, A1, A2, A3, A4, A5, A6, Letter, Legal, Tabloid, Ledger), `landscape`, `scale` opcionales

* `options.location`
  * Tipo: `LocationConfig`
  * Usar cuando: necesites scraping según la ubicación o el idioma.

* `options.skipTlsVerification`
  * Tipo: Boolean
  * Usar cuando: necesites omitir la verificación TLS.

* `options.removeBase64Images`
  * Tipo: Boolean
  * Usar cuando: quieras eliminar imágenes en base64 de la salida markdown.

* `options.blockAds`
  * Tipo: Boolean
  * Usar cuando: quieras bloquear anuncios y ventanas emergentes de cookies.

* `options.proxy`
  * Tipo: String
  * Usar cuando: necesites controlar el proxy.
  * Valores confirmados: `"basic"`, `"stealth"`, `"enhanced"`, `"auto"` o una cadena con una URL de proxy personalizada

* `options.maxAge`
  * Tipo: Long
  * Usar cuando: quieras datos en caché con una antigüedad máxima (milisegundos).

* `options.storeInCache`
  * Tipo: Boolean
  * Usar cuando: quieras que Firecrawl almacene el resultado en caché.

* `options.integration`
  * Tipo: String
  * Usar cuando: la API espere un identificador de integración en la solicitud.

<div id="interact">
  ## Interact
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa interact cuando una página requiera acciones del navegador o la ejecución de código una vez iniciado el scraping.

<div id="preferred-sdk-methods">
  ### Métodos recomendados del SDK
</div>

* `client.interact(jobId, code)` — usa el lenguaje predeterminado `node` y el timeout de ejecución predeterminado de la API
* `client.interact(jobId, code, language, timeout)` — `timeout` está en segundos (1–300), o `null` para omitirlo y usar el valor predeterminado de la API (30 segundos)
* `client.interact(jobId, code, language, timeout, origin)` — la cadena opcional `origin` se envía solo cuando no es `null` (atribución de la solicitud)

<div id="simple-example">
  ### Ejemplo simple
</div>

```java
import com.firecrawl.models.BrowserExecuteResponse;

BrowserExecuteResponse result = client.interact(
    "<scrapeJobId>",
    "console.log(await page.title());",
    "node",
    60
);
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```java
BrowserExecuteResponse result = client.interact(
    "<scrapeJobId>",
    "// Usa los métodos de página de Playwright aquí",
    "node",
    120
);
```

<div id="stop-interactive-browser">
  ### Detener el navegador interactivo
</div>

Finaliza la sesión del navegador asociada al scraping al terminar.

**Método recomendado del SDK:** `client.stopInteractiveBrowser(jobId)`

```java
import com.firecrawl.models.BrowserDeleteResponse;

BrowserDeleteResponse stopped = client.stopInteractiveBrowser("<scrapeJobId>");
```

<div id="interact-response-browserexecuteresponse">
  ### Respuesta de Interact (`BrowserExecuteResponse`)
</div>

* `isSuccess()`: boolean
* `getStdout()`, `getStderr()`, `getResult()`, `getError()`: String (puede ser `null`)
* `getExitCode()`: Integer (puede ser `null`)
* `getKilled()`: Boolean (puede ser `null`) — true cuando la ejecución se detuvo por timeout

<div id="stop-response-browserdeleteresponse">
  ### Respuesta al detener (`BrowserDeleteResponse`)
</div>

* `isSuccess()`: boolean
* `getSessionDurationMs()`: Long (puede ser null)
* `getCreditsBilled()`: Integer (puede ser null)
* `getError()`: String (puede ser null)

<div id="parameters">
  ### Parámetros
</div>

* `jobId`
  * Tipo: String
  * Úsalo cuando: tengas un ID de trabajo de scraping.

* `code`
  * Tipo: String
  * Úsalo cuando: quieras ejecutar código en la sesión del navegador.

* `language`
  * Tipo: String
  * Úsalo cuando: necesites un runtime específico.
  * Valores admitidos: `"python"`, `"node"`, `"bash"`
  * Nota: en el SDK, el valor predeterminado es `"node"` si es `null`.

* `timeout`
  * Tipo: Integer
  * Úsalo cuando: necesites un tiempo de espera de ejecución en segundos (1–300). `null` omite el campo y usa el valor predeterminado de la API.

* `origin`
  * Tipo: String
  * Úsalo cuando: necesites una etiqueta opcional de origen en la solicitud. Es preferible omitirla salvo que tu integración la requiera.

<div id="notes">
  ## Notas
</div>

* Alias en desuso: `scrapeExecute` → `interact`, `deleteScrapeBrowser` → `stopInteractiveBrowser` (y los auxiliares `*Async` correspondientes).
* El SDK de Java solo expone interacciones mediante código: no hay ningún parámetro `prompt` en `interact` (a diferencia de algunos SDK de otros lenguajes).
