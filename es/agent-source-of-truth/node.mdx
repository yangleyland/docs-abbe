Fuente canónica de Firecrawl en Node.js para agentes. Alineada con `firecrawl` **v4.18.2** (`firecrawl/apps/js-sdk/firecrawl`) y la especificación OpenAPI v2. Los nombres de los métodos, los parámetros y los tipos coinciden con la API pública del SDK.

<div id="install">
  ## Instalación
</div>

```bash
npm install firecrawl
```

<div id="authenticate">
  ## Autenticación
</div>

```ts
import { Firecrawl } from "firecrawl";

const client = new Firecrawl({
  apiKey: process.env.FIRECRAWL_API_KEY,
  // apiUrl: "https://api.firecrawl.dev" // opcional; usa FIRECRAWL_API_URL o el valor predeterminado en la nube como alternativa
});
```

<div id="when-to-use-what">
  ## Cuándo usar cada opción
</div>

* `search`: úsalo cuando empieces con una consulta y necesites descubrir resultados.
* `scrape`: úsalo cuando ya tengas una URL y quieras el contenido de la página.
* `interact`: úsalo cuando la página requiera clics, formularios u otras acciones del navegador después de que un scraping haya creado una sesión. Para flujos interactivos de varios pasos, prefiere `interact` en lugar de las `actions` durante el scraping.
* `support/ask`: úsalo cuando una llamada a la API de Firecrawl falle o devuelva resultados inesperados y necesites un diagnóstico.
* `support/docs-search`: úsalo cuando necesites consultar la documentación de Firecrawl.

<div id="search">
  ## Search
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa search para descubrir páginas relevantes a partir de una consulta y luego elegir las URLs que quieras usar para hacer scraping o interactuar con ellas. Puedes restringir los resultados a un sitio con `site:`, por ejemplo `site:docs.firecrawl.dev crawl webhooks`.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`client.search(query, options?)` → `Promise<SearchData>`

<div id="simple-example">
  ### Ejemplo simple
</div>

```ts
const results = await client.search("site:docs.firecrawl.dev webhook retries");
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```ts
const results = await client.search("site:docs.firecrawl.dev crawl webhooks", {
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:m",
  location: "San Francisco,California,United States",
  ignoreInvalidURLs: true,
  timeout: 60000,
  scrapeOptions: {
    formats: [
      "markdown",
      "links",
      { type: "json", prompt: "Extract title and key endpoints." }
    ],
    onlyMainContent: true,
    includeTags: ["main", "article"],
    excludeTags: ["nav", "footer"],
    waitFor: 1000
  }
});
```

<div id="return-value">
  ### Valor devuelto
</div>

`SearchData` es un objeto con arrays opcionales (cada elemento es un objeto de resultado ligero o un `Document` completo cuando `scrapeOptions` hidrató la coincidencia):

* `web`: coincidencias del índice web
* `news`: resultados de noticias
* `images`: resultados de imágenes

**Error que debes evitar:** `search()` no devuelve `{ data: [...] }`. No accedas a `result.data`. Los resultados web están en `result.web`, los de noticias en `result.news` y los de imágenes en `result.images`.

<div id="parameters">
  ### Parámetros
</div>

* `query`
  * Tipo: string
  * Úsalo cuando: necesites una consulta de búsqueda.
  * Notas: usa `site:example.com` para limitar los resultados a un dominio.

* `options` (opcional; el segundo argumento usa `{}` de forma predeterminada)
  * Tipo: `Omit<SearchRequest, "query">`

* `options.sources`
  * Tipo: array de nombres de fuentes u objetos de fuente con tipo
  * Úsalo cuando: quieras controlar en qué fuentes se busca.
  * Valores confirmados:
    * `"web"`: resultados del índice web
    * `"news"`: resultados de noticias
    * `"images"`: resultados de imágenes
    * `{ type: "web" | "news" | "images" }`: formato de objeto de fuente con tipo

* `options.categories`
  * Tipo: array de nombres de categorías u objetos de categoría con tipo
  * Úsalo cuando: quieras filtrar los resultados por categoría.
  * Valores confirmados:
    * `"github"`: resultados centrados en GitHub
    * `"research"`: resultados de investigación y académicos
    * `"pdf"`: resultados centrados en PDF
    * `{ type: "github" | "research" | "pdf" }`: formato de objeto de categoría con tipo

* `options.limit`
  * Tipo: number
  * Úsalo cuando: quieras limitar los resultados.

* `options.tbs`
  * Tipo: string
  * Úsalo cuando: necesites un filtro temporal (por ejemplo `qdr:d`, `qdr:w`, `sbd:1,qdr:m`).

* `options.location`
  * Tipo: string
  * Úsalo cuando: quieras resultados localizados.

* `options.ignoreInvalidURLs`
  * Tipo: boolean
  * Úsalo cuando: quieras omitir las URL que otros endpoints no pueden extraer.

* `options.timeout`
  * Tipo: number
  * Úsalo cuando: necesites un tiempo de espera de la solicitud en milisegundos.

* `options.scrapeOptions`
  * Tipo: `ScrapeOptions`
  * Úsalo cuando: quieras extraer cada resultado de búsqueda (consulta los parámetros de scraping para ver los campos). El SDK aplica la misma validación que para `scrape` (por ejemplo, se rechaza la cadena simple `"json"` en `formats`).

<div id="scrape">
  ## Scraping
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa scraping cuando ya tengas una URL y quieras contenido estructurado en uno o más formatos.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`client.scrape(url, options?)` → `Promise<Document>`

<div id="simple-example">
  ### Ejemplo simple
</div>

```ts
const doc = await client.scrape("https://docs.firecrawl.dev", {
  formats: ["markdown"]
});
```

### Ejemplo avanzado

```ts
const doc = await client.scrape("https://example.com/pricing", {
  formats: [
    "markdown",
    "links",
    { type: "json", prompt: "Extract plan names and prices." },
    { type: "screenshot", fullPage: true, quality: 80, viewport: { width: 1280, height: 720 } },
    { type: "changeTracking", modes: ["git-diff"], tag: "pricing" },
    { type: "attributes", selectors: [{ selector: "a", attribute: "href" }] }
  ],
  headers: {
    "User-Agent": "FirecrawlDocsBot/1.0"
  },
  onlyMainContent: true,
  waitFor: 1000,
  parsers: [{ type: "pdf", mode: "auto", maxPages: 5 }],
  actions: [
    { type: "click", selector: "#accept" },
    { type: "wait", milliseconds: 750 },
    { type: "scrape" }
  ],
  location: { country: "US", languages: ["en-US"] },
  removeBase64Images: true,
  fastMode: true,
  blockAds: true,
  proxy: "auto",
  maxAge: 86400000,
  minAge: 1,
  storeInCache: true,
  profile: { name: "docs-session", saveChanges: true }
});
```

<div id="parameters">
  ### Parámetros
</div>

* `url`
  * Tipo: string
  * Úsalo cuando: quieras hacer scraping de una página específica.

* `options.formats`
  * Tipo: array de cadenas de formato u objetos de formato
  * Usar cuando: necesitas varios formatos de salida.
  * Formatos de cadena simples (la unión `FormatString` también incluye `"json"`, pero el SDK rechaza `"json"` como cadena simple; usa un objeto `json` a continuación):
    * `"markdown"`: contenido en markdown
    * `"html"`: HTML limpio
    * `"rawHtml"`: HTML sin procesar
    * `"links"`: enlaces de la página
    * `"images"`: URL de imágenes
    * `"screenshot"`: salida de captura de pantalla
    * `"summary"`: salida de resumen
    * `"changeTracking"`: salida de seguimiento de cambios (para opciones como `modes`, usa `{ type: "changeTracking", modes: [...] }`; `modes` es obligatorio en ese objeto en los tipos)
    * `"attributes"`: extracción de atributos (usa `{ type: "attributes", selectors: [...] }` al pasar selectores)
    * `"branding"`: salida del perfil de marca
    * `"audio"`: extracción de audio
    * `"video"`: extracción de video
  * Tipos de formato solo como objeto (como mínimo `type`, como se muestra):
    * `{ type: "json", prompt?: string, schema?: JSON schema or Zod schema }`: se requiere al menos uno de `prompt` o `schema` (validación del SDK).
    * `{ type: "question", question: string }`: extracción de estilo pregunta-respuesta.
    * `{ type: "highlights", query: string }`: extracción de texto fuente relevante.
    * `{ type: "screenshot", fullPage?, quality?, viewport? }`: las mismas opciones que la forma de cadena, pero como objeto.
    * `{ type: "changeTracking", modes: ("git-diff" | "json")[], schema?, prompt?, tag? }`: `modes` es obligatorio.
    * `{ type: "attributes", selectors: Array<{ selector, attribute }> }`
  * Campos compartidos de objeto donde corresponda:
    * `schema`: esquema JSON o esquema Zod para `json`, o para `changeTracking` en modo `json` (el SDK convierte Zod a JSON Schema).
    * `modes`: solo para `changeTracking`: `"git-diff"` y/o `"json"`.
    * `tag`: etiqueta de rama opcional para seguimiento de cambios.
    * `fullPage`, `quality`, `viewport`: opciones de captura de pantalla.

* `options.headers`
  * Tipo: registro de cadenas a cadenas
  * Úsalo cuando: necesites cabeceras de solicitud personalizadas.

* `options.includeTags`
  * Tipo: array de cadenas
  * Úsalo si: quieres incluir solo etiquetas HTML específicas.

* `options.excludeTags`
  * Tipo: array de cadenas
  * Úsalo cuando quieras excluir etiquetas HTML específicas.

* `options.onlyMainContent`
  * Tipo: boolean
  * Úsalo cuando: quieras eliminar los menús de navegación, el pie de página y otros elementos repetitivos.

* `options.timeout`
  * Tipo: número
  * Úsalo cuando: necesites un timeout en milisegundos.

* `options.waitFor`
  * Tipo: número
  * Úsalo cuando: necesites esperar a que la página termine de renderizarse (milisegundos).

* `options.mobile`
  * Tipo: boolean
  * Úsalo cuando: quieras un viewport móvil.

* `options.parsers`
  * Tipo: array de nombres de analizadores o de objetos de analizador
  * Usar cuando: necesites controlar el análisis de archivos (por ejemplo, de PDF).
  * Valores admitidos:
    * `"pdf"`: habilita el análisis de PDF
    * `{ type: "pdf", mode?: "fast" | "auto" | "ocr", maxPages?: number }`: opciones del analizador de PDF

* `options.actions`
  * Tipo: array de objetos de acción
  * Úsalo cuando: necesites acciones ligeras previas al scraping.
  * Tipos de acción admitidos:
    * `wait`: se requiere `milliseconds` o `selector`
    * `screenshot`: `fullPage`, `quality`, `viewport` son opcionales
    * `click`: se requiere `selector`
    * `write`: se requiere `text` (haz clic primero para enfocar el campo de entrada)
    * `press`: se requiere `key`
    * `scroll`: se requiere `direction` (`up` o `down`), `selector` es opcional
    * `scrape`: sin campos adicionales
    * `executeJavascript`: se requiere `script`
    * `pdf`: `format` (A0, A1, A2, A3, A4, A5, A6, Letter, Legal, Tabloid, Ledger), `landscape`, `scale` son opcionales

* `options.location`
  * Tipo: objeto con `country` y `languages`
  * Úsalo cuando: necesites scraping según la ubicación o el idioma.

* `options.skipTlsVerification`
  * Tipo: boolean
  * Úsalo cuando necesites omitir la verificación de TLS.

* `options.removeBase64Images`
  * Tipo: boolean
  * Úsalo cuando: quieras excluir las imágenes en base64 de la salida Markdown.

* `options.fastMode`
  * Tipo: booleano
  * Úsalo cuando: quieras hacer scraping más rápido con menor fidelidad.

* `options.blockAds`
  * Tipo: boolean
  * Se usa cuando: quieres bloquear anuncios y ventanas emergentes de cookies.

* `options.proxy`
  * Tipo: string
  * Úsalo cuando: necesites controlar el proxy.
  * Valores admitidos:
    * `"basic"`
    * `"stealth"`
    * `"enhanced"`
    * `"auto"`
    * cadena de URL de proxy personalizada

* `options.maxAge`
  * Tipo: number
  * Úsalo cuando: quieras datos en caché de hasta una antigüedad máxima (en milisegundos).

* `options.minAge`
  * Type: number
  * Úsalo cuando: quieras datos en caché solo si tienen al menos esta antigüedad (milisegundos).

* `options.storeInCache`
  * Tipo: boolean
  * Úsalo cuando: quieras que Firecrawl guarde el resultado en caché.

* `options.profile`
  * Tipo: objeto con `name` y `saveChanges` opcional
  * Úsalo cuando: quieras un perfil de navegador persistente compartido entre operaciones de scraping e interacciones.

<div id="interact">
  ## Interact
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa `interact` para controlar, mediante código o lenguaje natural, la sesión del navegador asociada a un trabajo de scraping (a través de `metadata.scrapeId`). El SDK requiere al menos uno de estos dos campos: `code` o `prompt`. Para flujos que van más allá de ajustes rápidos antes del scraping, es preferible `interact` a las `actions` de scraping.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`client.interact(jobId, args)` → `Promise<ScrapeExecuteResponse>`

<div id="simple-example">
  ### Ejemplo simple
</div>

```ts
const doc = await client.scrape("https://example.com", { formats: ["markdown"] });
const jobId = doc.metadata?.scrapeId;
if (!jobId) throw new Error("Falta scrapeId en la respuesta del scraping");

const result = await client.interact(jobId, {
  prompt: "Haz clic en la pestaña de precios y resume los planes."
});
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```ts
const result = await client.interact("<scrapeJobId>", {
  code: "console.log(await page.title());",
  language: "node",
  timeout: 60
});
```

<div id="parameters">
  ### Parámetros
</div>

* `jobId`
  * Tipo: string
  * Úsalo cuando: tengas un ID de trabajo de scraping de `document.metadata.scrapeId`.

* `args.code`
  * Tipo: string
  * Úsalo cuando: quieras ejecutar código en la sesión del navegador (por ejemplo, usar `page` de Playwright en el entorno de ejecución `node`).

* `args.prompt`
  * Tipo: string
  * Úsalo cuando: quieras que el agente del navegador siga una instrucción en lenguaje natural.

* Al menos uno de `args.code` o `args.prompt` no debe estar vacío (de lo contrario, el SDK genera una excepción).

* `args.language`
  * Tipo: string
  * Úsalo cuando: necesites un entorno de ejecución específico.
  * Valores confirmados: `"python"`, `"node"`, `"bash"`
  * Valor predeterminado cuando se omite: `"node"` (establecido por el SDK en el cuerpo de la solicitud).

* `args.timeout`
  * Tipo: number
  * Úsalo cuando: necesites un tiempo de espera de ejecución en segundos.

<div id="return-value-interact">
  ### Valor devuelto (`interact`)
</div>

`ScrapeExecuteResponse` coincide con `BrowserExecuteResponse`. Los campos confirmados incluyen:

* `success`: boolean
* `output`: string (salida agregada cuando está disponible)
* `stdout`: string
* `result`: string (campo alias junto con stdout en los typings)
* `stderr`: string
* `exitCode`: number
* `killed`: boolean
* `liveViewUrl`: string
* `interactiveLiveViewUrl`: string
* `error`: string

<div id="stop-session">
  ### Detener sesión
</div>

`client.stopInteraction(jobId)` → `Promise<ScrapeBrowserDeleteResponse>`

* `jobId`: ID del trabajo de scraping (el mismo ID que se usa para `interact`).
* Los campos de respuesta en las definiciones de tipos incluyen `success`, `sessionDurationMs`, `creditsBilled`, `error`.

<div id="ask-agentic-debugging">
  ## Ask (Depuración con agentes)
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa Ask cuando una llamada a la API de Firecrawl falle o devuelva resultados inesperados. El agente de soporte con IA diagnostica el problema, propone parámetros para corregirlo y, opcionalmente, valida la solución con la API real. Latencia típica: 15-30 segundos.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

Aún no está disponible en el SDK de Node.js. Usa `fetch` directamente.

### Ejemplo básico

```ts
const response = await fetch("https://api.firecrawl.dev/v2/support/ask", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.FIRECRAWL_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    question: "my scrape returned empty markdown for https://example.com",
  }),
});
const result = await response.json();
console.log(result.answer);
console.log(result.fixParameters);
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```ts
const response = await fetch("https://api.firecrawl.dev/v2/support/ask", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.FIRECRAWL_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    question: "crawl returned 3 pages but I expected 50",
    rationale: "user is on their third failed crawl attempt today",
    context: { userPlan: "standard", retryCount: 3 },
  }),
});
const result = await response.json();
```

<div id="agent-retry-pattern">
  ### Patrón de reintento de Agent
</div>

```ts
import { Firecrawl } from "firecrawl";

const client = new Firecrawl({ apiKey: process.env.FIRECRAWL_API_KEY });

const doc = await client.scrape("https://example.com/pricing", {
  formats: ["markdown"],
});

if (!doc.markdown || doc.markdown.length < 100) {
  const diagResponse = await fetch(
    "https://api.firecrawl.dev/v2/support/ask",
    {
      method: "POST",
      headers: {
        Authorization: `Bearer ${process.env.FIRECRAWL_API_KEY}`,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        question: `scrape returned only ${doc.markdown?.length ?? 0} chars for https://example.com/pricing`,
      }),
    }
  );
  const diagnosis = await diagResponse.json();

  if (diagnosis.fixParameters) {
    const retryDoc = await client.scrape("https://example.com/pricing", {
      formats: ["markdown"],
      ...diagnosis.fixParameters,
    });
  }
}
```

<div id="parameters">
  ### Parámetros
</div>

* `question`
  * Tipo: string (obligatorio, 1-8000 caracteres)
  * Úsalo cuando: necesites describir el problema.

* `rationale`
  * Tipo: string (1-2000 caracteres)
  * Úsalo cuando: seas un agente de IA que hace la llamada en nombre de un usuario.

* `context`
  * Tipo: objeto (formato libre)
  * Úsalo cuando: quieras pasar metadatos al prompt de depuración.

<div id="docs-search">
  ## Búsqueda en la documentación
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa docs-search para buscar en la documentación de Firecrawl.

### Ejemplo básico

```ts
const response = await fetch(
  "https://api.firecrawl.dev/v2/support/docs-search",
  {
    method: "POST",
    headers: {
      Authorization: `Bearer ${process.env.FIRECRAWL_API_KEY}`,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      question: "how do I verify webhook signatures?",
    }),
  }
);
const result = await response.json();
console.log(result.answer);
```

<div id="parameters">
  ### Parámetros
</div>

* `question`
  * Tipo: string (obligatorio, 1-8000 caracteres)
  * Úsalo cuando: necesites una respuesta fundamentada en la documentación.

<div id="notes">
  ## Notas
</div>

* Alias de cliente obsoletos: `scrapeExecute` → `interact`; `stopInteractiveBrowser` y `deleteScrapeBrowser` → `stopInteraction`.
* La exportación predeterminada de `Firecrawl` es el cliente de v2; la versión heredada de v1 permanece en `client.v1`.
* Los esquemas de Zod que se pasan a `formats` (para `json` o `changeTracking`) el SDK los convierte a JSON Schema.
* El paquete declara **Node.js &gt;= 22** en `engines`.
