Fuente canónica de Firecrawl en Python para agentes. Generada a partir del código fuente del SDK (`firecrawl-py` / `firecrawl` **4.22.1**) y de la especificación OpenAPI v2. Los nombres de los métodos, los parámetros y los tipos de retorno coinciden con el cliente v2 de `firecrawl/v2/client.py`, salvo que se indique lo contrario.

<div id="install">
  ## Instalación
</div>

```bash
pip install firecrawl-py
```

<div id="authenticate">
  ## Autenticación
</div>

```py
import os
from firecrawl import Firecrawl

client = Firecrawl(api_key=os.environ.get("FIRECRAWL_API_KEY"))
# client = Firecrawl(api_key="fc-...", api_url="https://api.firecrawl.dev")
```

<div id="when-to-use-what">
  ## Cuándo usar cada uno
</div>

* `search`: úsalo cuando partes de una consulta y necesitas encontrar resultados.
* `scrape`: úsalo cuando ya tienes una URL y quieres el contenido de la página.
* `interact`: úsalo cuando la página requiere clics, formularios o acciones del navegador después del scraping.
* `support/ask`: úsalo cuando una llamada a la API de Firecrawl falla o devuelve resultados inesperados y necesitas un diagnóstico.
* `support/docs-search`: úsalo cuando necesitas buscar en la documentación de Firecrawl.

<div id="search">
  ## Search
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa search para descubrir páginas relevantes a partir de una consulta y luego elige las URL para hacer scraping o interactuar con ellas. Puedes limitar los resultados a un sitio con `site:`, por ejemplo `site:docs.firecrawl.dev crawl webhooks`.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`client.search(query, **options)` → `SearchData` (`firecrawl.v2.types`)

<div id="return-value">
  ### Valor devuelto
</div>

Devuelve un modelo `SearchData` con listas opcionales:

* `web` — resultados web (`SearchResultWeb` o `Document` completo cuando `scrape_options` hidrata el contenido)
* `news` — resultados de noticias (`SearchResultNews` o `Document`)
* `images` — resultados de imágenes (`SearchResultImages` o `Document`)

Los campos omitidos son `None` cuando la API no devolvió esa clave.

**Error que debes evitar:** `search()` no devuelve `{ data: [...] }`. No accedas a `result.data`. Los resultados web están en `result.web`, las noticias en `result.news` y las imágenes en `result.images`.

<div id="simple-example">
  ### Ejemplo simple
</div>

```py
results = client.search("site:docs.firecrawl.dev webhook retries")
for item in results.web or []:
    print(getattr(item, "url", None), getattr(item, "title", None))
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```py
from firecrawl.v2.types import ScrapeOptions

results = client.search(
    "site:docs.firecrawl.dev crawl webhooks",
    sources=["web", "news"],
    categories=["research"],
    limit=10,
    tbs="qdr:m",
    location="San Francisco,California,United States",
    ignore_invalid_urls=True,
    timeout=300000,
    scrape_options=ScrapeOptions(
        formats=[
            "markdown",
            "links",
            {"type": "json", "prompt": "Extract title and key endpoints."},
        ],
        only_main_content=True,
        include_tags=["main", "article"],
        exclude_tags=["nav", "footer"],
        wait_for=1000,
        actions=[
            {"type": "click", "selector": "button#accept"},
            {"type": "wait", "milliseconds": 750},
            {"type": "scrape"},
        ],
    ),
)
```

<div id="parameters">
  ### Parámetros
</div>

* `query`
  * Tipo: str
  * Usar cuando: necesites una consulta de búsqueda.
  * Notas: usa `site:example.com` para limitar los resultados a un dominio.

* `sources`
  * Tipo: lista de nombres de fuentes u objetos `Source`
  * Usar cuando: quieras controlar en qué fuentes se busca.
  * Valores confirmados:
    * `"web"`: resultados del índice web
    * `"news"`: resultados de noticias
    * `"images"`: resultados de imágenes
    * `Source(type="web" | "news" | "images")`: forma de objeto `Source` tipado

* `categories`
  * Tipo: lista de nombres de categorías u objetos `Category`
  * Usar cuando: quieras filtrar los resultados por categoría.
  * Valores confirmados:
    * `"github"`: resultados centrados en GitHub
    * `"research"`: resultados de investigación y académicos
    * `"pdf"`: resultados centrados en PDF
    * `Category(type="github" | "research" | "pdf")`: forma de objeto `Category` tipado

* `limit`
  * Tipo: int
  * Usar cuando: quieras limitar la cantidad de resultados.
  * Notas: el valor predeterminado es `5` en el modelo del SDK.

* `tbs`
  * Tipo: str
  * Usar cuando: necesites un filtro basado en tiempo (por ejemplo, `qdr:d`, `qdr:w`, `sbd:1,qdr:m`).

* `location`
  * Tipo: str
  * Usar cuando: quieras resultados localizados.

* `ignore_invalid_urls`
  * Tipo: bool
  * Usar cuando: quieras omitir las URL que otros endpoints no pueden extraer.

* `timeout`
  * Tipo: int
  * Usar cuando: necesites un tiempo de espera de la solicitud en milisegundos.
  * Notas: el valor predeterminado es `300000` en el modelo del SDK.

* `scrape_options`
  * Tipo: `ScrapeOptions`
  * Usar cuando: quieras hacer scraping de cada resultado de búsqueda (consulta los parámetros de scraping para ver los campos).

<div id="scrape">
  ## Scraping
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa el scraping cuando ya tengas una URL y quieras contenido estructurado en uno o más formatos.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`client.scrape(url, **options)` → `Document` (`firecrawl.v2.types`)

<div id="simple-example">
  ### Ejemplo simple
</div>

```py
doc = client.scrape("https://docs.firecrawl.dev", formats=["markdown"])
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```py
doc = client.scrape(
    "https://example.com/pricing",
    formats=[
        "markdown",
        "links",
        {"type": "json", "prompt": "Extract plan names and prices."},
        {"type": "screenshot", "full_page": True, "quality": 80, "viewport": {"width": 1280, "height": 720}},
        {"type": "changeTracking", "modes": ["git-diff"], "tag": "pricing"},
        {"type": "attributes", "selectors": [{"selector": "a", "attribute": "href"}]},
    ],
    headers={"User-Agent": "FirecrawlDocsBot/1.0"},
    only_main_content=True,
    wait_for=1000,
    parsers=[{"type": "pdf", "mode": "auto", "max_pages": 5}],
    actions=[
        {"type": "click", "selector": "#accept"},
        {"type": "wait", "milliseconds": 750},
        {"type": "scrape"},
    ],
    location={"country": "US", "languages": ["en-US"]},
    remove_base64_images=True,
    fast_mode=True,
    block_ads=True,
    proxy="auto",
    max_age=86400000,
    store_in_cache=True,
    profile={"name": "docs-session", "save_changes": True},
)
```

<div id="parameters">
  ### Parámetros
</div>

* `url`
  * Tipo: str
  * Úsalo cuando: quieras hacer scraping de una página específica.

* `formatos`
  * Tipo: lista de cadenas de formato o de objetos de formato
  * Úsalo cuando: quieras varios formatos de salida.
  * Cadenas de formato confirmadas:
    * `"markdown"`: contenido en markdown
    * `"html"`: HTML limpio
    * `"rawHtml"` o `"raw_html"`: HTML sin procesar
    * `"links"`: enlaces de la página
    * `"images"`: URL de imágenes
    * `"screenshot"`: salida de captura de pantalla
    * `"summary"`: salida de resumen
    * `"changeTracking"` o `"change_tracking"`: salida de seguimiento de cambios
    * `"attributes"`: extracción de atributos
    * `"branding"`: salida de perfil de marca
    * `"audio"`: extracción de audio
    * `"video"`: extracción de video
  * Tipos de formato solo como objeto:
    * `{"type": "json", ...}`: extracción JSON. Usa un objeto, no la cadena simple `"json"`.
    * `{"type": "question", "question": "..."}`: salida de preguntas y respuestas.
    * `{"type": "highlights", "query": "..."}`: salida de texto fuente relevante.
  * Campos del objeto de formato:
    * `type`: una de las cadenas de formato anteriores, o `"json"`, `"question"` o `"highlights"` para los formatos solo como objeto
    * `question`: para `type: "question"`
    * `query`: para `type: "highlights"`
    * `prompt`: opcional para `type: "json"`
    * `schema`: esquema JSON para `type: "json"` o para el modo JSON de seguimiento de cambios
    * `modes`: array de `"git-diff"` o `"json"` para `type: "changeTracking"`
    * `tag`: etiqueta de seguimiento de cambios para `type: "changeTracking"`
    * `full_page`, `quality`, `viewport`: opciones de captura de pantalla para `type: "screenshot"`
    * `selectors`: array de `{selector, attribute}` para `type: "attributes"`

* `cabeceras`
  * Tipo: dict de str a str
  * Úsalo cuando: necesites encabezados de solicitud personalizados.

* `include_tags`
  * Tipo: lista de str
  * Úsalo cuando: quieras incluir solo etiquetas HTML específicas.

* `exclude_tags`
  * Tipo: lista de str
  * Úsalo cuando: quieras excluir etiquetas HTML concretas.

* `only_main_content`
  * Tipo: bool
  * Úsalo cuando: quieras eliminar la navegación, el pie de página y otros elementos superfluos.

* `timeout`
  * Tipo: int
  * Úsalo cuando: necesites un timeout en milisegundos.

* `wait_for`
  * Tipo: int
  * Úsalo cuando: necesites esperar a que la página se renderice (milisegundos).

* `mobile`
  * Tipo: bool
  * Úsalo cuando: necesites un viewport móvil.

* `parsers`
  * Tipo: lista de nombres de analizadores o de objetos de analizador
  * Úsalo cuando: necesites controlar el análisis de archivos (por ejemplo, el análisis de PDF).
  * Valores admitidos:
    * `"pdf"`: habilita el análisis de PDF
    * `{ "type": "pdf", "mode": "fast" | "auto" | "ocr", "max_pages": int }`: opciones del analizador de PDF

* `acciones`
  * Tipo: lista de objetos de acción
  * Úsalo cuando: necesites acciones previas al scraping ligeras.
  * Tipos de acciones admitidos:
    * `wait`: `milliseconds` o `selector` obligatorios
    * `screenshot`: `full_page`, `quality`, `viewport` opcionales
    * `click`: `selector` obligatorio
    * `write`: `text` obligatorio (haz clic primero para enfocar el campo de entrada)
    * `press`: `key` obligatorio
    * `scroll`: `direction` (`up` o `down`) obligatorio, `selector` opcional
    * `scrape`: sin campos adicionales
    * `executeJavascript`: `script` obligatorio
    * `pdf`: `format` (A0, A1, A2, A3, A4, A5, A6, Letter, Legal, Tabloid, Ledger), `landscape`, `scale` opcionales

* `location`
  * Tipo: dict con `country` y `languages`
  * Úsalo cuando: necesites hacer scraping según la ubicación geográfica o el idioma.

* `skip_tls_verification`
  * Tipo: bool
  * Úsalo cuando: necesites omitir la verificación de TLS.

* `remove_base64_images`
  * Type: bool
  * Úsalo cuando: quieras quitar las imágenes en base64 de la salida en Markdown.

* `fast_mode`
  * Tipo: bool
  * Úsalo cuando: quieras scrapes más rápidos con fidelidad reducida.

* `block_ads`
  * Tipo: bool
  * Úsalo cuando quieras bloquear anuncios y ventanas emergentes de cookies.

* `proxy`
  * Tipo: str
  * Úsalo cuando: necesites controlar el proxy.
  * Valores permitidos: `"basic"`, `"stealth"`, `"enhanced"`, `"auto"`

* `max_age`
  * Type: int
  * Úsalo cuando quieras datos en caché con una antigüedad máxima (en milisegundos).

* `min_age`
  * Tipo: int
  * Úsalo cuando: quieras datos en caché solo si tienen al menos esta antigüedad (milisegundos).
  * Notas: solo en `ScrapeOptions` pasado como `scrape_options` a `client.search`. El método `client.scrape` no acepta `min_age`.

* `store_in_cache`
  * Tipo: bool
  * Úsalo cuando: quieras que Firecrawl guarde el resultado en caché.

* `profile`
  * Tipo: `dict` con `name` y `save_changes` o `saveChanges` opcionales
  * Úsalo cuando: quieras un perfil de navegador persistente compartido entre operaciones de scraping e interacciones.

<div id="interact">
  ## Interact
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa interact cuando una página requiera acciones del navegador o ejecutar código una vez iniciado el scraping.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`client.interact(job_id, code=None, *, prompt=None, language="node", timeout=None)`

`prompt` solo puede pasarse como argumento con nombre: llama a `client.interact(job_id, prompt="...")` o `client.interact(job_id, code="...")`. Al menos uno de `code` o `prompt` debe tener un valor no vacío (se valida en `firecrawl/v2/methods/scrape.py`).

<div id="simple-example">
  ### Ejemplo simple
</div>

```py
doc = client.scrape("https://example.com", formats=["markdown"])
job_id = doc.metadata.scrape_id if doc.metadata else None
if not job_id:
    raise RuntimeError("Missing scrape_id from scrape response")

result = client.interact(job_id, prompt="Click the pricing tab and summarize the plans.")
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```py
result = client.interact(
    "<scrapeJobId>",
    code="print(await page.title())",
    language="python",
    timeout=60,
)
```

<div id="stop-session">
  ### Detener sesión
</div>

`client.stop_interaction(job_id)` finaliza la sesión del navegador asociada al scraping. Devuelve `BrowserDeleteResponse` (`success`, `session_duration_ms` opcional, `credits_billed`, `error`).

<div id="parameters">
  ### Parámetros
</div>

* `job_id`
  * Tipo: str
  * Usar cuando: tienes un ID de trabajo de scraping de `document.metadata.scrape_id`.

* `code`
  * Tipo: str
  * Usar cuando: quieres ejecutar código en la sesión del navegador (opcional si se establece `prompt`).

* `prompt`
  * Tipo: str
  * Usar cuando: quieres que el agente del navegador siga una instrucción en lenguaje natural (opcional si se establece `code`).

* `language`
  * Tipo: str
  * Usar cuando: necesitas un entorno de ejecución específico.
  * Valores confirmados: `"python"`, `"node"`, `"bash"`
  * Notas: el valor predeterminado es `"node"` en el SDK.

* `timeout`
  * Tipo: int
  * Usar cuando: necesitas un límite de tiempo de ejecución en segundos.

<div id="return-value">
  ### Valor devuelto
</div>

Devuelve `BrowserExecuteResponse`: `success`, `live_view_url` (opcional), `interactive_live_view_url`, `output`, `stdout`, `result`, `stderr`, `exit_code`, `killed`, `error` (el camelCase de la API se normaliza a snake&#95;case en el modelo).

<div id="ask-agentic-debugging">
  ## Ask (Depuración con agentes)
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa `ask` cuando una llamada a la API de Firecrawl falle o devuelva resultados inesperados. El agente de soporte con IA diagnostica el problema, propone parámetros para corregirlo y, opcionalmente, valida la solución con la API real. Latencia típica: 15-30 segundos.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

Aún no está disponible en el SDK de Python. Usa `requests` o `httpx` directamente.

<div id="simple-example">
  ### Ejemplo simple
</div>

```py
import requests

response = requests.post(
    "https://api.firecrawl.dev/v2/support/ask",
    headers={
        "Authorization": f"Bearer {os.environ['FIRECRAWL_API_KEY']}",
        "Content-Type": "application/json",
    },
    json={"question": "my scrape returned empty markdown for https://example.com"},
)
result = response.json()
print(result["answer"])
print(result["fixParameters"])
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```py
import requests

response = requests.post(
    "https://api.firecrawl.dev/v2/support/ask",
    headers={
        "Authorization": f"Bearer {os.environ['FIRECRAWL_API_KEY']}",
        "Content-Type": "application/json",
    },
    json={
        "question": "crawl returned 3 pages but I expected 50",
        "rationale": "user is on their third failed crawl attempt today",
        "context": {"userPlan": "standard", "retryCount": 3},
    },
)
result = response.json()
```

<div id="agent-retry-pattern">
  ### Patrón de reintentos del agente
</div>

```py
from firecrawl import Firecrawl
import requests, os

client = Firecrawl(api_key=os.environ["FIRECRAWL_API_KEY"])

doc = client.scrape("https://example.com/pricing", formats=["markdown"])

if not doc.markdown or len(doc.markdown) < 100:
    diagnosis = requests.post(
        "https://api.firecrawl.dev/v2/support/ask",
        headers={
            "Authorization": f"Bearer {os.environ['FIRECRAWL_API_KEY']}",
            "Content-Type": "application/json",
        },
        json={
            "question": f"scrape returned only {len(doc.markdown or '')} chars for https://example.com/pricing",
        },
    ).json()

    if diagnosis.get("fixParameters"):
        doc = client.scrape(
            "https://example.com/pricing",
            formats=["markdown"],
            **diagnosis["fixParameters"],
        )
```

<div id="parameters">
  ### Parámetros
</div>

* `question`
  * Tipo: str (obligatorio, 1-8000 caracteres)
  * Úsalo cuando: necesites describir el problema.

* `rationale`
  * Tipo: str (1-2000 caracteres)
  * Úsalo cuando: seas un agente de IA que realiza la llamada en nombre de un usuario.

* `context`
  * Tipo: dict (formato libre)
  * Úsalo cuando: quieras pasar metadatos al prompt de depuración.

<div id="docs-search">
  ## Búsqueda en la documentación
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa `docs-search` para buscar en la documentación de Firecrawl.

<div id="simple-example">
  ### Ejemplo simple
</div>

```py
import requests

FIRECRAWL_API_KEY = "fc-YOUR_API_KEY"

response = requests.post(
    "https://api.firecrawl.dev/v2/support/docs-search",
    headers={"Authorization": f"Bearer {FIRECRAWL_API_KEY}"},
    json={"question": "how do I verify webhook signatures?"},
)
print(response.json()["answer"])
```

<div id="parameters">
  ### Parámetros
</div>

* `question`
  * Tipo: str (obligatorio, 1-8000 caracteres)
  * Úsalo cuando: necesites una respuesta basada en la documentación.

<div id="notes">
  ## Notas
</div>

* Alias obsoletos: `scrape_execute` → `interact`; `stop_interactive_browser` y `delete_scrape_browser` → `stop_interaction`.

- El cliente principal de `Firecrawl` expone directamente los métodos de v2; v1 sigue estando en `client.v1`.

* Es posible que el fragmento de OpenAPI v2 incluido para `POST /v2/scrape/{jobId}/interact` solo documente `code`; el SDK de Python y el servidor aceptan `code` o `prompt` para este endpoint.
