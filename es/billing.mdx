<div id="overview">
  ## Descripción general
</div>

Firecrawl utiliza un **sistema de facturación basado en créditos**. Cada llamada a la API que haces consume créditos, y la cantidad de créditos consumidos depende del endpoint y de las opciones que utilices. Recibes una asignación mensual de créditos según tu plan, y Smart Upgrade te mantiene cubierto a medida que crece tu uso.

Para ver los precios de los planes vigentes, visita la [página de precios de Firecrawl](https://www.firecrawl.dev/pricing).

<Note>
  Todas las facturas de Firecrawl se emiten en **dólares estadounidenses (USD)**, independientemente de tu dirección de facturación o método de pago.
</Note>

<div id="credits">
  ## Créditos
</div>

Los créditos son la unidad de consumo en Firecrawl. Cada plan incluye una cuota mensual de créditos que se reinicia al comienzo de cada ciclo de facturación. Los distintos endpoints de la API consumen diferentes cantidades de créditos.

<div id="credit-costs-per-endpoint">
  ### Costo de créditos por endpoint
</div>

| Endpoint     | Credit Cost                      | Notes                                                                                                                                                                                                                                                                                                                                  |
| ------------ | -------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Scrape**   | 1 crédito / página               | Convierte una única URL en markdown limpio, HTML o datos estructurados. Se aplican créditos adicionales al usar opciones de extracción (ver más abajo).                                                                                                                                                                                |
| **Crawl**    | 1 crédito / página               | Extrae un sitio web completo siguiendo enlaces desde una URL inicial. Los mismos costos por página de las opciones de extracción se aplican a cada página rastreada.                                                                                                                                                                   |
| **Map**      | 1 crédito / llamada              | Descubre todas las URL de un sitio web sin extraer su contenido.                                                                                                                                                                                                                                                                       |
| **Search**   | 2 créditos / 10 resultados       | Busca en la web y, opcionalmente, extrae los resultados. Redondeado hacia arriba por cada 10 resultados (p. ej., 11 resultados = 4 créditos). Se aplican costos adicionales de extracción por página a cada resultado que se extrae. Consulta [aquí](/es/features/search#zero-data-retention-zdr) los precios de búsqueda ZDR Enterprise. |
| **Interact** | 2 créditos / minuto de navegador | Sesión interactiva de navegador en sandbox, facturada por minuto.                                                                                                                                                                                                                                                                      |
| **Agent**    | Dinámico                         | Agente autónomo de investigación web. 5 ejecuciones diarias gratuitas; precios basados en uso a partir de ese límite.                                                                                                                                                                                                                  |

<div id="additional-credit-costs-for-scrape-options">
  ### Costos de crédito adicionales para las opciones de extracción
</div>

Ciertas opciones de extracción añaden créditos sobre el costo base por página:

| Option                       | Additional Cost         | Description                                                                                                                         |
| ---------------------------- | ----------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| PDF parsing                  | +1 crédito / página PDF | Extrae contenido de documentos PDF                                                                                                  |
| formato JSON (extracción con LLM) | +4 créditos / página    | Usa un LLM para extraer datos JSON estructurados de la página                                                                       |
| Enhanced Mode                | +4 créditos / página    | Extracción mejorada para páginas a las que es difícil acceder                                                                       |
| Zero Data Retention (ZDR)    | +1 crédito / página     | Garantiza que no se conserve ningún dato más allá de la solicitud (consulta [Scrape ZDR](/es/features/scrape#zero-data-retention-zdr)) |

Estos modificadores se acumulan. Por ejemplo, extraer una página con formato JSON (extracción con LLM) y Enhanced Mode cuesta **1 + 4 + 4 = 9 créditos** por página. Estos mismos modificadores se aplican a los endpoints Crawl y Search, ya que usan scrape internamente para cada página.

Las solicitudes a `x.com` y otras URL de X/Twitter usan la API de Grok y tienen precios independientes. Consulta [facturación de X (x.com)](#x-xcom-billing) al final de esta página.

<div id="when-credits-are-charged">
  ### Cuándo se cobran créditos
</div>

Los créditos se cobran siempre que la infraestructura de Firecrawl procesa una solicitud, incluso si el sitio de destino devuelve un código de estado de error HTTP como 403 Forbidden o 404 Not Found. Esto se debe a que la infraestructura de scraping (renderizado en el navegador, proxy, etc.) se utiliza por completo independientemente de la respuesta del sitio de destino. Puedes revisar el campo `metadata.statusCode` en la respuesta de la API para detectar estos casos y evitar reintentar URLs que se bloquean de forma constante.

Para trabajos de **batch scrape** y **crawl**, los créditos se cobran de forma asíncrona a medida que cada página termina de procesarse, no cuando se envía el trabajo. Esto significa que puede haber un retraso entre el envío de un trabajo y el momento en que se refleja el costo total en créditos en tu cuenta. Si un lote contiene muchas URLs o las páginas se ponen en cola durante períodos de alto tráfico, los créditos pueden seguir registrándose minutos u horas después del envío. Hacer *polling* o comprobar el estado del batch no consume créditos.

<Note>
  **Verificación previa de créditos para crawl:** Antes de que comience un trabajo de crawl, Firecrawl verifica que tu saldo de créditos disponible pueda cubrir por completo el parámetro `limit` que solicitaste. Si tu saldo es inferior a `limit`, la solicitud devuelve un 402 incluso si el crawl hubiera descubierto menos páginas. El valor predeterminado de `limit` es **10,000**, por lo que, si lo omites, debes tener 10,000 créditos disponibles por adelantado. Para evitarlo, pasa un `limit` explícito que coincida con la cantidad de páginas que realmente quieres recorrer (por ejemplo, `limit: 100`).
</Note>

<div id="tracking-your-usage">
  ### Seguimiento del uso
</div>

Puedes monitorear tu consumo de créditos de dos maneras:

* **Dashboard**: Consulta tu consumo actual e histórico en [firecrawl.dev/app](https://www.firecrawl.dev/app)
* **API**: Usa los endpoints [Credit Usage](/es/api-reference/endpoint/credit-usage) y [Credit Usage Historical](/es/api-reference/endpoint/credit-usage-historical) para comprobar tu consumo de forma programática

<Note>
  Estamos trabajando activamente en mejoras para que el uso de créditos sea más fácil de entender. Mantente al tanto de las novedades.
</Note>

<div id="plans">
  ## Planes
</div>

Firecrawl ofrece planes mensuales de suscripción. No existe una opción de pago por uso puro, pero Smart Upgrade (descrito más abajo) permite escalar de forma flexible por encima de tu plan base.

<div id="paid-plans">
  ### Planes de pago
</div>

| Plan         | Créditos mensuales          | Navegadores concurrentes |
| ------------ | --------------------------- | -----------------------: |
| **Hobby**    | 5,000 / 6,500 / 8,000       |                        5 |
| **Standard** | 100,000 / 130,000 / 160,000 |                       50 |
| **Growth**   | 500,000 / 650,000           |                      100 |
| **escala**   | 1,000,000                   |                      150 |

<Note>
  Para necesidades más allá del plan escala, Firecrawl ofrece planes **Enterprise** con créditos personalizados, soporte dedicado, SLAs, descuentos por volumen, sin retención de datos y SSO. Visita la [página Enterprise](https://www.firecrawl.dev/enterprise) para más detalles.
</Note>

Todos los planes de pago están disponibles con facturación **mensual** o **anual**. La facturación anual ofrece un descuento frente a pagar mes a mes. Para ver los precios actuales de cada plan, visita la [página de precios](https://www.firecrawl.dev/pricing).

<div id="billing-cycle">
  ### Ciclo de facturación
</div>

* **Planes mensuales**: Los créditos se restablecen en tu fecha de renovación mensual
* **Planes anuales**: Se te factura anualmente, pero los créditos igual se restablecen cada mes en tu fecha de renovación mensual virtual
* **Los créditos del plan no utilizados no se acumulan de forma predeterminada** — tu asignación mensual se restablece cada mes. **Los planes anuales Escala permiten acumular los créditos del plan no utilizados durante 1 mes**, y **los planes anuales Enterprise permiten acumularlos durante 2 meses**.

<div id="concurrent-browsers">
  ### Navegadores concurrentes
</div>

Los navegadores concurrentes representan cuántas páginas web puede procesar Firecrawl simultáneamente para ti. Tu plan determina este límite. Si lo superas, las tareas adicionales esperarán en una cola hasta que se libere un cupo. Consulta [Límites de tasa](/es/rate-limits) para obtener todos los detalles sobre concurrencia y límites de tasa de la API.

<div id="smart-upgrade">
  ## Smart Upgrade
</div>

<Note>
  A partir del **1 de junio de 2026**, Smart Upgrade sustituye a Auto-Recharge. En lugar de pagar excedentes, esos pagos ahora se destinan a tus cambios de nivel, para ofrecerte una experiencia fluida y sin interrupciones.
</Note>

Cuando tu saldo llega a cero, Smart Upgrade te sube automáticamente un nivel en la escala de créditos, incluidos los niveles intermedios dentro de cada plan, para que las solicitudes sigan fluyendo sin interrupciones. Solo se te factura la diferencia de precio prorrateada entre tu nivel actual y el siguiente, y recibes de inmediato la diferencia de créditos correspondiente. Puedes deshabilitar Smart Upgrade en la [configuración de facturación](https://www.firecrawl.dev/app/settings?tab=billing) en cualquier momento. Una vez que alcances el nivel Escala (1,000,000 créditos/mes), no hay niveles superiores a los que Smart Upgrade pueda pasar; [contacta con ventas](https://www.firecrawl.dev/enterprise) para consultar un plan Enterprise personalizado.

<div id="pro-ration-examples">
  ### Ejemplos de prorrateo
</div>

Supongamos que estás en **Hobby 6.5k** ($28/mes o $290/año) y Smart Upgrade te pasa a **Hobby 8k** ($37/mes o $390/año). Recibes **1,500 créditos adicionales** de inmediato (la diferencia hasta 8k), y el cargo por la actualización depende de tu periodicidad de facturación:

* **Plan mensual:** Hoy se te factura la diferencia mensual de $37 − $28 = **$9**. Tu fecha de renovación no cambia.
* **Plan anual, con 5 meses restantes de tu ciclo de 12 meses:** Hoy se te factura `(5 ÷ 12) × 100 ≈ $41.67`, donde la diferencia de precio anual es $390 − $290 = $100. Los 4 meses restantes en Hobby 8k quedan cubiertos por este cargo, y tu fecha de renovación no cambia.

Cuando agotas el nivel 8k, el siguiente paso es **Hobby 8k → Standard 100k**, y se te factura la diferencia entre esos dos niveles ($62/mes o $600/año, prorrateado de la misma manera), con **92,000 créditos adicionales** añadidos a tu saldo.

<div id="upgrading-and-downgrading">
  ## Actualizar o bajar de plan
</div>

* **Las actualizaciones de plan** surten efecto de inmediato. Hoy se te cobra el precio completo del nuevo plan (sin prorrateo) y tu ciclo de facturación se restablece: tu próxima renovación será un mes o un año después de la fecha de actualización. Los créditos no usados de tu plan anterior se transfieren, y tu nueva asignación de créditos y tus límites de concurrencia se aplican al instante.
* **Los cambios a un plan inferior** se programan para entrar en vigor en tu próxima fecha de renovación. Mantienes los créditos y límites de tu plan actual hasta entonces. El tiempo no usado de tu plan actual no se acredita ni se reembolsa. Puedes deshacer un cambio programado a un plan inferior desde tu [configuración de facturación](https://www.firecrawl.dev/app/settings?tab=billing) en cualquier momento antes de la fecha de entrada en vigor.

<div id="switching-between-monthly-and-yearly-billing">
  ### Cambio entre facturación mensual y anual
</div>

* **Mensual → Anual** en el mismo nivel de créditos o en uno superior se considera una mejora inmediata.
* **Anual → Mensual** se considera una mejora inmediata solo si cambias a un nivel de créditos estrictamente superior.

<div id="running-out-of-credits">
  ## Quedarse sin créditos
</div>

Si agotas tu asignación de créditos y no tienes Smart Upgrade habilitado, las solicitudes a la API que consumen créditos devolverán un error **HTTP 402 (Pago requerido)**.

Con **Smart Upgrade** habilitado, tu suscripción pasa automáticamente al siguiente nivel de créditos y recibes de inmediato la asignación de créditos del nuevo nivel, por lo que las solicitudes siguen procesándose sin interrupciones. Smart Upgrade deja de aplicarse cuando alcanzas el nivel Escala.

Para reanudar el uso después de un bloqueo total, puedes:

1. Habilitar [Smart Upgrade](#smart-upgrade) para subir de nivel automáticamente
2. Actualizar manualmente a un plan superior
3. Esperar a que tus créditos se restablezcan en el siguiente ciclo de facturación

<div id="coupons">
  ## Cupones
</div>

Firecrawl admite dos tipos de cupones:

* Los **cupones de suscripción** aplican un descuento a la suscripción de tu plan (p. ej., un porcentaje de descuento en el precio mensual o anual). Estos **solo** se pueden aplicar durante el proceso de pago de Stripe, cuando te suscribes por primera vez a un plan de pago o cambias de plan. No puedes aplicar un cupón de suscripción una vez completado el pago.
* Los **cupones de créditos** añaden créditos extra a tu cuenta. Estos se pueden canjear desde la sección de **Facturación** de tu panel de control en [firecrawl.dev/app/billing](https://www.firecrawl.dev/app/billing). Busca el campo de cupón en la página de facturación para aplicar tu código. Los créditos extra de los cupones de créditos son independientes de la asignación mensual de tu plan y se conservan aunque cambies a un plan superior o inferior.

<div id="faqs">
  ## Preguntas frecuentes
</div>

<AccordionGroup>
  <Accordion title="¿Los créditos no utilizados se trasladan al mes siguiente?">
    Los **créditos del plan** no se acumulan de forma predeterminada — tu asignación mensual se restablece cada mes. **Los planes anuales escala acumulan los créditos del plan no utilizados durante 1 mes**, y **los planes Enterprise anuales los acumulan durante 2 meses**.
  </Accordion>

  <Accordion title="¿Cómo se calcula el cargo de Smart Upgrade?">
    Smart Upgrade cobra la diferencia prorrateada entre tu nivel actual y el siguiente nivel. En los planes **mensuales**, esto es simplemente la diferencia de precio mensual. En los planes **anuales**, es `(meses restantes en tu ciclo de facturación ÷ 12) × diferencia de precio anual`, usando meses calendario completos desde el inicio de tu período actual. También recibes la diferencia de créditos entre los dos niveles, por lo que solo pagas por lo que obtienes.
  </Accordion>

  <Accordion title="¿Cómo sé cuántos créditos me quedan?">
    Consulta el panel de control en [firecrawl.dev/app](https://www.firecrawl.dev/app), o invoca de forma programática el [endpoint de la API de uso de créditos](/es/api-reference/endpoint/credit-usage).
  </Accordion>

  <Accordion title="¿Dónde aplico un código de cupón?">
    Depende del tipo de cupón. Los **cupones de créditos** se aplican en la sección facturación de tu panel. Los **cupones de suscripción** (descuentos en el precio de tu plan) solo se pueden aplicar en la página de pago de Stripe al suscribirte o cambiar de plan.
  </Accordion>

  <Accordion title="Necesito más concurrencia o un plan personalizado. ¿Con quién me contacto?">
    Ponte en contacto con [help@firecrawl.dev](mailto:help@firecrawl.dev), o visita la [página Enterprise](https://www.firecrawl.dev/enterprise) para obtener más información sobre planes personalizados.
  </Accordion>

  <Accordion title="¿En qué moneda se emiten las facturas?">
    Todas las facturas de Firecrawl se emiten en **dólares estadounidenses (USD)**, independientemente de tu dirección de facturación o método de pago.
  </Accordion>

  <Accordion title="¿Cómo agrego un número de IVA, el nombre de la empresa o la dirección de facturación a mis facturas?">
    Ve a tu [configuración de facturación](https://www.firecrawl.dev/app/settings?tab=billing), haz clic en **Manage Subscriptions** y actualiza tu dirección de facturación, el nombre de la empresa y el número de IVA en el portal de Stripe. Las próximas facturas incluirán automáticamente la información actualizada.

    Para volver a generar una factura pagada anterior con la nueva información:

    1. Actualiza primero tus datos de facturación en el portal de Stripe (ver arriba).
    2. Abre la pestaña **Historial de facturas** en el portal de Stripe y descarga el PDF de la factura que quieras; Stripe la vuelve a renderizar con tu información de facturación actual.
    3. Si una factura no refleja los datos actualizados, envía un correo a [help@firecrawl.dev](mailto:help@firecrawl.dev) con los números de factura y las volveremos a generar por ti.
  </Accordion>
</AccordionGroup>

<div id="x-xcom-billing">
  ## Facturación de X (x.com)
</div>

Firecrawl usa la **API oficial de Grok** de [xAI](https://x.ai/) para ofrecer resúmenes con IA, extracción estructurada y acceso en tiempo real al contenido público de X. Las solicitudes a las URL de perfiles y publicaciones de `x.com`, `twitter.com` y `mobile.twitter.com` se procesan mediante las herramientas internas autorizadas de Grok (`x_search`, recuperación de hilos y búsqueda web restringida a x.com), en lugar del web scraping tradicional.

<div id="credit-costs">
  ### Costos de créditos
</div>

| Componente       | Costo en créditos        | Descripción                                                     |
| ---------------- | ------------------------ | --------------------------------------------------------------- |
| **Costo base**   | 1 crédito / solicitud    | Procesamiento estándar de solicitudes de scraping               |
| **Grok X Query** | +29 créditos / solicitud | Uso de la API de Grok (tokens + tool calls) para contenido de X |

Por ejemplo, procesar una solicitud típica de publicación o hilo cuesta **30 créditos** (`1` base + `29` Grok X Query) y devuelve datos estructurados generados por Grok, contexto del hilo y resúmenes. Si también está habilitado el formato JSON (extracción con LLM), el total es de **34 créditos** por solicitud.

Este método cumple con las interfaces publicadas de X a través de la asociación con xAI y proporciona una salida razonada y de mayor calidad, en lugar de hacer scraping directo de la página.

<Note>
  **Las capacidades difieren del scraping estándar.** Grok devuelve resultados procesados por IA, que pueden incluir resúmenes, métricas clave, contexto del hilo y más. Para obtener datos estructurados sin procesar a escala, usa la [API oficial X Enterprise](https://developer.x.com/).
</Note>
