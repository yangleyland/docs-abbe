Source de référence curl officielle de Firecrawl pour les agents. Alignée sur `api-reference/v2-openapi.json` (URL de base du serveur `https://api.firecrawl.dev/v2`).

<div id="authenticate">
  ## Authentification
</div>

```bash
export FIRECRAWL_API_KEY="fc-your-api-key"
```

<div id="when-to-use-what">
  ## Quand utiliser quoi
</div>

* `search` : à utiliser si vous partez d’une requête et avez besoin de découverte.
* `scrape` : à utiliser si vous avez déjà une URL et souhaitez récupérer le contenu d’une page.
* `interact` : à utiliser si la page nécessite des clics, des formulaires ou des actions dans le navigateur après le scrape.
* `support/ask` : à utiliser lorsqu’un appel à l’API Firecrawl échoue ou renvoie des résultats inattendus et que vous avez besoin d’un diagnostic.
* `support/docs-search` : à utiliser lorsque vous devez rechercher dans la documentation de Firecrawl.

<div id="search">
  ## Search
</div>

<div id="why-use-it">
  ### Pourquoi l&#39;utiliser
</div>

Utilisez la recherche pour découvrir des pages pertinentes à partir d&#39;une requête, puis choisissez des URLs à scraper ou avec lesquelles interagir. Vous pouvez restreindre les résultats à un site avec `site:`, par exemple `site:docs.firecrawl.dev crawl webhooks`.

<div id="endpoint">
  ### Point de terminaison
</div>

`POST /search`

<div id="simple-example">
  ### Exemple simple
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/search" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "site:docs.firecrawl.dev webhook retries"
  }'
```

<div id="complex-example">
  ### Exemple complexe
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/search" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "site:docs.firecrawl.dev crawl webhooks",
    "sources": [{"type": "web"}, {"type": "news"}],
    "categories": [{"type": "research"}],
    "limit": 10,
    "tbs": "qdr:m",
    "location": "San Francisco,California,United States",
    "country": "US",
    "ignoreInvalidURLs": true,
    "timeout": 60000,
    "enterprise": ["anon"],
    "scrapeOptions": {
      "formats": [
        "markdown",
        "links",
        {"type": "json", "prompt": "Extract title and key endpoints."}
      ],
      "onlyMainContent": true,
      "includeTags": ["main", "article"],
      "excludeTags": ["nav", "footer"],
      "waitFor": 1000
    }
  }'
```

<div id="response">
  ### Réponse
</div>

Les réponses en cas de succès incluent `success`, `data`, éventuellement `warning`, `id` et `creditsUsed`.

* `data.web`, `data.images`, `data.news` : tableaux de résultats ; les clés présentes dépendent de `sources` (par défaut, seul `data.web` est renseigné).
* Les éléments web et d’actualité incluent des champs tels que `title`, `url` et (lorsque `scrapeOptions` / les formats le demandent) `markdown`, `html`, `rawHtml`, `links`, `screenshot`, `audio`, `video` et `metadata`.
* Les éléments d’image incluent des champs tels que `imageUrl`, `url` et les dimensions lorsqu’elles sont disponibles.
* `warning` : message facultatif lisible par un humain.
* `id` : chaîne contenant l’ID de tâche de recherche.
* `creditsUsed` : nombre entier de crédits facturés pour l’appel.

<div id="parameters">
  ### Paramètres
</div>

* `query`
  * Type : string (obligatoire, longueur maximale : 500)
  * À utiliser lorsque : vous avez besoin d’une requête de recherche.
  * Remarques : utilisez `site:example.com` pour limiter les résultats à un domaine.

* `sources`
  * Type : tableau d’objets source typés
  * À utiliser lorsque : vous voulez contrôler les sources dans lesquelles effectuer la recherche.
  * Formes d’objet confirmées :
    * `{ "type": "web" }` avec `tbs` et `location` facultatifs pour chaque source
    * `{ "type": "news" }`
    * `{ "type": "images" }`

* `categories`
  * Type : tableau d’objets de catégorie typés
  * À utiliser lorsque : vous voulez filtrer les résultats par catégorie.
  * Formes d’objet confirmées :
    * `{ "type": "github" }`
    * `{ "type": "research" }`
    * `{ "type": "pdf" }`

* `limit`
  * Type : entier (minimum : 1, maximum : 100, valeur par défaut : 5)
  * À utiliser lorsque : vous voulez limiter le nombre de résultats.

* `tbs`
  * Type : string
  * À utiliser lorsque : vous avez besoin d’un filtre temporel (par exemple `qdr:d`, `qdr:w`, `sbd:1,qdr:m`).

* `location`
  * Type : string
  * À utiliser lorsque : vous voulez des résultats localisés.

* `country`
  * Type : string (valeur par défaut : `"US"`)
  * À utiliser lorsque : vous voulez un ciblage ISO 3166-1 alpha-2 (par exemple `"US"`).

* `ignoreInvalidURLs`
  * Type : booléen (valeur par défaut : false)
  * À utiliser lorsque : vous voulez ignorer les URL qui ne peuvent pas être extraites par d’autres points de terminaison.

* `timeout`
  * Type : entier (en millisecondes, valeur par défaut : 60000)
  * À utiliser lorsque : vous avez besoin d’un délai d’expiration de requête en millisecondes.

* `enterprise`
  * Type : tableau de strings (`"anon"` ou `"zdr"` par élément)
  * À utiliser lorsque : vous avez besoin de contrôles de recherche Enterprise.
  * Valeurs :
    * `"zdr"`: zéro conservation des données de bout en bout
    * `"anon"`: zéro conservation des données avec anonymisation

* `scrapeOptions`
  * Type : objet
  * À utiliser lorsque : vous voulez extraire chaque résultat de recherche (voir les paramètres de Scrape pour les champs).

<div id="scrape">
  ## Scrape
</div>

### Pourquoi l’utiliser

Utilisez scrape lorsque vous avez déjà une URL et souhaitez du contenu structuré dans un ou plusieurs formats.

<div id="endpoint">
  ### Point de terminaison
</div>

`POST /scrape`

<div id="simple-example">
  ### Exemple simple
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://docs.firecrawl.dev",
    "formats": ["markdown"]
  }'
```

<div id="complex-example">
  ### Exemple complexe
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/pricing",
    "formats": [
      "markdown",
      "links",
      {"type": "json", "prompt": "Extract plan names and prices."},
      {"type": "screenshot", "fullPage": true, "quality": 80, "viewport": {"width": 1280, "height": 720}},
      {"type": "changeTracking", "modes": ["git-diff"], "tag": "pricing"}
    ],
    "headers": {"User-Agent": "FirecrawlDocsBot/1.0"},
    "onlyMainContent": true,
    "waitFor": 1000,
    "parsers": [{"type": "pdf", "mode": "auto", "maxPages": 5}],
    "actions": [
      {"type": "click", "selector": "#accept"},
      {"type": "wait", "milliseconds": 750},
      {"type": "scrape"}
    ],
    "location": {"country": "US", "languages": ["en-US"]},
    "removeBase64Images": true,
    "blockAds": true,
    "proxy": "auto",
    "maxAge": 86400000,
    "minAge": 1,
    "storeInCache": true,
    "profile": {"name": "docs-session", "saveChanges": true},
    "zeroDataRetention": false
  }'
```

<div id="response">
  ### Réponse
</div>

Les réponses réussies incluent `success` et `data`. Champs `data` courants (selon les `formats` et les options) :

* `markdown`, `summary`, `html`, `rawHtml`, `screenshot`, `audio`, `video`, `links`
* `actions` : lorsque la requête incluait des `actions` lors du scrape, contient les résultats dans l&#39;ordre, tels que `screenshots`, `scrapes`, `javascriptReturns` et `pdfs`
* `metadata` : métadonnées de la page (`title`, `sourceURL`, `url`, `statusCode`, `error` et d&#39;autres champs extraits)
* `warning` : avertissement facultatif concernant l&#39;extraction ou le formatage
* `changeTracking` : présent lorsque le format `changeTracking` est demandé

<div id="parameters">
  ### Paramètres
</div>

* `url`
  * Type : string
  * À utiliser lorsque : vous souhaitez scraper une page spécifique.

* `formats`
  * Type : tableau de chaînes de format ou d&#39;objets de format
  * À utiliser lorsque : vous souhaitez plusieurs formats de sortie.
  * Chaînes de format confirmées :
    * `"markdown"`: contenu markdown
    * `"html"`: HTML nettoyé
    * `"rawHtml"`: HTML brut
    * `"links"`: liens de la page
    * `"images"`: URL d&#39;image
    * `"screenshot"`: sortie screenshot
    * `"summary"`: sortie de résumé
    * `"changeTracking"`: sortie de suivi des modifications
    * `"json"`: extraction JSON
    * `"branding"`: sortie du profil de branding
    * `"audio"`: extraction audio
    * `"video"`: extraction vidéo
  * Champs de l&#39;objet format :
    * `type`: l&#39;une des chaînes de format ci-dessus
    * `prompt`, `schema`: options d&#39;extraction JSON pour `type: "json"`
    * `modes`, `schema`, `prompt`, `tag`: options de suivi des modifications pour `type: "changeTracking"`
    * `fullPage`, `quality`, `viewport`: options de screenshot pour `type: "screenshot"`

* `headers`
  * Type : object
  * À utiliser lorsque : vous avez besoin d&#39;en-têtes de requête personnalisés.

* `includeTags`
  * Type : tableau de chaînes
  * À utiliser lorsque : vous souhaitez inclure uniquement certaines balises HTML.

* `excludeTags`
  * Type : tableau de chaînes
  * À utiliser lorsque : vous souhaitez exclure certaines balises HTML.

* `onlyMainContent`
  * Type : boolean
  * À utiliser lorsque : vous souhaitez supprimer la navigation, le pied de page et les autres éléments répétitifs.

* `timeout`
  * Type : number
  * À utiliser lorsque : vous avez besoin d&#39;un délai d&#39;expiration en millisecondes.

* `waitFor`
  * Type : number
  * À utiliser lorsque : vous devez attendre le rendu de la page (en millisecondes).

* `mobile`
  * Type : boolean
  * À utiliser lorsque : vous souhaitez un viewport mobile.

* `parsers`
  * Type : tableau d&#39;objets
  * À utiliser lorsque : vous avez besoin de contrôler l&#39;analyse des fichiers.
  * Structure confirmée : `{ "type": "pdf", "mode": "fast" | "auto" | "ocr", "maxPages": number }` (`type` requis ; autres champs facultatifs avec valeurs par défaut selon la spécification)

* `actions`
  * Type : tableau d&#39;objets d&#39;action
  * À utiliser lorsque : vous avez besoin d&#39;actions légères avant le scraping.
  * Types d&#39;action confirmés :
    * `wait`: `milliseconds` ou `selector` requis
    * `screenshot`: `fullPage`, `quality`, `viewport` facultatifs
    * `click`: `selector` requis, `all` facultatif
    * `write`: `text` requis (cliquez d&#39;abord pour donner le focus au champ de saisie)
    * `press`: `key` requis
    * `scroll`: `type` requis ; `direction` (`up` ou `down`, valeur par défaut `down`) ; `selector` facultatif
    * `scrape`: aucun champ supplémentaire
    * `executeJavascript`: `script` requis
    * `pdf`: `format` (A0, A1, A2, A3, A4, A5, A6, Letter, Legal, Tabloid, Ledger), `landscape`, `scale` facultatifs

* `location`
  * Type : objet avec `country` et `languages`
  * À utiliser lorsque : vous avez besoin d&#39;un scraping tenant compte de la géolocalisation ou de la langue.

* `skipTlsVerification`
  * Type : boolean
  * À utiliser lorsque : vous devez ignorer la vérification TLS.

* `removeBase64Images`
  * Type : boolean
  * À utiliser lorsque : vous souhaitez supprimer les images en base64 de la sortie markdown.

* `blockAds`
  * Type : boolean
  * À utiliser lorsque : vous souhaitez bloquer les publicités et les pop-ups de cookies.

* `proxy`
  * Type : string
  * À utiliser lorsque : vous avez besoin de contrôler le proxy.
  * Valeurs confirmées : `"basic"`, `"enhanced"`, `"auto"`

* `maxAge`
  * Type : number
  * À utiliser lorsque : vous souhaitez des données en cache jusqu&#39;à un âge maximal (en millisecondes).

* `minAge`
  * Type : number
  * À utiliser lorsque : vous souhaitez des données en cache uniquement si elles ont au moins cet âge (en millisecondes).

* `storeInCache`
  * Type : boolean
  * À utiliser lorsque : vous souhaitez que Firecrawl mette le résultat en cache.

* `profile`
  * Type : objet avec `name` et `saveChanges` facultatif
  * À utiliser lorsque : vous souhaitez un profil de navigateur persistant partagé entre les scrapes et les interactions.

* `zeroDataRetention`
  * Type : boolean
  * À utiliser lorsque : vous souhaitez une rétention nulle des données pour ce scrape.

<div id="interact">
  ## Interact
</div>

### Pourquoi l’utiliser

Utilisez Interact lorsqu’une page nécessite des actions dans le navigateur ou l’exécution de code après le lancement d’un scrape.

<div id="endpoint">
  ### Point de terminaison
</div>

`POST /scrape/{jobId}/interact`

<div id="simple-example">
  ### Exemple simple
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape/<jobId>/interact" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "code": "console.log(await page.title());"
  }'
```

<div id="complex-example">
  ### Exemple complexe
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape/<jobId>/interact" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "code": "const title = await page.title();\nconst h1 = await page.locator(\"h1\").first().textContent().catch(() => null);\nconsole.log(JSON.stringify({ title, h1 }));",
    "language": "node",
    "timeout": 120
  }'
```

<div id="parameters">
  ### Paramètres
</div>

* `jobId` (chemin)
  * Type : string (UUID)
  * À utiliser si : vous disposez de l’ID de la tâche de scraping pour la session de navigateur active.

* `code` (corps JSON)
  * Type : string (obligatoire dans OpenAPI ; longueur min. 1, longueur max. 100000)
  * À utiliser si : vous souhaitez exécuter du code dans le sandbox de navigateur lié au scraping.

* `language` (corps JSON)
  * Type : string
  * À utiliser si : vous avez besoin d’un environnement d’exécution spécifique.
  * Valeurs confirmées : `"python"`, `"node"`, `"bash"` (par défaut `"node"`)

* `timeout` (corps JSON)
  * Type : integer (secondes ; minimum 1, maximum 300, par défaut 30)
  * À utiliser si : vous avez besoin d’un délai maximal d’exécution.

<div id="response">
  ### Réponse
</div>

Les réponses en cas de succès incluent `success` ainsi que des champs d’exécution comme `stdout`, `result` (alias de `stdout`), `stderr`, `exitCode`, `killed` et `error` (nullable).

### DELETE /scrape/{jobId}/interact

<div id="example">
  ### Exemple
</div>

```bash
curl -X DELETE "https://api.firecrawl.dev/v2/scrape/<jobId>/interact" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY"
```

Pas de corps JSON. Une réponse réussie inclut `success`.

<div id="ask-agentic-debugging">
  ## Ask (débogage agentique)
</div>

### Pourquoi l’utiliser

Utilisez Ask lorsqu’un appel à l’API Firecrawl échoue ou renvoie des résultats inattendus. L’agent d’assistance IA diagnostique le problème, propose des paramètres de correction et peut, si nécessaire, valider la correction sur l’API réelle. Latence typique : 15 à 30 secondes.

<div id="endpoint">
  ### Point de terminaison
</div>

`POST /support/ask`

<div id="simple-example">
  ### Exemple simple
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/support/ask" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "my scrape returned empty markdown for https://example.com"
  }'
```

<div id="complex-example">
  ### Exemple complexe
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/support/ask" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "crawl returned 3 pages but I expected 50",
    "rationale": "user is on their third failed crawl attempt today",
    "context": {"userPlan": "standard", "retryCount": 3}
  }'
```

<div id="response">
  ### Réponse
</div>

Les réponses en cas de succès incluent `requestId`, `answer`, `confidence`, `fixParameters`, `validation`, `usage` et `durationMs`.

* `answer` : un texte rédigé en 2 à 4 phrases présentant le diagnostic et la correction.
* `confidence` : `high`, `medium` ou `low`.
* `fixParameters` : des paramètres d’API exploitables par une machine pour appliquer la correction (`null` si aucune correction ne s’applique).
* `validation.tested` : indique si l’agent a testé la correction sur l’API en production.
* `validation.result` : `success`, `failure` ou `skipped`.
* `feedback` : présent lorsque l’agent est bloqué ; `null` en cas de succès.

<div id="parameters">
  ### Paramètres
</div>

* `question`
  * Type : string (obligatoire, 1-8000 caractères)
  * À utiliser lorsque : vous devez décrire le problème.

* `rationale`
  * Type : string (1-2000 caractères)
  * À utiliser lorsque : vous êtes un agent IA agissant au nom d&#39;un utilisateur. Décrivez ce que l&#39;utilisateur cherche à accomplir.

* `context`
  * Type : object (forme libre)
  * À utiliser lorsque : vous souhaitez transmettre des métadonnées depuis votre agent vers le prompt de débogage.

<div id="docs-search">
  ## Recherche dans la documentation
</div>

### Pourquoi l’utiliser

Utilisez docs-search pour rechercher dans la documentation de Firecrawl.

<div id="endpoint">
  ### Point de terminaison
</div>

`POST /support/docs-search`

<div id="example">
  ### Exemple
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/support/docs-search" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "how do I verify webhook signatures?"
  }'
```

<div id="parameters">
  ### Paramètres
</div>

* `question`
  * Type : string (obligatoire, 1-8000 caractères)
  * À utiliser lorsque : vous avez besoin d&#39;une réponse basée sur la documentation.

<div id="notes">
  ## Remarques
</div>

* Les lignes de résultats de recherche sont imbriquées dans `data.web`, `data.images` ou `data.news`, et non dans un tableau `data` à plat.
* Utilisez `POST /scrape/{jobId}/interact` pour les workflows en plusieurs étapes dans le navigateur ; les `actions` de scrape conviennent mieux aux petites étapes préalables au scrape.
* Le schéma OpenAPI publié exige `code` pour `interact`. Certains SDK et clients acceptent également un champ `prompt` pour des instructions en langage naturel ; cet alias n’apparaît pas dans `v2-openapi.json`.
