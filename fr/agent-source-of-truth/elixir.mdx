Référence canonique Elixir de Firecrawl pour les agents. Générée à partir du code source du SDK et de la spécification OpenAPI v2.

<div id="install">
  ## Installation
</div>

Ajoutez à `mix.exs` :

```elixir
{:firecrawl, "~> 1.0.0"}
```

<div id="authenticate">
  ## Authentification
</div>

```elixir
# config/runtime.exs ou config.exs
config :firecrawl, api_key: System.get_env("FIRECRAWL_API_KEY")

# ou transmettez api_key à chaque appel
{:ok, res} = Firecrawl.search_and_scrape([query: "site:docs.firecrawl.dev webhook retries"], api_key: "fc-your-api-key")
```

<div id="when-to-use-what">
  ## Quand utiliser quoi
</div>

* `search` : à utiliser lorsque vous partez d’une requête et devez découvrir des résultats.
* `scrape` : à utiliser lorsque vous avez déjà une URL et voulez le contenu de la page.
* `interact` : à utiliser lorsque la page nécessite des clics, des formulaires ou des actions Browser après le scrape.

<div id="search">
  ## Recherche
</div>

<div id="why-use-it">
  ### Pourquoi l’utiliser
</div>

Utilisez la recherche pour découvrir des pages pertinentes à partir d’une requête, puis choisissez des URL à scraper ou avec lesquelles interagir. Vous pouvez limiter les résultats à un site avec `site:`, par exemple `site:docs.firecrawl.dev crawl webhooks`.

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

`Firecrawl.search_and_scrape(params \\ [], opts \\ [])`

<div id="simple-example">
  ### Exemple simple
</div>

```elixir
{:ok, res} = Firecrawl.search_and_scrape(query: "site:docs.firecrawl.dev webhook retries")
```

<div id="complex-example">
  ### Exemple complexe
</div>

```elixir
{:ok, res} = Firecrawl.search_and_scrape(
  query: "site:docs.firecrawl.dev crawl webhooks",
  sources: [:web, :news],
  categories: [:research],
  limit: 10,
  tbs: "qdr:m",
  location: "San Francisco,California,United States",
  country: "US",
  ignore_invalid_urls: true,
  timeout: 60000,
  scrape_options: [
    formats: [
      "markdown",
      "links",
      %{type: "json", prompt: "Extract title and key endpoints."}
    ],
    only_main_content: true,
    include_tags: ["main", "article"],
    exclude_tags: ["nav", "footer"],
    wait_for: 1000
  ]
)
```

<div id="parameters">
  ### Paramètres
</div>

* `query`
  * Type : chaîne
  * À utiliser lorsque : vous avez besoin d&#39;une requête de recherche.
  * Remarques : utilisez `site:example.com` pour limiter les résultats à un domaine.

* `sources`
  * Type : liste d&#39;atomes, de chaînes ou de maps
  * À utiliser lorsque : vous souhaitez contrôler les sources dans lesquelles effectuer la recherche.
  * Valeurs confirmées :
    * `"web"` ou `:web`
    * `"news"` ou `:news`
    * `"images"` ou `:images`
    * `%{type: "web" | "news" | "images"}`

* `categories`
  * Type : liste d&#39;atomes, de chaînes ou de maps
  * À utiliser lorsque : vous souhaitez filtrer les résultats par catégorie.
  * Valeurs confirmées :
    * `"github"` ou `:github`
    * `"research"` ou `:research`
    * `"pdf"` ou `:pdf`
    * `%{type: "github" | "research" | "pdf"}`

* `limit`
  * Type : entier
  * À utiliser lorsque : vous souhaitez limiter le nombre de résultats.

* `tbs`
  * Type : chaîne
  * À utiliser lorsque : vous avez besoin d&#39;un filtre temporel (par exemple `qdr:d`, `qdr:w`, `sbd:1,qdr:m`).

* `location`
  * Type : chaîne
  * À utiliser lorsque : vous souhaitez obtenir des résultats localisés.

* `country`
  * Type : chaîne
  * À utiliser lorsque : vous souhaitez un ciblage ISO 3166-1 alpha-2 (par exemple `"US"`).

* `ignore_invalid_urls`
  * Type : booléen
  * À utiliser lorsque : vous souhaitez ignorer les URL qui ne peuvent pas être scrapées par d&#39;autres points de terminaison.

* `timeout`
  * Type : entier
  * À utiliser lorsque : vous avez besoin d&#39;un délai d&#39;expiration de requête en millisecondes.

* `enterprise`
  * Type : liste de chaînes
  * À utiliser lorsque : vous avez besoin de contrôles de recherche Enterprise.
  * Valeurs confirmées :
    * `"zdr"`: rétention nulle des données de bout en bout
    * `"anon"`: rétention nulle des données anonymisée

* `scrape_options`
  * Type : liste de mots-clés
  * À utiliser lorsque : vous souhaitez scraper chaque résultat de recherche (voir les paramètres de Scrape pour les champs).

<div id="scrape">
  ## Scrape
</div>

<div id="why-use-it">
  ### Pourquoi l’utiliser
</div>

Utilisez `scrape` lorsque vous avez déjà une URL et que vous souhaitez du contenu structuré dans un ou plusieurs formats.

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

`Firecrawl.scrape_and_extract_from_url(params \\ [], opts \\ [])`

<div id="simple-example">
  ### Exemple simple
</div>

```elixir
{:ok, res} = Firecrawl.scrape_and_extract_from_url(
  url: "https://docs.firecrawl.dev",
  formats: ["markdown"]
)
```

<div id="complex-example">
  ### Exemple complexe
</div>

```elixir
{:ok, res} = Firecrawl.scrape_and_extract_from_url(
  url: "https://example.com/pricing",
  formats: [
    "markdown",
    "links",
    %{type: "json", prompt: "Extract plan names and prices."},
    %{type: "screenshot", fullPage: true, quality: 80, viewport: %{width: 1280, height: 720}},
    %{type: "changeTracking", modes: ["git-diff"], tag: "pricing"}
  ],
  headers: %{"User-Agent" => "FirecrawlDocsBot/1.0"},
  only_main_content: true,
  wait_for: 1000,
  parsers: [%{type: "pdf", mode: "auto", maxPages: 5}],
  actions: [
    %{type: "click", selector: "#accept"},
    %{type: "wait", milliseconds: 750},
    %{type: "scrape"}
  ],
  location: [country: "US", languages: ["en-US"]],
  remove_base64_images: true,
  block_ads: true,
  proxy: :auto,
  max_age: 86400000,
  min_age: 1,
  store_in_cache: true,
  profile: [name: "docs-session", save_changes: true],
  zero_data_retention: false
)
```

<div id="parameters">
  ### Paramètres
</div>

* `url`
  * Type : string
  * À utiliser si : vous voulez scraper une page précise.

* `formats`
  * Type : liste de chaînes de format ou de maps de format
  * À utiliser si : vous voulez plusieurs formats de sortie.
  * Chaînes de format confirmées :
    * `"markdown"`: contenu markdown
    * `"html"`: HTML nettoyé
    * `"rawHtml"`: HTML brut
    * `"links"`: liens de la page
    * `"images"`: URL d’images
    * `"screenshot"`: sortie screenshot
    * `"summary"`: sortie de résumé
    * `"changeTracking"`: sortie de suivi des modifications
    * `"json"`: extraction JSON
    * `"branding"`: sortie du profil de branding
    * `"audio"`: extraction audio
    * `"video"`: extraction vidéo
  * Champs de la map de format :
    * `type`: l’une des chaînes de format ci-dessus
    * `prompt`, `schema`: options d’extraction JSON pour `type: "json"`
    * `modes`, `schema`, `prompt`, `tag`: options de suivi des modifications pour `type: "changeTracking"`
    * `fullPage`, `quality`, `viewport`: options de screenshot pour `type: "screenshot"`

* `headers`
  * Type : map
  * À utiliser si : vous avez besoin d’en-têtes de requête personnalisés.

* `include_tags`
  * Type : liste de strings
  * À utiliser si : vous voulez inclure uniquement certaines balises HTML.

* `exclude_tags`
  * Type : liste de strings
  * À utiliser si : vous voulez exclure certaines balises HTML.

* `only_main_content`
  * Type : boolean
  * À utiliser si : vous voulez supprimer la navigation, le pied de page et les autres éléments parasites.

* `timeout`
  * Type : integer
  * À utiliser si : vous avez besoin d’un délai d’expiration en millisecondes.

* `wait_for`
  * Type : integer
  * À utiliser si : vous devez attendre le rendu de la page (en millisecondes).

* `mobile`
  * Type : boolean
  * À utiliser si : vous voulez un viewport mobile.

* `parsers`
  * Type : liste de chaînes de parseur ou de maps
  * À utiliser si : vous avez besoin de contrôler l’analyse des fichiers.
  * Valeurs confirmées :
    * `"pdf"`
    * `%{type: "pdf", mode: "fast" | "auto" | "ocr", maxPages: integer}`

* `actions`
  * Type : liste de maps d’action
  * À utiliser si : vous avez besoin d’actions légères avant le scrape.
  * Types d’action confirmés :
    * `wait`: `milliseconds` ou `selector` requis
    * `screenshot`: `fullPage`, `quality`, `viewport` facultatifs
    * `click`: `selector` requis, `all` facultatif
    * `write`: `text` requis (cliquez d’abord pour donner le focus au champ de saisie)
    * `press`: `key` requis
    * `scroll`: `direction` (`up` ou `down`) requis, `selector` facultatif
    * `scrape`: aucun champ supplémentaire
    * `executeJavascript`: `script` requis
    * `pdf`: `format` (A0, A1, A2, A3, A4, A5, A6, Letter, Legal, Tabloid, Ledger), `landscape`, `scale` facultatifs

* `location`
  * Type : liste de mots-clés (par exemple `country:` et `languages:`)
  * À utiliser si : vous avez besoin d’un scrape tenant compte de la zone géographique ou de la langue.

* `skip_tls_verification`
  * Type : boolean
  * À utiliser si : vous devez ignorer la vérification TLS.

* `remove_base64_images`
  * Type : boolean
  * À utiliser si : vous voulez supprimer les images base64 de la sortie markdown.

* `block_ads`
  * Type : boolean
  * À utiliser si : vous voulez bloquer les publicités et les fenêtres contextuelles de cookies.

* `proxy`
  * Type : atom ou string
  * À utiliser si : vous avez besoin de contrôler le proxy.
  * Valeurs confirmées : `:basic`, `:enhanced`, `:auto`

* `max_age`
  * Type : integer
  * À utiliser si : vous voulez des données en cache jusqu’à un âge maximal (en millisecondes).

* `min_age`
  * Type : integer
  * À utiliser si : vous voulez des données en cache uniquement si elles ont au moins cet âge (en millisecondes).

* `store_in_cache`
  * Type : boolean
  * À utiliser si : vous voulez que Firecrawl mette le résultat en cache.

* `profile`
  * Type : liste de mots-clés avec `name:` et `save_changes:` facultatif (ou `saveChanges:`)
  * À utiliser si : vous voulez un profil Browser persistant partagé entre les scrapes et les interactions.

* `zero_data_retention`
  * Type : boolean
  * À utiliser si : vous voulez une conservation nulle des données pour ce scrape.

<div id="interact">
  ## Interact
</div>

<div id="why-use-it">
  ### Pourquoi l’utiliser
</div>

Utilisez Interact lorsqu’une page nécessite des actions dans le navigateur ou l’exécution de code une fois le scrape lancé.

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

`Firecrawl.interact_with_scrape_browser_session(job_id, params \\ [], opts \\ [])`

<div id="simple-example">
  ### Exemple simple
</div>

```elixir
{:ok, res} = Firecrawl.interact_with_scrape_browser_session(
  "<scrapeJobId>",
  code: "console.log(await page.title());",
  language: :node,
  timeout: 60
)
```

<div id="complex-example">
  ### Exemple complexe
</div>

```elixir
{:ok, res} = Firecrawl.interact_with_scrape_browser_session(
  "<scrapeJobId>",
  code: "// Utilisez les méthodes de page Playwright ici",
  language: :node,
  timeout: 120
)
```

<div id="parameters">
  ### Paramètres
</div>

* `job_id`
  * Type : string
  * À utiliser si vous disposez d’un ID de tâche de scraping.

* `code`
  * Type : string
  * À utiliser si vous souhaitez exécuter du code dans la session de navigateur.

* `language`
  * Type : atom ou string
  * À utiliser si vous avez besoin d’un environnement d’exécution spécifique.
  * Valeurs prises en charge : `:python`, `:node`, `:bash` (ou leurs équivalents sous forme de string)

* `timeout`
  * Type : integer
  * À utiliser si vous avez besoin d’un délai d’exécution maximal en secondes.

* `origin`
  * Type : string
  * À utiliser si vous souhaitez un libellé d’origine facultatif pour la télémétrie d’exécution.

<div id="stop-interactive-session">
  ### Arrêter la session interactive
</div>

`Firecrawl.stop_interactive_scrape_browser_session(job_id, opts \\ [])` envoie `DELETE /scrape/{jobId}/interact` et met fin à la session de navigateur associée à cette tâche de scraping. Une variante bang `stop_interactive_scrape_browser_session!/2` est également disponible.

```elixir
{:ok, res} = Firecrawl.stop_interactive_scrape_browser_session("<scrapeJobId>")
```

<div id="notes">
  ## Remarques
</div>

* Le client Elixir suit la structure d’OpenAPI ; les noms de fonctions et les clés de paramètres sont générés à partir de la spécification.
* Chaque fonction publique a une variante bang (`!`) qui lève une exception en cas d’erreur au lieu de renvoyer `{:error, _}`.
* Ce SDK n’expose que des interactions par code (pas de paramètre `prompt` sur `interact_with_scrape_browser_session`).
