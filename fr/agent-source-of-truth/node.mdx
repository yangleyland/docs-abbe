Source de référence canonique de Firecrawl pour Node.js destinée aux agents. Alignée sur `firecrawl` **v4.18.2** (`firecrawl/apps/js-sdk/firecrawl`) et la spécification OpenAPI v2. Les noms de méthode, les paramètres et les types correspondent à l&#39;API publique du SDK.

<div id="install">
  ## Installation
</div>

```bash
npm install firecrawl
```

<div id="authenticate">
  ## Authentification
</div>

```ts
import { Firecrawl } from "firecrawl";

const client = new Firecrawl({
  apiKey: process.env.FIRECRAWL_API_KEY,
  // apiUrl: "https://api.firecrawl.dev" // optionnel ; utilise FIRECRAWL_API_URL ou le cloud par défaut
});
```

<div id="when-to-use-what">
  ## Quand utiliser quoi
</div>

* `search` : à utiliser lorsque vous partez d’une requête et devez découvrir des résultats.
* `scrape` : à utiliser lorsque vous avez déjà une URL et souhaitez récupérer le contenu de la page.
* `interact` : à utiliser lorsque la page nécessite des clics, des formulaires ou d’autres actions du navigateur une fois qu’un scraping a créé une session. Pour les flux interactifs en plusieurs étapes, privilégiez `interact` plutôt que les `actions` au moment du scraping.

- `support/ask` : à utiliser lorsqu’un appel à l’API Firecrawl échoue ou renvoie des résultats inattendus et que vous avez besoin d’un diagnostic.
- `support/docs-search` : à utiliser lorsque vous devez consulter la documentation de Firecrawl.

<div id="search">
  ## Search
</div>

<div id="why-use-it">
  ### Pourquoi l&#39;utiliser
</div>

Utilisez la recherche pour découvrir des pages pertinentes à partir d&#39;une requête, puis choisissez des URLs à scraper ou avec lesquelles interagir. Vous pouvez restreindre les résultats à un site avec `site:`, par exemple `site:docs.firecrawl.dev crawl webhooks`.

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

`client.search(query, options?)` → `Promise<SearchData>`

<div id="simple-example">
  ### Exemple simple
</div>

```ts
const results = await client.search("site:docs.firecrawl.dev webhook retries");
```

<div id="complex-example">
  ### Exemple complexe
</div>

```ts
const results = await client.search("site:docs.firecrawl.dev crawl webhooks", {
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:m",
  location: "San Francisco,California,United States",
  ignoreInvalidURLs: true,
  timeout: 60000,
  scrapeOptions: {
    formats: [
      "markdown",
      "links",
      { type: "json", prompt: "Extract title and key endpoints." }
    ],
    onlyMainContent: true,
    includeTags: ["main", "article"],
    excludeTags: ["nav", "footer"],
    waitFor: 1000
  }
});
```

<div id="return-value">
  ### Valeur de retour
</div>

`SearchData` est un objet avec des tableaux optionnels (chaque entrée est soit un objet de résultat léger, soit un `Document` complet lorsque `scrapeOptions` a hydraté le résultat) :

* `web` : résultats de l’index web
* `news` : résultats d’actualité
* `images` : résultats d’images

**Erreur à éviter :** `search()` ne renvoie pas `{ data: [...] }`. N’accédez pas à `result.data`. Les résultats web se trouvent dans `result.web`, les actualités dans `result.news` et les images dans `result.images`.

<div id="parameters">
  ### Paramètres
</div>

* `query`
  * Type : string
  * À utiliser lorsque : vous avez besoin d’une requête de recherche.
  * Remarques : utilisez `site:example.com` pour limiter les résultats à un domaine.

* `options` (facultatif ; le deuxième argument prend par défaut la valeur `{}`)
  * Type : `Omit<SearchRequest, "query">`

* `options.sources`
  * Type : tableau de noms de sources ou d’objets source typés
  * À utiliser lorsque : vous souhaitez contrôler les sources à interroger.
  * Valeurs confirmées :
    * `"web"`: résultats de l’index web
    * `"news"`: résultats d’actualités
    * `"images"`: résultats d’images
    * `{ type: "web" | "news" | "images" }`: forme d’objet source typé

* `options.categories`
  * Type : tableau de noms de catégories ou d’objets catégorie typés
  * À utiliser lorsque : vous souhaitez filtrer les résultats par catégorie.
  * Valeurs confirmées :
    * `"github"`: résultats axés sur GitHub
    * `"research"`: résultats de recherche et universitaires
    * `"pdf"`: résultats axés sur les PDF
    * `{ type: "github" | "research" | "pdf" }`: forme d’objet catégorie typé

* `options.limit`
  * Type : number
  * À utiliser lorsque : vous souhaitez limiter le nombre de résultats.

* `options.tbs`
  * Type : string
  * À utiliser lorsque : vous avez besoin d’un filtre temporel (par exemple `qdr:d`, `qdr:w`, `sbd:1,qdr:m`).

* `options.location`
  * Type : string
  * À utiliser lorsque : vous souhaitez obtenir des résultats localisés.

* `options.ignoreInvalidURLs`
  * Type : boolean
  * À utiliser lorsque : vous souhaitez ignorer les URL qui ne peuvent pas être scrapées par d’autres points de terminaison.

* `options.timeout`
  * Type : number
  * À utiliser lorsque : vous avez besoin d’un délai d’expiration de requête en millisecondes.

* `options.scrapeOptions`
  * Type : `ScrapeOptions`
  * À utiliser lorsque : vous souhaitez scraper chaque résultat de recherche (voir les paramètres de Scrape pour les champs). Le SDK exécute la même validation que pour `scrape` (par exemple, la chaîne simple `"json"` dans `formats` est rejetée).

<div id="scrape">
  ## Scrape
</div>

### Pourquoi l’utiliser

Utilisez scrape lorsque vous avez déjà une URL et souhaitez un contenu structuré dans un ou plusieurs formats.

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

`client.scrape(url, options?)` → `Promise<Document>`

<div id="simple-example">
  ### Exemple simple
</div>

```ts
const doc = await client.scrape("https://docs.firecrawl.dev", {
  formats: ["markdown"]
});
```

<div id="complex-example">
  ### Exemple complexe
</div>

```ts
const doc = await client.scrape("https://example.com/pricing", {
  formats: [
    "markdown",
    "links",
    { type: "json", prompt: "Extract plan names and prices." },
    { type: "screenshot", fullPage: true, quality: 80, viewport: { width: 1280, height: 720 } },
    { type: "changeTracking", modes: ["git-diff"], tag: "pricing" },
    { type: "attributes", selectors: [{ selector: "a", attribute: "href" }] }
  ],
  headers: {
    "User-Agent": "FirecrawlDocsBot/1.0"
  },
  onlyMainContent: true,
  waitFor: 1000,
  parsers: [{ type: "pdf", mode: "auto", maxPages: 5 }],
  actions: [
    { type: "click", selector: "#accept" },
    { type: "wait", milliseconds: 750 },
    { type: "scrape" }
  ],
  location: { country: "US", languages: ["en-US"] },
  removeBase64Images: true,
  fastMode: true,
  blockAds: true,
  proxy: "auto",
  maxAge: 86400000,
  minAge: 1,
  storeInCache: true,
  profile: { name: "docs-session", saveChanges: true }
});
```

<div id="parameters">
  ### Paramètres
</div>

* `url`
  * Type : string
  * À utiliser lorsque : vous souhaitez scraper une page précise.

* `options.formats`
  * Type : tableau de chaînes de format ou d’objets de format
  * À utiliser lorsque : vous souhaitez plusieurs formats de sortie.
  * Formats sous forme de chaînes simples (l’union `FormatString` inclut aussi `"json"`, mais le SDK rejette `"json"` comme chaîne simple — utilisez un objet `json` ci-dessous) :
    * `"markdown"`: contenu markdown
    * `"html"`: HTML nettoyé
    * `"rawHtml"`: HTML brut
    * `"links"`: liens de la page
    * `"images"`: URL d’images
    * `"screenshot"`: sortie de capture d’écran
    * `"summary"`: sortie de résumé
    * `"changeTracking"`: sortie de suivi des modifications (pour des options comme `modes`, utilisez `{ type: "changeTracking", modes: [...] }` — `modes` est requis sur cet objet dans les typages)
    * `"attributes"`: extraction d’attributs (utilisez `{ type: "attributes", selectors: [...] }` lorsque vous passez des sélecteurs)
    * `"branding"`: sortie du profil de branding
    * `"audio"`: extraction audio
    * `"video"`: extraction vidéo
  * Types de format disponibles uniquement sous forme d’objet (avec au minimum `type`, comme indiqué) :
    * `{ type: "json", prompt?: string, schema?: JSON schema or Zod schema }`: au moins l’un de `prompt` ou `schema` est requis (validation du SDK).
    * `{ type: "question", question: string }`: extraction de type questions-réponses.
    * `{ type: "highlights", query: string }`: extraction de texte source pertinent.
    * `{ type: "screenshot", fullPage?, quality?, viewport? }`: mêmes options que la forme chaîne, mais sous forme d’objet.
    * `{ type: "changeTracking", modes: ("git-diff" | "json")[], schema?, prompt?, tag? }`: `modes` est requis.
    * `{ type: "attributes", selectors: Array<{ selector, attribute }> }`
  * Champs d’objet partagés, le cas échéant :
    * `schema`: schéma JSON ou schéma Zod pour `json`, ou pour `changeTracking` en mode `json` (Zod est converti en JSON Schema par le SDK).
    * `modes`: pour `changeTracking` uniquement : `"git-diff"` et/ou `"json"`.
    * `tag`: balise de branche facultative pour le suivi des modifications.
    * `fullPage`, `quality`, `viewport`: options de capture d’écran.

* `options.headers`
  * Type : dictionnaire de chaînes de caractères à chaînes de caractères
  * À utiliser lorsque : vous avez besoin d’en-têtes de requête personnalisés.

* `options.includeTags`
  * Type : tableau de chaînes
  * À utiliser lorsque : vous souhaitez inclure uniquement certaines balises HTML.

* `options.excludeTags`
  * Type : tableau de chaînes de caractères
  * À utiliser si : vous souhaitez exclure certaines balises HTML.

* `options.onlyMainContent`
  * Type : booléen
  * À utiliser si vous souhaitez supprimer la barre de navigation, le pied de page et d’autres éléments récurrents.

* `options.timeout`
  * Type : number
  * À utiliser si vous avez besoin d’un délai d’expiration en millisecondes.

* `options.waitFor`
  * Type : nombre
  * À utiliser lorsque vous devez attendre le rendu de la page (en millisecondes).

* `options.mobile`
  * Type : booléen
  * À utiliser si vous souhaitez un viewport mobile.

* `options.parsers`
  * Type : tableau de noms de parseur ou d’objets parseur
  * À utiliser si vous avez besoin d’options de parsing de fichiers (par exemple pour les PDF).
  * Valeurs prises en charge :
    * `"pdf"` : active le parsing des PDF
    * `{ type: "pdf", mode?: "fast" | "auto" | "ocr", maxPages?: number }` : options du parseur PDF

* `options.actions`
  * Type : tableau d’objets d’action
  * À utiliser lorsque : vous avez besoin d’actions légères avant le scraping.
  * Types d’action pris en charge :
    * `wait` : `milliseconds` ou `selector` requis
    * `screenshot` : `fullPage`, `quality`, `viewport` facultatifs
    * `click` : `selector` requis
    * `write` : `text` requis (cliquez d’abord pour mettre le champ de saisie au focus)
    * `press` : `key` requis
    * `scroll` : `direction` (`up` ou `down`) requise, `selector` facultatif
    * `scrape` : aucun champ supplémentaire
    * `executeJavascript` : `script` requis
    * `pdf` : `format` (A0, A1, A2, A3, A4, A5, A6, Letter, Legal, Tabloid, Ledger), `landscape`, `scale` facultatifs

* `options.location`
  * Type : objet avec `country` et `languages`
  * À utiliser lorsque : vous avez besoin d’un scraping tenant compte du pays ou de la langue.

* `options.skipTlsVerification`
  * Type : booléen
  * À utiliser si vous devez ignorer la vérification TLS.

* `options.removeBase64Images`
  * Type : booléen
  * À utiliser lorsque : vous souhaitez exclure les images en base64 de la sortie Markdown.

* `options.fastMode`
  * Type : booléen
  * À utiliser si : vous souhaitez des scrapes plus rapides avec une fidélité réduite.

* `options.blockAds`
  * Type : boolean
  * À utiliser si vous souhaitez bloquer les publicités et les pop-up de cookies.

* `options.proxy`
  * Type : string
  * À utiliser lorsque : vous devez contrôler le proxy.
  * Valeurs confirmées :
    * `"basic"`
    * `"stealth"`
    * `"enhanced"`
    * `"auto"`
    * chaîne d’URL de proxy personnalisée

* `options.maxAge`
  * Type : number
  * À utiliser lorsque : vous souhaitez des données mises en cache jusqu’à une ancienneté maximale (en millisecondes).

* `options.minAge`
  * Type : number
  * À utiliser lorsque : vous souhaitez des données en cache uniquement si elles ont au moins cet âge (en millisecondes).

* `options.storeInCache`
  * Type : booléen
  * À utiliser si : vous souhaitez que Firecrawl mette le résultat en cache.

* `options.profile`
  * Type : objet avec `name` et `saveChanges` en option
  * À utiliser lorsque : vous souhaitez un profil de navigateur persistant, partagé entre les scrapes et les interactions.

<div id="interact">
  ## Interact
</div>

### Pourquoi l’utiliser

Utilisez `interact` pour contrôler, par code ou en langage naturel, la session de navigateur associée à une tâche de scraping (via `metadata.scrapeId`). Le SDK exige au moins l’un des deux paramètres `code` ou `prompt`. Pour les flux qui vont au-delà de simples ajustements rapides avant le scraping, privilégiez `interact` aux `actions` exécutées au moment du scraping.

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

`client.interact(jobId, args)` → `Promise<ScrapeExecuteResponse>`

<div id="simple-example">
  ### Exemple simple
</div>

```ts
const doc = await client.scrape("https://example.com", { formats: ["markdown"] });
const jobId = doc.metadata?.scrapeId;
if (!jobId) throw new Error("Missing scrapeId from scrape response");

const result = await client.interact(jobId, {
  prompt: "Click the pricing tab and summarize the plans."
});
```

<div id="complex-example">
  ### Exemple complexe
</div>

```ts
const result = await client.interact("<scrapeJobId>", {
  code: "console.log(await page.title());",
  language: "node",
  timeout: 60
});
```

<div id="parameters">
  ### Paramètres
</div>

* `jobId`
  * Type : string
  * À utiliser lorsque : vous disposez d’un ID de tâche de scraping issu de `document.metadata.scrapeId`.

* `args.code`
  * Type : string
  * À utiliser lorsque : vous souhaitez exécuter du code dans la session de navigateur (par exemple, utiliser `page` de Playwright dans l’environnement d’exécution `node`).

* `args.prompt`
  * Type : string
  * À utiliser lorsque : vous voulez que l’agent de navigateur suive une instruction en langage naturel.

* Au moins un des champs `args.code` ou `args.prompt` doit être renseigné (sinon, le SDK génère une erreur).

* `args.language`
  * Type : string
  * À utiliser lorsque : vous avez besoin d’un environnement d’exécution spécifique.
  * Valeurs acceptées : `"python"`, `"node"`, `"bash"`
  * Valeur par défaut lorsqu’il est omis : `"node"` (définie par le SDK dans le corps de la requête).

* `args.timeout`
  * Type : number
  * À utiliser lorsque : vous devez définir un délai d’expiration en secondes pour l’exécution.

<div id="return-value-interact">
  ### Valeur de retour (`interact`)
</div>

`ScrapeExecuteResponse` est identique à `BrowserExecuteResponse`. Les champs confirmés incluent :

* `success` : booléen
* `output` : chaîne (sortie agrégée lorsqu’elle est disponible)
* `stdout` : chaîne
* `result` : chaîne (champ alias aux côtés de stdout dans les définitions de types)
* `stderr` : chaîne
* `exitCode` : nombre
* `killed` : booléen
* `liveViewUrl` : chaîne
* `interactiveLiveViewUrl` : chaîne
* `error` : chaîne

<div id="stop-session">
  ### Arrêter une session
</div>

`client.stopInteraction(jobId)` → `Promise<ScrapeBrowserDeleteResponse>`

* `jobId` : ID de la tâche de scraping (même ID que celui utilisé pour `interact`).
* Les champs de la réponse définis dans le typage incluent `success`, `sessionDurationMs`, `creditsBilled`, `error`.

<div id="ask-agentic-debugging">
  ## Ask (Débogage par agent)
</div>

### Pourquoi l’utiliser

Utilisez ask lorsqu’un appel à l’API Firecrawl échoue ou renvoie des résultats inattendus. L’agent d’assistance IA diagnostique le problème, propose des paramètres correctifs et peut, si besoin, valider la correction sur l’API réelle. Latence typique : 15 à 30 secondes.

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

Pas encore disponible dans le SDK Node.js. Utilisez `fetch` directement.

<div id="simple-example">
  ### Exemple simple
</div>

```ts
const response = await fetch("https://api.firecrawl.dev/v2/support/ask", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.FIRECRAWL_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    question: "my scrape returned empty markdown for https://example.com",
  }),
});
const result = await response.json();
console.log(result.answer);
console.log(result.fixParameters);
```

<div id="complex-example">
  ### Exemple complexe
</div>

```ts
const response = await fetch("https://api.firecrawl.dev/v2/support/ask", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.FIRECRAWL_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    question: "crawl returned 3 pages but I expected 50",
    rationale: "user is on their third failed crawl attempt today",
    context: { userPlan: "standard", retryCount: 3 },
  }),
});
const result = await response.json();
```

<div id="agent-retry-pattern">
  ### Schéma de réessai de l’Agent
</div>

```ts
import { Firecrawl } from "firecrawl";

const client = new Firecrawl({ apiKey: process.env.FIRECRAWL_API_KEY });

const doc = await client.scrape("https://example.com/pricing", {
  formats: ["markdown"],
});

if (!doc.markdown || doc.markdown.length < 100) {
  const diagResponse = await fetch(
    "https://api.firecrawl.dev/v2/support/ask",
    {
      method: "POST",
      headers: {
        Authorization: `Bearer ${process.env.FIRECRAWL_API_KEY}`,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        question: `scrape returned only ${doc.markdown?.length ?? 0} chars for https://example.com/pricing`,
      }),
    }
  );
  const diagnosis = await diagResponse.json();

  if (diagnosis.fixParameters) {
    const retryDoc = await client.scrape("https://example.com/pricing", {
      formats: ["markdown"],
      ...diagnosis.fixParameters,
    });
  }
}
```

<div id="parameters">
  ### Paramètres
</div>

* `question`
  * Type : string (obligatoire, 1-8000 caractères)
  * À utiliser lorsque : vous devez décrire le problème.

* `rationale`
  * Type : string (1-2000 caractères)
  * À utiliser lorsque : vous êtes un agent IA qui appelle au nom d’un utilisateur.

* `context`
  * Type : object (forme libre)
  * À utiliser lorsque : vous souhaitez transmettre des métadonnées au prompt de débogage.

<div id="docs-search">
  ## Recherche dans la documentation
</div>

### Pourquoi l’utiliser

Utilisez docs-search pour rechercher dans la documentation de Firecrawl.

<div id="simple-example">
  ### Exemple simple
</div>

```ts
const response = await fetch(
  "https://api.firecrawl.dev/v2/support/docs-search",
  {
    method: "POST",
    headers: {
      Authorization: `Bearer ${process.env.FIRECRAWL_API_KEY}`,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      question: "how do I verify webhook signatures?",
    }),
  }
);
const result = await response.json();
console.log(result.answer);
```

<div id="parameters">
  ### Paramètres
</div>

* `question`
  * Type : chaîne de caractères (obligatoire, 1-8000 caractères)
  * À utiliser si : vous avez besoin d’une réponse basée sur la documentation.

<div id="notes">
  ## Remarques
</div>

* Alias obsolètes : `scrapeExecute`, `stopInteractiveBrowser` et `deleteScrapeBrowser` renvoient à `interact` et `stopInteraction`.
* L’export `Firecrawl` par défaut correspond au client v2 ; v1 reste disponible via `client.v1`.
* Les schémas Zod passés à `formats` (pour `json` ou `changeTracking`) sont convertis en JSON Schema par le SDK.
* Le package déclare **Node.js &gt;= 22** dans `engines`.
