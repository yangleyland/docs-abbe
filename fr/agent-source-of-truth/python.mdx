source de référence Python officielle de Firecrawl pour les agents. Générée à partir du code source du SDK (`firecrawl-py` / `firecrawl` **4.22.1**) et de la spécification OpenAPI v2. Les noms de méthode, les paramètres et les types de retour correspondent à ceux du client v2 dans `firecrawl/v2/client.py`, sauf indication contraire.

<div id="install">
  ## Installation
</div>

```bash
pip install firecrawl-py
```

<div id="authenticate">
  ## Authentification
</div>

```py
import os
from firecrawl import Firecrawl

client = Firecrawl(api_key=os.environ.get("FIRECRAWL_API_KEY"))
# client = Firecrawl(api_key="fc-...", api_url="https://api.firecrawl.dev")
```

<div id="when-to-use-what">
  ## Quand utiliser chaque option
</div>

* `search` : à utiliser lorsque vous partez d’une requête et avez besoin de découvrir des résultats.
* `scrape` : à utiliser lorsque vous avez déjà une URL et souhaitez récupérer le contenu de la page.
* `interact` : à utiliser lorsque la page nécessite des clics, des formulaires ou des actions du navigateur après le scrape.

- `support/ask` : à utiliser lorsqu’un appel à l’API Firecrawl échoue ou renvoie des résultats inattendus et que vous avez besoin d’un diagnostic.
- `support/docs-search` : à utiliser lorsque vous devez consulter la documentation Firecrawl.

<div id="search">
  ## Recherche
</div>

<div id="why-use-it">
  ### Pourquoi l&#39;utiliser
</div>

Utilisez la recherche pour découvrir des pages pertinentes à partir d&#39;une requête, puis choisissez les URLs à scraper ou avec lesquelles interagir. Vous pouvez restreindre les résultats à un site avec `site:`, par exemple `site:docs.firecrawl.dev crawl webhooks`.

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

`client.search(query, **options)` → `SearchData` (`firecrawl.v2.types`)

<div id="return-value">
  ### Valeur de retour
</div>

Renvoie un modèle `SearchData` avec des listes facultatives :

* `web` — résultats web (`SearchResultWeb` ou `Document` complet lorsque `scrape_options` hydrate le contenu)
* `news` — résultats d’actualité (`SearchResultNews` ou `Document`)
* `images` — résultats d’images (`SearchResultImages` ou `Document`)

Les groupes omis valent `None` lorsque l’API n’a pas renvoyé cette clé.

**Erreur à éviter :** `search()` ne renvoie pas `{ data: [...] }`. N’accédez pas à `result.data`. Les résultats web sont dans `result.web`, ceux d’actualité dans `result.news`, et les images dans `result.images`.

<div id="simple-example">
  ### Exemple simple
</div>

```py
results = client.search("site:docs.firecrawl.dev webhook retries")
for item in results.web or []:
    print(getattr(item, "url", None), getattr(item, "title", None))
```

<div id="complex-example">
  ### Exemple complexe
</div>

```py
from firecrawl.v2.types import ScrapeOptions

results = client.search(
    "site:docs.firecrawl.dev crawl webhooks",
    sources=["web", "news"],
    categories=["research"],
    limit=10,
    tbs="qdr:m",
    location="San Francisco,California,United States",
    ignore_invalid_urls=True,
    timeout=300000,
    scrape_options=ScrapeOptions(
        formats=[
            "markdown",
            "links",
            {"type": "json", "prompt": "Extract title and key endpoints."},
        ],
        only_main_content=True,
        include_tags=["main", "article"],
        exclude_tags=["nav", "footer"],
        wait_for=1000,
        actions=[
            {"type": "click", "selector": "button#accept"},
            {"type": "wait", "milliseconds": 750},
            {"type": "scrape"},
        ],
    ),
)
```

<div id="parameters">
  ### Paramètres
</div>

* `query`
  * Type : str
  * À utiliser lorsque : vous avez besoin d&#39;une requête de recherche.
  * Notes : utilisez `site:example.com` pour limiter les résultats à un domaine.

* `sources`
  * Type : liste de noms de source ou d&#39;objets `Source`
  * À utiliser lorsque : vous voulez définir les sources à interroger.
  * Valeurs confirmées :
    * `"web"` : résultats de l&#39;index web
    * `"news"` : résultats d&#39;actualités
    * `"images"` : résultats d&#39;images
    * `Source(type="web" | "news" | "images")` : forme d&#39;objet source typée

* `categories`
  * Type : liste de noms de catégorie ou d&#39;objets `Category`
  * À utiliser lorsque : vous souhaitez filtrer les résultats par catégorie.
  * Valeurs confirmées :
    * `"github"` : résultats axés sur GitHub
    * `"research"` : résultats de recherche et académiques
    * `"pdf"` : résultats axés sur les PDF
    * `Category(type="github" | "research" | "pdf")` : forme d&#39;objet catégorie typée

* `limit`
  * Type : int
  * À utiliser lorsque : vous souhaitez limiter le nombre de résultats.
  * Notes : la valeur par défaut est `5` dans le modèle SDK.

* `tbs`
  * Type : str
  * À utiliser lorsque : vous avez besoin d&#39;un filtre temporel (par exemple `qdr:d`, `qdr:w`, `sbd:1,qdr:m`).

* `location`
  * Type : str
  * À utiliser lorsque : vous souhaitez des résultats localisés.

* `ignore_invalid_urls`
  * Type : bool
  * À utiliser lorsque : vous souhaitez exclure les URL qui ne peuvent pas être extraites par d&#39;autres points de terminaison.

* `timeout`
  * Type : int
  * À utiliser lorsque : vous avez besoin d&#39;un délai d&#39;expiration de la requête en millisecondes.
  * Notes : la valeur par défaut est `300000` dans le modèle SDK.

* `scrape_options`
  * Type : `ScrapeOptions`
  * À utiliser lorsque : vous souhaitez extraire chaque résultat de recherche (voir les paramètres Scrape pour les champs).

<div id="scrape">
  ## Scrape
</div>

### Pourquoi l’utiliser

Utilisez scrape si vous avez déjà une URL et souhaitez un contenu structuré dans un ou plusieurs formats.

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

`client.scrape(url, **options)` → `Document` (`firecrawl.v2.types`)

<div id="simple-example">
  ### Exemple simple
</div>

```py
doc = client.scrape("https://docs.firecrawl.dev", formats=["markdown"])
```

<div id="complex-example">
  ### Exemple complexe
</div>

```py
doc = client.scrape(
    "https://example.com/pricing",
    formats=[
        "markdown",
        "links",
        {"type": "json", "prompt": "Extract plan names and prices."},
        {"type": "screenshot", "full_page": True, "quality": 80, "viewport": {"width": 1280, "height": 720}},
        {"type": "changeTracking", "modes": ["git-diff"], "tag": "pricing"},
        {"type": "attributes", "selectors": [{"selector": "a", "attribute": "href"}]},
    ],
    headers={"User-Agent": "FirecrawlDocsBot/1.0"},
    only_main_content=True,
    wait_for=1000,
    parsers=[{"type": "pdf", "mode": "auto", "max_pages": 5}],
    actions=[
        {"type": "click", "selector": "#accept"},
        {"type": "wait", "milliseconds": 750},
        {"type": "scrape"},
    ],
    location={"country": "US", "languages": ["en-US"]},
    remove_base64_images=True,
    fast_mode=True,
    block_ads=True,
    proxy="auto",
    max_age=86400000,
    store_in_cache=True,
    profile={"name": "docs-session", "save_changes": True},
)
```

<div id="parameters">
  ### Paramètres
</div>

* `url`
  * Type : str
  * À utiliser si vous souhaitez extraire une page spécifique.

* `formats`
  * Type : liste de chaînes de format ou d’objets de format
  * À utiliser si : vous souhaitez plusieurs formats de sortie.
  * Chaînes de format prises en charge :
    * `"markdown"`: contenu markdown
    * `"html"`: HTML nettoyé
    * `"rawHtml"` ou `"raw_html"`: HTML brut
    * `"links"`: liens de la page
    * `"images"`: URL d’images
    * `"screenshot"`: sortie de capture d’écran
    * `"summary"`: sortie de résumé
    * `"changeTracking"` ou `"change_tracking"`: sortie de suivi des modifications
    * `"attributes"`: extraction d’attributs
    * `"branding"`: sortie du profil de branding
    * `"audio"`: extraction audio
    * `"video"`: extraction vidéo
  * Types de format uniquement sous forme d’objet :
    * `{"type": "json", ...}`: extraction JSON. Utilisez un objet, et non la chaîne simple `"json"`.
    * `{"type": "question", "question": "..."}`: sortie de questions-réponses.
    * `{"type": "highlights", "query": "..."}`: sortie de texte source pertinent.
  * Champs de l’objet de format :
    * `type`: l’une des chaînes de format ci-dessus, ou `"json"`, `"question"` ou `"highlights"` pour les formats uniquement sous forme d’objet
    * `question`: pour `type: "question"`
    * `query`: pour `type: "highlights"`
    * `prompt`: facultatif pour `type: "json"`
    * `schema`: schéma JSON pour `type: "json"` ou pour le mode JSON du suivi des modifications
    * `modes`: tableau de `"git-diff"` ou `"json"` pour `type: "changeTracking"`
    * `tag`: balise de suivi des modifications pour `type: "changeTracking"`
    * `full_page`, `quality`, `viewport`: options de capture d’écran pour `type: "screenshot"`
    * `selectors`: tableau de `{selector, attribute}` pour `type: "attributes"`

* `headers`
  * Type : dictionnaire de chaînes de caractères à chaînes de caractères
  * À utiliser lorsque : vous avez besoin d’en-têtes de requête personnalisés.

* `include_tags`
  * Type : liste de str
  * À utiliser si : vous souhaitez inclure uniquement certaines balises HTML.

* `exclude_tags`
  * Type : liste de str
  * À utiliser lorsque : vous souhaitez exclure certaines balises HTML.

* `only_main_content`
  * Type : bool
  * À utiliser lorsque : vous souhaitez supprimer le menu de navigation, le pied de page et d’autres éléments standard.

* `timeout`
  * Type : int
  * À utiliser si vous avez besoin d’un délai d’expiration en millisecondes.

* `wait_for`
  * Type : int
  * À utiliser lorsque : vous devez attendre que la page s’affiche complètement (en millisecondes).

* `mobile`
  * Type : bool
  * À utiliser si : vous souhaitez un viewport mobile.

* `parsers`
  * Type : liste de noms de parseurs ou d’objets parseur
  * À utiliser si : vous avez besoin d’options de parsing de fichiers (par exemple, pour l’analyse de PDF).
  * Valeurs prises en charge :
    * `"pdf"`: active l’analyse PDF
    * `{ "type": "pdf", "mode": "fast" | "auto" | "ocr", "max_pages": int }`: options du parseur PDF

* `actions`
  * Type : liste d&#39;objets d&#39;action
  * À utiliser lorsque : vous avez besoin d&#39;actions légères de pré-scrape.
  * Types d&#39;action pris en charge :
    * `wait` : `milliseconds` ou `selector` requis
    * `screenshot` : `full_page`, `quality`, `viewport` facultatifs
    * `click` : `selector` requis
    * `write` : `text` requis (cliquez d&#39;abord pour activer le champ de saisie)
    * `press` : `key` requis
    * `scroll` : `direction` (`up` ou `down`) requise, `selector` facultatif
    * `scrape` : aucun champ supplémentaire
    * `executeJavascript` : `script` requis
    * `pdf` : `format` (A0, A1, A2, A3, A4, A5, A6, Letter, Legal, Tabloid, Ledger), `landscape`, `scale` facultatifs

* `location`
  * Type : dict avec `country` et `languages`
  * À utiliser lorsque : vous avez besoin d’un scraping tenant compte de la géolocalisation ou de la langue.

* `skip_tls_verification`
  * Type : bool
  * À utiliser si vous devez ignorer la vérification TLS.

* `remove_base64_images`
  * Type : bool
  * À utiliser si : vous souhaitez supprimer les images en base64 de la sortie Markdown.

* `fast_mode`
  * Type : bool
  * À utiliser si : vous voulez des extractions plus rapides, avec une fidélité réduite.

* `block_ads`
  * Type : bool
  * À utiliser si : vous souhaitez bloquer les publicités et les bannières de cookies.

* `proxy`
  * Type: str
  * À utiliser lorsque : vous avez besoin de contrôler le proxy.
  * Valeurs prises en charge : `"basic"`, `"stealth"`, `"enhanced"`, `"auto"`

* `max_age`
  * Type : int
  * À utiliser si : vous souhaitez des données mises en cache jusqu’à un âge maximal (en millisecondes).

* `min_age`
  * Type : int
  * À utiliser si : vous souhaitez des données en cache uniquement si elles ont au moins cet âge (en millisecondes).
  * Remarques : uniquement dans `ScrapeOptions` passé comme `scrape_options` à `client.search`. La méthode `client.scrape` n’accepte pas `min_age`.

* `store_in_cache`
  * Type : bool
  * À utiliser si : vous souhaitez que Firecrawl mette le résultat en cache.

* `profile`
  * Type : dictionnaire avec `name`, et `save_changes` ou `saveChanges` en option
  * À utiliser lorsque : vous souhaitez un profil de navigateur persistant, réutilisé entre les scrapes et les interactions.

<div id="interact">
  ## Interact
</div>

### Pourquoi l’utiliser

Utilisez interact lorsqu’une page nécessite des actions dans le navigateur ou l’exécution de code une fois le scrape lancé.

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

`client.interact(job_id, code=None, *, prompt=None, language="node", timeout=None)`

`prompt` doit être passé comme argument nommé : appelez `client.interact(job_id, prompt="...")` ou `client.interact(job_id, code="...")`. Au moins l’un de `code` ou `prompt` doit être renseigné (validation effectuée dans `firecrawl/v2/methods/scrape.py`).

<div id="simple-example">
  ### Exemple simple
</div>

```py
doc = client.scrape("https://example.com", formats=["markdown"])
job_id = doc.metadata.scrape_id if doc.metadata else None
if not job_id:
    raise RuntimeError("Missing scrape_id from scrape response")

result = client.interact(job_id, prompt="Click the pricing tab and summarize the plans.")
```

<div id="complex-example">
  ### Exemple complexe
</div>

```py
result = client.interact(
    "<scrapeJobId>",
    code="print(await page.title())",
    language="python",
    timeout=60,
)
```

<div id="stop-session">
  ### Arrêter la session
</div>

`client.stop_interaction(job_id)` met fin à la session de navigateur associée au scrape. Renvoie `BrowserDeleteResponse` (`success`, optional `session_duration_ms`, `credits_billed`, `error`).

<div id="parameters">
  ### Paramètres
</div>

* `job_id`
  * Type : str
  * À utiliser lorsque : vous disposez d’un ID de tâche de scraping issu de `document.metadata.scrape_id`.

* `code`
  * Type : str
  * À utiliser lorsque : vous souhaitez exécuter du code dans la session de navigateur (facultatif si `prompt` est défini).

* `prompt`
  * Type : str
  * À utiliser lorsque : vous souhaitez que l’agent du navigateur suive une instruction en langage naturel (facultatif si `code` est défini).

* `language`
  * Type : str
  * À utiliser lorsque : vous avez besoin d’un environnement d’exécution spécifique.
  * Valeurs confirmées : `"python"`, `"node"`, `"bash"`
  * Remarques : la valeur par défaut est `"node"` dans le SDK.

* `timeout`
  * Type : int
  * À utiliser lorsque : vous avez besoin d’un délai d’exécution maximal en secondes.

<div id="return-value">
  ### Valeur de retour
</div>

Renvoie `BrowserExecuteResponse` : `success`, `live_view_url` (facultatif), `interactive_live_view_url`, `output`, `stdout`, `result`, `stderr`, `exit_code`, `killed`, `error` (le camelCase de l’API est normalisé en snake&#95;case dans le modèle).

<div id="ask-agentic-debugging">
  ## Ask (Débogage par agent)
</div>

<div id="why-use-it">
  ### Pourquoi l&#39;utiliser
</div>

Utilisez `ask` lorsqu&#39;un appel à l&#39;API Firecrawl échoue ou renvoie des résultats inattendus. L&#39;agent d&#39;assistance IA diagnostique le problème, propose les paramètres à corriger et, si besoin, valide la correction sur l&#39;API réelle. Latence typique : 15-30 secondes.

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

Pas encore disponible dans le SDK Python. Utilisez directement `requests` ou `httpx`.

<div id="simple-example">
  ### Exemple simple
</div>

```py
import requests

response = requests.post(
    "https://api.firecrawl.dev/v2/support/ask",
    headers={
        "Authorization": f"Bearer {os.environ['FIRECRAWL_API_KEY']}",
        "Content-Type": "application/json",
    },
    json={"question": "my scrape returned empty markdown for https://example.com"},
)
result = response.json()
print(result["answer"])
print(result["fixParameters"])
```

<div id="complex-example">
  ### Exemple complexe
</div>

```py
import requests

response = requests.post(
    "https://api.firecrawl.dev/v2/support/ask",
    headers={
        "Authorization": f"Bearer {os.environ['FIRECRAWL_API_KEY']}",
        "Content-Type": "application/json",
    },
    json={
        "question": "crawl returned 3 pages but I expected 50",
        "rationale": "user is on their third failed crawl attempt today",
        "context": {"userPlan": "standard", "retryCount": 3},
    },
)
result = response.json()
```

<div id="agent-retry-pattern">
  ### Stratégie de nouvelle tentative de l’agent
</div>

```py
from firecrawl import Firecrawl
import requests, os

client = Firecrawl(api_key=os.environ["FIRECRAWL_API_KEY"])

doc = client.scrape("https://example.com/pricing", formats=["markdown"])

if not doc.markdown or len(doc.markdown) < 100:
    diagnosis = requests.post(
        "https://api.firecrawl.dev/v2/support/ask",
        headers={
            "Authorization": f"Bearer {os.environ['FIRECRAWL_API_KEY']}",
            "Content-Type": "application/json",
        },
        json={
            "question": f"scrape returned only {len(doc.markdown or '')} chars for https://example.com/pricing",
        },
    ).json()

    if diagnosis.get("fixParameters"):
        doc = client.scrape(
            "https://example.com/pricing",
            formats=["markdown"],
            **diagnosis["fixParameters"],
        )
```

<div id="parameters">
  ### Paramètres
</div>

* `question`
  * Type : str (requis, 1-8000 caractères)
  * À utiliser lorsque : vous devez décrire le problème.

* `rationale`
  * Type : str (1-2000 caractères)
  * À utiliser lorsque : vous êtes un agent IA qui appelle au nom d’un utilisateur.

* `context`
  * Type : dict (forme libre)
  * À utiliser lorsque : vous souhaitez transmettre des métadonnées dans le prompt de débogage.

<div id="docs-search">
  ## Recherche dans la documentation
</div>

### Pourquoi l’utiliser

Utilisez docs-search pour rechercher dans la documentation de Firecrawl.

<div id="simple-example">
  ### Exemple simple
</div>

```py
import requests

FIRECRAWL_API_KEY = "fc-YOUR_API_KEY"

response = requests.post(
    "https://api.firecrawl.dev/v2/support/docs-search",
    headers={"Authorization": f"Bearer {FIRECRAWL_API_KEY}"},
    json={"question": "how do I verify webhook signatures?"},
)
print(response.json()["answer"])
```

<div id="parameters">
  ### Paramètres
</div>

* `question`
  * Type : str (obligatoire, 1-8000 caractères)
  * À utiliser lorsque : vous avez besoin d’une réponse basée sur la documentation.

<div id="notes">
  ## Remarques
</div>

* Alias dépréciés : `scrape_execute` → `interact` ; `stop_interactive_browser` et `delete_scrape_browser` → `stop_interaction`.
* Le client `Firecrawl` principal expose directement les méthodes v2 ; la v1 reste disponible sous `client.v1`.
* Il est possible que l’extrait OpenAPI v2 fourni pour `POST /v2/scrape/{jobId}/interact` ne documente que `code` ; le Python SDK et le serveur acceptent `code` ou `prompt` pour ce point de terminaison.
