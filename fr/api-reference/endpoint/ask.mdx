Le point de terminaison `/support/ask` est un agent d’assistance IA qui diagnostique les problèmes liés à vos jobs Firecrawl, à votre compte et à votre utilisation de l’API. Envoyez une question et recevez une réponse vérifiée avec des paramètres de correction concrets — généralement en 15–30 secondes.

<div id="designed-for-ai-agents">
  ## Conçu pour les agents IA
</div>

`/support/ask` est conçu pour une communication **d&#39;agent à agent**. Si vous créez un agent IA qui utilise Firecrawl, intégrez ce point de terminaison à votre flux de gestion des erreurs afin que votre agent puisse diagnostiquer lui-même les échecs de scraping, les problèmes de crawl et les erreurs de configuration, sans intervention humaine.

Transmettez un champ `rationale` pour donner à l&#39;agent d&#39;assistance du contexte sur ce que votre utilisateur final cherche à accomplir. Cela permet de prioriser la collecte d&#39;éléments probants.

<div id="how-it-works">
  ## Comment ça marche
</div>

1. **Vous décrivez le problème** — une question en langage naturel qui décrit le problème.
2. **L’agent enquête** — il examine les journaux d’exécution, l’état du compte, la documentation et le code source.
3. **L’agent valide** — lorsque c’est possible, l’agent teste un correctif sur l’API Firecrawl réelle (par ex., en relançant un `scrape` avec des paramètres ajustés).
4. **Vous obtenez une réponse vérifiée** — la réponse inclut un `answer` rédigé en prose, des `fixParameters` lisibles par machine que vous pouvez appliquer directement, ainsi que des résultats de `validation` indiquant si le correctif a été testé.

<div id="authentication">
  ## Authentification
</div>

Utilise votre clé API Firecrawl comme jeton Bearer. La requête est automatiquement restreinte à votre équipe — vous ne pouvez consulter que vos propres jobs et les données de votre compte.

```bash
curl -X POST https://api.firecrawl.dev/v2/support/ask \
  -H "Authorization: Bearer fc-YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "my crawl returned 3 pages but I expected 50",
    "rationale": "user is on their third failed crawl attempt today"
  }'
```

<div id="response-fields">
  ## Champs de la réponse
</div>

| Champ           | Type    | Description                                                     |                                                                                                 |
| --------------- | ------- | --------------------------------------------------------------- | ----------------------------------------------------------------------------------------------- |
| `answer`        | string  | Texte de 2 à 4 phrases expliquant le diagnostic et le correctif |                                                                                                 |
| `confidence`    | string  | `high`, `medium` ou `low`                                       |                                                                                                 |
| `fixParameters` | object  | null                                                            | Paramètres d&#39;API à appliquer pour le correctif (par ex., `{"waitFor": 5000}`)               |
| `validation`    | object  | null                                                            | Indique si le correctif a été testé : `tested`, `result` (success/failure/skipped), `evidence`  |
| `feedback`      | object  | null                                                            | Présent lorsque l&#39;agent est bloqué ; `{ blockedBy, attempted }`. `null` en cas de réussite. |
| `durationMs`    | integer | Temps d&#39;exécution total en millisecondes                    |                                                                                                 |

<div id="status-codes">
  ## Codes d’état
</div>

| Code  | Signification                                                       |
| ----- | ------------------------------------------------------------------- |
| `200` | Réponse fournie ou bloqué (enveloppe toujours renvoyée)             |
| `400` | JSON invalide ou non-conformité au schéma                           |
| `401` | Jeton Bearer manquant ou invalide                                   |
| `504` | Délai maximal strict de 60 s atteint — enveloppe partielle renvoyée |
