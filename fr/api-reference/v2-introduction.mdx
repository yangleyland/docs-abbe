<div id="base-url">
  ## URL de base
</div>

Toutes les requêtes utilisent l’URL de base suivante :

```bash
https://api.firecrawl.dev
```

<div id="authentication">
  ## Authentification
</div>

Chaque requête doit inclure un en-tête `Authorization` contenant votre clé d&#39;API :

```bash
Authorization: Bearer fc-YOUR-API-KEY
```

Incluez cet en-tête dans tous les appels à l&#39;API. Vous pouvez trouver votre clé d’API dans le [tableau de bord Firecrawl](https://www.firecrawl.dev/app/api-keys).

Si vous êtes un agent sans clé d’API, commencez par [Obtenir les identifiants](/fr/ai-onboarding#get-credentials). Si votre plateforme prend en charge WorkOS ID-JAG, utilisez [`auth.md`](https://www.firecrawl.dev/auth.md) pour les instructions d’inscription.

<CodeGroup>
  ```bash cURL
  curl -X POST "https://api.firecrawl.dev/v2/scrape" \
    -H "Authorization: Bearer fc-YOUR-API-KEY" \
    -H "Content-Type: application/json" \
    -d '{"url": "https://example.com"}'
  ```

  ```python Python
  from firecrawl import Firecrawl

  firecrawl = Firecrawl(api_key="fc-YOUR-API-KEY")

  result = firecrawl.scrape("https://example.com")
  ```

  ```js Node
  import { Firecrawl } from 'firecrawl';

  const firecrawl = new Firecrawl({ apiKey: "fc-YOUR-API-KEY" });

  const result = await firecrawl.scrape('https://example.com');
  ```
</CodeGroup>

<div id="response-codes">
  ## Codes de réponse
</div>

Firecrawl utilise des codes d&#39;état HTTP standard pour indiquer le résultat de vos requêtes. Les codes de la plage `2xx` indiquent une réussite, les codes `4xx` indiquent des erreurs côté client et les codes `5xx` indiquent des erreurs côté serveur.

Voir [Errors](/fr/api-reference/errors) pour la référence complète, y compris la chaîne `error` renvoyée pour chaque type d&#39;échec, des recommandations de réessai et un extrait de backoff exponentiel à copier-coller.

<div id="429-responses">
  ## Réponses 429
</div>

Lorsque vous dépassez les limites de débit ou de concurrence de votre offre, l’API renvoie un code d’état `429`. Consultez [Limites de débit](/fr/rate-limits) pour connaître les limites par offre et [Errors](/fr/api-reference/errors) pour savoir comment réessayer.
