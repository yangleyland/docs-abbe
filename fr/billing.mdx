<div id="overview">
  ## Vue d&#39;ensemble
</div>

Firecrawl utilise un **système de facturation basé sur des crédits**. Chaque appel à l&#39;API que vous effectuez consomme des crédits, et le nombre de crédits consommés dépend du point de terminaison et des options que vous utilisez. Vous disposez d&#39;un quota mensuel de crédits en fonction de votre offre, et Smart Upgrade vous accompagne à mesure que votre utilisation augmente.

Pour consulter les tarifs actuels, rendez-vous sur la [page de tarification Firecrawl](https://www.firecrawl.dev/pricing).

<Note>
  Toutes les factures Firecrawl sont émises en **dollars américains (USD)**, quelle que soit votre adresse de facturation ou votre mode de paiement.
</Note>

<div id="credits">
  ## Crédits
</div>

Les crédits sont l’unité d’utilisation dans Firecrawl. Chaque offre inclut un quota mensuel de crédits qui est réinitialisé au début de chaque cycle de facturation. Différents endpoints d’API consomment un nombre de crédits différent.

<div id="credit-costs-per-endpoint">
  ### Coût en crédits par point de terminaison
</div>

| Point de terminaison | Coût en crédits                  | Notes                                                                                                                                                                                                                                                                                                                                                             |
| -------------------- | -------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Scrape**           | 1 crédit / page                  | Convertit une URL unique en markdown propre, HTML ou données structurées. Des crédits supplémentaires s’appliquent lors de l’utilisation des options d’extraction (voir ci-dessous).                                                                                                                                                                              |
| **Crawl**            | 1 crédit / page                  | Scrape un site web entier en suivant les liens à partir d’une URL de départ. Les mêmes coûts des options d’extraction par page s’appliquent à chaque page explorée.                                                                                                                                                                                               |
| **Map**              | 1 crédit / appel                 | Découvre toutes les URL d’un site web sans scraper leur contenu.                                                                                                                                                                                                                                                                                                  |
| **Search**           | 2 crédits / 10 résultats         | Recherche sur le web et, en option, scrape les résultats. Arrondi à la tranche supérieure par groupe de 10 résultats (par ex., 11 résultats = 4 crédits). Des coûts supplémentaires de scraping par page s’appliquent à chaque résultat qui est scrapé. Voir [ici](/fr/features/search#zero-data-retention-zdr) pour la tarification de la recherche ZDR Enterprise. |
| **Interact**         | 2 crédits / minute de navigateur | Session de bac à sable navigateur interactive, facturée à la minute.                                                                                                                                                                                                                                                                                              |
| **Agent**            | Dynamique                        | Agent autonome de recherche web. 5 exécutions quotidiennes gratuites ; tarification à l’usage au-delà de ce quota.                                                                                                                                                                                                                                                |

<div id="additional-credit-costs-for-scrape-options">
  ### Coûts de crédits supplémentaires pour les options d’extraction
</div>

Certaines options d’extraction ajoutent des crédits en plus du coût de base par page :

| Option                       | Coût supplémentaire  | Description                                                                                                                   |
| ---------------------------- | -------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| PDF parsing                  | +1 crédit / page PDF | Extraire le contenu de documents PDF                                                                                          |
| JSON format (LLM extraction) | +4 crédits / page    | Utiliser un LLM pour extraire des données JSON structurées à partir de la page                                                |
| Enhanced Mode                | +4 crédits / page    | Scrape amélioré pour les pages difficiles d’accès                                                                             |
| Zero Data Retention (ZDR)    | +1 crédit / page     | Garantit qu’aucune donnée n’est conservée au-delà de la requête (voir [Scrape ZDR](/fr/features/scrape#zero-data-retention-zdr)) |

Ces modificateurs se cumulent. Par exemple, scraper une page avec à la fois le format JSON et le mode Enhanced coûte **1 + 4 + 4 = 9 crédits** par page. Ces mêmes modificateurs s’appliquent aux points de terminaison Crawl et Search, puisqu’ils utilisent `scrape` en interne pour chaque page.

Les requêtes vers `x.com` et les autres URL X/Twitter utilisent l’API Grok et font l’objet d’une tarification distincte. Voir [la facturation de X (x.com)](#x-xcom-billing) en bas de cette page.

<div id="when-credits-are-charged">
  ### Quand les crédits sont débités
</div>

Les crédits sont débités chaque fois que l’infrastructure de Firecrawl traite une requête, même si le site cible renvoie un code d’erreur HTTP tel que 403 Forbidden ou 404 Not Found. En effet, l’infrastructure de scraping (rendu du navigateur, proxy, etc.) est entièrement mobilisée, quelle que soit la réponse du site cible. Vous pouvez vérifier le champ `metadata.statusCode` dans la réponse de l’API pour détecter ces cas et éviter de relancer des URL systématiquement bloquées.

Pour les tâches de **batch scrape** et de **crawl**, les crédits sont facturés de manière asynchrone à mesure que chaque page est traitée — et non au moment où la tâche est soumise. Cela signifie qu’il peut y avoir un délai entre l’envoi d’une tâche et le moment où le coût total en crédits apparaît sur votre compte. Si un batch contient de nombreuses URL ou si des pages sont mises en file d’attente pendant des périodes de fort trafic, des crédits peuvent continuer à être débités et à apparaître minutes ou heures après la soumission. Le polling ou la consultation du statut d’un batch ne consomment pas de crédits.

<Note>
  **Vérification préalable des crédits pour le crawl :** Avant qu’une tâche de crawl ne démarre, Firecrawl vérifie que votre solde de crédits restant peut couvrir l’intégralité du paramètre `limit` demandé. Si votre solde est inférieur à `limit`, la requête renvoie un 402, même si le crawl aurait découvert moins de pages. La valeur par défaut de `limit` est **10,000** ; l’omettre exige donc de disposer de 10,000 crédits à l’avance. Pour éviter cela, transmettez un `limit` explicite correspondant au nombre de pages que vous avez réellement l’intention de parcourir (par exemple, `limit: 100`).
</Note>

<div id="tracking-your-usage">
  ### Suivi de votre consommation de crédits
</div>

Vous pouvez suivre votre consommation de crédits de deux manières :

* **Tableau de bord** : consultez votre consommation actuelle et historique sur [firecrawl.dev/app](https://www.firecrawl.dev/app)
* **API** : utilisez les points de terminaison [Credit Usage](/fr/api-reference/endpoint/credit-usage) et [Credit Usage Historical](/fr/api-reference/endpoint/credit-usage-historical) pour vérifier votre consommation par programmation

<Note>
  Nous travaillons activement sur des améliorations pour rendre la consommation de crédits plus facile à comprendre. Restez informé des prochaines mises à jour.
</Note>

<div id="plans">
  ## Offres
</div>

Firecrawl propose des offres mensuelles par abonnement. Il n&#39;existe pas d&#39;offre purement à l&#39;usage, mais Smart Upgrade (décrit ci‑dessous) permet une mise à l&#39;échelle flexible au‑delà de votre offre de base.

<div id="paid-plans">
  ### Offres payantes
</div>

| Offre        | Crédits mensuels            | Navigateur concurrent |
| ------------ | --------------------------- | --------------------: |
| **Hobby**    | 5,000 / 6,500 / 8,000       |                     5 |
| **Standard** | 100,000 / 130,000 / 160,000 |                    50 |
| **Growth**   | 500,000 / 650,000           |                   100 |
| **Scale**    | 1,000,000                   |                   150 |

<Note>
  Pour des besoins au‑delà de Scale, Firecrawl propose des offres **Enterprise** avec des crédits personnalisés, une assistance dédiée, des SLA, des remises sur volume, aucune rétention de données et le SSO. Consultez la [page Enterprise](https://www.firecrawl.dev/enterprise) pour plus de détails.
</Note>

Toutes les offres payantes sont disponibles avec une facturation **mensuelle** ou **annuelle**. La facturation annuelle propose une remise par rapport au paiement mensuel. Pour les tarifs actuels de chaque offre, consultez la [page de tarification](https://www.firecrawl.dev/pricing).

<div id="billing-cycle">
  ### Cycle de facturation
</div>

* **Offres mensuelles** : les crédits sont réinitialisés à votre date de renouvellement mensuelle
* **Offres annuelles** : vous êtes facturé une fois par an, mais les crédits sont tout de même réinitialisés chaque mois à votre date de renouvellement mensuelle virtuelle
* **Par défaut, les crédits d’offre inutilisés ne sont pas reportés** — votre quota mensuel est réinitialisé chaque mois. **Les offres annuelles Scale reportent les crédits d’offre inutilisés pendant 1 mois**, et **les offres annuelles Enterprise les reportent pendant 2 mois**.

<div id="concurrent-browsers">
  ### Navigateurs simultanés
</div>

Les navigateurs simultanés correspondent au nombre de pages web que Firecrawl peut traiter en parallèle pour vous. Votre offre détermine cette limite. Si vous la dépassez, les tâches supplémentaires attendent dans une file d’attente jusqu’à ce qu’un créneau se libère. Consultez [Limites de débit](/fr/rate-limits) pour tous les détails sur la concurrence et les limites de débit de l’API.

<div id="smart-upgrade">
  ## Smart Upgrade
</div>

<Note>
  À partir du **1er juin 2026**, Smart Upgrade remplace Auto-Recharge. Au lieu de payer des dépassements, ces paiements sont désormais appliqués à vos passages à l’offre supérieure, pour une expérience fluide et sans interruption.
</Note>

Lorsque votre solde atteint zéro, Smart Upgrade vous fait automatiquement passer au niveau supérieur dans l’échelle des crédits, y compris aux paliers intermédiaires de chaque offre, afin que les requêtes continuent d’être traitées sans interruption. Seule la différence de prix calculée au prorata entre votre palier actuel et le suivant vous est facturée, et vous recevez immédiatement le delta de crédits correspondant. Vous pouvez désactiver Smart Upgrade à tout moment dans les [paramètres de facturation](https://www.firecrawl.dev/app/settings?tab=billing). Une fois le palier Scale atteint (1 000 000 crédits/mois), il n’existe plus de palier supérieur vers lequel Smart Upgrade peut vous faire passer ; [contactez l’équipe commerciale](https://www.firecrawl.dev/enterprise) pour une offre Enterprise personnalisée.

<div id="pro-ration-examples">
  ### Exemples de calcul au prorata
</div>

Supposons que vous soyez sur **Hobby 6.5k** ($28/mois ou $290/an) et que Smart Upgrade vous fasse passer à **Hobby 8k** ($37/mois ou $390/an). Vous recevez immédiatement **1 500 crédits supplémentaires** (l’écart jusqu’à 8k), et les frais de mise à niveau dépendent de votre mode de facturation :

* **Offre mensuelle :** la différence mensuelle de $37 − $28 = **$9** vous est facturée aujourd’hui. Votre date de renouvellement ne change pas.
* **Offre annuelle, avec 5 mois restants sur votre cycle de 12 mois :** le montant `(5 ÷ 12) × 100 ≈ $41.67` vous est facturé aujourd’hui, la différence de prix annuelle étant de $390 − $290 = $100. Les 4 mois restants sur Hobby 8k sont couverts par ces frais, et votre date de renouvellement ne change pas.

Lorsque vous épuisez le palier 8k, l’étape suivante est **Hobby 8k → Standard 100k**, facturée sur la base de la différence entre ces deux paliers ($62/mois ou $600/an, calculés au prorata de la même manière), avec **92 000 crédits supplémentaires** ajoutés à votre solde.

<div id="upgrading-and-downgrading">
  ## Mise à niveau et rétrogradation
</div>

* Les **mises à niveau** prennent effet immédiatement. Le prix complet de la nouvelle offre vous est facturé aujourd’hui (sans prorata), et votre cycle de facturation est réinitialisé — votre prochain renouvellement aura lieu un mois ou un an après la date de mise à niveau. Tous les crédits inutilisés de votre offre précédente sont reportés, et votre nouveau quota de crédits ainsi que vos limites de concurrence s’appliquent immédiatement.
* Les **rétrogradations** sont programmées pour prendre effet à votre prochaine date de renouvellement. Vous conservez les crédits et limites de votre offre actuelle jusqu’à cette date, et le temps inutilisé sur votre offre actuelle n’est ni crédité ni remboursé. Vous pouvez annuler une rétrogradation programmée depuis vos [paramètres de facturation](https://www.firecrawl.dev/app/settings?tab=billing) à tout moment avant la date d’effet.

<div id="switching-between-monthly-and-yearly-billing">
  ### Basculer entre la facturation mensuelle et annuelle
</div>

* **Mensuel → Annuel** au même niveau de crédits ou à un niveau supérieur est considéré comme une mise à niveau immédiate.
* **Annuel → Mensuel** est considéré comme une mise à niveau immédiate uniquement si vous passez à un niveau de crédits strictement supérieur.

<div id="running-out-of-credits">
  ## Vous n’avez plus de crédits
</div>

Si vous épuisez votre quota de crédits et que Smart Upgrade n’est pas activé, les requêtes API qui consomment des crédits renverront une erreur **HTTP 402 (Paiement requis)**.

Avec **Smart Upgrade** activé, votre abonnement passe automatiquement au palier de crédits supérieur et vous recevez immédiatement le quota de crédits de ce nouveau palier, afin que les requêtes continuent sans interruption. Smart Upgrade s’arrête une fois le palier Scale atteint.

Pour reprendre l’utilisation après un arrêt complet, vous pouvez :

1. Activer [Smart Upgrade](#smart-upgrade) pour passer automatiquement au niveau supérieur
2. Passer manuellement à une offre supérieure
3. Attendre que vos crédits soient réinitialisés au prochain cycle de facturation

<div id="coupons">
  ## Coupons
</div>

Firecrawl prend en charge deux types de coupons :

* **Les coupons d’abonnement** appliquent une réduction à votre abonnement (par ex. un pourcentage de réduction sur votre tarif mensuel ou annuel). Ils peuvent **uniquement** être appliqués pendant le processus de paiement Stripe, lorsque vous souscrivez pour la première fois à une offre payante ou changez d’offre. Vous ne pouvez pas appliquer un coupon d’abonnement une fois le paiement terminé.
* **Les coupons de crédit** ajoutent des crédits bonus à votre compte. Ils peuvent être utilisés depuis la section **Facturation** de votre tableau de bord à l’adresse [firecrawl.dev/app/billing](https://www.firecrawl.dev/app/billing). Recherchez le champ de saisie du coupon sur la page de facturation pour appliquer votre code. Les crédits bonus issus des coupons de crédit sont distincts du quota mensuel de votre offre et sont conservés même si vous passez à une offre supérieure ou inférieure.

<AccordionGroup>
  <Accordion title="Les crédits non utilisés sont-ils reportés au mois suivant ?">
    **Les crédits de l’offre** ne sont pas reportés par défaut — votre quota mensuel est réinitialisé chaque mois. **Les offres annuelles Scale reportent les crédits d’offre non utilisés pendant 1 mois**, et **les offres annuelles Enterprise les reportent pendant 2 mois**.
  </Accordion>

  <Accordion title="Comment le coût de Smart Upgrade est-il calculé ?">
    Smart Upgrade facture la différence au prorata entre votre palier actuel et le palier suivant. Pour les offres **mensuelles**, il s’agit de la différence de prix mensuelle. Pour les offres **annuelles**, c’est `(mois restants dans votre cycle de facturation ÷ 12) × différence de prix annuel`, en utilisant des mois calendaires complets à partir du début de votre période actuelle. Vous recevez également l’écart de crédits entre les deux paliers, vous ne payez donc que ce que vous gagnez.
  </Accordion>

<div id="x-xcom-billing">
  ## Facturation X (x.com)
</div>

Firecrawl utilise l&#39;**API Grok** officielle de [xAI](https://x.ai/) pour fournir une synthèse par IA, une extraction structurée et un accès en temps réel au contenu public de X. Les requêtes visant des URL de profils et de publications sur `x.com`, `twitter.com` et `mobile.twitter.com` sont traitées via les outils internes autorisés de Grok (`x_search`, récupération de fils et recherche web limitée à x.com) plutôt que par du scraping web traditionnel.

<div id="credit-costs">
  ### Coût en crédits
</div>

| Composant        | Coût en crédits       | Description                                                            |
| ---------------- | --------------------- | ---------------------------------------------------------------------- |
| **Coût de base** | 1 crédit / requête    | Traitement standard d’une requête de scraping                          |
| **Grok X Query** | +29 crédits / requête | Utilisation de l’API Grok (tokens + appels d’outils) pour le contenu X |

Par exemple, le traitement d’une requête typique sur une publication ou un fil coûte **30 crédits** (`1` de base + `29` Grok X Query) et renvoie des données structurées générées par Grok, le contexte du fil et des résumés. Si le format JSON (extraction LLM) est également activé, le total est de **34 crédits** par requête.

Cette méthode respecte les interfaces publiées par X dans le cadre de son partenariat avec xAI et fournit des résultats de meilleure qualité, plus réfléchis, plutôt qu’un simple scraping brut de la page.

<Note>
  **Les capacités diffèrent du scraping standard.** Grok renvoie des résultats traités par l’IA, qui peuvent inclure des résumés, des indicateurs clés, le contexte du fil, etc. Pour obtenir des données structurées brutes à grande échelle, utilisez [l’API officielle X Enterprise](https://developer.x.com/).
</Note>
