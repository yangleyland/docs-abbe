<div id="introducing-extract-beta">
  ## Présentation de /extract (bêta)
</div>

<Warning>
  [Veuillez utiliser la nouvelle documentation Open Beta de /extract en cliquant ici 🔥](/fr/features/extract)
</Warning>

Extrayez des données structurées à partir d’une ou de plusieurs URL, ou de sites entiers, à l’aide de modèles de langage (LLMs). Notre nouveau point de terminaison `/extract` vous permet de :

* Extraire d’un coup des données structurées depuis des sites web complets
* Connecter ou créer des applications d’enrichissement de données nécessitant des données structurées issues de sites web
* Développer des applications d’IA ayant besoin de données propres provenant de plusieurs sites web

<div id="considerations">
  ## Considérations
</div>

Le point de terminaison `/extract` offre une extraction de données flexible avec des schémas personnalisables. Les résultats peuvent être améliorés en ajustant le prompt. Il est actuellement en bêta et nous accueillons volontiers vos retours.

<div id="extracting-data">
  ## Extraction de données
</div>

<div id="extract-endpoint">
  ### point de terminaison /extract
</div>

Utilisé pour extraire des données structurées à partir de sites web entiers.

Lors de la saisie des URL, vous pouvez ajouter `/*` à l’URL pour extraire des informations sur l’ensemble du chemin du site plutôt que sur une seule page.

Par exemple, `https://firecrawl.dev/*` tentera d’extraire des données de toutes les pages du domaine firecrawl.dev. La prise en charge de `/*` est encore en test ; signalez tout problème à help@firecrawl.com.

<div id="pending-response">
  #### Réponse en attente
</div>

Les tâches d’extraction peuvent avoir l’un des états suivants :

* `completed` : la tâche d’extraction s’est terminée avec succès.
* `processing` : la tâche d’extraction est toujours en cours.
* `failed` : la tâche d’extraction a rencontré une erreur et n’a pas abouti.
* `cancelled` : la tâche d’extraction a été annulée par l’utilisateur.

<div id="improving-results-with-web-search-external-links">
  ### Améliorer les résultats avec la recherche web et les liens externes
</div>

Si vous souhaitez améliorer les résultats de l’extraction, vous pouvez transmettre le paramètre `enableWebSearch` au point de terminaison. Cela permettra de chercher les données via des liens externes — en dehors du périmètre des URL fournies.

<div id="billing">
  ### Facturation
</div>

Pendant que `/extract` est en bêta, nous facturons 5 crédits par URL scrappée utilisée pour composer la réponse finale. Cela vise à prévenir les abus. Cela sera amené à changer à l’avenir.
