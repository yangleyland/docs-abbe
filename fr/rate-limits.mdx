Les limites de débit plafonnent le nombre de requêtes que votre équipe peut effectuer par minute, tandis que les limites de concurrence plafonnent le nombre de jobs pouvant s’exécuter en parallèle. Les deux sont définies par votre offre. Lorsque vous dépassez une limite de débit ou de concurrence, l’API renvoie une réponse `429`. Consultez [Erreurs](/fr/api-reference/errors) pour la liste complète des erreurs et un exemple de nouvelle tentative avec backoff exponentiel.

<div id="billing-model">
  ## Modèle de facturation
</div>

Firecrawl utilise des abonnements mensuels. Il n’existe pas de modèle de paiement à l&#39;usage, mais la **fonction de rechargement automatique** offre une mise à l’échelle flexible. Une fois que vous avez souscrit à une offre, vous pouvez acheter automatiquement des crédits supplémentaires lorsque vous passez sous un certain seuil. Les packs de rechargement automatique plus importants offrent de meilleurs tarifs. Pour tester avant de vous engager sur une offre de niveau supérieur, commencez par l’offre Free ou Hobby.

Les changements vers une offre inférieure prennent effet au prochain renouvellement. Aucun avoir correspondant au temps non utilisé n’est émis.

<div id="concurrent-browser-limits">
  ## Limites de navigateurs concurrents
</div>

Les navigateurs concurrents déterminent le nombre de pages que Firecrawl peut traiter pour vous en parallèle. Votre offre fixe la limite ; les tâches au-delà de celle-ci patienteront dans une file d’attente jusqu’à ce qu’un navigateur se libère.

Le temps passé dans la file d’attente est décompté du paramètre [`timeout`](/fr/advanced-scraping-guide#timing-and-cache) de la requête, vous pouvez donc définir un délai d’expiration plus court pour provoquer un échec rapide plutôt que d’attendre. Pour vérifier la disponibilité actuelle avant d’envoyer une tâche, appelez l’endpoint [Queue Status](/fr/api-reference/endpoint/queue-status). Les tâches en attente dans votre file d’attente de concurrence expireront au maximum après 48 heures.

Chaque équipe a un nombre maximal de tâches pouvant attendre dans la file d’attente de concurrence. Si vous dépassez cette limite, les nouvelles tâches seront rejetées avec un code d’état `429` jusqu’à ce que les tâches existantes soient terminées. Pour les offres plus élevées avec des limites de concurrence personnalisées, le nombre maximal de tâches en file d’attente est égal à 2 000 fois votre limite de concurrence, avec un plafond de 2 000 000.

Si vous avez besoin de limites de concurrence plus élevées, [contactez-nous pour les offres Enterprise](https://firecrawl.dev/enterprise).

<div id="api-rate-limits">
  ## Limites de débit de l&#39;API
</div>

Les limites de débit sont mesurées en requêtes par minute et servent principalement à prévenir les abus. Lorsqu’elles sont correctement configurées, votre véritable goulot d’étranglement sera le nombre de navigateurs concurrents. Les limites de débit s’appliquent par équipe, de sorte que toutes les clés API d’une même équipe partagent les mêmes compteurs de limite de débit.
