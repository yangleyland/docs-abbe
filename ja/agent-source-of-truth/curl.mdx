エージェント向けの、Firecrawl の curl に関する信頼できる唯一の参照元です。`api-reference/v2-openapi.json` (サーバーのベース URL は `https://api.firecrawl.dev/v2`) と整合しています。

<div id="authenticate">
  ## 認証
</div>

```bash
export FIRECRAWL_API_KEY="fc-your-api-key"
```

<div id="when-to-use-what">
  ## 用途別の使い分け
</div>

* `search`: クエリから始めて、見つける必要がある場合に使用します。
* `scrape`: すでに URL があり、ページのコンテンツを取得したい場合に使用します。
* `interact`: クリックやフォーム入力、またはスクレイピング後のブラウザアクションが必要なページで使用します。
* `support/ask`: Firecrawl API の呼び出しが失敗した場合や、想定外の結果が返ってきて原因の診断が必要な場合に使用します。
* `support/docs-search`: Firecrawl のドキュメントを調べたい場合に使用します。

<div id="search">
  ## Search
</div>

<div id="why-use-it">
  ### 使用する理由
</div>

クエリから関連ページを見つけるには検索を使用し、その後スクレイピングまたはInteractするURLを選択します。`site:` を使って結果をサイトに絞り込むことができます。たとえば `site:docs.firecrawl.dev crawl webhooks` です。

<div id="endpoint">
  ### エンドポイント
</div>

`POST /search`

<div id="simple-example">
  ### シンプルな例
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/search" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "site:docs.firecrawl.dev webhook retries"
  }'
```

<div id="complex-example">
  ### 複雑な例
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/search" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "site:docs.firecrawl.dev crawl webhooks",
    "sources": [{"type": "web"}, {"type": "news"}],
    "categories": [{"type": "research"}],
    "limit": 10,
    "tbs": "qdr:m",
    "location": "San Francisco,California,United States",
    "country": "US",
    "ignoreInvalidURLs": true,
    "timeout": 60000,
    "enterprise": ["anon"],
    "scrapeOptions": {
      "formats": [
        "markdown",
        "links",
        {"type": "json", "prompt": "Extract title and key endpoints."}
      ],
      "onlyMainContent": true,
      "includeTags": ["main", "article"],
      "excludeTags": ["nav", "footer"],
      "waitFor": 1000
    }
  }'
```

<div id="response">
  ### レスポンス
</div>

成功したレスポンスには、`success`、`data`、任意の `warning`、`id`、`creditsUsed` が含まれます。

* `data.web`、`data.images`、`data.news`: 結果の配列です。どのキーが含まれるかは `sources` によって異なります (デフォルトでは `data.web` のみが設定されます) 。
* `web` および `news` の項目には、`title`、`url`、および (`scrapeOptions` / フォーマットで要求された場合) `markdown`、`html`、`rawHtml`、`links`、`screenshot`、`audio`、`video`、`metadata` などのフィールドが含まれます。
* 画像項目には、`imageUrl`、`url`、および利用可能な場合はサイズなどのフィールドが含まれます。
* `warning`: 任意の人間が読める通知です。
* `id`: 検索ジョブ ID の文字列です。
* `creditsUsed`: この呼び出しで消費されたクレジット数を表す整数です。

<div id="parameters">
  ### パラメータ
</div>

* `query`
  * 型: string (必須、最大長 500) 
  * 使用する場合: 検索クエリが必要な場合。
  * 注記: 結果を特定のドメインに限定するには、`site:example.com` を使用します。

* `sources`
  * 型: 型付きソースオブジェクトの配列
  * 使用する場合: 検索対象のソースを制御したい場合。
  * 確認済みのオブジェクト形式:
    * `{ "type": "web" }` (ソースごとの `tbs` と `location` は任意) 
    * `{ "type": "news" }`
    * `{ "type": "images" }`

* `categories`
  * 型: 型付きカテゴリオブジェクトの配列
  * 使用する場合: カテゴリで結果を絞り込みたい場合。
  * 確認済みのオブジェクト形式:
    * `{ "type": "github" }`
    * `{ "type": "research" }`
    * `{ "type": "pdf" }`

* `limit`
  * 型: integer (最小 1、最大 100、デフォルト 5) 
  * 使用する場合: 結果数に上限を設定したい場合。

* `tbs`
  * 型: string
  * 使用する場合: 時間ベースの絞り込みが必要な場合 (例: `qdr:d`、`qdr:w`、`sbd:1,qdr:m`) 。

* `location`
  * 型: string
  * 使用する場合: ローカライズされた結果を取得したい場合。

* `country`
  * 型: string (デフォルト `"US"`) 
  * 使用する場合: ISO 3166-1 alpha-2 で対象国を指定したい場合 (例: `"US"`) 。

* `ignoreInvalidURLs`
  * 型: boolean (デフォルト false) 
  * 使用する場合: 他のエンドポイントでスクレイピングできない URL を除外したい場合。

* `timeout`
  * 型: integer (ミリ秒、デフォルト 60000) 
  * 使用する場合: ミリ秒単位のリクエストタイムアウトが必要な場合。

* `enterprise`
  * 型: 文字列の配列 (各項目は `"anon"` または `"zdr"`) 
  * 使用する場合: エンタープライズ検索の制御が必要な場合。
  * 値:
    * `"zdr"`: エンドツーエンドのゼロデータ保持
    * `"anon"`: 匿名化されたゼロデータ保持

* `scrapeOptions`
  * 型: object
  * 使用する場合: 各検索結果をスクレイピングしたい場合 (フィールドについては Scrape パラメータを参照) 。

<div id="scrape">
  ## スクレイピング
</div>

### 使う理由

URL がすでにあり、1 つ以上のフォーマットで構造化されたコンテンツを取得したい場合は、スクレイピングを使用します。

<div id="endpoint">
  ### エンドポイント
</div>

`POST /scrape`

<div id="simple-example">
  ### シンプルな例
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://docs.firecrawl.dev",
    "formats": ["markdown"]
  }'
```

### 高度な例

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/pricing",
    "formats": [
      "markdown",
      "links",
      {"type": "json", "prompt": "Extract plan names and prices."},
      {"type": "screenshot", "fullPage": true, "quality": 80, "viewport": {"width": 1280, "height": 720}},
      {"type": "changeTracking", "modes": ["git-diff"], "tag": "pricing"}
    ],
    "headers": {"User-Agent": "FirecrawlDocsBot/1.0"},
    "onlyMainContent": true,
    "waitFor": 1000,
    "parsers": [{"type": "pdf", "mode": "auto", "maxPages": 5}],
    "actions": [
      {"type": "click", "selector": "#accept"},
      {"type": "wait", "milliseconds": 750},
      {"type": "scrape"}
    ],
    "location": {"country": "US", "languages": ["en-US"]},
    "removeBase64Images": true,
    "blockAds": true,
    "proxy": "auto",
    "maxAge": 86400000,
    "minAge": 1,
    "storeInCache": true,
    "profile": {"name": "docs-session", "saveChanges": true},
    "zeroDataRetention": false
  }'
```

<div id="response">
  ### レスポンス
</div>

成功したレスポンスには `success` と `data` が含まれます。主な `data` フィールド (`formats` と options に応じて異なります) :

* `markdown`, `summary`, `html`, `rawHtml`, `screenshot`, `audio`, `video`, `links`
* `actions`: リクエストにスクレイピング時の `actions` が含まれていた場合、`screenshots`、`scrapes`、`javascriptReturns`、`pdfs` などの順序どおりの結果が含まれます
* `metadata`: ページのメタデータ (`title`、`sourceURL`、`url`、`statusCode`、`error`、およびその他の抽出されたフィールド)
* `warning`: 抽出またはフォーマットに関する任意の通知
* `changeTracking`: `changeTracking` フォーマットがリクエストされた場合に含まれます

<div id="parameters">
  ### パラメータ
</div>

* `url`
  * Type: string
  * 使用する場面: 特定のページをスクレイピングしたい場合。

* `formats`
  * Type: フォーマット文字列またはフォーマットオブジェクトの配列
  * 使用する場面: 複数の出力フォーマットが必要な場合。
  * 確認済みのフォーマット文字列:
    * `"markdown"`: Markdownコンテンツ
    * `"html"`: クリーンなHTML
    * `"rawHtml"`: 生のHTML
    * `"links"`: ページリンク
    * `"images"`: 画像URL
    * `"screenshot"`: スクリーンショット出力
    * `"summary"`: 要約出力
    * `"changeTracking"`: changeTracking出力
    * `"json"`: JSON抽出
    * `"branding"`: ブランドプロファイル出力
    * `"audio"`: 音声抽出
    * `"video"`: 動画抽出
  * フォーマットオブジェクトのフィールド:
    * `type`: 上記のフォーマット文字列のいずれか
    * `prompt`, `schema`: `type: "json"` のJSON抽出オプション
    * `modes`, `schema`, `prompt`, `tag`: `type: "changeTracking"` のchangeTrackingオプション
    * `fullPage`, `quality`, `viewport`: `type: "screenshot"` のスクリーンショットオプション

* `headers`
  * Type: object
  * 使用する場面: カスタムのリクエストヘッダーが必要な場合。

* `includeTags`
  * Type: stringの配列
  * 使用する場面: 特定のHTMLタグだけを含めたい場合。

* `excludeTags`
  * Type: stringの配列
  * 使用する場面: 特定のHTMLタグを除外したい場合。

* `onlyMainContent`
  * Type: boolean
  * 使用する場面: nav、footer、その他の定型要素を取り除きたい場合。

* `timeout`
  * Type: number
  * 使用する場面: ミリ秒単位のタイムアウトが必要な場合。

* `waitFor`
  * Type: number
  * 使用する場面: ページのレンダリング完了を待つ必要がある場合 (ミリ秒) 。

* `mobile`
  * Type: boolean
  * 使用する場面: モバイル用のビューポートを使いたい場合。

* `parsers`
  * Type: オブジェクトの配列
  * 使用する場面: ファイルのパースを制御したい場合。
  * 確認済みの形式: `{ "type": "pdf", "mode": "fast" | "auto" | "ocr", "maxPages": number }` (`type` は必須、他のフィールドは仕様に応じたデフォルト値付きで任意)

* `actions`
  * Type: アクションオブジェクトの配列
  * 使用する場面: 軽量なスクレイピング前アクションが必要な場合。
  * 確認済みのアクションタイプ:
    * `wait`: `milliseconds` または `selector` が必須
    * `screenshot`: `fullPage`, `quality`, `viewport` は任意
    * `click`: `selector` が必須、`all` は任意
    * `write`: `text` が必須 (先にクリックして入力欄へフォーカス)
    * `press`: `key` が必須
    * `scroll`: `type` が必須; `direction` (`up` または `down`、デフォルトは `down`); `selector` は任意
    * `scrape`: 追加フィールドなし
    * `executeJavascript`: `script` が必須
    * `pdf`: `format` (A0, A1, A2, A3, A4, A5, A6, Letter, Legal, Tabloid, Ledger) 、`landscape`、`scale` は任意

* `location`
  * Type: `country` と `languages` を含むobject
  * 使用する場面: 地理情報または言語を考慮したスクレイピングが必要な場合。

* `skipTlsVerification`
  * Type: boolean
  * 使用する場面: TLSの確認をスキップする必要がある場合。

* `removeBase64Images`
  * Type: boolean
  * 使用する場面: Markdown出力からbase64画像を除外したい場合。

* `blockAds`
  * Type: boolean
  * 使用する場面: 広告やCookieポップアップをブロックしたい場合。

* `proxy`
  * Type: string
  * 使用する場面: プロキシを制御したい場合。
  * 確認済みの値: `"basic"`, `"enhanced"`, `"auto"`

* `maxAge`
  * Type: number
  * 使用する場面: 指定した最大経過時間までのキャッシュデータを使いたい場合 (ミリ秒) 。

* `minAge`
  * Type: number
  * 使用する場面: 少なくともこの時間が経過したキャッシュデータのみを使いたい場合 (ミリ秒) 。

* `storeInCache`
  * Type: boolean
  * 使用する場面: Firecrawlに結果をキャッシュさせたい場合。

* `profile`
  * Type: `name` と任意の `saveChanges` を含むobject
  * 使用する場面: スクレイピングや操作のあいだで共有される永続的なブラウザプロファイルを使いたい場合。

* `zeroDataRetention`
  * Type: boolean
  * 使用する場面: このスクレイピングでゼロデータ保持を有効にしたい場合。

<div id="interact">
  ## Interact
</div>

### なぜ使うのか

ページで、スクレイピングの開始後にブラウザのアクションやコードの実行が必要な場合は、Interact を使用します。

<div id="endpoint">
  ### エンドポイント
</div>

`POST /scrape/{jobId}/interact`

### 簡単な例

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape/<jobId>/interact" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "code": "console.log(await page.title());"
  }'
```

<div id="complex-example">
  ### 複雑な例
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape/<jobId>/interact" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "code": "const title = await page.title();\nconst h1 = await page.locator(\"h1\").first().textContent().catch(() => null);\nconsole.log(JSON.stringify({ title, h1 }));",
    "language": "node",
    "timeout": 120
  }'
```

<div id="parameters">
  ### パラメータ
</div>

* `jobId` (path)
  * Type: string (UUID)
  * 使用する場合: ライブのブラウザセッション用のスクレイピングジョブ ID がある場合。

* `code` (JSON body)
  * Type: string (OpenAPI では必須。最小長 1、最大長 100000)
  * 使用する場合: スクレイピングに関連付けられたブラウザサンドボックスでコードを実行したい場合。

* `language` (JSON body)
  * Type: string
  * 使用する場合: 特定のランタイムが必要な場合。
  * 確認済みの値: `"python"`, `"node"`, `"bash"` (デフォルトは `"node"`)

* `timeout` (JSON body)
  * Type: integer (秒。最小 1、最大 300、デフォルト 30)
  * 使用する場合: 実行タイムアウトを指定する必要がある場合。

<div id="response">
  ### レスポンス
</div>

成功時のレスポンスには、`success` に加えて、`stdout`、`result` (`stdout` の別名) 、`stderr`、`exitCode`、`killed`、`error` (null 許容) などの実行フィールドが含まれます。

### DELETE /scrape/{jobId}/interact

<div id="example">
  ### 例
</div>

```bash
curl -X DELETE "https://api.firecrawl.dev/v2/scrape/<jobId>/interact" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY"
```

JSONボディはありません。成功時のレスポンスには `success` が含まれます。

<div id="ask-agentic-debugging">
  ## Ask (エージェント型デバッグ)
</div>

<div id="why-use-it">
  ### 使用する理由
</div>

Firecrawl API の呼び出しが失敗した場合や、想定外の結果が返された場合は、`ask` を使用します。AI サポートエージェントが問題を診断し、修正用のパラメータを提案し、必要に応じてその修正を実際の API に対して検証します。通常のレイテンシ: 15〜30 秒。

<div id="endpoint">
  ### エンドポイント
</div>

`POST /support/ask`

### 基本的な例

```bash
curl -X POST "https://api.firecrawl.dev/v2/support/ask" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "my scrape returned empty markdown for https://example.com"
  }'
```

### 高度な例

```bash
curl -X POST "https://api.firecrawl.dev/v2/support/ask" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "crawl returned 3 pages but I expected 50",
    "rationale": "user is on their third failed crawl attempt today",
    "context": {"userPlan": "standard", "retryCount": 3}
  }'
```

<div id="response">
  ### レスポンス
</div>

成功したレスポンスには、`requestId`、`answer`、`confidence`、`fixParameters`、`validation`、`usage`、`durationMs` が含まれます。

* `answer`: 診断内容と修正方法を説明する 2〜4 文の文章。
* `confidence`: `high`、`medium`、または `low`。
* `fixParameters`: 修正を適用するための、機械的に処理可能な API パラメータ (適用可能な修正がない場合は null) 。
* `validation.tested`: エージェントが修正をライブ API でテストしたかどうか。
* `validation.result`: `success`、`failure`、または `skipped`。
* `feedback`: エージェントが行き詰まった場合に含まれ、成功時は null。

<div id="parameters">
  ### パラメータ
</div>

* `question`
  * 型: string (必須、1〜8000文字) 
  * 使用する場面: 問題の内容を説明する必要がある場合。

* `rationale`
  * 型: string (1〜2000文字) 
  * 使用する場面: AIエージェントがユーザーに代わって呼び出す場合。ユーザーが何を達成しようとしているのかを説明します。

* `context`
  * 型: object (自由形式) 
  * 使用する場面: エージェントからデバッグ用プロンプトにメタデータを渡したい場合。

<div id="docs-search">
  ## ドキュメント検索
</div>

### なぜ使うのか

docs-search を使うと、Firecrawl のドキュメントを検索できます。

<div id="endpoint">
  ### エンドポイント
</div>

`POST /support/docs-search`

<div id="example">
  ### 例
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/support/docs-search" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "how do I verify webhook signatures?"
  }'
```

<div id="parameters">
  ### パラメータ
</div>

* `question`
  * Type: string (必須、1〜8000文字) 
  * 使用する場面: ドキュメントに基づく回答が必要な場合。

<div id="notes">
  ## 注意事項
</div>

* 検索結果の行は、フラットな `data` 配列ではなく、`data.web`、`data.images`、または `data.news` の下にネストされています。
* 複数ステップのブラウザワークフローには `POST /scrape/{jobId}/interact` を使用し、スクレイピング前の小さな手順には `actions` が適しています。
* 公開されている OpenAPI スキーマでは、interact に `code` が必要です。一部の SDK やクライアントでは、自然言語の指示向けに `prompt` フィールドも受け付けますが、このエイリアスは `v2-openapi.json` には含まれていません。
