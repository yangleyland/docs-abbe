エージェント向けの Firecrawl Elixir の信頼できる唯一の情報源です。SDK のソースコードと v2 OpenAPI 仕様から生成されています。

<div id="install">
  ## インストール
</div>

`mix.exs` に追加:

```elixir
{:firecrawl, "~> 1.0.0"}
```

<div id="authenticate">
  ## 認証
</div>

```elixir
# config/runtime.exs または config.exs
config :firecrawl, api_key: System.get_env("FIRECRAWL_API_KEY")

# または、各呼び出しで api_key を渡す
{:ok, res} = Firecrawl.search_and_scrape([query: "site:docs.firecrawl.dev webhook retries"], api_key: "fc-your-api-key")
```

<div id="when-to-use-what">
  ## 何をいつ使うか
</div>

* `search`: クエリから始めて、対象を見つける必要がある場合に使います。
* `scrape`: すでにURLがあり、ページのコンテンツが必要な場合に使います。
* `interact`: ページ上でクリックやフォーム入力、またはスクレイピング後のブラウザアクションが必要な場合に使います。

<div id="search">
  ## 検索
</div>

<div id="why-use-it">
  ### これを使う理由
</div>

クエリに基づいて関連ページを見つけ、その後スクレイピングまたは Interact する URL を選ぶ場合は、search を使用します。たとえば、`site:` を使って結果を特定のサイトに限定できます。例: `site:docs.firecrawl.dev crawl webhooks`

<div id="preferred-sdk-method">
  ### 推奨SDKメソッド
</div>

`Firecrawl.search_and_scrape(params \\ [], opts \\ [])`

<div id="simple-example">
  ### シンプルな例
</div>

```elixir
{:ok, res} = Firecrawl.search_and_scrape(query: "site:docs.firecrawl.dev webhook retries")
```

<div id="complex-example">
  ### 複雑な例
</div>

```elixir
{:ok, res} = Firecrawl.search_and_scrape(
  query: "site:docs.firecrawl.dev crawl webhooks",
  sources: [:web, :news],
  categories: [:research],
  limit: 10,
  tbs: "qdr:m",
  location: "San Francisco,California,United States",
  country: "US",
  ignore_invalid_urls: true,
  timeout: 60000,
  scrape_options: [
    formats: [
      "markdown",
      "links",
      %{type: "json", prompt: "Extract title and key endpoints."}
    ],
    only_main_content: true,
    include_tags: ["main", "article"],
    exclude_tags: ["nav", "footer"],
    wait_for: 1000
  ]
)
```

<div id="parameters">
  ### パラメータ
</div>

* `query`
  * Type: string
  * Use when: 検索クエリが必要な場合。
  * Notes: 結果を特定のドメインに限定するには `site:example.com` を使用します。

* `sources`
  * Type: list of atoms, strings, or maps
  * Use when: 検索対象のソースを制御したい場合。
  * Confirmed values:
    * `"web"` or `:web`
    * `"news"` or `:news`
    * `"images"` or `:images`
    * `%{type: "web" | "news" | "images"}`

* `categories`
  * Type: list of atoms, strings, or maps
  * Use when: 結果をカテゴリで絞り込みたい場合。
  * Confirmed values:
    * `"github"` or `:github`
    * `"research"` or `:research`
    * `"pdf"` or `:pdf`
    * `%{type: "github" | "research" | "pdf"}`

* `limit`
  * Type: integer
  * Use when: 結果数の上限を設定したい場合。

* `tbs`
  * Type: string
  * Use when: 時間ベースの絞り込みが必要な場合 (例: `qdr:d`, `qdr:w`, `sbd:1,qdr:m`) 。

* `location`
  * Type: string
  * Use when: ロケーションに応じた結果を取得したい場合。

* `country`
  * Type: string
  * Use when: ISO 3166-1 alpha-2 でターゲティングしたい場合 (例: `"US"`) 。

* `ignore_invalid_urls`
  * Type: boolean
  * Use when: 他のエンドポイントでスクレイピングできない URL を除外したい場合。

* `timeout`
  * Type: integer
  * Use when: ミリ秒単位のリクエストタイムアウトが必要な場合。

* `enterprise`
  * Type: list of strings
  * Use when: エンタープライズ向けの検索制御が必要な場合。
  * Confirmed values:
    * `"zdr"`: エンドツーエンドのゼロデータ保持
    * `"anon"`: 匿名化されたゼロデータ保持

* `scrape_options`
  * Type: keyword list
  * Use when: 各検索結果をスクレイピングしたい場合 (フィールドについては `Scrape` のパラメータを参照) 。

<div id="scrape">
  ## スクレイピング
</div>

<div id="why-use-it">
  ### これを使う理由
</div>

URL がすでに分かっており、1 つ以上のフォーマットで構造化されたコンテンツを取得したい場合は、スクレイピングを使用します。

<div id="preferred-sdk-method">
  ### 推奨SDKメソッド
</div>

`Firecrawl.scrape_and_extract_from_url(params \\ [], opts \\ [])`

### 基本的な例

```elixir
{:ok, res} = Firecrawl.scrape_and_extract_from_url(
  url: "https://docs.firecrawl.dev",
  formats: ["markdown"]
)
```

### 応用例

```elixir
{:ok, res} = Firecrawl.scrape_and_extract_from_url(
  url: "https://example.com/pricing",
  formats: [
    "markdown",
    "links",
    %{type: "json", prompt: "Extract plan names and prices."},
    %{type: "screenshot", fullPage: true, quality: 80, viewport: %{width: 1280, height: 720}},
    %{type: "changeTracking", modes: ["git-diff"], tag: "pricing"}
  ],
  headers: %{"User-Agent" => "FirecrawlDocsBot/1.0"},
  only_main_content: true,
  wait_for: 1000,
  parsers: [%{type: "pdf", mode: "auto", maxPages: 5}],
  actions: [
    %{type: "click", selector: "#accept"},
    %{type: "wait", milliseconds: 750},
    %{type: "scrape"}
  ],
  location: [country: "US", languages: ["en-US"]],
  remove_base64_images: true,
  block_ads: true,
  proxy: :auto,
  max_age: 86400000,
  min_age: 1,
  store_in_cache: true,
  profile: [name: "docs-session", save_changes: true],
  zero_data_retention: false
)
```

<div id="parameters">
  ### パラメータ
</div>

* `url`
  * Type: string
  * Use when: 特定のページをスクレイピングしたい場合。

* `formats`
  * Type: フォーマット文字列またはフォーマットマップのリスト
  * Use when: 複数の出力フォーマットが必要な場合。
  * 確認済みのフォーマット文字列:
    * `"markdown"`: Markdownコンテンツ
    * `"html"`: 整形済みHTML
    * `"rawHtml"`: 生のHTML
    * `"links"`: ページリンク
    * `"images"`: 画像URL
    * `"screenshot"`: スクリーンショット出力
    * `"summary"`: 要約出力
    * `"changeTracking"`: changeTracking出力
    * `"json"`: JSON抽出
    * `"branding"`: ブランドプロファイル出力
    * `"audio"`: 音声抽出
    * `"video"`: 動画抽出
  * フォーマットマップのフィールド:
    * `type`: 上記のフォーマット文字列のいずれか
    * `prompt`, `schema`: `type: "json"` のJSON抽出オプション
    * `modes`, `schema`, `prompt`, `tag`: `type: "changeTracking"` のchangeTrackingオプション
    * `fullPage`, `quality`, `viewport`: `type: "screenshot"` のスクリーンショットオプション

* `headers`
  * Type: map
  * Use when: カスタムのリクエストヘッダーが必要な場合。

* `include_tags`
  * Type: 文字列のリスト
  * Use when: 特定のHTMLタグのみを含めたい場合。

* `exclude_tags`
  * Type: 文字列のリスト
  * Use when: 特定のHTMLタグを除外したい場合。

* `only_main_content`
  * Type: boolean
  * Use when: ナビゲーション、フッター、その他の定型要素を取り除きたい場合。

* `timeout`
  * Type: integer
  * Use when: ミリ秒単位のタイムアウトが必要な場合。

* `wait_for`
  * Type: integer
  * Use when: ページのレンダリングを待つ必要がある場合 (ミリ秒) 。

* `mobile`
  * Type: boolean
  * Use when: モバイル向けビューポートを使用したい場合。

* `parsers`
  * Type: parser文字列またはマップのリスト
  * Use when: ファイル解析の制御が必要な場合。
  * 確認済みの値:
    * `"pdf"`
    * `%{type: "pdf", mode: "fast" | "auto" | "ocr", maxPages: integer}`

* `actions`
  * Type: アクションマップのリスト
  * Use when: 軽量なスクレイピング前アクションが必要な場合。
  * 確認済みのアクションタイプ:
    * `wait`: `milliseconds` または `selector` が必須
    * `screenshot`: `fullPage`, `quality`, `viewport` は任意
    * `click`: `selector` が必須、`all` は任意
    * `write`: `text` が必須 (先にクリックして入力欄にフォーカス)
    * `press`: `key` が必須
    * `scroll`: `direction` (`up` または `down`) が必須、`selector` は任意
    * `scrape`: 追加フィールドなし
    * `executeJavascript`: `script` が必須
    * `pdf`: `format` (A0, A1, A2, A3, A4, A5, A6, Letter, Legal, Tabloid, Ledger) 、`landscape`、`scale` は任意

* `location`
  * Type: キーワードリスト (たとえば `country:` や `languages:`)
  * Use when: 地域または言語を考慮したスクレイピングが必要な場合。

* `skip_tls_verification`
  * Type: boolean
  * Use when: TLS確認をスキップする必要がある場合。

* `remove_base64_images`
  * Type: boolean
  * Use when: Markdown出力からbase64画像を除外したい場合。

* `block_ads`
  * Type: boolean
  * Use when: 広告やCookieポップアップのブロックが必要な場合。

* `proxy`
  * Type: atom または string
  * Use when: プロキシ制御が必要な場合。
  * 確認済みの値: `:basic`, `:enhanced`, `:auto`

* `max_age`
  * Type: integer
  * Use when: 指定した最大期間までのキャッシュデータを使いたい場合 (ミリ秒) 。

* `min_age`
  * Type: integer
  * Use when: 少なくともこの期間が経過したキャッシュデータのみを使いたい場合 (ミリ秒) 。

* `store_in_cache`
  * Type: boolean
  * Use when: Firecrawlに結果をキャッシュさせたい場合。

* `profile`
  * Type: `name:` と任意の `save_changes:` (`saveChanges:` も可) を含むキーワードリスト
  * Use when: スクレイピングや操作の間で共有される永続的なブラウザープロファイルを使いたい場合。

* `zero_data_retention`
  * Type: boolean
  * Use when: このスクレイピングでゼロデータ保持を有効にしたい場合。

<div id="interact">
  ## Interact
</div>

<div id="why-use-it">
  ### これを使う理由
</div>

ページで、スクレイピング開始後にブラウザのアクションやコードの実行が必要な場合は、Interact を使用します。

<div id="preferred-sdk-method">
  ### 推奨SDKメソッド
</div>

`Firecrawl.interact_with_scrape_browser_session(job_id, params \\ [], opts \\ [])`

<div id="simple-example">
  ### シンプルな例
</div>

```elixir
{:ok, res} = Firecrawl.interact_with_scrape_browser_session(
  "<scrapeJobId>",
  code: "console.log(await page.title());",
  language: :node,
  timeout: 60
)
```

<div id="complex-example">
  ### 複雑な例
</div>

```elixir
{:ok, res} = Firecrawl.interact_with_scrape_browser_session(
  "<scrapeJobId>",
  code: "// ここでPlaywrightのページメソッドを使用してください",
  language: :node,
  timeout: 120
)
```

<div id="parameters">
  ### パラメータ
</div>

* `job_id`
  * 型: string
  * 使用するケース: スクレイピングジョブ ID がある場合。

* `code`
  * 型: string
  * 使用するケース: ブラウザセッションでコードを実行したい場合。

* `language`
  * 型: atom または string
  * 使用するケース: 特定のランタイムが必要な場合。
  * 確認済みの値: `:python`、`:node`、`:bash` (または対応する文字列)

* `timeout`
  * 型: integer
  * 使用するケース: 秒単位の実行タイムアウトが必要な場合。

* `origin`
  * 型: string
  * 使用するケース: 実行テレメトリ用の任意の origin ラベルを指定したい場合。

<div id="stop-interactive-session">
  ### インタラクティブセッションの停止
</div>

`Firecrawl.stop_interactive_scrape_browser_session(job_id, opts \\ [])` は `DELETE /scrape/{jobId}/interact` を実行し、そのスクレイピングジョブのブラウザセッションを終了します。bang variant の `stop_interactive_scrape_browser_session!/2` も利用できます。

```elixir
{:ok, res} = Firecrawl.stop_interactive_scrape_browser_session("<scrapeJobId>")
```

<div id="notes">
  ## 注記
</div>

* Elixir クライアントは OpenAPI の構成に沿っており、関数名とパラメータキーは仕様から自動生成されます。
* 各公開関数には、`{:error, _}` を返す代わりにエラー時に例外を発生させる bang (`!`) variant があります。
* この SDK が提供するのはコードベースの操作のみです (`interact_with_scrape_browser_session` に `prompt` パラメータはありません) 。
