エージェント向けの、Firecrawl Node.js の信頼できる唯一の参照元です。`firecrawl` **v4.18.2** (`firecrawl/apps/js-sdk/firecrawl`) および v2 OpenAPI 仕様に対応しています。メソッド名、パラメータ、型は SDK の公開 API と一致しています。

<div id="install">
  ## インストール
</div>

```bash
npm install firecrawl
```

<div id="authenticate">
  ## 認証
</div>

```ts
import { Firecrawl } from "firecrawl";

const client = new Firecrawl({
  apiKey: process.env.FIRECRAWL_API_KEY,
  // apiUrl: "https://api.firecrawl.dev" // 任意; 未指定時はFIRECRAWL_API_URLまたはクラウドのデフォルトにフォールバック
});
```

<div id="when-to-use-what">
  ## どれを使い分けるべきか
</div>

* `search`: クエリから始めて、対象を見つける必要がある場合に使います。
* `scrape`: すでに URL があり、ページのコンテンツが必要な場合に使います。
* `interact`: スクレイピングによってセッションが作成された後に、ページ上でクリックやフォーム操作、またはその他のブラウザアクションが必要な場合に使います。複数ステップの対話型フローでは、スクレイピング時の `actions` より `interact` を優先してください。
* `support/ask`: Firecrawl API の呼び出しが失敗した場合や、想定外の結果が返ってきた場合に、原因を診断する必要があるときに使います。
* `support/docs-search`: Firecrawl のドキュメントを調べる必要がある場合に使います。

<div id="search">
  ## Search
</div>

### 使用する理由

クエリから関連ページを見つけ、そこからスクレイピングまたは Interact する URL を選びたい場合に使用します。`site:` を使って結果をサイトに絞り込めます。例：`site:docs.firecrawl.dev crawl webhooks`

<div id="preferred-sdk-method">
  ### 推奨SDKメソッド
</div>

`client.search(query, options?)` → `Promise<SearchData>`

<div id="simple-example">
  ### シンプルな例
</div>

```ts
const results = await client.search("site:docs.firecrawl.dev webhook retries");
```

<div id="complex-example">
  ### 複雑な例
</div>

```ts
const results = await client.search("site:docs.firecrawl.dev crawl webhooks", {
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:m",
  location: "San Francisco,California,United States",
  ignoreInvalidURLs: true,
  timeout: 60000,
  scrapeOptions: {
    formats: [
      "markdown",
      "links",
      { type: "json", prompt: "Extract title and key endpoints." }
    ],
    onlyMainContent: true,
    includeTags: ["main", "article"],
    excludeTags: ["nav", "footer"],
    waitFor: 1000
  }
});
```

<div id="return-value">
  ### 戻り値
</div>

`SearchData` は任意の配列を含むオブジェクトです (各エントリは軽量な結果オブジェクト、または `scrapeOptions` によってヒットが hydrate されている場合は完全な `Document` です) :

* `web`: web インデックスのヒット
* `news`: ニュースのヒット
* `images`: 画像のヒット

**避けるべき間違い:** `search()` は `{ data: [...] }` を返しません。`result.data` にはアクセスしないでください。Web の結果は `result.web`、ニュースは `result.news`、画像は `result.images` に入ります。

<div id="parameters">
  ### パラメータ
</div>

* `query`
  * 型: string
  * 使用する場面: 検索クエリが必要な場合。
  * 注記: 結果を特定のドメインに限定するには、`site:example.com` を使用します。

* `options` (任意。第 2 引数のデフォルトは `{}`)
  * 型: `Omit<SearchRequest, "query">`

* `options.sources`
  * 型: ソース名の配列、または型付きソースオブジェクトの配列
  * 使用する場面: どのソースを検索対象にするかを制御したい場合。
  * 確認済みの値:
    * `"web"`: web インデックスの結果
    * `"news"`: ニュース結果
    * `"images"`: 画像結果
    * `{ type: "web" | "news" | "images" }`: 型付きソースオブジェクト形式

* `options.categories`
  * 型: カテゴリ名の配列、または型付きカテゴリオブジェクトの配列
  * 使用する場面: カテゴリで結果を絞り込みたい場合。
  * 確認済みの値:
    * `"github"`: GitHub に特化した結果
    * `"research"`: 研究・学術関連の結果
    * `"pdf"`: PDF に特化した結果
    * `{ type: "github" | "research" | "pdf" }`: 型付きカテゴリオブジェクト形式

* `options.limit`
  * 型: number
  * 使用する場面: 結果数の上限を設定したい場合。

* `options.tbs`
  * 型: string
  * 使用する場面: 時間ベースの絞り込みが必要な場合 (例: `qdr:d`, `qdr:w`, `sbd:1,qdr:m`) 。

* `options.location`
  * 型: string
  * 使用する場面: 地域に応じた結果を取得したい場合。

* `options.ignoreInvalidURLs`
  * 型: boolean
  * 使用する場面: 他のエンドポイントでスクレイピングできない URL を除外したい場合。

* `options.timeout`
  * 型: number
  * 使用する場面: ミリ秒単位のリクエストタイムアウトが必要な場合。

* `options.scrapeOptions`
  * 型: `ScrapeOptions`
  * 使用する場面: 各検索結果をスクレイピングしたい場合 (フィールドについては Scrape のパラメータを参照) 。SDK は `scrape` と同じ検証を実行します (たとえば、`formats` 内の単なる文字列 `"json"` は拒否されます)。

<div id="scrape">
  ## スクレイピング
</div>

<div id="why-use-it">
  ### どのような場合に使うか
</div>

URLがすでにあり、1つ以上のフォーマットで構造化されたコンテンツを取得したい場合は、スクレイピングを使用します。

<div id="preferred-sdk-method">
  ### 推奨SDKメソッド
</div>

`client.scrape(url, options?)` → `Promise<Document>`

<div id="simple-example">
  ### シンプルな例
</div>

```ts
const doc = await client.scrape("https://docs.firecrawl.dev", {
  formats: ["markdown"]
});
```

### 高度な例

```ts
const doc = await client.scrape("https://example.com/pricing", {
  formats: [
    "markdown",
    "links",
    { type: "json", prompt: "Extract plan names and prices." },
    { type: "screenshot", fullPage: true, quality: 80, viewport: { width: 1280, height: 720 } },
    { type: "changeTracking", modes: ["git-diff"], tag: "pricing" },
    { type: "attributes", selectors: [{ selector: "a", attribute: "href" }] }
  ],
  headers: {
    "User-Agent": "FirecrawlDocsBot/1.0"
  },
  onlyMainContent: true,
  waitFor: 1000,
  parsers: [{ type: "pdf", mode: "auto", maxPages: 5 }],
  actions: [
    { type: "click", selector: "#accept" },
    { type: "wait", milliseconds: 750 },
    { type: "scrape" }
  ],
  location: { country: "US", languages: ["en-US"] },
  removeBase64Images: true,
  fastMode: true,
  blockAds: true,
  proxy: "auto",
  maxAge: 86400000,
  minAge: 1,
  storeInCache: true,
  profile: { name: "docs-session", saveChanges: true }
});
```

<div id="parameters">
  ### パラメータ
</div>

* `url`
  * 型: string
  * 使用する場面: 特定のページをスクレイピングしたい場合

* `options.formats`
  * Type: フォーマット文字列またはフォーマットオブジェクトの配列
  * Use when: 複数の出力フォーマットが必要な場合。
  * プレーンな文字列フォーマット (`FormatString` のユニオンには `"json"` も含まれますが、SDK は `"json"` をプレーンな文字列としては受け付けません。代わりに下記の `json` オブジェクトを使用してください):
    * `"markdown"`: Markdownコンテンツ
    * `"html"`: クリーンアップされたHTML
    * `"rawHtml"`: 生のHTML
    * `"links"`: ページのリンク
    * `"images"`: 画像URL
    * `"screenshot"`: スクリーンショット出力
    * `"summary"`: 要約出力
    * `"changeTracking"`: changeTracking出力 (`modes` などのオプションには `{ type: "changeTracking", modes: [...] }` を使用します — 型定義ではこのオブジェクトで `modes` が必須です)
    * `"attributes"`: 属性抽出 (セレクターを渡す場合は `{ type: "attributes", selectors: [...] }` を使用)
    * `"branding"`: ブランディングプロファイル出力
    * `"audio"`: 音声抽出
    * `"video"`: 動画抽出
  * オブジェクトでのみ指定可能なフォーマットタイプ (最低限、表示されている `type` が必要):
    * `{ type: "json", prompt?: string, schema?: JSON schema or Zod schema }`: `prompt` または `schema` の少なくとも一方が必須です (SDK のバリデーション)。
    * `{ type: "question", question: string }`: 質問応答形式の抽出。
    * `{ type: "highlights", query: string }`: 関連するソーステキストの抽出。
    * `{ type: "screenshot", fullPage?, quality?, viewport? }`: 文字列形式と同じオプションですが、オブジェクトとして指定します。
    * `{ type: "changeTracking", modes: ("git-diff" | "json")[], schema?, prompt?, tag? }`: `modes` は必須です。
    * `{ type: "attributes", selectors: Array<{ selector, attribute }> }`
  * 該当する場合に共通のオブジェクトフィールド:
    * `schema`: `json` 用、または `json` モードの `changeTracking` 用の JSON schema または Zod schema (Zod は SDK によって JSON Schema に変換されます)。
    * `modes`: `changeTracking` のみ: `"git-diff"` および/または `"json"`。
    * `tag`: 任意の changeTracking ブランチタグ。
    * `fullPage`, `quality`, `viewport`: スクリーンショットオプション。

* `options.headers`
  * 型: 文字列をキー、文字列を値とするレコード
  * 使用するケース: カスタムのリクエストヘッダーが必要な場合。

* `options.includeTags`
  * 型: 文字列の配列
  * 使用するケース: 特定のHTMLタグのみを含めたい場合。

* `options.excludeTags`
  * 型: 文字列の配列
  * 使用する場合: 特定のHTMLタグを除外したいとき。

* `options.onlyMainContent`
  * 型: boolean
  * 使用する場合: ナビゲーション、フッター、その他の定型要素を削除したいとき。

* `options.timeout`
  * 型: number
  * 使用する場面: ミリ秒単位のタイムアウトが必要な場合。

* `options.waitFor`
  * 型: number
  * 使用する場面: ページのレンダリングが完了するまで待機する必要がある場合 (ミリ秒) 。

* `options.mobile`
  * 型: boolean
  * 使用する場面: モバイルのビューポートが必要な場合。

* `options.parsers`
  * 型: パーサー名またはパーサーオブジェクトの配列
  * 使用する場面: ファイル解析を制御する必要がある場合 (たとえば PDF の解析) 。
  * 使用可能な値:
    * `"pdf"`: PDF 解析を有効にする
    * `{ type: "pdf", mode?: "fast" | "auto" | "ocr", maxPages?: number }`: PDF パーサーのオプション

* `options.actions`
  * 種別: アクションオブジェクトの配列
  * 使用する場面: 軽量なスクレイピング前のアクションが必要な場合。
  * 確認済みのアクションタイプ:
    * `wait`: `milliseconds` または `selector` が必要
    * `screenshot`: `fullPage`、`quality`、`viewport` は任意
    * `click`: `selector` が必要
    * `write`: `text` が必要 (最初にクリックして入力欄にフォーカスしてください)
    * `press`: `key` が必要
    * `scroll`: `direction` (`up` または `down`) が必要、`selector` は任意
    * `scrape`: 追加フィールドなし
    * `executeJavascript`: `script` が必要
    * `pdf`: `format` (A0、A1、A2、A3、A4、A5、A6、Letter、Legal、Tabloid、Ledger) 、`landscape`、`scale` は任意

* `options.location`
  * 型: `country` と `languages` を持つオブジェクト
  * 使用する場合: 地理情報または言語を考慮したスクレイピングが必要な場合。

* `options.skipTlsVerification`
  * 型: boolean
  * 使用する場面: TLS の確認をスキップする必要がある場合。

* `options.removeBase64Images`
  * 型: boolean
  * 使用する場合: Markdown出力からbase64画像を除外したいとき。

* `options.fastMode`
  * 型: boolean
  * 使用する場面: 精度をやや落としてでも、より高速なスクレイピングを行いたいとき。

* `options.blockAds`
  * 型: boolean
  * 使用する場面: 広告および Cookie ポップアップをブロックしたい場合。

* `options.proxy`
  * 型: string
  * 使用する場面: プロキシ制御が必要な場合。
  * 指定可能な値:
    * `"basic"`
    * `"stealth"`
    * `"enhanced"`
    * `"auto"`
    * カスタムプロキシURL文字列

* `options.maxAge`
  * 型: number
  * 使用する場合: 最大経過時間 (ミリ秒) までのキャッシュデータを使いたいとき。

* `options.minAge`
  * 型: number
  * 使用する場合: キャッシュデータが少なくともこの値以上古い場合にのみ使用したいとき (ミリ秒) 。

* `options.storeInCache`
  * 型: boolean
  * 使用するケース: Firecrawl に結果をキャッシュさせたい場合。

* `options.profile`
  * 型: `name` と任意の `saveChanges` を持つオブジェクト
  * 使用する場合: スクレイピングや操作のあいだで共有される、永続的なブラウザプロファイルが必要な場合。

<div id="interact">
  ## Interact
</div>

### これを使う理由

`interact` は、スクレイピングジョブに紐づくブラウザセッション (`metadata.scrapeId` 経由) を、コードまたは自然言語で操作するために使います。SDK では、`code` または `prompt` の少なくとも一方が必要です。スクレイピング前の簡単な調整を超えるフローでは、スクレイピング時の `actions` よりも `interact` を使うことを推奨します。

<div id="preferred-sdk-method">
  ### 推奨SDKメソッド
</div>

`client.interact(jobId, args)` → `Promise<ScrapeExecuteResponse>`

<div id="simple-example">
  ### シンプルな例
</div>

```ts
const doc = await client.scrape("https://example.com", { formats: ["markdown"] });
const jobId = doc.metadata?.scrapeId;
if (!jobId) throw new Error("スクレイピングレスポンスにscrapeIdがありません");

const result = await client.interact(jobId, {
  prompt: "pricingタブをクリックして、プランを要約してください。"
});
```

<div id="complex-example">
  ### 複雑な例
</div>

```ts
const result = await client.interact("<scrapeJobId>", {
  code: "console.log(await page.title());",
  language: "node",
  timeout: 60
});
```

<div id="parameters">
  ### パラメータ
</div>

* `jobId`
  * 型: string
  * 使用する場合: `document.metadata.scrapeId` から取得したスクレイピングジョブ ID がある場合。

* `args.code`
  * 型: string
  * 使用する場合: ブラウザセッション内でコードを実行したい場合 (たとえば、`node` ランタイムで Playwright の `page` を使用する場合) 。

* `args.prompt`
  * 型: string
  * 使用する場合: ブラウザエージェントに自然言語の指示を実行させたい場合。

* `args.code` または `args.prompt` の少なくとも一方は空でない必要があります (そうでない場合は SDK が例外をスローします) 。

* `args.language`
  * 型: string
  * 使用する場合: 特定のランタイムを指定する必要がある場合。
  * 確認済みの値: `"python"`, `"node"`, `"bash"`
  * 省略時のデフォルト値: `"node"` (リクエストボディで SDK によって設定されます) 。

* `args.timeout`
  * 型: number
  * 使用する場合: 秒単位で実行タイムアウトを指定する必要がある場合。

<div id="return-value-interact">
  ### 戻り値 (`interact`)
</div>

`ScrapeExecuteResponse` は `BrowserExecuteResponse` と一致します。確認済みのフィールドは以下のとおりです。

* `success`: boolean
* `output`: string (存在する場合は集約された出力) 
* `stdout`: string
* `result`: string (型定義では `stdout` と並ぶエイリアス的なフィールド) 
* `stderr`: string
* `exitCode`: number
* `killed`: boolean
* `liveViewUrl`: string
* `interactiveLiveViewUrl`: string
* `error`: string

<div id="stop-session">
  ### セッションの停止
</div>

`client.stopInteraction(jobId)` → `Promise<ScrapeBrowserDeleteResponse>`

* `jobId`: スクレイピングジョブのID (`interact`で使用するIDと同じ) 。
* 型定義に含まれるレスポンスフィールドは、`success`、`sessionDurationMs`、`creditsBilled`、`error` です。

<div id="ask-agentic-debugging">
  ## Ask (エージェント型デバッグ)
</div>

<div id="why-use-it">
  ### どのような場合に使うか
</div>

Firecrawl API の呼び出しが失敗したり、想定外の結果が返ってきたりした場合は、ask を使用します。AI サポートエージェントが問題を診断し、修正に必要なパラメータを提案し、必要に応じて実際の API に対して修正内容を検証します。通常のレイテンシ: 15〜30 秒。

<div id="preferred-sdk-method">
  ### 推奨SDKメソッド
</div>

Node.js SDKではまだ利用できません。`fetch`を直接使用してください。

<div id="simple-example">
  ### シンプルな例
</div>

```ts
const response = await fetch("https://api.firecrawl.dev/v2/support/ask", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.FIRECRAWL_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    question: "my scrape returned empty markdown for https://example.com",
  }),
});
const result = await response.json();
console.log(result.answer);
console.log(result.fixParameters);
```

<div id="complex-example">
  ### 複雑な例
</div>

```ts
const response = await fetch("https://api.firecrawl.dev/v2/support/ask", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.FIRECRAWL_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    question: "crawl returned 3 pages but I expected 50",
    rationale: "user is on their third failed crawl attempt today",
    context: { userPlan: "standard", retryCount: 3 },
  }),
});
const result = await response.json();
```

<div id="agent-retry-pattern">
  ### Agent のリトライパターン
</div>

```ts
import { Firecrawl } from "firecrawl";

const client = new Firecrawl({ apiKey: process.env.FIRECRAWL_API_KEY });

const doc = await client.scrape("https://example.com/pricing", {
  formats: ["markdown"],
});

if (!doc.markdown || doc.markdown.length < 100) {
  const diagResponse = await fetch(
    "https://api.firecrawl.dev/v2/support/ask",
    {
      method: "POST",
      headers: {
        Authorization: `Bearer ${process.env.FIRECRAWL_API_KEY}`,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        question: `scrape returned only ${doc.markdown?.length ?? 0} chars for https://example.com/pricing`,
      }),
    }
  );
  const diagnosis = await diagResponse.json();

  if (diagnosis.fixParameters) {
    const retryDoc = await client.scrape("https://example.com/pricing", {
      formats: ["markdown"],
      ...diagnosis.fixParameters,
    });
  }
}
```

<div id="parameters">
  ### パラメータ
</div>

* `question`
  * 型: string (必須、1～8000文字) 
  * 使用する場面: 問題の内容を説明する必要がある場合。

* `rationale`
  * 型: string (1～2000文字) 
  * 使用する場面: ユーザーに代わって呼び出すAIエージェントである場合。

* `context`
  * 型: object (自由形式) 
  * 使用する場面: デバッグ用のプロンプトにメタデータを渡したい場合。

<div id="docs-search">
  ## ドキュメント検索
</div>

### なぜ使うのか

docs-search を使うと、Firecrawl のドキュメントを検索できます。

### 簡単な例

```ts
const response = await fetch(
  "https://api.firecrawl.dev/v2/support/docs-search",
  {
    method: "POST",
    headers: {
      Authorization: `Bearer ${process.env.FIRECRAWL_API_KEY}`,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      question: "how do I verify webhook signatures?",
    }),
  }
);
const result = await response.json();
console.log(result.answer);
```

<div id="parameters">
  ### パラメータ
</div>

* `question`
  * 型: string (必須、1～8000文字) 
  * 使用する場面: ドキュメントに基づいた回答が必要な場合。

<div id="notes">
  ## 注意事項
</div>

* 非推奨のクライアントエイリアス: `scrapeExecute` → `interact`、`stopInteractiveBrowser` と `deleteScrapeBrowser` → `stopInteraction`。
* デフォルトの `Firecrawl` エクスポートは v2 クライアントで、v1 は引き続き `client.v1` で利用できます。
* `formats` (`json` または `changeTracking` 用) に渡された Zod スキーマは、SDK によって JSON Schema に変換されます。
* パッケージの `engines` では **Node.js &gt;= 22** が宣言されています。
