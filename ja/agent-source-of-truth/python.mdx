エージェント向けの Firecrawl Python の信頼できる唯一の参照元です。SDK のソースコード (`firecrawl-py` / `firecrawl` **4.22.1**) と v2 OpenAPI 仕様から生成されています。特記がない限り、メソッド名、パラメータ、戻り値の型は `firecrawl/v2/client.py` の v2 クライアントと一致します。

<div id="install">
  ## インストール
</div>

```bash
pip install firecrawl-py
```

<div id="authenticate">
  ## 認証
</div>

```py
import os
from firecrawl import Firecrawl

client = Firecrawl(api_key=os.environ.get("FIRECRAWL_API_KEY"))
# client = Firecrawl(api_key="fc-...", api_url="https://api.firecrawl.dev")
```

<div id="when-to-use-what">
  ## 何をいつ使うか
</div>

* `search`: クエリから始めて、対象を見つける必要がある場合に使います。
* `scrape`: すでにURLがあり、ページのコンテンツを取得したい場合に使います。
* `interact`: ページでクリックやフォーム操作、またはスクレイピング後のブラウザアクションが必要な場合に使います。
* `support/ask`: Firecrawl API の呼び出しが失敗した場合や、予期しない結果が返ってきて原因の切り分けが必要な場合に使います。
* `support/docs-search`: Firecrawl のドキュメントを調べる必要がある場合に使います。

<div id="search">
  ## 検索
</div>

### 使用する理由

クエリから関連ページを見つけ、スクレイピングまたは Interact の対象にするURLを選ぶには、検索を使用します。`site:` で結果を特定のサイトに絞り込むことができます (例：`site:docs.firecrawl.dev crawl webhooks`) 。

<div id="preferred-sdk-method">
  ### 推奨SDKメソッド
</div>

`client.search(query, **options)` → `SearchData` (`firecrawl.v2.types`)

<div id="return-value">
  ### 戻り値
</div>

任意のリストを持つ `SearchData` モデルを返します。

* `web` — Web の検索結果 (`scrape_options` によって content が補完される場合は `SearchResultWeb` または完全な `Document`)
* `news` — ニュースの検索結果 (`SearchResultNews` または `Document`)
* `images` — 画像の検索結果 (`SearchResultImages` または `Document`)

API がそのキーを返さなかった場合、省略されたバケットは `None` です。

**避けるべき誤り:** `search()` は `{ data: [...] }` を返しません。`result.data` にはアクセスしないでください。Web の検索結果は `result.web`、ニュースは `result.news`、画像は `result.images` に格納されます。

<div id="simple-example">
  ### シンプルな例
</div>

```py
results = client.search("site:docs.firecrawl.dev webhook retries")
for item in results.web or []:
    print(getattr(item, "url", None), getattr(item, "title", None))
```

<div id="complex-example">
  ### 複雑な例
</div>

```py
from firecrawl.v2.types import ScrapeOptions

results = client.search(
    "site:docs.firecrawl.dev crawl webhooks",
    sources=["web", "news"],
    categories=["research"],
    limit=10,
    tbs="qdr:m",
    location="San Francisco,California,United States",
    ignore_invalid_urls=True,
    timeout=300000,
    scrape_options=ScrapeOptions(
        formats=[
            "markdown",
            "links",
            {"type": "json", "prompt": "Extract title and key endpoints."},
        ],
        only_main_content=True,
        include_tags=["main", "article"],
        exclude_tags=["nav", "footer"],
        wait_for=1000,
        actions=[
            {"type": "click", "selector": "button#accept"},
            {"type": "wait", "milliseconds": 750},
            {"type": "scrape"},
        ],
    ),
)
```

<div id="parameters">
  ### パラメータ
</div>

* `query`
  * 型: str
  * 使用する場面: 検索クエリが必要な場合。
  * 注記: 結果を特定のドメインに限定するには、`site:example.com` を使用します。

* `sources`
  * 型: ソース名または `Source` オブジェクトのリスト
  * 使用する場面: 検索対象のソースを制御したい場合。
  * 確認済みの値:
    * `"web"`: Web インデックスの結果
    * `"news"`: ニュース結果
    * `"images"`: 画像結果
    * `Source(type="web" | "news" | "images")`: 型付きソースオブジェクト形式

* `categories`
  * 型: カテゴリ名または `Category` オブジェクトのリスト
  * 使用する場面: カテゴリで結果を絞り込みたい場合。
  * 確認済みの値:
    * `"github"`: GitHub に特化した結果
    * `"research"`: 研究・学術関連の結果
    * `"pdf"`: PDF に特化した結果
    * `Category(type="github" | "research" | "pdf")`: 型付きカテゴリオブジェクト形式

* `limit`
  * 型: int
  * 使用する場面: 結果数の上限を設定したい場合。
  * 注記: SDK モデルでのデフォルトは `5` です。

* `tbs`
  * 型: str
  * 使用する場面: 時間ベースの絞り込みが必要な場合 (例: `qdr:d`、`qdr:w`、`sbd:1,qdr:m`) 。

* `location`
  * 型: str
  * 使用する場面: ローカライズされた結果を取得したい場合。

* `ignore_invalid_urls`
  * 型: bool
  * 使用する場面: 他のエンドポイントでスクレイピングできない URL を除外したい場合。

* `timeout`
  * 型: int
  * 使用する場面: ミリ秒単位でリクエストのタイムアウトを指定したい場合。
  * 注記: SDK モデルでのデフォルトは `300000` です。

* `scrape_options`
  * 型: `ScrapeOptions`
  * 使用する場面: 各検索結果をスクレイピングしたい場合 (フィールドについては スクレイピングのパラメータを参照) 。

<div id="scrape">
  ## スクレイピング
</div>

### 使用する理由

URLがすでに分かっていて、1つ以上のフォーマットで構造化されたコンテンツを取得したい場合は、スクレイピングを使用します。

<div id="preferred-sdk-method">
  ### 推奨SDKメソッド
</div>

`client.scrape(url, **options)` → `Document` (`firecrawl.v2.types`)

### 簡単な例

```py
doc = client.scrape("https://docs.firecrawl.dev", formats=["markdown"])
```

<div id="complex-example">
  ### 複雑な例
</div>

```py
doc = client.scrape(
    "https://example.com/pricing",
    formats=[
        "markdown",
        "links",
        {"type": "json", "prompt": "Extract plan names and prices."},
        {"type": "screenshot", "full_page": True, "quality": 80, "viewport": {"width": 1280, "height": 720}},
        {"type": "changeTracking", "modes": ["git-diff"], "tag": "pricing"},
        {"type": "attributes", "selectors": [{"selector": "a", "attribute": "href"}]},
    ],
    headers={"User-Agent": "FirecrawlDocsBot/1.0"},
    only_main_content=True,
    wait_for=1000,
    parsers=[{"type": "pdf", "mode": "auto", "max_pages": 5}],
    actions=[
        {"type": "click", "selector": "#accept"},
        {"type": "wait", "milliseconds": 750},
        {"type": "scrape"},
    ],
    location={"country": "US", "languages": ["en-US"]},
    remove_base64_images=True,
    fast_mode=True,
    block_ads=True,
    proxy="auto",
    max_age=86400000,
    store_in_cache=True,
    profile={"name": "docs-session", "save_changes": True},
)
```

<div id="parameters">
  ### パラメータ
</div>

* `url`
  * 型: str
  * 使用する場面: 特定のページをスクレイピングしたい場合

* `フォーマット`
  * Type: フォーマット文字列またはフォーマットオブジェクトのリスト
  * Use when: 複数の出力フォーマットが必要な場合に使用します。
  * Confirmed format strings:
    * `"markdown"`: Markdownコンテンツ
    * `"html"`: 整形済みHTML
    * `"rawHtml"` or `"raw_html"`: 生のHTML
    * `"links"`: ページリンク
    * `"images"`: 画像URL
    * `"screenshot"`: スクリーンショット出力
    * `"summary"`: 要約出力
    * `"changeTracking"` or `"change_tracking"`: changeTracking出力
    * `"attributes"`: 属性抽出
    * `"branding"`: ブランドプロファイル出力
    * `"audio"`: 音声抽出
    * `"video"`: 動画抽出
  * Object-only format types:
    * `{"type": "json", ...}`: JSON抽出。プレーンな文字列 `"json"` ではなく、オブジェクトを使用してください。
    * `{"type": "question", "question": "..."}`: 質問応答出力。
    * `{"type": "highlights", "query": "..."}`: 関連するソーステキスト出力。
  * Format object fields:
    * `type`: 上記のフォーマット文字列のいずれか、またはオブジェクト専用フォーマットの `"json"`、`"question"`、`"highlights"`
    * `question`: `type: "question"` で使用
    * `query`: `type: "highlights"` で使用
    * `prompt`: `type: "json"` では任意
    * `schema`: `type: "json"`、または changeTracking の JSONモードで使用する JSON schema
    * `modes`: `type: "changeTracking"` で使用する `"git-diff"` または `"json"` の配列
    * `tag`: `type: "changeTracking"` で使用する changeTracking タグ
    * `full_page`, `quality`, `viewport`: `type: "screenshot"` のスクリーンショットオプション
    * `selectors`: `type: "attributes"` で使用する `{selector, attribute}` の配列

* `headers`
  * 型: str をキー、str を値とする dict
  * 使用する場面: カスタムリクエストヘッダーが必要な場合。

* `include_tags`
  * 型: str のリスト
  * 使用する場面: 特定の HTML タグのみを含めたい場合。

* `exclude_tags`
  * 型: str のリスト
  * 使用する場面: 特定の HTML タグを除外したい場合。

* `only_main_content`
  * 型: bool
  * 使用する場合: ナビゲーション、フッター、その他の定型要素を削除したいとき。

* `timeout`
  * 型: int
  * 使用する場面: ミリ秒単位のタイムアウトが必要な場合。

* `wait_for`
  * 型: int
  * 使用する場面: ページのレンダリング完了を待つ必要がある場合 (ミリ秒) 。

* `mobile`
  * 型: bool
  * 使用する場合: モバイルビューポートが必要なとき。

* `parsers`
  * 型: パーサー名またはパーサーオブジェクトのリスト
  * 使用する場面: ファイルのパースを制御する必要がある場合 (例: PDF のパース) 。
  * 確認済みの値:
    * `"pdf"`: PDF のパースを有効化
    * `{ "type": "pdf", "mode": "fast" | "auto" | "ocr", "max_pages": int }`: PDF パーサーのオプション

* `アクション`
  * Type: アクションオブジェクトのリスト
  * Use when: 軽量なスクレイピング前アクションが必要な場合
  * 利用可能なアクションタイプ:
    * `wait`: `milliseconds` または `selector` が必要
    * `screenshot`: `full_page`、`quality`、`viewport` は任意
    * `click`: `selector` が必要
    * `write`: `text` が必要 (先にクリックして入力欄にフォーカスしてください)
    * `press`: `key` が必要
    * `scroll`: `direction` (`up` または `down`) が必要、`selector` は任意
    * `scrape`: 追加フィールドなし
    * `executeJavascript`: `script` が必要
    * `pdf`: `format` (A0, A1, A2, A3, A4, A5, A6, Letter, Legal, Tabloid, Ledger) 、`landscape`、`scale` は任意

* `ロケーション`
  * 型: `country` と `languages` を含む dict
  * 使用する場面: 地域情報や言語を考慮したスクレイピングが必要な場合。

* `skip_tls_verification`
  * 型: bool
  * 使用する場合: TLS の確認をスキップする必要がある場合。

* `remove_base64_images`
  * 型: bool
  * 使用する場合: Markdown出力からbase64画像を除外したいとき。

* `fast_mode`
  * 型: bool
  * 使用する場面: 忠実度を抑えて、より高速にスクレイピングしたい場合。

* `block_ads`
  * 型: bool
  * 使用する場合: 広告やCookieのポップアップをブロックしたいとき。

* `proxy`
  * 型: str
  * 使用する場面: プロキシ制御が必要な場合。
  * 使用可能な値: `"basic"`, `"stealth"`, `"enhanced"`, `"auto"`

* `max_age`
  * 型: int
  * 使用する場合: 最大経過時間 (ミリ秒) までのキャッシュデータを使用したいとき。

* `min_age`
  * 型: int
  * 使用する場面: キャッシュデータが少なくともこの値以上古い場合にのみ使いたいとき (ミリ秒) 。
  * 注記: `client.search` に `scrape_options` として渡す `ScrapeOptions` でのみ使用できます。`client.scrape` メソッドは `min_age` を受け付けません。

* `store_in_cache`
  * 型: bool
  * 使用する場面: Firecrawl で結果をキャッシュしたい場合。

* `profile`
  * 種類: `name` と、任意の `save_changes` または `saveChanges` を含む dict
  * 使用する場面: スクレイピングや操作の間で共有される永続的なブラウザプロファイルが必要な場合。

<div id="interact">
  ## Interact
</div>

### なぜ使うのか

ページで、スクレイピング開始後にブラウザのアクションやコード実行が必要な場合は、interact を使用します。

<div id="preferred-sdk-method">
  ### 推奨SDKメソッド
</div>

`client.interact(job_id, code=None, *, prompt=None, language="node", timeout=None)`

`prompt` はキーワード引数専用です。`client.interact(job_id, prompt="...")` または `client.interact(job_id, code="...")` のように呼び出してください。`code` または `prompt` の少なくとも一方には、空でない値を指定する必要があります (`firecrawl/v2/methods/scrape.py` で検証されます) 。

<div id="simple-example">
  ### シンプルな例
</div>

```py
doc = client.scrape("https://example.com", formats=["markdown"])
job_id = doc.metadata.scrape_id if doc.metadata else None
if not job_id:
    raise RuntimeError("Missing scrape_id from scrape response")

result = client.interact(job_id, prompt="Click the pricing tab and summarize the plans.")
```

<div id="complex-example">
  ### 複雑な例
</div>

```py
result = client.interact(
    "<scrapeJobId>",
    code="print(await page.title())",
    language="python",
    timeout=60,
)
```

<div id="stop-session">
  ### セッションを停止
</div>

`client.stop_interaction(job_id)` は、スクレイピングに関連付けられたブラウザセッションを終了します。戻り値は `BrowserDeleteResponse` (`success`、任意の `session_duration_ms`、`credits_billed`、`error`) です。

<div id="parameters">
  ### パラメータ
</div>

* `job_id`
  * 型: str
  * 使用する場合: `document.metadata.scrape_id` から取得したスクレイピングジョブ ID がある場合。

* `code`
  * 型: str
  * 使用する場合: ブラウザセッション内でコードを実行したい場合 (`prompt` が設定されている場合は任意) 。

* `prompt`
  * 型: str
  * 使用する場合: ブラウザエージェントに自然言語の指示を与えたい場合 (`code` が設定されている場合は任意) 。

* `language`
  * 型: str
  * 使用する場合: 特定のランタイムが必要な場合。
  * 使用可能な値: `"python"`, `"node"`, `"bash"`
  * 注: SDK ではデフォルトで `"node"` が使用されます。

* `timeout`
  * 型: int
  * 使用する場合: 秒単位の実行タイムアウトが必要な場合。

<div id="return-value">
  ### 戻り値
</div>

戻り値は `BrowserExecuteResponse` です: `success`、任意の `live_view_url`、`interactive_live_view_url`、`output`、`stdout`、`result`、`stderr`、`exit_code`、`killed`、`error` (モデルでは API の camelCase は snake&#95;case に正規化されます) 。

<div id="ask-agentic-debugging">
  ## Ask (エージェントによるデバッグ)
</div>

### 使用する理由

Firecrawl API の呼び出しが失敗した場合や、想定外の結果が返ってきた場合は、`ask` を使用します。AI サポートエージェントが問題を診断し、修正に必要なパラメータを提案し、必要に応じて実際の API に対して修正内容を検証します。通常の応答時間は 15〜30 秒です。

<div id="preferred-sdk-method">
  ### 推奨SDKメソッド
</div>

Python SDKではまだ利用できません。`requests`または`httpx`を直接使ってください。

### 基本的な例

```py
import requests

response = requests.post(
    "https://api.firecrawl.dev/v2/support/ask",
    headers={
        "Authorization": f"Bearer {os.environ['FIRECRAWL_API_KEY']}",
        "Content-Type": "application/json",
    },
    json={"question": "my scrape returned empty markdown for https://example.com"},
)
result = response.json()
print(result["answer"])
print(result["fixParameters"])
```

<div id="complex-example">
  ### 複雑な例
</div>

```py
import requests

response = requests.post(
    "https://api.firecrawl.dev/v2/support/ask",
    headers={
        "Authorization": f"Bearer {os.environ['FIRECRAWL_API_KEY']}",
        "Content-Type": "application/json",
    },
    json={
        "question": "crawl returned 3 pages but I expected 50",
        "rationale": "user is on their third failed crawl attempt today",
        "context": {"userPlan": "standard", "retryCount": 3},
    },
)
result = response.json()
```

<div id="agent-retry-pattern">
  ### Agentのリトライパターン
</div>

```py
from firecrawl import Firecrawl
import requests, os

client = Firecrawl(api_key=os.environ["FIRECRAWL_API_KEY"])

doc = client.scrape("https://example.com/pricing", formats=["markdown"])

if not doc.markdown or len(doc.markdown) < 100:
    diagnosis = requests.post(
        "https://api.firecrawl.dev/v2/support/ask",
        headers={
            "Authorization": f"Bearer {os.environ['FIRECRAWL_API_KEY']}",
            "Content-Type": "application/json",
        },
        json={
            "question": f"scrape returned only {len(doc.markdown or '')} chars for https://example.com/pricing",
        },
    ).json()

    if diagnosis.get("fixParameters"):
        doc = client.scrape(
            "https://example.com/pricing",
            formats=["markdown"],
            **diagnosis["fixParameters"],
        )
```

<div id="parameters">
  ### パラメータ
</div>

* `question`
  * 型: str (必須、1～8000文字) 
  * 使用する場面: 問題の内容を説明する必要がある場合。

* `rationale`
  * 型: str (1～2000文字) 
  * 使用する場面: ユーザーに代わって AI agent が呼び出す場合。

* `context`
  * 型: dict (自由形式) 
  * 使用する場面: デバッグ用の prompt にメタデータを渡したい場合。

<div id="docs-search">
  ## ドキュメント検索
</div>

<div id="why-use-it">
  ### これを使う理由
</div>

docs-search を使って Firecrawl のドキュメントを検索します。

### 基本的な例

```py
import requests

FIRECRAWL_API_KEY = "fc-YOUR_API_KEY"

response = requests.post(
    "https://api.firecrawl.dev/v2/support/docs-search",
    headers={"Authorization": f"Bearer {FIRECRAWL_API_KEY}"},
    json={"question": "how do I verify webhook signatures?"},
)
print(response.json()["answer"])
```

<div id="parameters">
  ### パラメータ
</div>

* `question`
  * 型: str (必須、1〜8000文字) 
  * 使用する場面: ドキュメントに基づいた回答が必要な場合。

<div id="notes">
  ## 注記
</div>

* 非推奨のエイリアス: `scrape_execute` → `interact`、`stop_interactive_browser` と `delete_scrape_browser` → `stop_interaction`。
* トップレベルの `Firecrawl` クライアントでは v2 メソッドを直接利用でき、v1 は `client.v1` にあります。
* `POST /v2/scrape/{jobId}/interact` 用にバンドルされている v2 OpenAPI スニペットでは `code` のみが記載されている場合がありますが、Python SDK とサーバーはこのエンドポイントで `code` または `prompt` のいずれも受け付けます。
