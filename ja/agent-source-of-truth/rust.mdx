エージェント向けの Firecrawl Rust の正規の情報源です。SDK のソースコードと v2 OpenAPI 仕様から生成されています。

<div id="install">
  ## インストール
</div>

```bash
cargo add firecrawl
```

Crate: crates.io の **`firecrawl`**。現在の SDK バージョンは **2.0.0** です (固定する前に、crates.io で最新リリースを確認してください) 。

<div id="authenticate">
  ## 認証
</div>

```rust
use firecrawl::Client;

let client = Client::new("fc-your-api-key")?;
// セルフホストの例:
// let client = Client::new_selfhosted("http://localhost:3002", Some("fc-your-api-key"))?;
```

<div id="when-to-use-what">
  ## 使い分けの目安
</div>

* `search`: クエリから始めて、対象を見つける必要がある場合に使います。
* `scrape`: すでに URL があり、ページコンテンツを取得したい場合に使います。
* `interact`: ページ上でのクリック、フォーム入力、またはスクレイピング後のブラウザアクションが必要な場合に使います。

- `support/ask`: Firecrawl API の呼び出しが失敗した場合や、想定外の結果が返ってきて原因の切り分けが必要な場合に使います。
- `support/docs-search`: Firecrawl のドキュメントを調べたい場合に使います。

<div id="search">
  ## 検索
</div>

<div id="why-use-it">
  ### どのような場合に使うか
</div>

クエリから関連するページを見つけ、スクレイピングまたは Interact する URL を選ぶには search を使用します。たとえば、`site:` を使うと、`site:docs.firecrawl.dev crawl webhooks` のように結果を特定のサイトに絞り込めます。

<div id="preferred-sdk-method">
  ### 推奨SDKメソッド
</div>

`client.search(query, options)` → `Result<SearchResponse, FirecrawlError>`

<div id="simple-example">
  ### シンプルな例
</div>

```rust
use firecrawl::Client;

let results = client
    .search("site:docs.firecrawl.dev webhook retries", None)
    .await?;
```

### 複雑な例

```rust
use firecrawl::{
    Client, SearchOptions, SearchSource, SearchCategory, ScrapeOptions, Format, JsonOptions,
};

let options = SearchOptions {
    sources: Some(vec![SearchSource::Web, SearchSource::News]),
    categories: Some(vec![SearchCategory::Research]),
    limit: Some(10),
    tbs: Some("qdr:m".to_string()),
    location: Some("San Francisco,California,United States".to_string()),
    ignore_invalid_urls: Some(true),
    timeout: Some(60000),
    scrape_options: Some(ScrapeOptions {
        formats: Some(vec![Format::Markdown, Format::Links, Format::Json]),
        json_options: Some(JsonOptions {
            prompt: Some("Extract title and key endpoints.".to_string()),
            ..Default::default()
        }),
        only_main_content: Some(true),
        ..Default::default()
    }),
    ..Default::default()
};

let results = client
    .search("site:docs.firecrawl.dev crawl webhooks", options)
    .await?;
```

<div id="return-type">
  ### 戻り値の型
</div>

`client.search(...)` は `Result<SearchResponse, FirecrawlError>` を返します。

* `SearchResponse`
  * `success: bool`
  * `data: SearchData`
  * `warning: Option<String>`
* `SearchData`
  * `web: Option<Vec<SearchResultOrDocument>>` — 各項目は `SearchResultOrDocument::WebResult(SearchResultWeb)` (url、title、description、…) または `SearchResultOrDocument::Document(Document)` のいずれかです。後者は、その行に対して API がスクレイピング済みコンテンツを返した場合に該当します (markdown、html、rawHtml、または metadata が存在することで判別されます) 。
  * `news: Option<Vec<SearchResultNews>>`
  * `images: Option<Vec<SearchResultImage>>`

<div id="parameters">
  ### パラメータ
</div>

* `query`
  * 型: `impl AsRef<str>`
  * 使用する場面: 検索クエリが必要な場合。
  * 注: 結果を特定のドメインに限定するには `site:example.com` を使用します。

* `options.sources`
  * 型: `Vec<SearchSource>`
  * 使用する場面: 検索対象のソースを制御したい場合。
  * 確認済みの値: `Web`, `News`, `Images`

* `options.categories`
  * 型: `Vec<SearchCategory>`
  * 使用する場面: カテゴリで結果を絞り込みたい場合。
  * 確認済みの値: `Github`, `Research`, `Pdf`

* `options.limit`
  * 型: u32
  * 使用する場面: 結果数の上限を設定したい場合。

* `options.tbs`
  * 型: String
  * 使用する場面: 時間ベースの絞り込みが必要な場合 (例: `qdr:d`, `qdr:w`, `sbd:1,qdr:m`) 。

* `options.location`
  * 型: String
  * 使用する場面: ローカライズされた結果を取得したい場合。

* `options.ignore_invalid_urls`
  * 型: bool
  * 使用する場面: 他のエンドポイントでスクレイピングできない URL を除外したい場合。

* `options.timeout`
  * 型: u32
  * 使用する場面: ミリ秒単位のリクエストタイムアウトが必要な場合。

* `options.scrape_options`
  * 型: `ScrapeOptions`
  * 使用する場面: 各検索結果をスクレイピングしたい場合 (フィールドについては Scrape のパラメータを参照) 。

* `options.integration`
  * 型: `Option<String>`
  * 使用する場面: サーバー側の追跡のためにインテグレーション識別子が必要な場合。
  * 注: 製品で意図的に設定している場合を除き、agent 指向の例では省略します。

<div id="scrape">
  ## スクレイピング
</div>

### これを使う理由

URLがすでにあり、1つ以上のフォーマットで構造化されたコンテンツを取得したい場合は、スクレイピングを使用します。

<div id="preferred-sdk-method">
  ### 推奨SDKメソッド
</div>

`client.scrape(url, options)` → `Result<Document, FirecrawlError>` (SDKではHTTPレスポンスの`data`フィールドが取り出された状態で返されます) 。

### 簡単な例

```rust
use firecrawl::{Client, ScrapeOptions, Format};

let doc = client
    .scrape("https://docs.firecrawl.dev", ScrapeOptions {
        formats: Some(vec![Format::Markdown]),
        ..Default::default()
    })
    .await?;
```

### 複雑な例

```rust
use firecrawl::{
    Client, ScrapeOptions, Format, JsonOptions, ScreenshotOptions, ChangeTrackingOptions,
    ChangeTrackingMode, AttributeSelector, Action, ParserConfig, ProxyType,
};

let doc = client
    .scrape("https://example.com/pricing", ScrapeOptions {
        formats: Some(vec![
            Format::Markdown,
            Format::Links,
            Format::Json,
            Format::Screenshot,
            Format::ChangeTracking,
            Format::Attributes,
        ]),
        json_options: Some(JsonOptions {
            prompt: Some("Extract plan names and prices.".to_string()),
            ..Default::default()
        }),
        screenshot_options: Some(ScreenshotOptions {
            full_page: Some(true),
            quality: Some(80),
            ..Default::default()
        }),
        change_tracking_options: Some(ChangeTrackingOptions {
            modes: Some(vec![ChangeTrackingMode::GitDiff]),
            tag: Some("pricing".to_string()),
            ..Default::default()
        }),
        attribute_selectors: Some(vec![AttributeSelector {
            selector: "a".to_string(),
            attribute: "href".to_string(),
        }]),
        parsers: Some(vec![ParserConfig::Pdf {
            parser_type: "pdf".to_string(),
            max_pages: Some(5),
        }]),
        actions: Some(vec![
            Action::Click { selector: "#accept".to_string() },
            Action::Wait { milliseconds: Some(750), selector: None },
            Action::Scrape,
        ]),
        proxy: Some(ProxyType::Auto),
        ..Default::default()
    })
    .await?;
```

<div id="parameters">
  ### パラメータ
</div>

* `url`
  * 型: `impl AsRef<str>`
  * 使用する場面: 特定のページをスクレイピングしたい場合。

* `options.formats`
  * 型: `Vec<Format>`
  * 使用する場合: 複数の出力フォーマットが必要なとき。
  * 確認済みの値: `Markdown`, `Html`, `RawHtml`, `Links`, `Images`, `Screenshot`, `Summary`, `ChangeTracking`, `Json`, `Attributes`, `Branding`, `Audio`, `Video`

* `options.headers`
  * 型: `HashMap<String, String>`
  * 使用する場面: カスタムのリクエストヘッダーが必要な場合。

* `options.include_tags`
  * 型: `Vec<String>`
  * 使用する場合: 特定のHTMLタグのみを含めたい場合。

* `options.exclude_tags`
  * 型: `Vec<String>`
  * 使用するケース: 特定のHTMLタグを除外したい場合。

* `options.only_main_content`
  * 型: bool
  * 使用する場面: nav、footer、その他のボイラープレートを除去したい場合。

* `options.timeout`
  * 型: u32
  * 使用するケース: ミリ秒単位のタイムアウトが必要な場合。

* `options.wait_for`
  * 型: u32
  * 使用する場面: ページのレンダリング完了を待つ必要がある場合 (ミリ秒) 。

* `options.mobile`
  * 型: bool
  * 使用する場面: モバイルのビューポートが必要な場合。

* `options.parsers`
  * 型: `Vec<ParserConfig>`
  * 使用するタイミング: ファイル解析の制御が必要な場合。
  * 確認されている値:
    * `ParserConfig::Simple("pdf".to_string())`
    * `ParserConfig::Pdf { parser_type: "pdf", max_pages: Some(n) }`

* `options.actions`
  * 種別: `Vec<Action>`
  * 使用する場面: 軽量なスクレイピング前アクションが必要な場合
  * サポートされているアクションタイプ:
    * `Wait`: `milliseconds` または `selector` が必須
    * `Screenshot`: `full_page`、`quality`、`viewport` は任意
    * `Click`: `selector` が必須
    * `Write`: `text` が必須 (先にクリックして入力欄にフォーカス)
    * `Press`: `key` が必須
    * `Scroll`: `direction` (`Up` または `Down`) が必須、`selector` は任意
    * `Scrape`: 追加フィールドなし
    * `ExecuteJavascript`: `script` が必須
    * `Pdf`: `format` (A0、A1、A2、A3、A4、A5、A6、Letter、Legal、Tabloid、Ledger) 、`landscape`、`scale` は任意

* `options.location`
  * 型: `LocationConfig`
  * 使用する場面: 地域や言語に応じたスクレイピングが必要な場合。
  * 確認済みのフィールド: `country`, `languages`

* `options.skip_tls_verification`
  * Type: bool
  * 使用する場面: TLS の確認をスキップする必要がある場合。

* `options.remove_base64_images`
  * 型: bool
  * 使用する場合: Markdown 出力から base64 画像を除外したいとき。

* `options.fast_mode`
  * 型: bool
  * 使用する場合: 忠実度を下げて、より高速にスクレイピングしたいとき。

* `options.block_ads`
  * 型: bool
  * 使用する場面: 広告や Cookie ポップアップのブロックが必要な場合。

* `options.proxy`
  * タイプ: `ProxyType`
  * 使用する場合: プロキシを制御する必要がある場合。
  * 使用可能な値: `Basic`, `Stealth`, `Enhanced`, `Auto`

* `options.max_age`
  * 型: u32
  * 使用する場合: 最大経過時間 (ミリ秒) までのキャッシュ済みデータを使いたいとき。

* `options.min_age`
  * 型: u32
  * 使用する場面: この値 (ミリ秒) 以上古いキャッシュ済みデータのみを使用したい場合。

* `options.store_in_cache`
  * 型: bool
  * 使用する場合: Firecrawl に結果をキャッシュさせたいとき。

* `options.profile`
  * Type: `ProfileConfig`
  * 使用する場面: スクレイピングや操作の間で共有される永続的なブラウザプロファイルが必要な場合。
  * 確認済みフィールド: `name`, `save_changes`

* `options.integration`
  * 型: `Option<String>`
  * 使用する場合: サーバー側での追跡にインテグレーション識別子が必要なとき。
  * 注意: エージェント向けの例では、製品で意図的に設定する場合を除き、省略してください。

* `options.json_options`
  * 型: `JsonOptions`
  * 使用するケース: JSON抽出を設定したい場合。
  * 確認済みのフィールド: `schema`, `system_prompt`, `prompt`

* `options.screenshot_options`
  * 型: `ScreenshotOptions`
  * 使用するケース: スクリーンショット出力を設定したい場合。
  * 確認済みフィールド: `full_page`, `quality`, `viewport`

* `options.change_tracking_options`
  * Type: `ChangeTrackingOptions`
  * 使用する場合: 変更追跡の出力を設定したいとき。
  * 確認済みフィールド: `modes` (`GitDiff` または `Json`), `schema`, `prompt`, `tag`

* `options.attribute_selectors`
  * 型: `Vec<AttributeSelector>`
  * 使用する場合: 属性提取の出力を取得したいとき。
  * 確認済みフィールド: `selector`, `attribute`

<div id="interact">
  ## Interact
</div>

### なぜこれを使うのか

ページで、スクレイピング開始後にブラウザのアクションやコードの実行が必要な場合は、interact を使用します。

<div id="preferred-sdk-method">
  ### 推奨SDKメソッド
</div>

`client.interact(job_id, options)`

ブラウザセッションを終了するには、`client.stop_interaction(job_id)` を使用します (`/v2/scrape/{jobId}/interact` に対する HTTP `DELETE`) 。

### 簡単な例

```rust
use firecrawl::{Client, ScrapeExecuteOptions};

let result = client
    .interact(
        "<scrapeJobId>",
        ScrapeExecuteOptions {
            prompt: Some("Click the pricing tab and summarize the plans.".to_string()),
            ..Default::default()
        },
    )
    .await?;
```

<div id="complex-example">
  ### 複雑な使用例
</div>

```rust
use firecrawl::{Client, ScrapeExecuteOptions, ScrapeExecuteLanguage};

let result = client
    .interact(
        "<scrapeJobId>",
        ScrapeExecuteOptions {
            code: Some("console.log(await page.title());".to_string()),
            language: Some(ScrapeExecuteLanguage::Node),
            timeout: Some(60),
            ..Default::default()
        },
    )
    .await?;
```

<div id="stop-session">
  ### セッション停止
</div>

```rust
let stopped = client.stop_interaction("<scrapeJobId>").await?;
// stopped: ScrapeBrowserDeleteResponse { success, session_duration_ms, credits_billed, error, ... }
```

<div id="parameters">
  ### パラメータ
</div>

* `job_id`
  * 型: `impl AsRef<str>`
  * 使用する場面: スクレイピングジョブ ID がある場合。

* `options.code`
  * 型: `Option<String>`
  * 使用する場面: ブラウザセッション内でコードを実行したい場合。

* `options.prompt`
  * 型: `Option<String>`
  * 使用する場面: ブラウザエージェントに自然言語の指示を与えたい場合。

* `options.language`
  * 型: `ScrapeExecuteLanguage`
  * 使用する場面: 特定のランタイムが必要な場合。
  * 使用可能な値: `Python`, `Node`, `Bash`
  * 注: 省略した場合、SDK ではデフォルトで `Node` が使用されます。

* `options.timeout`
  * 型: u32
  * 使用する場面: 秒単位で実行タイムアウトを指定する必要がある場合。

* `options.origin`
  * 型: `Option<String>`
  * 使用する場面: 実行テレメトリー用に任意の origin ラベルが必要な場合。
  * 注: プロダクト側で意図的に設定している場合を除き、エージェント向けの例では省略してください。

`options.code` または `options.prompt` の少なくとも 1 つは空でない必要があります。そうでない場合、SDK は API を呼び出す前に `FirecrawlError::Misuse` を返します。

<div id="response-types">
  ### レスポンスタイプ
</div>

* `interact` → `Result<ScrapeExecuteResponse, FirecrawlError>`
  * `success: bool` は常に含まれます。`live_view_url`、`interactive_live_view_url`、`output`、`stdout`、`result`、`stderr`、`exit_code`、`killed`、`error` は、いずれも構造体上の `Option` フィールドです。
* `stop_interaction` → `Result<ScrapeBrowserDeleteResponse, FirecrawlError>`
  * `success: bool` は常に含まれます。`session_duration_ms`、`credits_billed`、`error` は任意です。

v2 OpenAPI spec では現在、interact のリクエストボディで `code` が必須として定義されています。Rust SDK はサーバーの挙動に合わせて、**`code`** または **`prompt`** のどちらかを受け付けます (`scrape.rs` の `ScrapeExecuteOptions` を参照) 。

<div id="notes">
  ## 注意
</div>

* 非推奨エイリアス: `scrape_execute`、`stop_interactive_browser`、`delete_scrape_browser` は、`interact` と `stop_interaction` に対応しています。
* `ScrapeOptions` には、高度なフォーマット向けの専用オプションとして、`json_options`、`screenshot_options`、`change_tracking_options` が含まれています。
* `search_and_scrape(query, limit)` は便利なヘルパーです。デフォルトの `ScrapeOptions` を指定して `search` を呼び出し、`data.web` 内の `SearchResultOrDocument::Document` エントリから構築した `Vec<Document>` を返します (`search.rs` を参照) 。
* v2 SDK では、すべての型が crate ルートからエクスポートされます: `use firecrawl::Client` (`use firecrawl::v2::Client` ではありません) 。
* v2 ではエラー型が簡略化されています: `CrawlJobFailed(String, CrawlStatus)` → `JobFailed(String)`、`Missuse` → `Misuse`。
