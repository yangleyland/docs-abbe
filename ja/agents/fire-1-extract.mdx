FIRE-1 は、賢い Web 回遊と操作によって Firecrawl のスクレイピング機能を拡張する AI エージェントです。ページネーションの処理、ブラウザのアクション制御、複雑なサイト構造の横断に対応し、従来のスクレイピングを超える包括的なデータ抽出を実現します。

<div id="what-fire-1-can-do">
  ### FIRE-1 ができること:
</div>

- ページ分割されたコンテンツを自動でたどります。
- ボタン、リンク、入力欄、動的要素を操作します。
- 複数ページにわたる高度な抽出タスクを実行します。

<div id="enabling-fire-1-agent">
  ## FIRE-1 エージェントの有効化
</div>

FIRE-1 エージェントを有効化するには、`scrape` または `extract` エンドポイント向けの API リクエストのペイロードに `agent` オブジェクトを含めます。

`agent` オブジェクトには次のプロパティがあります:

*   `model` (string, 任意): 使用する AI モデルを指定します。指定がない場合は `FIRE-1` がデフォルトです。現在利用可能なモデルは `FIRE-1` のみです。
*   `prompt` (string, `scrape` エンドポイントでは必須): AI エージェントへの指示。探すべきコンテンツやサイトの操作方法（例: ページネーションの扱い、クリックするボタンなど）を記述します。`/extract` では `prompt` パラメータで渡されたプロンプトが使用されます。

<div id="using-fire-1-with-the-scrape-endpoint">
  ### /scrape エンドポイントでの FIRE-1 の利用
</div>

`/v1/scrape` エンドポイントで FIRE-1（エージェント）を使うと、最終的なコンテンツをスクレイプする前にインテリジェントなナビゲーションを実行できます。

**例（cURL）：**

```bash
curl -X POST https://api.firecrawl.dev/v1/scrape \
    -H 'Content-Type: application/json' \
    -H 'Authorization: Bearer YOUR_API_KEY' \
    -d '{
      "url": "https://example.com/products?page=1",
      "formats": ["markdown"],
      "agent": {
        "model": "FIRE-1",
        "prompt": "「次へ」ボタンが無効になるまでクリックして商品一覧を順に移動し、訪れた各ページのコンテンツをスクレイピングしてください。"
      }
    }'
```

この例では、FIRE-1（エージェント）に、最終的なスクレイプを行う前に商品リストをページ送り（ページネーション）するよう指示しています。

<div id="using-fire-1-with-the-extract-endpoint">
  ### Extract エンドポイントで FIRE-1 を使用する
</div>

同様に、複数のページにまたがるナビゲーションや要素とのインタラクションが必要な複雑な抽出タスクでは、`/v1/extract` エンドポイントで FIRE-1 エージェントを活用できます。

**例 (cURL) :**

```bash
curl -X POST https://api.firecrawl.dev/v1/extract \
    -H 'Content-Type: application/json' \
    -H 'Authorization: Bearer YOUR_API_KEY' \
    -d '{
      "urls": ["https://example-forum.com/topic/123"],
      "prompt": "Extract all user comments from this forum thread.",
      "schema": {
        "type": "object",
        "properties": {
          "comments": {
            "type": "array",
            "items": {
              "type": "object",
              "properties": {
                "author": {"type": "string"},
                "comment_text": {"type": "string"}
              },
              "required": ["author", "comment_text"]
            }
          }
        },
        "required": ["comments"]
      },
      "agent": {
        "model": "FIRE-1"
      }
    }'
```

ここでは、指定されたschemaとpromptに基づく抽出プロセスが始まる前に、agentが&quot;さらにコメントを読み込む&quot;ボタンを操作して、ページ上のすべてのコメントが読み込まれるようにします。
**注:** FIRE-1 agentは強力な機能を備えていますが、ナビゲーション指示の複雑さや操作対象のページ数によっては、より多くのクレジットを消費する可能性があります。

<div id="meet-fire-1-intelligent-navigation-and-interaction">
  ### FIRE-1 の紹介: インテリジェントなナビゲーションと操作
</div>

![FIRE-1 Agent Visualization](/launch-week/lw3-d2-3.webp)

FIRE-1 は Firecrawl に新たなインテリジェンスをもたらし、複雑な Web サイト構造のナビゲーション、ページネーションの処理、動的コンテンツの操作などを通じて、スクレイピング作業を強化します。この強力な AI エージェントにより、従来のスクレイピング手法では難しかった、より包括的なデータ抽出が可能になります。

### FIRE-1でできること:

- ページ分割されたコンテンツを自動で巡回します。
- ボタンやリンクなどの動的要素を操作します。
- 複数ページにまたがる高度なデータ抽出を実行します。

<div id="how-to-enable-fire-1">
  ## FIRE-1 を有効化する方法
</div>

FIRE-1 の有効化は簡単です。/scrape エンドポイントまたは /extract エンドポイントへの API リクエストに、agent オブジェクトを含めるだけです。

```json
"agent": {
  "model": "FIRE-1",
  "prompt": "ここに詳しいナビゲーション手順を入力してください。"
}
```

*注記:* `prompt` フィールドはスクレイプリクエストで必須で、FIRE-1 にウェブページとの具体的なインタラクション方法を正確に指示します。

<div id="example-usage-with-scrape-endpoint">
  ## /scrape エンドポイントを使った使用例
</div>

以下は、FIRE-1 と /scrape エンドポイントを使って商品一覧をページネーションしながらたどる簡単な例です。

```bash
curl -X POST https://api.firecrawl.dev/v1/scrape \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -d '{
    "url": "https://example.com/products?page=1",
    "formats": ["markdown"],
    "agent": {
      "model": "FIRE-1",
      "prompt": "Navigate through the product listings by clicking the \'Next Page\' button until disabled. Scrape each page."
    }
  }'
```

このシナリオでは、FIRE-1 はページネーションを適切にたどり、すべての商品を収集します。

<div id="considerations">
  ## 注意点
</div>

* FIRE-1 の使用では、タスクの複雑さややり取りの深さに応じて、より多くのクレジットを消費する場合があります。
* 結果と効率を最適化できるよう、FIRE-1 に対する指示が明確になるようプロンプトを作成してください。
