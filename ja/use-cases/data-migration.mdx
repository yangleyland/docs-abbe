あらゆる Web サイトからコンテンツ、構造、メタデータを抽出し、新しいプラットフォームにインポートできる形式に変換します。スクレイピングは Firecrawl が担うため、ターゲットシステム向けのデータマッピングに集中できます。

<div id="start-with-a-template">
  ## テンプレートから始める
</div>

<Card title="Firecrawl Migrator" icon="github" href="https://github.com/firecrawl/firecrawl-migrator">
  プラットフォームやシステム間のデータ移行を効率化
</Card>

<div id="how-it-works">
  ## 仕組み
</div>

Firecrawl で移行元の Web サイトを指定すると、すべてのページをクロールし、整形された構造化データを返します。次に、その出力を移行先プラットフォームのスキーマに合わせて変換し、そのプラットフォームの API または一括インポートツールを使って取り込みます。

<div id="what-you-can-migrate">
  ## 移行可能な内容
</div>

* **コンテンツ**: ページ、投稿、記事、メディアファイル、メタデータ
* **構造**: 階層、カテゴリー、タグ、タクソノミー
* **ユーザー**: プロフィールおよび公開されているユーザー関連データ
* **設定**: 設定項目、カスタムフィールド、ワークフロー
* **EC**: 商品、カタログ、在庫、注文

<div id="common-migration-use-cases">
  ## 一般的な移行ユースケース
</div>

ユーザーは Firecrawl を使って、さまざまなプラットフォームからデータを抽出するための移行ツールを構築しています。

<CardGroup cols={2}>
  <Card title="CMSコンテンツの抽出">
    WordPress、Drupal、Joomla のサイトや独自の CMS プラットフォームからコンテンツを抽出します。コンテンツ構造とメタデータを保持したまま、新しいシステム (Contentful、Strapi、Sanity など) にインポートできる形式でエクスポートします。
  </Card>

  <Card title="ECデータの抽出">
    Magento や WooCommerce ストアから、在庫、価格、説明、仕様を含む商品カタログを抽出します。Shopify、BigCommerce、その他のプラットフォームにインポートできる形式にデータを整えます。
  </Card>
</CardGroup>

<div id="faqs">
  ## よくある質問
</div>

<AccordionGroup>
  <Accordion title="大規模な移行にはどのように対応しますか？">
    当社のインフラは大規模な移行に合わせて自動的にスケールします。バッチ処理と並列抽出による段階的処理に対応しており、進捗を追跡しながら管理しやすい単位に分割することで、数百万ページ規模でも移行できます。
  </Accordion>

  <Accordion title="移行中にSEOの価値は維持できますか？">
    はい。URL、タイトル、ディスクリプションなどのSEOメタデータをすべて抽出し、適切なリダイレクトを実装します。移行期間中も検索順位の維持を支援します。
  </Accordion>

  <Accordion title="メディアファイルや添付ファイルはどう扱いますか？">
    Firecrawlはすべてのメディアファイルを抽出・カタログ化できます。新しいプラットフォームに再アップロードするためにダウンロードすることも、同一のCDNを継続利用する場合は直接参照することも可能です。
  </Accordion>

  <Accordion title="移行の検証はどのように行いますか？">
    詳細な抽出レポートを提供し、比較ツールにも対応しています。コンテンツの網羅性の確認、リンク切れのチェック、データ整合性の検証が行えます。
  </Accordion>

  <Accordion title="ユーザー生成コンテンツやコメントも移行できますか？">
    はい。コメント、レビュー、フォーラム投稿など、公開されているユーザー生成コンテンツを抽出できます。非公開のユーザーデータには、適切な認証と権限が必要です。
  </Accordion>
</AccordionGroup>
