Fonte oficial canônica de curl do Firecrawl para agentes. Alinhada com `api-reference/v2-openapi.json` (URL base do servidor `https://api.firecrawl.dev/v2`).

<div id="authenticate">
  ## Autenticação
</div>

```bash
export FIRECRAWL_API_KEY="fc-your-api-key"
```

<div id="when-to-use-what">
  ## Quando usar cada um
</div>

* `search`: use quando você começa com uma consulta e precisa descobrir resultados.
* `scrape`: use quando você já tem uma URL e quer o conteúdo da página.
* `interact`: use quando a página exige cliques, formulários ou ações no navegador após o scraping.
* `support/ask`: use quando uma chamada da API do Firecrawl falha ou retorna resultados inesperados e você precisa de um diagnóstico.
* `support/docs-search`: use quando você precisa consultar a documentação do Firecrawl.

<div id="search">
  ## Busca
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use a busca para descobrir páginas relevantes a partir de uma consulta e, em seguida, escolher URLs para fazer scraping ou interagir. Você pode restringir os resultados a um site com `site:`, por exemplo `site:docs.firecrawl.dev crawl webhooks`.

<div id="endpoint">
  ### Endpoint
</div>

`POST /search`

<div id="simple-example">
  ### Exemplo simples
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/search" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "site:docs.firecrawl.dev webhook retries"
  }'
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/search" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "site:docs.firecrawl.dev crawl webhooks",
    "sources": [{"type": "web"}, {"type": "news"}],
    "categories": [{"type": "research"}],
    "limit": 10,
    "tbs": "qdr:m",
    "location": "San Francisco,California,United States",
    "country": "US",
    "ignoreInvalidURLs": true,
    "timeout": 60000,
    "enterprise": ["anon"],
    "scrapeOptions": {
      "formats": [
        "markdown",
        "links",
        {"type": "json", "prompt": "Extract title and key endpoints."}
      ],
      "onlyMainContent": true,
      "includeTags": ["main", "article"],
      "excludeTags": ["nav", "footer"],
      "waitFor": 1000
    }
  }'
```

<div id="response">
  ### Resposta
</div>

Respostas bem-sucedidas incluem `success`, `data`, `warning` opcional, `id` e `creditsUsed`.

* `data.web`, `data.images`, `data.news`: arrays de resultados; as chaves exibidas dependem de `sources` (por padrão, apenas `data.web` é preenchido).
* Itens de web e de notícias incluem campos como `title`, `url` e (quando `scrapeOptions` / formatos os solicitam) `markdown`, `html`, `rawHtml`, `links`, `screenshot`, `audio`, `video` e `metadata`.
* Itens de imagem incluem campos como `imageUrl`, `url` e dimensões, quando disponíveis.
* `warning`: aviso opcional legível por humanos.
* `id`: string com o ID do job de busca.
* `creditsUsed`: número inteiro de créditos cobrados pela chamada.

<div id="parameters">
  ### Parâmetros
</div>

* `query`
  * Tipo: string (obrigatório, comprimento máximo de 500)
  * Use quando: você precisar de uma consulta de busca.
  * Observações: use `site:example.com` para limitar os resultados a um domínio.

* `sources`
  * Tipo: array de objetos de fonte tipados
  * Use quando: você quiser controlar quais fontes são pesquisadas.
  * Formatos de objeto confirmados:
    * `{ "type": "web" }` com `tbs` e `location` opcionais por fonte
    * `{ "type": "news" }`
    * `{ "type": "images" }`

* `categories`
  * Tipo: array de objetos de categoria tipados
  * Use quando: você quiser filtrar os resultados por categoria.
  * Formatos de objeto confirmados:
    * `{ "type": "github" }`
    * `{ "type": "research" }`
    * `{ "type": "pdf" }`

* `limit`
  * Tipo: inteiro (mínimo 1, máximo 100, padrão 5)
  * Use quando: você quiser limitar a quantidade de resultados.

* `tbs`
  * Tipo: string
  * Use quando: você precisar de um filtro baseado em tempo (por exemplo, `qdr:d`, `qdr:w`, `sbd:1,qdr:m`).

* `location`
  * Tipo: string
  * Use quando: você quiser resultados localizados.

* `country`
  * Tipo: string (padrão `"US"`)
  * Use quando: você quiser direcionamento ISO 3166-1 alpha-2 (por exemplo, `"US"`).

* `ignoreInvalidURLs`
  * Tipo: boolean (padrão false)
  * Use quando: você quiser descartar URLs que não podem ser extraídas por outros endpoints.

* `timeout`
  * Tipo: inteiro (milissegundos, padrão 60000)
  * Use quando: você precisar de um tempo limite da requisição em milissegundos.

* `enterprise`
  * Tipo: array de strings (`"anon"` ou `"zdr"` por item)
  * Use quando: você precisar de controles de busca empresariais.
  * Valores:
    * `"zdr"`: zero data retention de ponta a ponta
    * `"anon"`: zero data retention anônimo

* `scrapeOptions`
  * Tipo: objeto
  * Use quando: você quiser fazer scraping de cada resultado de busca (consulte os parâmetros de Scrape para os campos).

<div id="scrape">
  ## Scraping
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use o scraping quando você já tiver uma URL e quiser conteúdo estruturado em um ou mais formatos.

<div id="endpoint">
  ### Endpoint
</div>

`POST /scrape`

<div id="simple-example">
  ### Exemplo simples
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://docs.firecrawl.dev",
    "formats": ["markdown"]
  }'
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/pricing",
    "formats": [
      "markdown",
      "links",
      {"type": "json", "prompt": "Extract plan names and prices."},
      {"type": "screenshot", "fullPage": true, "quality": 80, "viewport": {"width": 1280, "height": 720}},
      {"type": "changeTracking", "modes": ["git-diff"], "tag": "pricing"}
    ],
    "headers": {"User-Agent": "FirecrawlDocsBot/1.0"},
    "onlyMainContent": true,
    "waitFor": 1000,
    "parsers": [{"type": "pdf", "mode": "auto", "maxPages": 5}],
    "actions": [
      {"type": "click", "selector": "#accept"},
      {"type": "wait", "milliseconds": 750},
      {"type": "scrape"}
    ],
    "location": {"country": "US", "languages": ["en-US"]},
    "removeBase64Images": true,
    "blockAds": true,
    "proxy": "auto",
    "maxAge": 86400000,
    "minAge": 1,
    "storeInCache": true,
    "profile": {"name": "docs-session", "saveChanges": true},
    "zeroDataRetention": false
  }'
```

<div id="response">
  ### Resposta
</div>

As respostas bem-sucedidas incluem `success` e `data`. Campos comuns de `data` (dependendo de `formats` e das opções):

* `markdown`, `summary`, `html`, `rawHtml`, `screenshot`, `audio`, `video`, `links`
* `ações`: quando a requisição incluiu `ações` durante o scraping, contém resultados em ordem, como `screenshots`, `scrapes`, `javascriptReturns` e `pdfs`
* `metadata`: metadados da página (`title`, `sourceURL`, `url`, `statusCode`, `error` e outros campos extraídos)
* `warning`: aviso opcional de extração ou formatação
* `changeTracking`: presente quando o formato `changeTracking` é solicitado

<div id="parameters">
  ### Parâmetros
</div>

* `url`
  * Tipo: string
  * Use quando: você quiser fazer scraping de uma página específica.

* `formats`
  * Tipo: array de strings de formato ou objetos de formato
  * Use quando: você quiser vários formatos de resultado.
  * Strings de formato confirmadas:
    * `"markdown"`: conteúdo em markdown
    * `"html"`: HTML limpo
    * `"rawHtml"`: HTML bruto
    * `"links"`: links da página
    * `"images"`: URLs de imagens
    * `"screenshot"`: resultado de captura de tela
    * `"summary"`: resultado de resumo
    * `"changeTracking"`: resultado de rastreamento de mudanças
    * `"json"`: extração JSON
    * `"branding"`: resultado do perfil de marca
    * `"audio"`: extração de áudio
    * `"video"`: extração de vídeo
  * Campos do objeto de formato:
    * `type`: uma das strings de formato acima
    * `prompt`, `schema`: opções de extração JSON para `type: "json"`
    * `modes`, `schema`, `prompt`, `tag`: opções de rastreamento de mudanças para `type: "changeTracking"`
    * `fullPage`, `quality`, `viewport`: opções de captura de tela para `type: "screenshot"`

* `headers`
  * Tipo: object
  * Use quando: você precisar de cabeçalhos de requisição personalizados.

* `includeTags`
  * Tipo: array de strings
  * Use quando: você quiser incluir apenas tags HTML específicas.

* `excludeTags`
  * Tipo: array de strings
  * Use quando: você quiser excluir tags HTML específicas.

* `onlyMainContent`
  * Tipo: boolean
  * Use quando: você quiser remover navegação, rodapé e outros elementos repetitivos.

* `timeout`
  * Tipo: number
  * Use quando: você precisar de um tempo limite em milissegundos.

* `waitFor`
  * Tipo: number
  * Use quando: você precisar aguardar a renderização da página (em milissegundos).

* `mobile`
  * Tipo: boolean
  * Use quando: você quiser uma viewport para dispositivos móveis.

* `parsers`
  * Tipo: array de objetos
  * Use quando: você precisar de controles de análise de arquivos.
  * Formato confirmado: `{ "type": "pdf", "mode": "fast" | "auto" | "ocr", "maxPages": number }` (`type` obrigatório; outros campos opcionais com padrões conforme a especificação)

* `actions`
  * Tipo: array de objetos de ação
  * Use quando: você precisar de ações leves antes do scraping.
  * Tipos de ação confirmados:
    * `wait`: `milliseconds` ou `selector` obrigatórios
    * `screenshot`: `fullPage`, `quality`, `viewport` opcionais
    * `click`: `selector` obrigatório, `all` opcional
    * `write`: `text` obrigatório (clique para focar o campo de entrada primeiro)
    * `press`: `key` obrigatório
    * `scroll`: `type` obrigatório; `direction` (`up` ou `down`, padrão `down`); `selector` opcional
    * `scrape`: sem campos adicionais
    * `executeJavascript`: `script` obrigatório
    * `pdf`: `format` (A0, A1, A2, A3, A4, A5, A6, Letter, Legal, Tabloid, Ledger), `landscape`, `scale` opcionais

* `location`
  * Tipo: object com `country` e `languages`
  * Use quando: você precisar de scraping com reconhecimento de localidade ou idioma.

* `skipTlsVerification`
  * Tipo: boolean
  * Use quando: você precisar ignorar a verificação de TLS.

* `removeBase64Images`
  * Tipo: boolean
  * Use quando: você quiser remover imagens em base64 do resultado em markdown.

* `blockAds`
  * Tipo: boolean
  * Use quando: você quiser bloqueio de anúncios e pop-ups de cookies.

* `proxy`
  * Tipo: string
  * Use quando: você precisar de controle de proxy.
  * Valores confirmados: `"basic"`, `"enhanced"`, `"auto"`

* `maxAge`
  * Tipo: number
  * Use quando: você quiser dados em cache até uma idade máxima (em milissegundos).

* `minAge`
  * Tipo: number
  * Use quando: você quiser dados em cache apenas se tiverem pelo menos essa idade (em milissegundos).

* `storeInCache`
  * Tipo: boolean
  * Use quando: você quiser que o Firecrawl armazene o resultado em cache.

* `profile`
  * Tipo: object com `name` e `saveChanges` opcional
  * Use quando: você quiser um perfil persistente de navegador compartilhado entre operações de scraping e interações.

* `zeroDataRetention`
  * Tipo: boolean
  * Use quando: você quiser retenção zero de dados para este scraping.

<div id="interact">
  ## Interagir
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use o recurso de interagir quando uma página exigir ações no navegador ou execução de código após o início de um scraping.

<div id="endpoint">
  ### Endpoint
</div>

`POST /scrape/{jobId}/interact`

<div id="simple-example">
  ### Exemplo simples
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape/<jobId>/interact" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "code": "console.log(await page.title());"
  }'
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape/<jobId>/interact" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "code": "const title = await page.title();\nconst h1 = await page.locator(\"h1\").first().textContent().catch(() => null);\nconsole.log(JSON.stringify({ title, h1 }));",
    "language": "node",
    "timeout": 120
  }'
```

<div id="parameters">
  ### Parâmetros
</div>

* `jobId` (path)
  * Tipo: string (UUID)
  * Use quando: você tiver o ID do job de scraping da sessão ativa do navegador.

* `code` (JSON body)
  * Tipo: string (obrigatório no OpenAPI; comprimento mínimo 1, comprimento máximo 100000)
  * Use quando: quiser executar código no sandbox do navegador vinculado ao scraping.

* `language` (JSON body)
  * Tipo: string
  * Use quando: precisar de um runtime específico.
  * Valores confirmados: `"python"`, `"node"`, `"bash"` (padrão: `"node"`)

* `timeout` (JSON body)
  * Tipo: integer (segundos; mínimo 1, máximo 300, padrão: 30)
  * Use quando: precisar de um tempo limite de execução.

<div id="response">
  ### Resposta
</div>

As respostas bem-sucedidas incluem `success` e campos de execução, como `stdout`, `result` (alias de `stdout`), `stderr`, `exitCode`, `killed` e `error` (pode ser nulo).

### DELETE /scrape/{jobId}/interact

<div id="example">
  ### Exemplo
</div>

```bash
curl -X DELETE "https://api.firecrawl.dev/v2/scrape/<jobId>/interact" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY"
```

Sem corpo JSON. Uma resposta bem-sucedida inclui `success`.

<div id="ask-agentic-debugging">
  ## Perguntar (Depuração com agente)
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use o ask quando uma chamada da API Firecrawl falhar ou retornar resultados inesperados. O agente de suporte por IA diagnostica o problema, propõe parâmetros de correção e, opcionalmente, valida a correção na API em produção. Latência típica: 15-30 segundos.

<div id="endpoint">
  ### Endpoint
</div>

`POST /support/ask`

### Exemplo básico

```bash
curl -X POST "https://api.firecrawl.dev/v2/support/ask" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "my scrape returned empty markdown for https://example.com"
  }'
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/support/ask" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "crawl returned 3 pages but I expected 50",
    "rationale": "user is on their third failed crawl attempt today",
    "context": {"userPlan": "standard", "retryCount": 3}
  }'
```

<div id="response">
  ### Resposta
</div>

As respostas bem-sucedidas incluem `requestId`, `answer`, `confidence`, `fixParameters`, `validation`, `usage` e `durationMs`.

* `answer`: texto em prosa de 2 a 4 frases que abrange o diagnóstico e a correção.
* `confidence`: `high`, `medium` ou `low`.
* `fixParameters`: parâmetros de API acionáveis por máquina para aplicar a correção (`null` se nenhuma correção for aplicável).
* `validation.tested`: se o agente testou a correção na API em produção.
* `validation.result`: `success`, `failure` ou `skipped`.
* `feedback`: presente quando o agente fica preso; `null` em caso de sucesso.

<div id="parameters">
  ### Parâmetros
</div>

* `question`
  * Tipo: string (obrigatório, 1-8000 caracteres)
  * Use quando: precisar descrever o problema.

* `rationale`
  * Tipo: string (1-2000 caracteres)
  * Use quando: você for um agente de IA fazendo a chamada em nome de um usuário. Descreva o que o usuário está tentando fazer.

* `context`
  * Tipo: object (livre)
  * Use quando: quiser enviar metadados do seu agente para o prompt de depuração.

<div id="docs-search">
  ## Busca na documentação
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use `docs-search` para pesquisar na documentação do Firecrawl.

<div id="endpoint">
  ### Endpoint
</div>

`POST /support/docs-search`

<div id="example">
  ### Exemplo
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/support/docs-search" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "how do I verify webhook signatures?"
  }'
```

<div id="parameters">
  ### Parâmetros
</div>

* `question`
  * Tipo: string (obrigatório, 1-8000 caracteres)
  * Use quando: precisar de uma resposta fundamentada na documentação.

<div id="notes">
  ## Notas
</div>

* As linhas de resultado de busca ficam aninhadas em `data.web`, `data.images` ou `data.news`, não em um array `data` simples.
* Use `POST /scrape/{jobId}/interact` para fluxos de trabalho no navegador com várias etapas; as `ações` de scraping são melhores para pequenas etapas antes do scraping.
* O esquema OpenAPI publicado exige `code` para `interact`. Alguns SDKs e clientes também aceitam um campo `prompt` para instruções em linguagem natural; esse alias não está em `v2-openapi.json`.
