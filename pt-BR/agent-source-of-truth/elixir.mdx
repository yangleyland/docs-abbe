Fonte oficial canônica do Firecrawl em Elixir para agentes. Gerada a partir do código-fonte do SDK e da especificação OpenAPI v2.

<div id="install">
  ## Instalação
</div>

Adicione ao `mix.exs`:

```elixir
{:firecrawl, "~> 1.0.0"}
```

<div id="authenticate">
  ## Autenticação
</div>

```elixir
# config/runtime.exs ou config.exs
config :firecrawl, api_key: System.get_env("FIRECRAWL_API_KEY")

# ou passe api_key em cada chamada
{:ok, res} = Firecrawl.search_and_scrape([query: "site:docs.firecrawl.dev webhook retries"], api_key: "fc-your-api-key")
```

<div id="when-to-use-what">
  ## Quando usar cada um
</div>

* `search`: use quando partir de uma consulta e precisar encontrar resultados.
* `scraping`: use quando já tiver uma URL e quiser o conteúdo da página.
* `interagir`: use quando a página exigir cliques, formulários ou ações do navegador após o scraping.

<div id="search">
  ## Busca
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use a busca para descobrir páginas relevantes com base em uma consulta e, depois, escolher URLs para fazer scraping ou interagir com elas. Você pode restringir os resultados a um site com `site:`, por exemplo `site:docs.firecrawl.dev crawl webhooks`.

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

`Firecrawl.search_and_scrape(params \\ [], opts \\ [])`

### Exemplo básico

```elixir
{:ok, res} = Firecrawl.search_and_scrape(query: "site:docs.firecrawl.dev webhook retries")
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```elixir
{:ok, res} = Firecrawl.search_and_scrape(
  query: "site:docs.firecrawl.dev crawl webhooks",
  sources: [:web, :news],
  categories: [:research],
  limit: 10,
  tbs: "qdr:m",
  location: "San Francisco,California,United States",
  country: "US",
  ignore_invalid_urls: true,
  timeout: 60000,
  scrape_options: [
    formats: [
      "markdown",
      "links",
      %{type: "json", prompt: "Extract title and key endpoints."}
    ],
    only_main_content: true,
    include_tags: ["main", "article"],
    exclude_tags: ["nav", "footer"],
    wait_for: 1000
  ]
)
```

<div id="parameters">
  ### Parâmetros
</div>

* `query`
  * Tipo: string
  * Use quando: você precisar de uma consulta de busca.
  * Observações: use `site:example.com` para limitar os resultados a um domínio.

* `sources`
  * Tipo: lista de atoms, strings ou maps
  * Use quando: você quiser controlar quais fontes são pesquisadas.
  * Valores confirmados:
    * `"web"` ou `:web`
    * `"news"` ou `:news`
    * `"images"` ou `:images`
    * `%{type: "web" | "news" | "images"}`

* `categories`
  * Tipo: lista de atoms, strings ou maps
  * Use quando: você quiser filtrar os resultados por categoria.
  * Valores confirmados:
    * `"github"` ou `:github`
    * `"research"` ou `:research`
    * `"pdf"` ou `:pdf`
    * `%{type: "github" | "research" | "pdf"}`

* `limit`
  * Tipo: integer
  * Use quando: você quiser limitar o número de resultados.

* `tbs`
  * Tipo: string
  * Use quando: você precisar de um filtro baseado em tempo (por exemplo, `qdr:d`, `qdr:w`, `sbd:1,qdr:m`).

* `location`
  * Tipo: string
  * Use quando: você quiser resultados localizados.

* `country`
  * Tipo: string
  * Use quando: você quiser segmentação por ISO 3166-1 alpha-2 (por exemplo, `"US"`).

* `ignore_invalid_urls`
  * Tipo: boolean
  * Use quando: você quiser descartar URLs que não podem ser extraídas por outros endpoints.

* `timeout`
  * Tipo: integer
  * Use quando: você precisar de um tempo limite da requisição em milissegundos.

* `enterprise`
  * Tipo: lista de strings
  * Use quando: você precisar de controles de busca empresariais.
  * Valores confirmados:
    * `"zdr"`: zero data retention de ponta a ponta
    * `"anon"`: zero data retention anonimizado

* `scrape_options`
  * Tipo: keyword list
  * Use quando: você quiser fazer scraping de cada resultado de busca (consulte os parâmetros de Scrape para ver os campos).

<div id="scrape">
  ## Scrape
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use o scraping quando você já tiver uma URL e quiser conteúdo estruturado em um ou mais formatos.

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

`Firecrawl.scrape_and_extract_from_url(params \\ [], opts \\ [])`

<div id="simple-example">
  ### Exemplo simples
</div>

```elixir
{:ok, res} = Firecrawl.scrape_and_extract_from_url(
  url: "https://docs.firecrawl.dev",
  formats: ["markdown"]
)
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```elixir
{:ok, res} = Firecrawl.scrape_and_extract_from_url(
  url: "https://example.com/pricing",
  formats: [
    "markdown",
    "links",
    %{type: "json", prompt: "Extract plan names and prices."},
    %{type: "screenshot", fullPage: true, quality: 80, viewport: %{width: 1280, height: 720}},
    %{type: "changeTracking", modes: ["git-diff"], tag: "pricing"}
  ],
  headers: %{"User-Agent" => "FirecrawlDocsBot/1.0"},
  only_main_content: true,
  wait_for: 1000,
  parsers: [%{type: "pdf", mode: "auto", maxPages: 5}],
  actions: [
    %{type: "click", selector: "#accept"},
    %{type: "wait", milliseconds: 750},
    %{type: "scrape"}
  ],
  location: [country: "US", languages: ["en-US"]],
  remove_base64_images: true,
  block_ads: true,
  proxy: :auto,
  max_age: 86400000,
  min_age: 1,
  store_in_cache: true,
  profile: [name: "docs-session", save_changes: true],
  zero_data_retention: false
)
```

<div id="parameters">
  ### Parâmetros
</div>

* `url`
  * Tipo: string
  * Use quando: você quiser fazer scraping de uma página específica.

* `formats`
  * Tipo: lista de strings de formato ou mapas de formato
  * Use quando: você quiser vários formatos de resultado.
  * Strings de formato confirmadas:
    * `"markdown"`: conteúdo em markdown
    * `"html"`: HTML limpo
    * `"rawHtml"`: HTML bruto
    * `"links"`: links da página
    * `"images"`: URLs de imagens
    * `"screenshot"`: resultado de captura de tela
    * `"summary"`: resultado de resumo
    * `"changeTracking"`: resultado de rastreamento de mudanças
    * `"json"`: extração JSON
    * `"branding"`: resultado do perfil de marca
    * `"audio"`: extração de áudio
    * `"video"`: extração de vídeo
  * Campos do mapa de formato:
    * `type`: uma das strings de formato acima
    * `prompt`, `schema`: opções de extração JSON para `type: "json"`
    * `modes`, `schema`, `prompt`, `tag`: opções de rastreamento de mudanças para `type: "changeTracking"`
    * `fullPage`, `quality`, `viewport`: opções de captura de tela para `type: "screenshot"`

* `headers`
  * Tipo: mapa
  * Use quando: você precisar de cabeçalhos de solicitação personalizados.

* `include_tags`
  * Tipo: lista de strings
  * Use quando: você quiser incluir apenas tags HTML específicas.

* `exclude_tags`
  * Tipo: lista de strings
  * Use quando: você quiser excluir tags HTML específicas.

* `only_main_content`
  * Tipo: boolean
  * Use quando: você quiser remover navegação, rodapé e outros elementos padrão.

* `timeout`
  * Tipo: integer
  * Use quando: você precisar de um tempo limite em milissegundos.

* `wait_for`
  * Tipo: integer
  * Use quando: você precisar aguardar a renderização da página (milissegundos).

* `mobile`
  * Tipo: boolean
  * Use quando: você quiser uma viewport para dispositivos móveis.

* `parsers`
  * Tipo: lista de strings de parser ou mapas
  * Use quando: você precisar de controles de análise de arquivos.
  * Valores confirmados:
    * `"pdf"`
    * `%{type: "pdf", mode: "fast" | "auto" | "ocr", maxPages: integer}`

* `actions`
  * Tipo: lista de mapas de ação
  * Use quando: você precisar de ações leves antes do scraping.
  * Tipos de ação confirmados:
    * `wait`: `milliseconds` ou `selector` obrigatório
    * `screenshot`: `fullPage`, `quality`, `viewport` opcionais
    * `click`: `selector` obrigatório, `all` opcional
    * `write`: `text` obrigatório (clique para focar o campo de entrada primeiro)
    * `press`: `key` obrigatório
    * `scroll`: `direction` (`up` ou `down`) obrigatório, `selector` opcional
    * `scrape`: sem campos adicionais
    * `executeJavascript`: `script` obrigatório
    * `pdf`: `format` (A0, A1, A2, A3, A4, A5, A6, Letter, Legal, Tabloid, Ledger), `landscape`, `scale` opcionais

* `location`
  * Tipo: lista de palavras-chave (por exemplo, `country:` e `languages:`)
  * Use quando: você precisar de scraping com reconhecimento de localização ou idioma.

* `skip_tls_verification`
  * Tipo: boolean
  * Use quando: você precisar ignorar a verificação de TLS.

* `remove_base64_images`
  * Tipo: boolean
  * Use quando: você quiser remover imagens em base64 do resultado em markdown.

* `block_ads`
  * Tipo: boolean
  * Use quando: você quiser bloquear anúncios e pop-ups de cookies.

* `proxy`
  * Tipo: atom ou string
  * Use quando: você precisar de controle de proxy.
  * Valores confirmados: `:basic`, `:enhanced`, `:auto`

* `max_age`
  * Tipo: integer
  * Use quando: você quiser dados em cache até uma idade máxima (milissegundos).

* `min_age`
  * Tipo: integer
  * Use quando: você quiser dados em cache apenas se tiverem pelo menos essa idade (milissegundos).

* `store_in_cache`
  * Tipo: boolean
  * Use quando: você quiser que o Firecrawl armazene o resultado em cache.

* `profile`
  * Tipo: lista de palavras-chave com `name:` e `save_changes:` opcional (ou `saveChanges:`)
  * Use quando: você quiser um perfil persistente de navegador compartilhado entre scrapes e interações.

* `zero_data_retention`
  * Tipo: boolean
  * Use quando: você quiser retenção zero de dados para este scraping.

<div id="interact">
  ## Interagir
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use `interagir` quando uma página exigir ações no navegador ou execução de código após o início do scraping.

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

`Firecrawl.interact_with_scrape_browser_session(job_id, params \\ [], opts \\ [])`

<div id="simple-example">
  ### Exemplo simples
</div>

```elixir
{:ok, res} = Firecrawl.interact_with_scrape_browser_session(
  "<scrapeJobId>",
  code: "console.log(await page.title());",
  language: :node,
  timeout: 60
)
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```elixir
{:ok, res} = Firecrawl.interact_with_scrape_browser_session(
  "<scrapeJobId>",
  code: "// Use os métodos de página do Playwright aqui",
  language: :node,
  timeout: 120
)
```

<div id="parameters">
  ### Parâmetros
</div>

* `job_id`
  * Tipo: string
  * Use quando: você tiver um ID de job de scraping.

* `code`
  * Tipo: string
  * Use quando: quiser executar código na sessão do navegador.

* `language`
  * Tipo: átomo ou string
  * Use quando: precisar de um runtime específico.
  * Valores aceitos: `:python`, `:node`, `:bash` (ou os equivalentes em string)

* `timeout`
  * Tipo: inteiro
  * Use quando: precisar de um tempo limite de execução em segundos.

* `origin`
  * Tipo: string
  * Use quando: quiser um rótulo de origem opcional para a telemetria de execução.

<div id="stop-interactive-session">
  ### Parar sessão interativa
</div>

`Firecrawl.stop_interactive_scrape_browser_session(job_id, opts \\ [])` envia `DELETE /scrape/{jobId}/interact` e encerra a sessão do navegador desse job de scraping. Uma variante com bang, `stop_interactive_scrape_browser_session!/2`, também está disponível.

```elixir
{:ok, res} = Firecrawl.stop_interactive_scrape_browser_session("<scrapeJobId>")
```

<div id="notes">
  ## Observações
</div>

* O cliente para Elixir segue o formato do OpenAPI; os nomes das funções e as chaves de parâmetro são gerados a partir da especificação.
* Cada função pública tem uma variante com bang (`!`) que gera uma exceção em caso de erro em vez de retornar `{:error, _}`.
* Este SDK expõe apenas interações baseadas em código (sem o parâmetro `prompt` em `interact_with_scrape_browser_session`).
