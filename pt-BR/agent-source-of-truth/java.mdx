Fonte oficial canônica do Firecrawl em Java para agentes. Gerada a partir do código-fonte do SDK e da especificação OpenAPI v2.

<div id="install">
  ## Instalação
</div>

Maven:

```xml
<dependency>
  <groupId>com.firecrawl</groupId>
  <artifactId>firecrawl-java</artifactId>
  <version>1.2.0</version>
</dependency>
```

Gradle:

```gradle
implementation("com.firecrawl:firecrawl-java:1.2.0")
```

<div id="authenticate">
  ## Autentique-se
</div>

```java
import com.firecrawl.client.FirecrawlClient;

FirecrawlClient client = FirecrawlClient.builder()
    .apiKey(System.getenv("FIRECRAWL_API_KEY"))
    .build();
```

<div id="when-to-use-what">
  ## Quando usar cada um
</div>

* `search`: use quando você começa com uma consulta e precisa descobrir resultados.
* `scrape`: use quando você já tem uma URL e quer o conteúdo da página.
* `interact`: use quando a página exige cliques, formulários ou ações no navegador após o scraping.
* `support/ask`: use quando uma chamada da API do Firecrawl falha ou retorna resultados inesperados e você precisa de um diagnóstico.
* `support/docs-search`: use quando você precisa consultar a documentação do Firecrawl.

<div id="search">
  ## Busca
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use a busca para descobrir páginas relevantes a partir de uma consulta e, em seguida, escolher URLs para fazer scraping ou interagir com elas. Você pode restringir os resultados a um site com `site:`, por exemplo `site:docs.firecrawl.dev crawl webhooks`.

<div id="preferred-sdk-methods">
  ### Métodos recomendados do SDK
</div>

* `client.search(query)`
* `client.search(query, options)`

<div id="return-value">
  ### Valor de retorno
</div>

`search` retorna `SearchData`. Acesse os buckets de resultados com `getWeb()`, `getNews()` e `getImages()` (cada um é `List<Map<String, Object>>` e pode ser nulo). Não trate `SearchData` como uma lista de resultados iterável diretamente.

<div id="simple-example">
  ### Exemplo básico
</div>

```java
import com.firecrawl.models.SearchData;
import java.util.List;
import java.util.Map;

SearchData results = client.search("site:docs.firecrawl.dev webhook retries");
List<Map<String, Object>> web = results.getWeb();
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```java
import com.firecrawl.models.SearchOptions;
import com.firecrawl.models.ScrapeOptions;
import com.firecrawl.models.JsonFormat;
import com.firecrawl.models.LocationConfig;

SearchOptions options = SearchOptions.builder()
    .sources(List.of("web", "news"))
    .categories(List.of("research"))
    .limit(10)
    .tbs("qdr:m")
    .location("San Francisco,California,United States")
    .ignoreInvalidURLs(true)
    .timeout(60000)
    .scrapeOptions(
        ScrapeOptions.builder()
            .formats(List.of(
                "markdown",
                "links",
                JsonFormat.builder().prompt("Extract title and key endpoints.").build()
            ))
            .onlyMainContent(true)
            .waitFor(1000)
            .build()
    )
    .build();

SearchData results = client.search("site:docs.firecrawl.dev crawl webhooks", options);
```

<div id="parameters">
  ### Parâmetros
</div>

* `query`
  * Tipo: String
  * Use quando: você precisa de uma consulta de busca.
  * Observações: use `site:example.com` para limitar os resultados a um domínio.

* `options.sources`
  * Tipo: Lista de strings de fontes ou mapas
  * Use quando: você quer controlar em quais fontes a busca será feita.
  * Valores confirmados:
    * `"web"`: resultados do índice da web
    * `"news"`: resultados de notícias
    * `"images"`: resultados de imagens
    * `{type: "web" | "news" | "images"}`: formato de mapa de fonte tipado

* `options.categories`
  * Tipo: Lista de strings de categorias ou mapas
  * Use quando: você quer filtrar os resultados por categoria.
  * Valores confirmados:
    * `"github"`: resultados focados em GitHub
    * `"research"`: resultados de pesquisa e acadêmicos
    * `"pdf"`: resultados focados em PDF
    * `{type: "github" | "research" | "pdf"}`: formato de mapa de categoria tipado

* `options.limit`
  * Tipo: Integer
  * Use quando: você quer limitar a quantidade de resultados.

* `options.tbs`
  * Tipo: String
  * Use quando: você precisa de um filtro baseado em tempo (por exemplo, `qdr:d`, `qdr:w`, `sbd:1,qdr:m`).

* `options.location`
  * Tipo: String
  * Use quando: você quer resultados localizados.

* `options.ignoreInvalidURLs`
  * Tipo: Boolean
  * Use quando: você quer descartar URLs que não podem ser processadas por outros endpoints.

* `options.timeout`
  * Tipo: Integer
  * Use quando: você precisa de um tempo limite de requisição em milissegundos.

* `options.scrapeOptions`
  * Tipo: `ScrapeOptions`
  * Use quando: você quer fazer scraping de cada resultado de busca (veja os parâmetros de Scrape para os campos).

* `options.integration`
  * Tipo: String
  * Use quando: a API espera um identificador de integração na requisição.

<div id="scrape">
  ## Scraping
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use scraping quando você já tiver uma URL e quiser conteúdo estruturado em um ou mais formatos.

<div id="preferred-sdk-methods">
  ### Métodos recomendados do SDK
</div>

* `client.scrape(url)`
* `client.scrape(url, options)`

<div id="return-value">
  ### Valor de retorno
</div>

`scrape` retorna `Document`. Os getters mais comuns incluem `getMarkdown()`, `getHtml()`, `getRawHtml()`, `getJson()`, `getMetadata()`, `getLinks()`, `getAudio()`, `getVideo()` e campos adicionais quando os formatos correspondentes são solicitados.

<div id="simple-example">
  ### Exemplo básico
</div>

```java
Document doc = client.scrape(
    "https://docs.firecrawl.dev",
    ScrapeOptions.builder().formats(List.of("markdown")).build()
);
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```java
import com.firecrawl.models.ScrapeOptions;
import com.firecrawl.models.JsonFormat;

List<Map<String, Object>> actions = List.of(
    Map.of("type", "click", "selector", "#accept"),
    Map.of("type", "wait", "milliseconds", 750),
    Map.of("type", "scrape")
);

LocationConfig location = LocationConfig.builder()
    .country("US")
    .languages(List.of("en-US"))
    .build();

ScrapeOptions options = ScrapeOptions.builder()
    .formats(List.of(
        "markdown",
        "links",
        JsonFormat.builder().prompt("Extract plan names and prices.").build(),
        Map.of("type", "screenshot", "fullPage", true, "quality", 80)
    ))
    .onlyMainContent(true)
    .waitFor(1000)
    .parsers(List.of(Map.of("type", "pdf", "maxPages", 5)))
    .actions(actions)
    .location(location)
    .removeBase64Images(true)
    .blockAds(true)
    .proxy("auto")
    .maxAge(86400000L)
    .storeInCache(true)
    .build();

Document doc = client.scrape("https://example.com/pricing", options);
```

<div id="parameters">
  ### Parâmetros
</div>

* `url`
  * Tipo: String
  * Usar quando: você quiser fazer scraping de uma página específica.

* `options.formats`
  * Tipo: Lista de strings de formato ou mapas de formato
  * Usar quando: você quiser vários formatos de resultado.
  * Strings de formato confirmadas:
    * `"markdown"`: conteúdo em markdown
    * `"html"`: HTML limpo
    * `"rawHtml"`: HTML bruto
    * `"links"`: links da página
    * `"images"`: URLs de imagens
    * `"screenshot"`: resultado de captura de tela
    * `"summary"`: resultado de resumo
    * `"changeTracking"`: resultado de rastreamento de mudanças
    * `"json"`: extração JSON
    * `"attributes"`: extração de atributos
    * `"branding"`: resultado do perfil da marca
    * `"audio"`: extração de áudio
    * `"video"`: extração de vídeo
  * Campos do objeto de formato:
    * `type`: uma das strings de formato acima
    * `prompt`, `schema`: opções de extração JSON para `type: "json"`
    * `modes`, `schema`, `prompt`, `tag`: opções de rastreamento de mudanças para `type: "changeTracking"`
    * `fullPage`, `quality`, `viewport`: opções de captura de tela para `type: "screenshot"`
    * `selectors`: array de `{selector, attribute}` para `type: "attributes"`

* `options.headers`
  * Tipo: `Map<String, String>`
  * Usar quando: você precisar de cabeçalhos de solicitação personalizados.

* `options.includeTags`
  * Tipo: `List<String>`
  * Usar quando: você quiser incluir apenas tags HTML específicas.

* `options.excludeTags`
  * Tipo: `List<String>`
  * Usar quando: você quiser excluir tags HTML específicas.

* `options.onlyMainContent`
  * Tipo: Boolean
  * Usar quando: você quiser remover navegação, rodapé e outros elementos padrão.

* `options.timeout`
  * Tipo: Integer
  * Usar quando: você precisar de um tempo limite em milissegundos.

* `options.waitFor`
  * Tipo: Integer
  * Usar quando: você precisar aguardar a renderização da página (milissegundos).

* `options.mobile`
  * Tipo: Boolean
  * Usar quando: você quiser uma viewport para dispositivos móveis.

* `options.parsers`
  * Tipo: `List<Object>`
  * Usar quando: você precisar de controles de análise de arquivos.
  * Valores confirmados:
    * `"pdf"`
    * `{type: "pdf", maxPages: number}`

* `options.actions`
  * Tipo: `List<Map<String, Object>>`
  * Usar quando: você precisar de ações leves antes do scraping.
  * Tipos de ação confirmados:
    * `wait`: `milliseconds` ou `selector` obrigatório
    * `screenshot`: `fullPage`, `quality`, `viewport` opcionais
    * `click`: `selector` obrigatório
    * `write`: `text` obrigatório (clique primeiro para focar o campo de entrada)
    * `press`: `key` obrigatório
    * `scroll`: `direction` (`up` ou `down`) obrigatório, `selector` opcional
    * `scrape`: sem campos adicionais
    * `executeJavascript`: `script` obrigatório
    * `pdf`: `format` (A0, A1, A2, A3, A4, A5, A6, Letter, Legal, Tabloid, Ledger), `landscape`, `scale` opcionais

* `options.location`
  * Tipo: `LocationConfig`
  * Usar quando: você precisar de scraping com reconhecimento de localização geográfica ou idioma.

* `options.skipTlsVerification`
  * Tipo: Boolean
  * Usar quando: você precisar ignorar a verificação de TLS.

* `options.removeBase64Images`
  * Tipo: Boolean
  * Usar quando: você quiser remover imagens em base64 do resultado em markdown.

* `options.blockAds`
  * Tipo: Boolean
  * Usar quando: você quiser bloquear anúncios e pop-ups de cookies.

* `options.proxy`
  * Tipo: String
  * Usar quando: você precisar de controle de proxy.
  * Valores confirmados: `"basic"`, `"stealth"`, `"enhanced"`, `"auto"` ou uma string de URL de proxy personalizada

* `options.maxAge`
  * Tipo: Long
  * Usar quando: você quiser dados em cache até uma idade máxima (milissegundos).

* `options.storeInCache`
  * Tipo: Boolean
  * Usar quando: você quiser que o Firecrawl armazene o resultado em cache.

* `options.integration`
  * Tipo: String
  * Usar quando: a API esperar um identificador de integração na solicitação.

<div id="interact">
  ## Interagir
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use interagir quando uma página exigir ações no navegador ou execução de código após o início do scraping.

<div id="preferred-sdk-methods">
  ### Métodos recomendados do SDK
</div>

* `client.interact(jobId, code)` — usa a linguagem padrão `node` e o tempo limite de execução padrão da API
* `client.interact(jobId, code, language, timeout)` — `timeout` é em segundos (1–300), ou null para omitir e usar o padrão da API (30 segundos)
* `client.interact(jobId, code, language, timeout, origin)` — a string `origin` opcional é enviada apenas quando não é nula (atribuição da requisição)

<div id="simple-example">
  ### Exemplo básico
</div>

```java
import com.firecrawl.models.BrowserExecuteResponse;

BrowserExecuteResponse result = client.interact(
    "<scrapeJobId>",
    "console.log(await page.title());",
    "node",
    60
);
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```java
BrowserExecuteResponse result = client.interact(
    "<scrapeJobId>",
    "// Use os métodos de página do Playwright aqui",
    "node",
    120
);
```

<div id="stop-interactive-browser">
  ### Parar navegador interativo
</div>

Encerre a sessão do navegador associada ao scraping quando concluir.

**Método recomendado do SDK:** `client.stopInteractiveBrowser(jobId)`

```java
import com.firecrawl.models.BrowserDeleteResponse;

BrowserDeleteResponse stopped = client.stopInteractiveBrowser("<scrapeJobId>");
```

<div id="interact-response-browserexecuteresponse">
  ### Resposta da interação (`BrowserExecuteResponse`)
</div>

* `isSuccess()`: boolean
* `getStdout()`, `getStderr()`, `getResult()`, `getError()`: String (pode ser null)
* `getExitCode()`: Integer (pode ser null)
* `getKilled()`: Boolean (pode ser null) — true quando a execução foi interrompida por tempo limite

<div id="stop-response-browserdeleteresponse">
  ### Resposta ao encerrar (`BrowserDeleteResponse`)
</div>

* `isSuccess()`: boolean
* `getSessionDurationMs()`: Long (pode ser nulo)
* `getCreditsBilled()`: Integer (pode ser nulo)
* `getError()`: String (pode ser nulo)

<div id="parameters">
  ### Parâmetros
</div>

* `jobId`
  * Tipo: String
  * Use quando: você tiver um ID de scrape job.

* `code`
  * Tipo: String
  * Use quando: você quiser executar código na sessão do navegador.

* `language`
  * Tipo: String
  * Use quando: você precisar de um runtime específico.
  * Valores confirmados: `"python"`, `"node"`, `"bash"`
  * Observações: o SDK usa `"node"` por padrão se o valor for null.

* `timeout`
  * Tipo: Integer
  * Use quando: você precisar de um tempo limite de execução em segundos (1–300). Null omite o campo e usa o padrão da API.

* `origin`
  * Tipo: String
  * Use quando: você precisar de um rótulo opcional de origin na requisição. Prefira omitir, a menos que sua integração exija isso.

<div id="notes">
  ## Notas
</div>

* Aliases descontinuados: `scrapeExecute` → `interact`, `deleteScrapeBrowser` → `stopInteractiveBrowser` (e os helpers `*Async` correspondentes).
* O SDK Java expõe apenas interações baseadas em código: não existe o parâmetro `prompt` em `interact` (ao contrário de alguns SDKs de outras linguagens).
