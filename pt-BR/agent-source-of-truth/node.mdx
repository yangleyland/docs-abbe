Fonte oficial canônica do Firecrawl em Node.js para agentes. Alinhada com `firecrawl` **v4.18.2** (`firecrawl/apps/js-sdk/firecrawl`) e a especificação OpenAPI v2. Os nomes de métodos, parâmetros e tipos correspondem à API pública do SDK.

<div id="install">
  ## Instalar
</div>

```bash
npm install firecrawl
```

<div id="authenticate">
  ## Autenticação
</div>

```ts
import { Firecrawl } from "firecrawl";

const client = new Firecrawl({
  apiKey: process.env.FIRECRAWL_API_KEY,
  // apiUrl: "https://api.firecrawl.dev" // opcional; usa FIRECRAWL_API_URL ou o padrão da nuvem como fallback
});
```

<div id="when-to-use-what">
  ## Quando usar cada opção
</div>

* `search`: use quando você começa com uma consulta e precisa descobrir resultados.
* `scrape`: use quando você já tem uma URL e quer o conteúdo da página.
* `interact`: use quando a página exige cliques, formulários ou outras ações do navegador depois que o scraping criou uma sessão. Para fluxos interativos de várias etapas, prefira `interact` em vez de `ações` durante o scraping.
* `support/ask`: use quando uma chamada de API do Firecrawl falha ou retorna resultados inesperados e você precisa de um diagnóstico.
* `support/docs-search`: use quando você precisa consultar a documentação do Firecrawl.

<div id="search">
  ## busca
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use a busca para descobrir páginas relevantes a partir de uma consulta e, em seguida, escolher URLs para fazer scraping ou interagir. Você pode restringir os resultados a um site com `site:`, por exemplo `site:docs.firecrawl.dev crawl webhooks`.

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

`client.search(query, options?)` → `Promise<SearchData>`

<div id="simple-example">
  ### Exemplo simples
</div>

```ts
const results = await client.search("site:docs.firecrawl.dev webhook retries");
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```ts
const results = await client.search("site:docs.firecrawl.dev crawl webhooks", {
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:m",
  location: "San Francisco,California,United States",
  ignoreInvalidURLs: true,
  timeout: 60000,
  scrapeOptions: {
    formats: [
      "markdown",
      "links",
      { type: "json", prompt: "Extract title and key endpoints." }
    ],
    onlyMainContent: true,
    includeTags: ["main", "article"],
    excludeTags: ["nav", "footer"],
    waitFor: 1000
  }
});
```

<div id="return-value">
  ### Valor de retorno
</div>

`SearchData` é um objeto com arrays opcionais (cada entrada é um objeto de resultado simplificado ou um `Document` completo quando `scrapeOptions` enriqueceu o resultado):

* `web`: resultados do índice web
* `news`: resultados de notícias
* `images`: resultados de imagens

**Erro a evitar:** `search()` não retorna `{ data: [...] }`. Não acesse `result.data`. Os resultados da web estão em `result.web`, os de notícias em `result.news` e os de imagens em `result.images`.

<div id="parameters">
  ### Parâmetros
</div>

* `query`
  * Tipo: string
  * Use quando: você precisar de um termo de busca.
  * Observações: use `site:example.com` para limitar os resultados a um domínio.

* `options` (opcional; o segundo argumento tem como padrão `{}`)
  * Tipo: `Omit<SearchRequest, "query">`

* `options.sources`
  * Tipo: array de nomes de fontes ou objetos de fonte tipados
  * Use quando: você quiser controlar em quais fontes a busca será feita.
  * Valores confirmados:
    * `"web"`: resultados do índice da web
    * `"news"`: resultados de notícias
    * `"images"`: resultados de imagens
    * `{ type: "web" | "news" | "images" }`: formato de objeto de fonte tipado

* `options.categories`
  * Tipo: array de nomes de categorias ou objetos de categoria tipados
  * Use quando: você quiser filtrar os resultados por categoria.
  * Valores confirmados:
    * `"github"`: resultados focados no GitHub
    * `"research"`: resultados de pesquisa e acadêmicos
    * `"pdf"`: resultados focados em PDF
    * `{ type: "github" | "research" | "pdf" }`: formato de objeto de categoria tipado

* `options.limit`
  * Tipo: number
  * Use quando: você quiser limitar a quantidade de resultados.

* `options.tbs`
  * Tipo: string
  * Use quando: você precisar de um filtro baseado em tempo (por exemplo, `qdr:d`, `qdr:w`, `sbd:1,qdr:m`).

* `options.location`
  * Tipo: string
  * Use quando: você quiser resultados localizados.

* `options.ignoreInvalidURLs`
  * Tipo: boolean
  * Use quando: você quiser descartar URLs que não podem ser extraídas por outros endpoints.

* `options.timeout`
  * Tipo: number
  * Use quando: você precisar de um tempo limite da requisição em milissegundos.

* `options.scrapeOptions`
  * Tipo: `ScrapeOptions`
  * Use quando: você quiser fazer scraping de cada resultado de busca (veja os parâmetros de Scrape para os campos). O SDK executa a mesma validação que para `scrape` (por exemplo, a string simples `"json"` em `formats` é rejeitada).

<div id="scrape">
  ## scraping
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use o scraping quando você já tiver uma URL e quiser conteúdo estruturado em um ou mais formatos.

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

`client.scrape(url, options?)` → `Promise<Document>`

<div id="simple-example">
  ### Exemplo simples
</div>

```ts
const doc = await client.scrape("https://docs.firecrawl.dev", {
  formats: ["markdown"]
});
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```ts
const doc = await client.scrape("https://example.com/pricing", {
  formats: [
    "markdown",
    "links",
    { type: "json", prompt: "Extract plan names and prices." },
    { type: "screenshot", fullPage: true, quality: 80, viewport: { width: 1280, height: 720 } },
    { type: "changeTracking", modes: ["git-diff"], tag: "pricing" },
    { type: "attributes", selectors: [{ selector: "a", attribute: "href" }] }
  ],
  headers: {
    "User-Agent": "FirecrawlDocsBot/1.0"
  },
  onlyMainContent: true,
  waitFor: 1000,
  parsers: [{ type: "pdf", mode: "auto", maxPages: 5 }],
  actions: [
    { type: "click", selector: "#accept" },
    { type: "wait", milliseconds: 750 },
    { type: "scrape" }
  ],
  location: { country: "US", languages: ["en-US"] },
  removeBase64Images: true,
  fastMode: true,
  blockAds: true,
  proxy: "auto",
  maxAge: 86400000,
  minAge: 1,
  storeInCache: true,
  profile: { name: "docs-session", saveChanges: true }
});
```

<div id="parameters">
  ### Parâmetros
</div>

* `url`
  * Tipo: string
  * Use quando: quiser fazer scraping de uma página específica.

* `options.formats`
  * Tipo: array de strings de formato ou objetos de formato
  * Use quando: você quiser vários formatos de resultado.
  * Formatos de string simples (a união `FormatString` também inclui `"json"`, mas o SDK rejeita `"json"` como string simples — use um objeto `json` abaixo):
    * `"markdown"`: conteúdo em markdown
    * `"html"`: HTML limpo
    * `"rawHtml"`: HTML bruto
    * `"links"`: links da página
    * `"images"`: URLs de imagens
    * `"screenshot"`: resultado de captura de tela
    * `"summary"`: resultado de resumo
    * `"changeTracking"`: resultado de rastreio de mudanças (para opções como `modes`, use `{ type: "changeTracking", modes: [...] }` — `modes` é obrigatório nesse objeto nas tipagens)
    * `"attributes"`: extração de atributos (use `{ type: "attributes", selectors: [...] }` ao informar seletores)
    * `"branding"`: resultado de perfil de branding
    * `"audio"`: extração de áudio
    * `"video"`: extração de vídeo
  * Tipos de formato aceitos apenas como objeto (no mínimo `type`, como mostrado):
    * `{ type: "json", prompt?: string, schema?: schema JSON ou schema Zod }`: pelo menos um entre `prompt` ou `schema` é obrigatório (validação do SDK).
    * `{ type: "question", question: string }`: extração no estilo de perguntas e respostas.
    * `{ type: "highlights", query: string }`: extração de trechos relevantes do texto-fonte.
    * `{ type: "screenshot", fullPage?, quality?, viewport? }`: mesmas opções da forma em string, mas como objeto.
    * `{ type: "changeTracking", modes: ("git-diff" | "json")[], schema?, prompt?, tag? }`: `modes` é obrigatório.
    * `{ type: "attributes", selectors: Array<{ selector, attribute }> }`
  * Campos de objeto compartilhados, quando aplicável:
    * `schema`: schema JSON ou schema Zod para `json`, ou para `changeTracking` no modo `json` (Zod é convertido em JSON Schema pelo SDK).
    * `modes`: apenas para `changeTracking`: `"git-diff"` e/ou `"json"`.
    * `tag`: tag opcional de branch para rastreio de mudanças.
    * `fullPage`, `quality`, `viewport`: opções de captura de tela.

* `options.headers`
  * Tipo: registro de string para string
  * Use quando: precisar de cabeçalhos de requisição personalizados.

* `options.includeTags`
  * Tipo: array de strings
  * Use quando: quiser incluir apenas tags HTML específicas.

* `options.excludeTags`
  * Tipo: array de strings
  * Use quando: quiser excluir tags HTML específicas.

* `options.onlyMainContent`
  * Tipo: boolean
  * Use quando: você quiser remover a navegação, o rodapé e outros elementos padrão.

* `options.timeout`
  * Tipo: number
  * Use quando: precisar de um tempo limite em milissegundos.

* `options.waitFor`
  * Tipo: number
  * Use quando: for necessário aguardar a renderização da página (milissegundos).

* `options.mobile`
  * Tipo: boolean
  * Use quando: você quiser uma viewport para dispositivos móveis.

* `options.parsers`
  * Tipo: array de nomes de parser ou objetos de parser
  * Use quando: precisar de controles de análise de arquivos (por exemplo, análise de PDF).
  * Valores aceitos:
    * `"pdf"`: habilita a análise de PDF
    * `{ type: "pdf", mode?: "fast" | "auto" | "ocr", maxPages?: number }`: opções do parser de PDF

* `options.actions`
  * Tipo: array de objetos de ações
  * Use quando: você precisar de ações leves de pré-scraping.
  * Tipos de ação aceitos:
    * `wait`: `milliseconds` ou `selector` obrigatórios
    * `screenshot`: `fullPage`, `quality`, `viewport` opcionais
    * `click`: `selector` obrigatório
    * `write`: `text` obrigatório (clique para focar no campo primeiro)
    * `press`: `key` obrigatório
    * `scroll`: `direction` (`up` ou `down`) obrigatório, `selector` opcional
    * `scrape`: nenhum campo adicional
    * `executeJavascript`: `script` obrigatório
    * `pdf`: `format` (A0, A1, A2, A3, A4, A5, A6, Letter, Legal, Tabloid, Ledger), `landscape`, `scale` opcionais

* `options.location`
  * Tipo: objeto com `country` e `languages`
  * Use quando: precisar de scraping com reconhecimento de localização ou idioma.

* `options.skipTlsVerification`
  * Tipo: boolean
  * Use quando: for necessário ignorar a verificação de TLS.

* `options.removeBase64Images`
  * Tipo: boolean
  * Use quando: quiser remover imagens codificadas em base64 do resultado em markdown.

* `options.fastMode`
  * Tipo: boolean
  * Use quando: você quiser operações de scraping mais rápidas, com fidelidade reduzida.

* `options.blockAds`
  * Tipo: boolean
  * Use quando: você quiser bloquear anúncios e pop-ups de cookies.

* `options.proxy`
  * Tipo: string
  * Use quando: precisar de controle sobre o proxy.
  * Valores aceitos:
    * `"basic"`
    * `"stealth"`
    * `"enhanced"`
    * `"auto"`
    * string de URL de proxy personalizada

* `options.maxAge`
  * Tipo: number
  * Use quando: você quiser dados em cache com no máximo de idade (milissegundos).

* `options.minAge`
  * Tipo: number
  * Use quando: você quiser usar apenas dados em cache se eles tiverem pelo menos essa idade (milissegundos).

* `options.storeInCache`
  * Tipo: boolean
  * Use quando: você quiser que o Firecrawl armazene o resultado em cache.

* `options.profile`
  * Tipo: objeto com `name` e `saveChanges` opcional
  * Use quando: você quiser um perfil persistente de navegador compartilhado entre operações de scraping e interações.

<div id="interact">
  ## Interagir
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use `interact` para controlar, com código ou linguagem natural, a sessão do navegador vinculada a um job de scraping (via `metadata.scrapeId`). O SDK exige pelo menos um entre `code` e `prompt`. Para fluxos que vão além de ajustes rápidos antes do scraping, prefira `interact` em vez de `ações` durante o scraping.

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

`client.interact(jobId, args)` → `Promise<ScrapeExecuteResponse>`

<div id="simple-example">
  ### Exemplo simples
</div>

```ts
const doc = await client.scrape("https://example.com", { formats: ["markdown"] });
const jobId = doc.metadata?.scrapeId;
if (!jobId) throw new Error("Missing scrapeId from scrape response");

const result = await client.interact(jobId, {
  prompt: "Click the pricing tab and summarize the plans."
});
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```ts
const result = await client.interact("<scrapeJobId>", {
  code: "console.log(await page.title());",
  language: "node",
  timeout: 60
});
```

<div id="parameters">
  ### Parâmetros
</div>

* `jobId`
  * Tipo: string
  * Use quando: você tiver o ID de um job de scraping em `document.metadata.scrapeId`.

* `args.code`
  * Tipo: string
  * Use quando: você quiser executar código na sessão do navegador (por exemplo, uso de `page` do Playwright no runtime `node`).

* `args.prompt`
  * Tipo: string
  * Use quando: você quiser que o agente do navegador siga uma instrução em linguagem natural.

* Pelo menos um de `args.code` ou `args.prompt` deve estar preenchido (caso contrário, o SDK gera uma exceção).

* `args.language`
  * Tipo: string
  * Use quando: você precisar de um ambiente de execução específico.
  * Valores confirmados: `"python"`, `"node"`, `"bash"`
  * Padrão quando omitido: `"node"` (definido pelo SDK no corpo da request).

* `args.timeout`
  * Tipo: number
  * Use quando: você precisar de um tempo limite de execução em segundos.

<div id="return-value-interact">
  ### Valor de retorno (`interact`)
</div>

`ScrapeExecuteResponse` corresponde a `BrowserExecuteResponse`. Os campos confirmados incluem:

* `success`: boolean
* `output`: string (resultado agregado, quando presente)
* `stdout`: string
* `result`: string (campo de alias ao lado de stdout nas tipagens)
* `stderr`: string
* `exitCode`: number
* `killed`: boolean
* `liveViewUrl`: string
* `interactiveLiveViewUrl`: string
* `error`: string

<div id="stop-session">
  ### Encerrar sessão
</div>

`client.stopInteraction(jobId)` → `Promise<ScrapeBrowserDeleteResponse>`

* `jobId`: ID do job de scraping (o mesmo ID usado para `interact`).
* Os campos de resposta na tipagem incluem `success`, `sessionDurationMs`, `creditsBilled`, `error`.

<div id="ask-agentic-debugging">
  ## Ask (Depuração com agentes)
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use `ask` quando uma chamada à API Firecrawl falhar ou retornar resultados inesperados. O agente de suporte com IA diagnostica o problema, sugere parâmetros para corrigir e, opcionalmente, valida a correção na API em produção. Latência típica: 15–30 segundos.

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

Ainda não está disponível no SDK do Node.js. Use `fetch` diretamente.

### Exemplo básico

```ts
const response = await fetch("https://api.firecrawl.dev/v2/support/ask", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.FIRECRAWL_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    question: "my scrape returned empty markdown for https://example.com",
  }),
});
const result = await response.json();
console.log(result.answer);
console.log(result.fixParameters);
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```ts
const response = await fetch("https://api.firecrawl.dev/v2/support/ask", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.FIRECRAWL_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    question: "crawl returned 3 pages but I expected 50",
    rationale: "user is on their third failed crawl attempt today",
    context: { userPlan: "standard", retryCount: 3 },
  }),
});
const result = await response.json();
```

<div id="agent-retry-pattern">
  ### Padrão de retentativa do agente
</div>

```ts
import { Firecrawl } from "firecrawl";

const client = new Firecrawl({ apiKey: process.env.FIRECRAWL_API_KEY });

const doc = await client.scrape("https://example.com/pricing", {
  formats: ["markdown"],
});

if (!doc.markdown || doc.markdown.length < 100) {
  const diagResponse = await fetch(
    "https://api.firecrawl.dev/v2/support/ask",
    {
      method: "POST",
      headers: {
        Authorization: `Bearer ${process.env.FIRECRAWL_API_KEY}`,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        question: `scrape returned only ${doc.markdown?.length ?? 0} chars for https://example.com/pricing`,
      }),
    }
  );
  const diagnosis = await diagResponse.json();

  if (diagnosis.fixParameters) {
    const retryDoc = await client.scrape("https://example.com/pricing", {
      formats: ["markdown"],
      ...diagnosis.fixParameters,
    });
  }
}
```

<div id="parameters">
  ### Parâmetros
</div>

* `question`
  * Tipo: string (obrigatório, 1-8000 caracteres)
  * Use quando: for preciso descrever o problema.

* `rationale`
  * Tipo: string (1-2000 caracteres)
  * Use quando: você for um agente de IA fazendo a chamada em nome de um usuário.

* `context`
  * Tipo: objeto (forma livre)
  * Use quando: quiser passar metadados para o prompt de depuração.

<div id="docs-search">
  ## Busca na documentação
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use o docs-search para consultar a documentação do Firecrawl.

### Exemplo básico

```ts
const response = await fetch(
  "https://api.firecrawl.dev/v2/support/docs-search",
  {
    method: "POST",
    headers: {
      Authorization: `Bearer ${process.env.FIRECRAWL_API_KEY}`,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      question: "how do I verify webhook signatures?",
    }),
  }
);
const result = await response.json();
console.log(result.answer);
```

<div id="parameters">
  ### Parâmetros
</div>

* `question`
  * Tipo: string (obrigatório, 1-8000 caracteres)
  * Use quando: precisar de uma resposta baseada na documentação.

<div id="notes">
  ## Observações
</div>

* Aliases descontinuados do cliente: `scrapeExecute` → `interact`; `stopInteractiveBrowser` e `deleteScrapeBrowser` → `stopInteraction`.
* A exportação padrão de `Firecrawl` é o cliente v2; a v1 continua disponível em `client.v1`.
* Schemas Zod passados para `formats` (para `json` ou `changeTracking`) são convertidos pelo SDK em schema JSON.
* O pacote declara **Node.js &gt;= 22** em `engines`.
