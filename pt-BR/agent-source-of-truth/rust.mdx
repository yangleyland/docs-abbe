Fonte de referência canônica do Firecrawl em Rust para agentes. Gerada a partir do código-fonte do SDK e da especificação OpenAPI v2.

<div id="install">
  ## Instalação
</div>

```bash
cargo add firecrawl
```

Crate: **`firecrawl`** no crates.io. A versão atual do SDK é **2.0.0** (verifique a versão mais recente no crates.io antes de fixar a versão).

<div id="authenticate">
  ## Autentique-se
</div>

```rust
use firecrawl::Client;

let client = Client::new("fc-your-api-key")?;
// Exemplo self-hosted:
// let client = Client::new_selfhosted("http://localhost:3002", Some("fc-your-api-key"))?;
```

<div id="when-to-use-what">
  ## Quando usar cada um
</div>

* `search`: use quando você parte de uma consulta e precisa descobrir resultados.
* `scrape`: use quando você já tem uma URL e quer o conteúdo da página.
* `interact`: use quando a página exige cliques, formulários ou ações no navegador após o scraping.

- `support/ask`: use quando uma chamada da API do Firecrawl falha ou retorna resultados inesperados e você precisa de um diagnóstico.
- `support/docs-search`: use quando você precisa consultar a documentação do Firecrawl.

<div id="search">
  ## Busca
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use a busca para descobrir páginas relevantes a partir de uma consulta e, em seguida, escolher URLs para fazer scraping ou interagir com elas. Você pode restringir os resultados a um domínio com o operador padrão `site:`, por exemplo `site:docs.firecrawl.dev crawl webhooks`.

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

`client.search(query, options)` → `Result<SearchResponse, FirecrawlError>`

<div id="simple-example">
  ### Exemplo simples
</div>

```rust
use firecrawl::Client;

let results = client
    .search("site:docs.firecrawl.dev webhook retries", None)
    .await?;
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```rust
use firecrawl::{
    Client, SearchOptions, SearchSource, SearchCategory, ScrapeOptions, Format, JsonOptions,
};

let options = SearchOptions {
    sources: Some(vec![SearchSource::Web, SearchSource::News]),
    categories: Some(vec![SearchCategory::Research]),
    limit: Some(10),
    tbs: Some("qdr:m".to_string()),
    location: Some("San Francisco,California,United States".to_string()),
    ignore_invalid_urls: Some(true),
    timeout: Some(60000),
    scrape_options: Some(ScrapeOptions {
        formats: Some(vec![Format::Markdown, Format::Links, Format::Json]),
        json_options: Some(JsonOptions {
            prompt: Some("Extract title and key endpoints.".to_string()),
            ..Default::default()
        }),
        only_main_content: Some(true),
        ..Default::default()
    }),
    ..Default::default()
};

let results = client
    .search("site:docs.firecrawl.dev crawl webhooks", options)
    .await?;
```

<div id="return-type">
  ### Tipo de retorno
</div>

`client.search(...)` retorna `Result<SearchResponse, FirecrawlError>`.

* `SearchResponse`
  * `success: bool`
  * `data: SearchData`
  * `warning: Option<String>`
* `SearchData`
  * `web: Option<Vec<SearchResultOrDocument>>` — cada item é `SearchResultOrDocument::WebResult(SearchResultWeb)` (url, title, description, …) ou `SearchResultOrDocument::Document(Document)` quando a API retorna conteúdo extraído para esse item (detectado quando `markdown`, `html`, `rawHtml` ou `metadata` está presente).
  * `news: Option<Vec<SearchResultNews>>`
  * `images: Option<Vec<SearchResultImage>>`

<div id="parameters">
  ### Parâmetros
</div>

* `query`
  * Tipo: `impl AsRef<str>`
  * Use quando: você precisar de uma consulta de busca.
  * Observações: use `site:example.com` para limitar os resultados a um domínio.

* `options.sources`
  * Tipo: `Vec<SearchSource>`
  * Use quando: você quiser controlar quais fontes serão pesquisadas.
  * Valores confirmados: `Web`, `News`, `Images`

* `options.categories`
  * Tipo: `Vec<SearchCategory>`
  * Use quando: você quiser filtrar os resultados por categoria.
  * Valores confirmados: `Github`, `Research`, `Pdf`

* `options.limit`
  * Tipo: u32
  * Use quando: você quiser limitar a quantidade de resultados.

* `options.tbs`
  * Tipo: String
  * Use quando: você precisar de um filtro baseado em tempo (por exemplo, `qdr:d`, `qdr:w`, `sbd:1,qdr:m`).

* `options.location`
  * Tipo: String
  * Use quando: você quiser resultados localizados.

* `options.ignore_invalid_urls`
  * Tipo: bool
  * Use quando: você quiser descartar URLs que não podem ser extraídas por outros endpoints.

* `options.timeout`
  * Tipo: u32
  * Use quando: você precisar de um tempo limite da requisição em milissegundos.

* `options.scrape_options`
  * Tipo: `ScrapeOptions`
  * Use quando: você quiser fazer scraping de cada resultado de busca (veja os parâmetros de Scrape para os campos).

* `options.integration`
  * Tipo: `Option<String>`
  * Use quando: você precisar de um identificador de integração para acompanhamento no servidor.
  * Observações: omita em exemplos orientados a agentes, a menos que seu produto o defina intencionalmente.

<div id="scrape">
  ## Scrape
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use o scrape quando você já tiver uma URL e quiser conteúdo estruturado em um ou mais formatos.

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

`client.scrape(url, options)` → `Result<Document, FirecrawlError>` (o campo `data` da resposta HTTP é retornado diretamente pelo SDK).

<div id="simple-example">
  ### Exemplo simples
</div>

```rust
use firecrawl::{Client, ScrapeOptions, Format};

let doc = client
    .scrape("https://docs.firecrawl.dev", ScrapeOptions {
        formats: Some(vec![Format::Markdown]),
        ..Default::default()
    })
    .await?;
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```rust
use firecrawl::{
    Client, ScrapeOptions, Format, JsonOptions, ScreenshotOptions, ChangeTrackingOptions,
    ChangeTrackingMode, AttributeSelector, Action, ParserConfig, ProxyType,
};

let doc = client
    .scrape("https://example.com/pricing", ScrapeOptions {
        formats: Some(vec![
            Format::Markdown,
            Format::Links,
            Format::Json,
            Format::Screenshot,
            Format::ChangeTracking,
            Format::Attributes,
        ]),
        json_options: Some(JsonOptions {
            prompt: Some("Extract plan names and prices.".to_string()),
            ..Default::default()
        }),
        screenshot_options: Some(ScreenshotOptions {
            full_page: Some(true),
            quality: Some(80),
            ..Default::default()
        }),
        change_tracking_options: Some(ChangeTrackingOptions {
            modes: Some(vec![ChangeTrackingMode::GitDiff]),
            tag: Some("pricing".to_string()),
            ..Default::default()
        }),
        attribute_selectors: Some(vec![AttributeSelector {
            selector: "a".to_string(),
            attribute: "href".to_string(),
        }]),
        parsers: Some(vec![ParserConfig::Pdf {
            parser_type: "pdf".to_string(),
            max_pages: Some(5),
        }]),
        actions: Some(vec![
            Action::Click { selector: "#accept".to_string() },
            Action::Wait { milliseconds: Some(750), selector: None },
            Action::Scrape,
        ]),
        proxy: Some(ProxyType::Auto),
        ..Default::default()
    })
    .await?;
```

<div id="parameters">
  ### Parâmetros
</div>

* `url`
  * Tipo: `impl AsRef<str>`
  * Use quando: quiser fazer scraping de uma página específica.

* `options.formats`
  * Tipo: `Vec<Format>`
  * Use quando: quiser vários formatos de resultado.
  * Valores aceitos: `Markdown`, `Html`, `RawHtml`, `Links`, `Images`, `Screenshot`, `Summary`, `ChangeTracking`, `Json`, `Attributes`, `Branding`, `Audio`, `Video`

* `options.headers`
  * Tipo: `HashMap<String, String>`
  * Use em casos em que: você precisa de cabeçalhos de solicitação personalizados.

* `options.include_tags`
  * Tipo: `Vec<String>`
  * Use nos casos em que: você quiser incluir apenas tags HTML específicas.

* `options.exclude_tags`
  * Tipo: `Vec<String>`
  * Use quando: você quiser excluir tags HTML específicas.

* `options.only_main_content`
  * Tipo: bool
  * Use quando: você quiser remover a navegação, o rodapé e outros elementos padrão.

* `options.timeout`
  * Tipo: u32
  * Use quando: precisar de um tempo limite em milissegundos.

* `options.wait_for`
  * Tipo: u32
  * Use quando: precisar aguardar a renderização da página (milissegundos).

* `options.mobile`
  * Tipo: bool
  * Use quando: você quiser um viewport móvel.

* `options.parsers`
  * Tipo: `Vec<ParserConfig>`
  * Use quando: precisar de controles de análise de arquivos.
  * Valores confirmados:
    * `ParserConfig::Simple("pdf".to_string())`
    * `ParserConfig::Pdf { parser_type: "pdf", max_pages: Some(n) }`

* `options.actions`
  * Tipo: `Vec<Action>`
  * Use quando: precisar de ações leves de pré-scraping.
  * Tipos de ação suportados:
    * `Wait`: `milliseconds` ou `selector` obrigatório
    * `Screenshot`: `full_page`, `quality`, `viewport` opcionais
    * `Click`: `selector` obrigatório
    * `Write`: `text` obrigatório (clique para focar o campo primeiro)
    * `Press`: `key` obrigatório
    * `Scroll`: `direction` (`Up` ou `Down`) obrigatório, `selector` opcional
    * `Scrape`: sem campos adicionais
    * `ExecuteJavascript`: `script` obrigatório
    * `Pdf`: `format` (A0, A1, A2, A3, A4, A5, A6, Letter, Legal, Tabloid, Ledger), `landscape`, `scale` opcionais

* `options.location`
  * Tipo: `LocationConfig`
  * Use quando: você precisar de scraping sensível à localização geográfica ou ao idioma.
  * Campos confirmados: `country`, `languages`

* `options.skip_tls_verification`
  * Tipo: bool
  * Use quando: for necessário ignorar a verificação de TLS.

* `options.remove_base64_images`
  * Tipo: bool
  * Use quando: quiser remover imagens em base64 da saída em markdown.

* `options.fast_mode`
  * Tipo: bool
  * Use quando: você quiser fazer scraping mais rápido, com menor fidelidade.

* `options.block_ads`
  * Tipo: bool
  * Use quando: você quiser o bloqueio de pop-ups de anúncios e cookies.

* `options.proxy`
  * Tipo: `ProxyType`
  * Use quando: você precisar de controle de proxy.
  * Valores confirmados: `Basic`, `Stealth`, `Enhanced`, `Auto`

* `options.max_age`
  * Tipo: u32
  * Use quando: você quiser dados em cache com uma idade máxima (milissegundos).

* `options.min_age`
  * Tipo: u32
  * Use quando: você quiser usar dados em cache apenas se eles tiverem pelo menos esta idade (milissegundos).

* `options.store_in_cache`
  * Tipo: bool
  * Use quando: você quiser que o Firecrawl mantenha o resultado em cache.

* `options.profile`
  * Tipo: `ProfileConfig`
  * Use quando: você quiser um perfil persistente de navegador compartilhado entre operações de scraping e interações.
  * Campos confirmados: `name`, `save_changes`

* `options.integration`
  * Tipo: `Option<String>`
  * Use quando: você precisar de um identificador de integração para rastreamento no servidor.
  * Observações: omita em exemplos voltados a agentes, a menos que seu produto o configure intencionalmente.

* `options.json_options`
  * Tipo: `JsonOptions`
  * Use nos casos em que: você quiser configurar a extração JSON.
  * Campos confirmados: `schema`, `system_prompt`, `prompt`

* `options.screenshot_options`
  * Tipo: `ScreenshotOptions`
  * Use quando: quiser configurar o resultado da captura de tela.
  * Campos confirmados: `full_page`, `quality`, `viewport`

* `options.change_tracking_options`
  * Tipo: `ChangeTrackingOptions`
  * Use quando: quiser configurar o resultado do rastreio de mudanças.
  * Campos confirmados: `modes` (`GitDiff` ou `Json`), `schema`, `prompt`, `tag`

* `options.attribute_selectors`
  * Tipo: `Vec<AttributeSelector>`
  * Use quando: você quiser o resultado da extração de atributos.
  * Campos confirmados: `selector`, `attribute`

<div id="interact">
  ## Interagir
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use `interact` quando uma página exigir ações no navegador ou execução de código após o início do scraping.

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

`client.interact(job_id, options)`

Para encerrar a sessão do navegador, use `client.stop_interaction(job_id)` (HTTP `DELETE` em `/v2/scrape/{jobId}/interact`).

<div id="simple-example">
  ### Exemplo simples
</div>

```rust
use firecrawl::{Client, ScrapeExecuteOptions};

let result = client
    .interact(
        "<scrapeJobId>",
        ScrapeExecuteOptions {
            prompt: Some("Click the pricing tab and summarize the plans.".to_string()),
            ..Default::default()
        },
    )
    .await?;
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```rust
use firecrawl::{Client, ScrapeExecuteOptions, ScrapeExecuteLanguage};

let result = client
    .interact(
        "<scrapeJobId>",
        ScrapeExecuteOptions {
            code: Some("console.log(await page.title());".to_string()),
            language: Some(ScrapeExecuteLanguage::Node),
            timeout: Some(60),
            ..Default::default()
        },
    )
    .await?;
```

<div id="stop-session">
  ### Encerrar sessão
</div>

```rust
let stopped = client.stop_interaction("<scrapeJobId>").await?;
// stopped: ScrapeBrowserDeleteResponse { success, session_duration_ms, credits_billed, error, ... }
```

<div id="parameters">
  ### Parâmetros
</div>

* `job_id`
  * Tipo: `impl AsRef<str>`
  * Use quando: você tiver um ID do job de scraping.

* `options.code`
  * Tipo: `Option<String>`
  * Use quando: quiser executar código na sessão do navegador.

* `options.prompt`
  * Tipo: `Option<String>`
  * Use quando: quiser que o agente do navegador siga uma instrução em linguagem natural.

* `options.language`
  * Tipo: `ScrapeExecuteLanguage`
  * Use quando: precisar de um runtime específico.
  * Valores confirmados: `Python`, `Node`, `Bash`
  * Observações: no SDK, o padrão é `Node` se for omitido.

* `options.timeout`
  * Tipo: u32
  * Use quando: precisar de um tempo limite de execução em segundos.

* `options.origin`
  * Tipo: `Option<String>`
  * Use quando: precisar de um rótulo de origem opcional para a telemetria de execução.
  * Observações: omita em exemplos orientados a agentes, a menos que seu produto o defina intencionalmente.

Pelo menos um entre `options.code` e `options.prompt` deve ser não vazio; caso contrário, o SDK retorna `FirecrawlError::Misuse` antes de chamar a API.

<div id="response-types">
  ### Tipos de resposta
</div>

* `interact` → `Result<ScrapeExecuteResponse, FirecrawlError>`
  * `success: bool` está sempre presente; `live_view_url`, `interactive_live_view_url`, `output`, `stdout`, `result`, `stderr`, `exit_code`, `killed` e `error` são campos `Option` da struct.
* `stop_interaction` → `Result<ScrapeBrowserDeleteResponse, FirecrawlError>`
  * `success: bool` está sempre presente; `session_duration_ms`, `credits_billed` e `error` são opcionais.

No momento, a especificação OpenAPI v2 modela o corpo da solicitação `interact` com `code` como obrigatório; o SDK de Rust acompanha o comportamento do servidor ao aceitar **ou** `code` **ou** `prompt` (veja `ScrapeExecuteOptions` em `scrape.rs`).

<div id="notes">
  ## Observações
</div>

* Aliases descontinuados: `scrape_execute`, `stop_interactive_browser` e `delete_scrape_browser` correspondem a `interact` e `stop_interaction`.
* `ScrapeOptions` inclui `json_options`, `screenshot_options` e `change_tracking_options` específicos para formatos avançados.
* `search_and_scrape(query, limit)` é uma função auxiliar de conveniência: chama `search` com `ScrapeOptions` padrão e retorna `Vec<Document>` criado a partir das entradas `SearchResultOrDocument::Document` em `data.web` (consulte `search.rs`).
* O SDK v2 exporta todos os tipos na raiz do crate: `use firecrawl::Client` (e não `use firecrawl::v2::Client`).
* Os tipos de erro foram simplificados no v2: `CrawlJobFailed(String, CrawlStatus)` → `JobFailed(String)`, `Missuse` → `Misuse`.
