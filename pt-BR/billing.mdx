<div id="overview">
  ## Visão geral
</div>

Firecrawl utiliza um **sistema de cobrança baseado em créditos**. Cada chamada de API que você faz consome créditos, e a quantidade de créditos consumidos depende do endpoint e das opções que você utiliza. Você recebe uma cota mensal de créditos com base no seu plano, e o upgrade inteligente garante cobertura à medida que seu uso cresce.

Para ver os preços atuais dos planos, visite a [página de preços do Firecrawl](https://www.firecrawl.dev/pricing).

<Note>
  Todas as faturas do Firecrawl são emitidas em **dólares americanos (USD)**, independentemente do seu endereço de cobrança ou método de pagamento.
</Note>

<div id="credits">
  ## Créditos
</div>

Créditos são a unidade de consumo no Firecrawl. Cada plano inclui uma cota mensal de créditos que é renovada no início de cada ciclo de cobrança. Diferentes endpoints da API consomem quantidades diferentes de créditos.

<div id="credit-costs-per-endpoint">
  ### Custos de crédito por endpoint
</div>

| Endpoint     | Custo de crédito                 | Observações                                                                                                                                                                                                                                                                                                                                                 |
| ------------ | -------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Scrape**   | 1 crédito / página               | Converte uma única URL em markdown limpo, HTML ou dados estruturados. Créditos adicionais se aplicam ao usar opções de scraping (consulte abaixo).                                                                                                                                                                                                          |
| **Crawl**    | 1 crédito / página               | Faz o scraping de um site inteiro seguindo links a partir de uma URL inicial. Os mesmos custos das opções de scraping por página se aplicam a cada página rastreada.                                                                                                                                                                                        |
| **Map**      | 1 crédito / chamada              | Descobre todas as URLs em um site sem fazer scraping do conteúdo.                                                                                                                                                                                                                                                                                           |
| **Search**   | 2 créditos / 10 resultados       | Faz uma busca na web e, opcionalmente, realiza o scraping dos resultados. Arredondado para cima a cada 10 resultados (por exemplo, 11 resultados = 4 créditos). Custos adicionais de scraping por página se aplicam a cada resultado que for raspado. Consulte [aqui](/pt-BR/features/search#zero-data-retention-zdr) para ver os preços da busca ZDR Enterprise. |
| **Interact** | 2 créditos / minuto de navegador | Sessão interativa de sandbox de navegador, cobrada por minuto.                                                                                                                                                                                                                                                                                              |
| **Agent**    | Dinâmico                         | Agente autônomo de pesquisa na web. 5 execuções diárias gratuitas; cobrança baseada em uso após isso.                                                                                                                                                                                                                                                       |

<div id="additional-credit-costs-for-scrape-options">
  ### Custos adicionais de crédito para opções de scraping
</div>

Certas opções de scraping adicionam créditos além do custo base por página:

| Opção                        | Custo adicional            | Descrição                                                                                                                |
| ---------------------------- | -------------------------- | ------------------------------------------------------------------------------------------------------------------------ |
| PDF parsing                  | +1 crédito / página de PDF | Extrair conteúdo de documentos PDF                                                                                       |
| JSON format (LLM extraction) | +4 créditos / página       | Usar um LLM para extrair dados JSON estruturados da página                                                               |
| Enhanced Mode                | +4 créditos / página       | Scraping aprimorado para páginas que são difíceis de acessar                                                             |
| Zero Data Retention (ZDR)    | +1 crédito / página        | Garante que nenhum dado seja persistido além da requisição (veja [Scrape ZDR](/pt-BR/features/scrape#zero-data-retention-zdr)) |

Esses modificadores se acumulam. Por exemplo, fazer scraping de uma página com JSON format e Enhanced Mode custa **1 + 4 + 4 = 9 créditos** por página. Esses mesmos modificadores se aplicam aos endpoints rastreamento e Search, já que eles usam scraping internamente para cada página.

Requisições para `x.com` e outras URLs do X/Twitter usam a API Grok e têm preços separados. Veja [cobrança do X (x.com)](#x-xcom-billing) no final desta página.

<div id="when-credits-are-charged">
  ### Quando os créditos são debitados
</div>

Os créditos são debitados sempre que a infraestrutura do Firecrawl processa uma request, mesmo que o site de destino retorne um código de status de erro HTTP, como 403 Forbidden ou 404 Not Found. Isso acontece porque a infraestrutura de scraping (renderização no navegador, proxy etc.) é totalmente utilizada, independentemente da resposta do site de destino. Você pode verificar o campo `metadata.statusCode` na resposta da API para detectar esses casos e evitar tentar novamente URLs que são bloqueadas de forma consistente.

Para jobs de **batch scrape** e **rastreamento**, os créditos são debitados de forma assíncrona à medida que cada página conclui o processamento — não quando o job é submetido. Isso significa que pode haver um atraso entre a submissão de um job e ver o custo total de créditos refletido na sua conta. Se um batch contiver muitas URLs ou se as páginas forem enfileiradas durante períodos de alto tráfego, os créditos podem continuar aparecendo minutos ou horas após a submissão. Realizar polling ou verificar o status do batch não consome créditos.

<Note>
  **Verificação prévia de créditos do rastreamento:** Antes de um job de rastreamento começar, o Firecrawl verifica se o saldo de créditos restante é suficiente para cobrir todo o parâmetro `limit` solicitado. Se o seu saldo for menor que `limit`, a request retorna um 402 mesmo que o rastreamento tivesse descoberto menos páginas. O `limit` padrão é **10.000**, portanto omiti-lo exige que 10.000 créditos estejam disponíveis antecipadamente. Para evitar isso, passe um `limit` explícito que corresponda ao número de páginas que você realmente pretende rastrear (por exemplo, `limit: 100`).
</Note>

<div id="tracking-your-usage">

* **API**: Use os endpoints [Credit Usage](/pt-BR/api-reference/endpoint/credit-usage) e [Credit Usage Historical](/pt-BR/api-reference/endpoint/credit-usage-historical) para verificar seu uso programaticamente

<div id="plans">
  ## Planos
</div>

Firecrawl oferece planos mensais por assinatura. Não há uma opção pura de pague‑conforme‑o‑uso (pay-as-you-go), mas o upgrade inteligente (descrito abaixo) oferece escalabilidade flexível adicional ao seu plano base.

<div id="paid-plans">
  ### Planos pagos
</div>

| Plano        | Créditos mensais            | Navegadores concorrentes |
| ------------ | --------------------------- | -----------------------: |
| **Hobby**    | 5,000 / 6,500 / 8,000       |                        5 |
| **Standard** | 100,000 / 130,000 / 160,000 |                       50 |
| **Growth**   | 500,000 / 650,000           |                      100 |
| **Scale**    | 1,000,000                   |                      150 |

<Note>
  Para necessidades além do plano Scale, Firecrawl oferece planos **Enterprise** com créditos personalizados, suporte dedicado, SLAs, descontos em volume, sem retenção de dados e SSO. Visite a [página Enterprise](https://www.firecrawl.dev/enterprise) para mais detalhes.
</Note>

Todos os planos pagos estão disponíveis com cobrança **mensal** ou **anual**. A cobrança anual oferece um desconto em comparação ao pagamento mês a mês. Para preços atualizados de cada plano, visite a [página de preços](https://www.firecrawl.dev/pricing).

<div id="billing-cycle">
  ### Ciclo de cobrança
</div>

* **Planos mensais**: Os créditos são restaurados na sua data de renovação mensal
* **Planos anuais**: A cobrança é anual, mas os créditos ainda são restaurados todo mês na sua data de renovação mensal virtual
* **Créditos do plano não utilizados não são transferidos por padrão** — sua cota mensal é restaurada a cada mês. **Planos anuais Scale transferem créditos do plano não utilizados por 1 mês**, e **planos anuais Enterprise os transferem por 2 meses**.

<div id="concurrent-browsers">
  ### Navegadores simultâneos
</div>

Navegadores simultâneos representam quantas páginas da web o Firecrawl pode processar para você ao mesmo tempo. Seu plano determina esse limite. Se você o exceder, tarefas adicionais ficam em uma fila de espera até que uma vaga seja liberada. Consulte [Limites de Taxa](/pt-BR/rate-limits) para ver todos os detalhes sobre concorrência e limites de taxa da API.

<div id="smart-upgrade">
  ## Upgrade inteligente
</div>

<Note>
  A partir de **1º de junho de 2026**, o Upgrade Inteligente substitui a Recarga Automática. Em vez de pagar por excedentes, esses valores agora são aplicados aos seus upgrades, proporcionando uma experiência contínua e sem interrupções.
</Note>

Quando seu saldo chega a zero, o Upgrade Inteligente automaticamente leva você um nível acima na escala de créditos, incluindo os níveis intermediários dentro de cada plano, para que as requisições continuem fluindo sem interrupção. Você paga apenas a diferença proporcional de preço entre seu nível atual e o próximo, e recebe imediatamente a diferença correspondente em créditos. Desative o Upgrade Inteligente em [configurações de cobrança](https://www.firecrawl.dev/app/settings?tab=billing) a qualquer momento. Ao atingir o nível Scale (1.000.000 créditos/mês), não há níveis mais altos para os quais o Upgrade Inteligente possa avançar; [fale com vendas](https://www.firecrawl.dev/enterprise) sobre um plano Enterprise personalizado.

<div id="pro-ration-examples">
  ### Exemplos de cobrança proporcional
</div>

Suponha que você esteja no **Hobby 6.5k** ($28/mês ou $290/ano) e que o upgrade inteligente mova você para o **Hobby 8k** ($37/mês ou $390/ano). Você recebe **1.500 créditos adicionais** imediatamente (a diferença até 8k), e a cobrança do upgrade depende da sua periodicidade de cobrança:

* **Plano mensal:** Hoje, será cobrada a diferença mensal de $37 − $28 = **$9**. Sua data de renovação permanece inalterada.
* **Plano anual, com 5 meses restantes no seu ciclo de 12 meses:** Hoje, será cobrado `(5 ÷ 12) × 100 ≈ &#36;41,67`, em que a diferença de preço anual é $390 − $290 = $100. Os 4 meses restantes no Hobby 8k são cobertos por essa cobrança, e sua data de renovação permanece inalterada.

Quando você esgota o nível 8k, o próximo passo na escala é **Hobby 8k → Standard 100k**, cobrado pela diferença entre esses dois níveis ($62/mês ou $600/ano, com cobrança proporcional da mesma forma), com **92.000 créditos adicionais** adicionados ao seu saldo.

<div id="upgrading-and-downgrading">
  ## Upgrades e downgrades de plano
</div>

* **Upgrades** entram em vigor imediatamente. O valor integral do novo plano é cobrado hoje (sem cobrança proporcional), e seu ciclo de cobrança é reiniciado — sua próxima renovação será em um mês ou um ano a partir da data do upgrade. Quaisquer créditos não utilizados do seu plano anterior são transferidos, e sua nova franquia de créditos e limites de concorrência se aplicam na hora.
* **Downgrades** são agendados para entrar em vigor na sua próxima data de renovação. Você mantém os créditos e limites do seu plano atual até lá, e o tempo não utilizado do seu plano atual não gera créditos nem reembolso. Você pode desfazer um downgrade agendado nas suas [configurações de cobrança](https://www.firecrawl.dev/app/settings?tab=billing) a qualquer momento antes da data de vigência.

<div id="switching-between-monthly-and-yearly-billing">
  ### Alternar entre cobrança mensal e anual
</div>

* **Mensal → Anual** no mesmo nível de créditos ou em um nível superior é considerado um upgrade imediato.
* **Anual → Mensal** é considerado um upgrade imediato apenas se você mudar para um nível de créditos estritamente superior.

<div id="running-out-of-credits">
  ## Sem créditos
</div>

Se você esgotar sua cota de créditos e não estiver com o **upgrade inteligente** ativado, as solicitações de API que consomem créditos retornarão um erro **HTTP 402 (Pagamento obrigatório)**.

Com o **upgrade inteligente** ativado, sua assinatura é movida automaticamente para o próximo nível de créditos, e você recebe imediatamente a cota de créditos desse novo nível, para que as solicitações continuem sem interrupção. O **upgrade inteligente** para quando você atinge o nível Scale.

Para retomar o uso após uma interrupção total, você pode:

1. Ativar o [upgrade inteligente](#smart-upgrade) para escalar automaticamente
2. Fazer upgrade manualmente para um plano superior
3. Aguardar seus créditos serem redefinidos no próximo ciclo de cobrança

<div id="coupons">
  ## Cupons
</div>

A Firecrawl oferece dois tipos de cupons:

* **Cupons de assinatura** aplicam um desconto à assinatura do seu plano (por exemplo, uma porcentagem de desconto no valor mensal ou anual). Eles **só** podem ser aplicados durante o fluxo de checkout do Stripe, quando você assina um plano pago pela primeira vez ou muda de plano. Não é possível aplicar um cupom de assinatura depois que o checkout for concluído.
* **Cupons de crédito** adicionam créditos bônus à sua conta. Eles podem ser resgatados na seção **cobrança** do seu painel em [firecrawl.dev/app/billing](https://www.firecrawl.dev/app/billing). Procure o campo de cupom na página de cobrança para aplicar seu código. Os créditos bônus de cupons de crédito são separados da cota mensal do seu plano e permanecem mesmo se você fizer upgrade ou downgrade do seu plano.

<div id="faqs">
  ## Perguntas frequentes
</div>

<AccordionGroup>
  <Accordion title="Créditos não utilizados são transferidos para o próximo mês?">
    **Créditos do plano** não são transferidos por padrão — sua cota mensal é redefinida a cada mês. **Planos anuais Scale transferem créditos do plano não utilizados por 1 mês**, e **planos anuais Enterprise os transferem por 2 meses**.
  </Accordion>

  <Accordion title="Como a cobrança do upgrade inteligente é calculada?">
    O upgrade inteligente cobra a diferença proporcional entre o seu nível atual e o próximo nível. Nos planos **mensais**, isso é simplesmente a diferença de preço mensal. Nos planos **anuais**, é `(meses restantes no seu ciclo de cobrança ÷ 12) × diferença de preço anual`, usando meses corridos completos a partir do início do seu período atual. Você também recebe a diferença de créditos entre os dois níveis, então paga apenas pelo que ganha.
  </Accordion>

  <Accordion title="Como eu sei quantos créditos ainda tenho?">
    Verifique o painel em [firecrawl.dev/app](https://www.firecrawl.dev/app), ou acesse o [endpoint da API de uso de créditos](/pt-BR/api-reference/endpoint/credit-usage) programaticamente.
  </Accordion>

  <Accordion title="Onde aplico um código de cupom?">
    Depende do tipo de cupom. **Cupons de crédito** são aplicados na seção cobrança do seu painel. **Cupons de assinatura** (descontos no preço do seu plano) só podem ser aplicados na página de checkout do Stripe ao assinar ou mudar de plano.
  </Accordion>

  <Accordion title="Preciso de mais concorrência ou de um plano personalizado. Com quem eu falo?">
    Entre em contato pelo e-mail [help@firecrawl.dev](mailto:help@firecrawl.dev), ou visite a [página Enterprise](https://www.firecrawl.dev/enterprise) para saber mais sobre planos personalizados.
  </Accordion>

  <Accordion title="Em qual moeda as faturas são cobradas?">
    Todas as faturas do Firecrawl são cobradas em **dólares americanos (USD)**, independentemente do seu endereço de cobrança ou método de pagamento.
  </Accordion>

  <Accordion title="Como adiciono um número de VAT, nome da empresa ou endereço de cobrança às minhas faturas?">
    Acesse suas [configurações de cobrança](https://www.firecrawl.dev/app/settings?tab=billing), clique em **Manage Subscriptions** e atualize seu endereço de cobrança, nome da empresa e número de VAT no portal da Stripe. As próximas faturas incluirão automaticamente as informações atualizadas.

    Para reemitir uma fatura paga anterior com as novas informações:

    1. Atualize primeiro seus dados de cobrança no portal da Stripe (veja acima).
    2. Abra a aba **Invoice history** no portal da Stripe e baixe o PDF da fatura desejada; a Stripe a renderiza novamente com base nas suas informações de cobrança atuais.
    3. Se alguma fatura não refletir os dados atualizados, envie um e-mail para [help@firecrawl.dev](mailto:help@firecrawl.dev) com os números das faturas e nós as reemitiremos para você.
  </Accordion>
</AccordionGroup>

<div id="x-xcom-billing">
  ## Cobrança do X (x.com)
</div>

O Firecrawl usa a **API oficial do Grok** da [xAI](https://x.ai/) para oferecer resumos com IA, extração estruturada e acesso em tempo real a conteúdo público do X. As requisições para URLs de perfis e publicações de `x.com`, `twitter.com` e `mobile.twitter.com` são processadas por meio das ferramentas internas autorizadas do Grok (`x_search`, obtenção de threads e busca na web restrita a x.com), em vez do scraping tradicional da web.

<div id="credit-costs">
  ### Custos em créditos
</div>

| Componente       | Custo em créditos         | Descrição                                                               |
| ---------------- | ------------------------- | ----------------------------------------------------------------------- |
| **Custo base**   | 1 crédito / requisição    | Processamento padrão de requisição de scraping                          |
| **Grok X Query** | +29 créditos / requisição | Uso da API do Grok (tokens + chamadas de ferramenta) para conteúdo do X |

Por exemplo, processar uma requisição típica de post ou thread custa **30 créditos** (`1` base + `29` Grok X Query) e retorna dados estruturados gerados pelo Grok, contexto da thread e resumos. Se o formato JSON (extração com LLM) também estiver habilitado, o total será de **34 créditos** por requisição.

Esse método está em conformidade com as interfaces publicadas pelo X por meio da parceria com a xAI e fornece resultados de maior qualidade e com raciocínio, em vez de fazer scraping bruto da página.

<Note>
  **Os recursos diferem do scraping padrão.** O Grok retorna resultados processados por IA, que podem incluir resumos, métricas importantes, contexto da thread e muito mais. Para dados estruturados brutos em escala, use a [API oficial X Enterprise](https://developer.x.com/).
</Note>
