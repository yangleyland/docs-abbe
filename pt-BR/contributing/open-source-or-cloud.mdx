O Firecrawl é open source sob a [licença AGPL-3.0](https://github.com/firecrawl/firecrawl/blob/main/LICENSE). Para entregar o melhor produto possível, também oferecemos uma versão em nuvem, hospedada em [firecrawl.dev](https://firecrawl.dev), com recursos adicionais e infraestrutura gerenciada.

<div id="feature-comparison">
  ## Comparação de recursos
</div>

A versão open source inclui o mecanismo principal de scraping e crawling. O Firecrawl Cloud adiciona funcionalidades em nível de infraestrutura, um painel de gerenciamento e recursos corporativos, além de tudo o que há na edição open source.

![Firecrawl Cloud vs Open Source](/images/open-source-cloud.png)

| Recurso | Open Source | Cloud |
|---------|:----------:|:-----:|
| Scrape | ✔ | ✔ |
| Crawl | ✔ | ✔ |
| Map | ✔ | ✔ |
| Search | ✔ | ✔ |
| Scrape em lote | ✔ | ✔ |
| Extract | ✔ | ✔ |
| Modo JSON | ✔ | ✔ |
| Formatos prontos para LLMs | ✔ | ✔ |
| Rastreamento de alterações | ✔ | ✔ |
| SDKs | ✔ | ✔ |
| Agente | ✘ | ✔ |
| Sandbox de navegador | ✘ | ✔ |
| ações | ✘ | ✔ |
| Proxies avançados | ✘ | ✔ |
| Rotações de proxies | ✘ | ✔ |
| Dashboard | ✘ | ✔ |
| Recursos corporativos | ✘ | ✔ |

<div id="why-both">
  ## Por que ambos?
</div>

A solução em nuvem nos permite inovar continuamente e manter um serviço sustentável e de alta qualidade para todos os usuários. A receita do Firecrawl Cloud financia o desenvolvimento contínuo do projeto open source.

* **Open source** — Execute o Firecrawl na sua própria infraestrutura com controle total sobre o mecanismo central de scraping. Veja [Executando localmente](/pt-BR/contributing/guide) e [Self-hosting](/pt-BR/contributing/self-host).
* **Cloud** — Comece imediatamente com um serviço gerenciado que inclui proxies aprimorados, um painel de uso e recursos empresariais. Cadastre-se em [firecrawl.dev](https://firecrawl.dev).
