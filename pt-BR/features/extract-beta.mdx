<div id="introducing-extract-beta">
  ## Apresentando /extract (Beta)
</div>

<Warning>
  [Use a nova documentação do Open Beta de /extract clicando aqui 🔥](/pt-BR/features/extract)
</Warning>

Extraia dados estruturados de uma única URL, de várias URLs ou de sites inteiros usando Large Language Models (LLMs). Nosso novo endpoint `/extract` permite que você:

* Extraia dados estruturados de sites completos de uma só vez
* Conecte ou crie aplicativos de enriquecimento de dados que precisem de dados estruturados de sites
* Desenvolva aplicativos de IA que precisem de dados limpos de vários sites

<div id="considerations">
  ## Considerações
</div>

O endpoint `/extract` oferece extração de dados flexível com esquemas personalizáveis. Os resultados podem ser aprimorados por meio do ajuste de prompt. Atualmente, está em beta e agradecemos seu feedback.

  ## Extração de dados

<div id="extract-endpoint">
  ### endpoint /extract
</div>

Usado para extrair dados estruturados de sites inteiros.

Ao especificar URLs, você pode adicionar `/*` à URL para extrair informações de todo o caminho do site, em vez de apenas uma única página.

Por exemplo, `https://firecrawl.dev/*` tentará extrair dados de todas as páginas no domínio firecrawl.dev. O `/*` ainda está em fase de testes, portanto, se tiver algum problema, envie um e-mail para help@firecrawl.com.

<div id="checking-extract-status">
  ### Verificando o status da extração
</div>

Você pode usar o endpoint `/extract/ID` para verificar o status de uma tarefa de extração.

<Note>Este endpoint só funciona para tarefas de extração que estão em andamento ou que foram concluídas recentemente (nas últimas 24 horas).</Note>

<CodeGroup>
  <CheckExtractJobCURL />
</CodeGroup>

<div id="pending-response">
  #### Resposta pendente
</div>

Tarefas de extração podem estar em um dos seguintes estados:

* `completed`: a tarefa de extração foi concluída com sucesso.
* `processing`: a tarefa de extração ainda está em processamento.
* `failed`: a tarefa de extração apresentou um erro e não foi concluída.
* `cancelled`: a tarefa de extração foi cancelada pelo usuário.

<div id="improving-results-with-web-search-external-links">
  ### Melhorando os resultados com pesquisa na web e links externos
</div>

Para melhorar os resultados da extração, você pode passar o parâmetro `enableWebSearch` para o endpoint. Isso permite que ele tente encontrar os dados em links externos — fora do escopo das URLs fornecidas.

<div id="billing">
  ### Cobrança
</div>

Enquanto o endpoint `/extract` estiver em beta, cobramos 5 créditos por URL raspada usada para compor a resposta final. Isso é para evitar abuso. Isso poderá mudar no futuro.
