<div id="how-it-works">
  ## Como funciona
</div>

Integre o Firecrawl diretamente ao seu fluxo de trabalho de desenvolvimento com IA por meio do Model Context Protocol. Depois de configurado, seu assistente de IA passa a ter acesso a um conjunto de ferramentas de scraping que pode acionar em seu nome:

| Ferramenta       | O que faz                                                             |
| ---------------- | --------------------------------------------------------------------- |
| **Scrape**       | Extrai conteúdo ou dados estruturados de uma única URL                |
| **Batch Scrape** | Extrai conteúdo de várias URLs conhecidas em paralelo                 |
| **Map**          | Descobre todas as URLs indexadas em um site                           |
| **Crawl**        | Percorre uma seção do site e extrai conteúdo de cada página           |
| **Search**       | Faz uma busca na web e, opcionalmente, extrai conteúdo dos resultados |

Seu assistente escolhe automaticamente a ferramenta certa. Peça para ele &quot;ler a documentação do Next.js&quot; e ele fará scraping. Peça para &quot;encontrar todas as postagens do blog em example.com&quot; e ele fará o mapeamento e depois a extração em lote.

<div id="faqs">
  ## FAQs
</div>

<AccordionGroup>
  <Accordion title="Quais assistentes de IA são compatíveis com MCP?">
    Atualmente, o Claude Desktop e o Cursor têm suporte nativo ao MCP. Outros assistentes de IA vêm adicionando suporte regularmente. Você também pode usar o MCP SDK para criar integrações personalizadas.
  </Accordion>

  <Accordion title="Posso usar MCP no VS Code ou em outras IDEs?">
    O VS Code e outras IDEs podem usar MCP por meio de extensões da comunidade ou integrações via terminal. O suporte nativo varia conforme a IDE. Confira nosso [repositório no GitHub](https://github.com/firecrawl/firecrawl-mcp-server) para guias de configuração específicos por IDE.
  </Accordion>

  <Accordion title="Como faço cache de documentos acessados com frequência?">
    O servidor MCP armazena respostas em cache automaticamente por 15 minutos. Você pode configurar a duração do cache nas configurações do servidor MCP ou implementar uma lógica de cache personalizada.
  </Accordion>

  <Accordion title="Existe limite de taxa para solicitações MCP?">
    As solicitações MCP usam seus limites padrão da API do Firecrawl. Recomendamos agrupar solicitações relacionadas e usar cache para documentação acessada com frequência.
  </Accordion>

  <Accordion title="Como configuro o MCP com minha chave de API do Firecrawl?">
    Siga nosso [guia de configuração](https://github.com/firecrawl/firecrawl-mcp-server#installation) para configurar o MCP. Você precisará adicionar sua chave de API do Firecrawl ao arquivo de configuração do MCP. O processo leva apenas alguns minutos.
  </Accordion>
</AccordionGroup>
