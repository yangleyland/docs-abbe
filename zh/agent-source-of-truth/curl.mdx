面向代理的 Firecrawl 规范 curl 权威来源。与 `api-reference/v2-openapi.json` 对齐 (服务器基础 URL 为 `https://api.firecrawl.dev/v2`) 。

<div id="authenticate">
  ## 身份验证
</div>

```bash
export FIRECRAWL_API_KEY="fc-your-api-key"
```

<div id="when-to-use-what">
  ## 何时使用哪一种
</div>

* `search`：当你从查询开始，需要先发现结果时使用。
* `scrape`：当你已经有 URL，并想获取页面内容时使用。
* `interact`：当页面需要点击、填写表单，或在抓取后执行浏览器 actions 时使用。
* `support/ask`：当 Firecrawl API 调用失败或返回了非预期结果，且你需要诊断问题时使用。
* `support/docs-search`：当你需要查阅 Firecrawl 文档时使用。

<div id="search">
  ## Search
</div>

### 使用场景

使用 search 从查询中发现相关页面，然后选择要抓取或交互的 URL。你可以使用标准的 `site:` 运算符将结果限定在某个站点范围内，例如 `site:docs.firecrawl.dev crawl webhooks`。

<div id="endpoint">
  ### 端点
</div>

`POST /search`

<div id="simple-example">
  ### 简单示例
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/search" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "site:docs.firecrawl.dev webhook retries"
  }'
```

<div id="complex-example">
  ### 复杂示例
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/search" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "site:docs.firecrawl.dev crawl webhooks",
    "sources": [{"type": "web"}, {"type": "news"}],
    "categories": [{"type": "research"}],
    "limit": 10,
    "tbs": "qdr:m",
    "location": "San Francisco,California,United States",
    "country": "US",
    "ignoreInvalidURLs": true,
    "timeout": 60000,
    "enterprise": ["anon"],
    "scrapeOptions": {
      "formats": [
        "markdown",
        "links",
        {"type": "json", "prompt": "Extract title and key endpoints."}
      ],
      "onlyMainContent": true,
      "includeTags": ["main", "article"],
      "excludeTags": ["nav", "footer"],
      "waitFor": 1000
    }
  }'
```

<div id="response">
  ### 响应
</div>

成功响应包括 `success`、`data`、可选的 `warning`、`id` 和 `creditsUsed`。

* `data.web`、`data.images`、`data.news`：结果数组；具体返回哪些键取决于 `sources` (默认仅返回 `data.web`) 。
* 网页和新闻条目包含 `title`、`url` 等字段，以及 (当 `scrapeOptions` / formats 请求这些内容时) `markdown`、`html`、`rawHtml`、`links`、`screenshot`、`audio`、`video` 和 `metadata`。
* 图片条目包含 `imageUrl`、`url` 等字段，以及在可用时返回的尺寸信息。
* `warning`：可选的、便于人类阅读的提示信息。
* `id`：搜索任务 ID 字符串。
* `creditsUsed`：此次调用扣除的整数额度。

<div id="parameters">
  ### 参数
</div>

* `query`
  * 类型：string (必填，最大长度 500) 
  * 适用场景：你需要提供搜索查询时。
  * 说明：使用 `site:example.com` 可将结果限定在某个域名内。

* `sources`
  * 类型：带类型标记的来源对象数组
  * 适用场景：你想控制要搜索哪些来源时。
  * 已确认的对象结构：
    * `{ "type": "web" }`，可为每个来源单独指定可选的 `tbs` 和 `location`
    * `{ "type": "news" }`
    * `{ "type": "images" }`

* `categories`
  * 类型：带类型标记的类别对象数组
  * 适用场景：你想按类别筛选结果时。
  * 已确认的对象结构：
    * `{ "type": "github" }`
    * `{ "type": "research" }`
    * `{ "type": "pdf" }`

* `limit`
  * 类型：integer (最小值 1，最大值 100，默认值 5) 
  * 适用场景：你想限制结果数量时。

* `tbs`
  * 类型：string
  * 适用场景：你需要按时间筛选时 (例如 `qdr:d`、`qdr:w`、`sbd:1,qdr:m`) 。

* `location`
  * 类型：string
  * 适用场景：你想获取本地化结果时。

* `country`
  * 类型：string (默认值 `"US"`) 
  * 适用场景：你想使用 ISO 3166-1 alpha-2 代码进行定向时 (例如 `"US"`) 。

* `ignoreInvalidURLs`
  * 类型：boolean (默认值 false) 
  * 适用场景：你想丢弃其他端点无法抓取的 URL 时。

* `timeout`
  * 类型：integer (毫秒，默认值 60000) 
  * 适用场景：你需要以毫秒为单位设置请求超时时间时。

* `enterprise`
  * 类型：字符串数组 (每项为 `"anon"` 或 `"zdr"`) 
  * 适用场景：你需要企业级搜索控制时。
  * 值：
    * `"zdr"`：端到端零数据保留
    * `"anon"`：匿名化零数据保留

* `scrapeOptions`
  * 类型：object
  * 适用场景：你想抓取每个搜索结果时 (字段请参见 Scrape 参数) 。

<div id="scrape">
  ## Scrape
</div>

### 为什么使用

如果你已经有一个 URL，并希望以一种或多种 formats 获取结构化内容，请使用 scrape。

<div id="endpoint">
  ### 端点
</div>

`POST /scrape`

<div id="simple-example">
  ### 简单示例
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://docs.firecrawl.dev",
    "formats": ["markdown"]
  }'
```

<div id="complex-example">
  ### 复杂示例
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/pricing",
    "formats": [
      "markdown",
      "links",
      {"type": "json", "prompt": "Extract plan names and prices."},
      {"type": "screenshot", "fullPage": true, "quality": 80, "viewport": {"width": 1280, "height": 720}},
      {"type": "changeTracking", "modes": ["git-diff"], "tag": "pricing"}
    ],
    "headers": {"User-Agent": "FirecrawlDocsBot/1.0"},
    "onlyMainContent": true,
    "waitFor": 1000,
    "parsers": [{"type": "pdf", "mode": "auto", "maxPages": 5}],
    "actions": [
      {"type": "click", "selector": "#accept"},
      {"type": "wait", "milliseconds": 750},
      {"type": "scrape"}
    ],
    "location": {"country": "US", "languages": ["en-US"]},
    "removeBase64Images": true,
    "blockAds": true,
    "proxy": "auto",
    "maxAge": 86400000,
    "minAge": 1,
    "storeInCache": true,
    "profile": {"name": "docs-session", "saveChanges": true},
    "zeroDataRetention": false
  }'
```

<div id="response">
  ### 响应
</div>

成功响应包含 `success` 和 `data`。常见的 `data` 字段 (取决于 `formats` 和选项) ：

* `markdown`、`summary`、`html`、`rawHtml`、`screenshot`、`audio`、`video`、`links`
* `actions`：当请求包含抓取时的 `actions` 时，会包含按顺序返回的结果，例如 `screenshots`、`scrapes`、`javascriptReturns` 和 `pdfs`
* `metadata`：页面元数据 (`title`、`sourceURL`、`url`、`statusCode`、`error` 以及其他提取出的字段)
* `warning`：可选的提取或格式化提示
* `changeTracking`：请求 `changeTracking` 格式时会出现

<div id="parameters">
  ### 参数
</div>

* `url`
  * 类型：string
  * 使用场景：想抓取某个特定页面时。

* `formats`
  * 类型：格式字符串数组或格式对象数组
  * 使用场景：需要多种输出格式时。
  * 已确认的格式字符串：
    * `"markdown"`：markdown 内容
    * `"html"`：清理后的 HTML
    * `"rawHtml"`：原始 HTML
    * `"links"`：页面链接
    * `"images"`：图片 URL
    * `"screenshot"`：截图输出
    * `"summary"`：摘要输出
    * `"changeTracking"`：变更追踪输出
    * `"json"`：JSON 提取
    * `"branding"`：品牌资料输出
    * `"audio"`：音频提取
    * `"video"`：视频提取
  * 格式对象字段：
    * `type`：上述格式字符串之一
    * `prompt`, `schema`：`type: "json"` 时的 JSON 提取选项
    * `modes`, `schema`, `prompt`, `tag`：`type: "changeTracking"` 时的变更追踪选项
    * `fullPage`, `quality`, `viewport`：`type: "screenshot"` 时的截图选项

* `headers`
  * 类型：object
  * 使用场景：需要自定义请求头时。

* `includeTags`
  * 类型：字符串数组
  * 使用场景：只想包含特定 HTML 标签时。

* `excludeTags`
  * 类型：字符串数组
  * 使用场景：想排除特定 HTML 标签时。

* `onlyMainContent`
  * 类型：boolean
  * 使用场景：想去除导航栏、页脚和其他模板化内容时。

* `timeout`
  * 类型：number
  * 使用场景：需要以毫秒为单位设置超时时间时。

* `waitFor`
  * 类型：number
  * 使用场景：需要等待页面渲染完成时 (毫秒) 。

* `mobile`
  * 类型：boolean
  * 使用场景：想使用移动端视口时。

* `parsers`
  * 类型：对象数组
  * 使用场景：需要控制文件解析时。
  * 已确认的结构：`{ "type": "pdf", "mode": "fast" | "auto" | "ocr", "maxPages": number }` (`type` 必填；其他字段为可选项，默认值遵循规范)

* `actions`
  * 类型：action 对象数组
  * 使用场景：需要轻量级抓取前操作时。
  * 已确认的 action 类型：
    * `wait`：必须提供 `milliseconds` 或 `selector`
    * `screenshot`：`fullPage`、`quality`、`viewport` 为可选项
    * `click`：必须提供 `selector`，`all` 为可选项
    * `write`：必须提供 `text` (先点击使输入框获得焦点)
    * `press`：必须提供 `key`
    * `scroll`：必须提供 `type`；`direction` (`up` 或 `down`，默认值为 `down`)；`selector` 为可选项
    * `scrape`：无额外字段
    * `executeJavascript`：必须提供 `script`
    * `pdf`：`format` (A0、A1、A2、A3、A4、A5、A6、Letter、Legal、Tabloid、Ledger) 、`landscape`、`scale` 为可选项

* `location`
  * 类型：包含 `country` 和 `languages` 的 object
  * 使用场景：需要按地理位置或语言进行抓取时。

* `skipTlsVerification`
  * 类型：boolean
  * 使用场景：需要跳过 TLS 验证时。

* `removeBase64Images`
  * 类型：boolean
  * 使用场景：想从 markdown 输出中移除 base64 图片时。

* `blockAds`
  * 类型：boolean
  * 使用场景：想启用广告和 Cookie 弹窗封禁时。

* `proxy`
  * 类型：string
  * 使用场景：需要控制代理时。
  * 已确认的值：`"basic"`、`"enhanced"`、`"auto"`

* `maxAge`
  * 类型：number
  * 使用场景：想使用不超过最大时长的缓存数据时 (毫秒) 。

* `minAge`
  * 类型：number
  * 使用场景：仅在缓存数据至少达到此时长时才使用时 (毫秒) 。

* `storeInCache`
  * 类型：boolean
  * 使用场景：希望 Firecrawl 缓存结果时。

* `profile`
  * 类型：包含 `name` 和可选 `saveChanges` 的 object
  * 使用场景：想使用可持久化的浏览器配置文件，并在多次抓取和交互之间共享时。

* `zeroDataRetention`
  * 类型：boolean
  * 使用场景：想为此次抓取启用零数据保留时。

<div id="interact">
  ## 交互
</div>

### 为什么使用

当页面在抓取开始后需要执行浏览器 actions 或运行代码时，请使用交互。

<div id="endpoint">
  ### 端点
</div>

`POST /scrape/{jobId}/interact`

<div id="simple-example">
  ### 简单示例
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape/<jobId>/interact" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "code": "console.log(await page.title());"
  }'
```

<div id="complex-example">
  ### 复杂示例
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape/<jobId>/interact" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "code": "const title = await page.title();\nconst h1 = await page.locator(\"h1\").first().textContent().catch(() => null);\nconsole.log(JSON.stringify({ title, h1 }));",
    "language": "node",
    "timeout": 120
  }'
```

<div id="parameters">
  ### 参数
</div>

* `jobId` (path)
  * 类型：string (UUID)
  * 使用时机：你已有实时浏览器会话对应的抓取任务 ID。

* `code` (JSON body)
  * 类型：string (在 OpenAPI 中必填；最小长度 1，最大长度 100000) 
  * 使用时机：你想在与抓取任务绑定的浏览器沙箱中运行代码。

* `language` (JSON body)
  * 类型：string
  * 使用时机：你需要指定特定的运行时。
  * 可选值：`"python"`、`"node"`、`"bash"` (默认 `"node"`) 

* `timeout` (JSON body)
  * 类型：integer (秒；最小值 1，最大值 300，默认值 30) 
  * 使用时机：你需要设置执行超时时间。

<div id="response">
  ### 响应
</div>

成功的响应包含 `success` 以及执行结果字段，例如 `stdout`、`result` (`stdout` 的别名) 、`stderr`、`exitCode`、`killed` 和 `error` (可为 null) 。

### DELETE /scrape/{jobId}/interact

<div id="example">
  ### 示例
</div>

```bash
curl -X DELETE "https://api.firecrawl.dev/v2/scrape/<jobId>/interact" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY"
```

无 JSON 请求体。成功响应中包含 `success`。

<div id="ask-agentic-debugging">
  ## Ask (代理调试)
</div>

<div id="why-use-it">
  ### 为什么使用它
</div>

当 Firecrawl API 调用失败或返回非预期结果时，可使用 ask。AI 支持代理会诊断问题、给出修复参数建议，并可选择通过实时 API 验证修复是否有效。典型延迟：15-30 秒。

<div id="endpoint">
  ### 端点
</div>

`POST /support/ask`

<div id="simple-example">
  ### 简单示例
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/support/ask" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "my scrape returned empty markdown for https://example.com"
  }'
```

<div id="complex-example">
  ### 复杂示例
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/support/ask" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "crawl returned 3 pages but I expected 50",
    "rationale": "user is on their third failed crawl attempt today",
    "context": {"userPlan": "standard", "retryCount": 3}
  }'
```

<div id="response">
  ### 响应
</div>

成功响应包含 `requestId`、`answer`、`confidence`、`fixParameters`、`validation`、`usage` 和 `durationMs`。

* `answer`：用 2–4 句话概述诊断结果和修复方法。
* `confidence`：`high`、`medium` 或 `low`。
* `fixParameters`：用于应用修复的机器可执行 API 参数 (如果没有适用的修复，则为 null) 。
* `validation.tested`：代理是否已针对实时 API 测试该修复。
* `validation.result`：`success`、`failure` 或 `skipped`。
* `feedback`：当代理陷入卡顿时出现；成功时为 null。

<div id="parameters">
  ### 参数
</div>

* `question`
  * 类型：字符串 (必填，1-8000 个字符) 
  * 适用场景：需要描述问题时。

* `rationale`
  * 类型：字符串 (1-2000 个字符) 
  * 适用场景：你是代表用户发起调用的 AI 代理时。请说明用户想要完成什么。

* `context`
  * 类型：对象 (任意结构) 
  * 适用场景：你想将代理的元数据传入调试 prompt 时。

<div id="docs-search">
  ## 文档搜索
</div>

### 为什么要用它

使用 docs-search 查找 Firecrawl 文档。

<div id="endpoint">
  ### 端点
</div>

`POST /support/docs-search`

<div id="example">
  ### 示例
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/support/docs-search" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "how do I verify webhook signatures?"
  }'
```

<div id="parameters">
  ### 参数
</div>

* `question`
  * 类型: string (必填，1-8000 个字符) 
  * 适用场景：需要基于文档作答时。

<div id="notes">
  ## 注意事项
</div>

* Search result 的结果行嵌套在 `data.web`、`data.images` 或 `data.news` 下，而不是扁平的 `data` 数组。
* 多步骤的浏览器工作流请使用 `POST /scrape/{jobId}/interact`；`scrape` 的 `actions` 更适合抓取前的小型准备步骤。
* 已发布的 OpenAPI schema 要求为 interact 提供 `code`。某些 SDK 和客户端也接受用于自然语言指令的 `prompt` 字段；但该别名不在 `v2-openapi.json` 中。
