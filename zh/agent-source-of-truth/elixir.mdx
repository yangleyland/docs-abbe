面向代理的 Firecrawl Elixir 权威信息源。基于 SDK 源代码和 v2 OpenAPI 规范生成。

<div id="install">
  ## 安装
</div>

添加到 `mix.exs`：

```elixir
{:firecrawl, "~> 1.0.0"}
```

<div id="authenticate">
  ## 认证
</div>

```elixir
# config/runtime.exs 或 config.exs
config :firecrawl, api_key: System.get_env("FIRECRAWL_API_KEY")

# 或者在每次请求中传入 api_key
{:ok, res} = Firecrawl.search_and_scrape([query: "site:docs.firecrawl.dev webhook retries"], api_key: "fc-your-api-key")
```

<div id="when-to-use-what">
  ## 何时使用哪种方式
</div>

* `search`：当你从查询词开始，需要先发现目标时使用。
* `scrape`：当你已经有 URL，并且想获取页面内容时使用。
* `interact`：当页面需要点击、填写表单，或在抓取后进行浏览器操作时使用。

<div id="search">
  ## 搜索
</div>

<div id="why-use-it">
  ### 为什么使用它
</div>

使用 search 可以根据查询词发现相关页面，然后选择要抓取或交互的 URL。你可以使用 `site:` 将结果限定在某个站点内，例如 `site:docs.firecrawl.dev crawl webhooks`。

<div id="preferred-sdk-method">
  ### 推荐的 SDK 方法
</div>

`Firecrawl.search_and_scrape(params \\ [], opts \\ [])`

<div id="simple-example">
  ### 简单示例
</div>

```elixir
{:ok, res} = Firecrawl.search_and_scrape(query: "site:docs.firecrawl.dev webhook retries")
```

<div id="complex-example">
  ### 复杂示例
</div>

```elixir
{:ok, res} = Firecrawl.search_and_scrape(
  query: "site:docs.firecrawl.dev crawl webhooks",
  sources: [:web, :news],
  categories: [:research],
  limit: 10,
  tbs: "qdr:m",
  location: "San Francisco,California,United States",
  country: "US",
  ignore_invalid_urls: true,
  timeout: 60000,
  scrape_options: [
    formats: [
      "markdown",
      "links",
      %{type: "json", prompt: "Extract title and key endpoints."}
    ],
    only_main_content: true,
    include_tags: ["main", "article"],
    exclude_tags: ["nav", "footer"],
    wait_for: 1000
  ]
)
```

<div id="parameters">
  ### 参数
</div>

* `query`
  * 类型：字符串
  * 使用场景：你需要搜索查询。
  * 说明：使用 `site:example.com` 可将结果限制在某个域名内。

* `sources`
  * 类型：原子、字符串或映射的列表
  * 使用场景：你想控制搜索哪些来源。
  * 已确认的值：
    * `"web"` 或 `:web`
    * `"news"` 或 `:news`
    * `"images"` 或 `:images`
    * `%{type: "web" | "news" | "images"}`

* `categories`
  * 类型：原子、字符串或映射的列表
  * 使用场景：你想按类别筛选结果。
  * 已确认的值：
    * `"github"` 或 `:github`
    * `"research"` 或 `:research`
    * `"pdf"` 或 `:pdf`
    * `%{type: "github" | "research" | "pdf"}`

* `limit`
  * 类型：整数
  * 使用场景：你想限制结果数量。

* `tbs`
  * 类型：字符串
  * 使用场景：你需要基于时间的筛选条件 (例如 `qdr:d`、`qdr:w`、`sbd:1,qdr:m`) 。

* `location`
  * 类型：字符串
  * 使用场景：你想获得本地化结果。

* `country`
  * 类型：字符串
  * 使用场景：你想使用 ISO 3166-1 alpha-2 进行定向 (例如 `"US"`) 。

* `ignore_invalid_urls`
  * 类型：布尔值
  * 使用场景：你想丢弃无法被其他端点抓取的 URL。

* `timeout`
  * 类型：整数
  * 使用场景：你需要以毫秒为单位设置请求超时。

* `enterprise`
  * 类型：字符串列表
  * 使用场景：你需要企业级搜索控制。
  * 已确认的值：
    * `"zdr"`：端到端零数据保留
    * `"anon"`：匿名化零数据保留

* `scrape_options`
  * 类型：关键字列表
  * 使用场景：你想抓取每个搜索结果 (字段请参见 Scrape 参数) 。

<div id="scrape">
  ## 抓取
</div>

<div id="why-use-it">
  ### 为什么使用它
</div>

如果你已经有 URL，并希望获取一种或多种 formats 的结构化内容，请使用 `scrape`。

<div id="preferred-sdk-method">
  ### 推荐的 SDK 方法
</div>

`Firecrawl.scrape_and_extract_from_url(params \\ [], opts \\ [])`

<div id="simple-example">
  ### 简单示例
</div>

```elixir
{:ok, res} = Firecrawl.scrape_and_extract_from_url(
  url: "https://docs.firecrawl.dev",
  formats: ["markdown"]
)
```

<div id="complex-example">
  ### 复杂示例
</div>

```elixir
{:ok, res} = Firecrawl.scrape_and_extract_from_url(
  url: "https://example.com/pricing",
  formats: [
    "markdown",
    "links",
    %{type: "json", prompt: "Extract plan names and prices."},
    %{type: "screenshot", fullPage: true, quality: 80, viewport: %{width: 1280, height: 720}},
    %{type: "changeTracking", modes: ["git-diff"], tag: "pricing"}
  ],
  headers: %{"User-Agent" => "FirecrawlDocsBot/1.0"},
  only_main_content: true,
  wait_for: 1000,
  parsers: [%{type: "pdf", mode: "auto", maxPages: 5}],
  actions: [
    %{type: "click", selector: "#accept"},
    %{type: "wait", milliseconds: 750},
    %{type: "scrape"}
  ],
  location: [country: "US", languages: ["en-US"]],
  remove_base64_images: true,
  block_ads: true,
  proxy: :auto,
  max_age: 86400000,
  min_age: 1,
  store_in_cache: true,
  profile: [name: "docs-session", save_changes: true],
  zero_data_retention: false
)
```

<div id="parameters">
  ### 参数
</div>

* `url`
  * 类型：string
  * 使用场景：当你想抓取某个特定页面时。

* `formats`
  * 类型：格式字符串列表或格式映射列表
  * 使用场景：当你需要多种输出格式时。
  * 已确认的格式字符串：
    * `"markdown"`：markdown 内容
    * `"html"`：清理后的 HTML
    * `"rawHtml"`：原始 HTML
    * `"links"`：页面链接
    * `"images"`：图片 URL
    * `"screenshot"`：截图输出
    * `"summary"`：摘要输出
    * `"changeTracking"`：变更追踪输出
    * `"json"`：JSON 提取
    * `"branding"`：品牌资料输出
    * `"audio"`：音频提取
    * `"video"`：视频提取
  * 格式映射字段：
    * `type`：上述格式字符串之一
    * `prompt`, `schema`：`type: "json"` 时的 JSON 提取选项
    * `modes`, `schema`, `prompt`, `tag`：`type: "changeTracking"` 时的变更追踪选项
    * `fullPage`, `quality`, `viewport`：`type: "screenshot"` 时的截图选项

* `headers`
  * 类型：map
  * 使用场景：当你需要自定义请求头时。

* `include_tags`
  * 类型：字符串列表
  * 使用场景：当你只想包含特定 HTML 标签时。

* `exclude_tags`
  * 类型：字符串列表
  * 使用场景：当你想排除特定 HTML 标签时。

* `only_main_content`
  * 类型：boolean
  * 使用场景：当你想去除导航、页脚和其他样板内容时。

* `timeout`
  * 类型：integer
  * 使用场景：当你需要设置以毫秒为单位的超时时间时。

* `wait_for`
  * 类型：integer
  * 使用场景：当你需要等待页面渲染完成时 (毫秒) 。

* `mobile`
  * 类型：boolean
  * 使用场景：当你想使用移动端视口时。

* `parsers`
  * 类型：解析器字符串列表或映射列表
  * 使用场景：当你需要控制文件解析方式时。
  * 已确认的值：
    * `"pdf"`
    * `%{type: "pdf", mode: "fast" | "auto" | "ocr", maxPages: integer}`

* `actions`
  * 类型：action 映射列表
  * 使用场景：当你需要轻量级的抓取前 actions 时。
  * 已确认的 action 类型：
    * `wait`：必须提供 `milliseconds` 或 `selector`
    * `screenshot`：`fullPage`、`quality`、`viewport` 为可选项
    * `click`：必须提供 `selector`，`all` 为可选项
    * `write`：必须提供 `text` (先点击以聚焦输入框)
    * `press`：必须提供 `key`
    * `scroll`：必须提供 `direction` (`up` 或 `down`) ，`selector` 为可选项
    * `scrape`：无额外字段
    * `executeJavascript`：必须提供 `script`
    * `pdf`：`format` (A0、A1、A2、A3、A4、A5、A6、Letter、Legal、Tabloid、Ledger) ，`landscape`、`scale` 为可选项

* `location`
  * 类型：关键字列表 (例如 `country:` 和 `languages:`)
  * 使用场景：当你需要支持地理位置或语言感知的抓取时。

* `skip_tls_verification`
  * 类型：boolean
  * 使用场景：当你需要跳过 TLS 验证时。

* `remove_base64_images`
  * 类型：boolean
  * 使用场景：当你想从 markdown 输出中移除 base64 图片时。

* `block_ads`
  * 类型：boolean
  * 使用场景：当你想启用广告和 Cookie 弹窗封禁时。

* `proxy`
  * 类型：atom 或 string
  * 使用场景：当你需要控制代理时。
  * 已确认的值：`:basic`, `:enhanced`, `:auto`

* `max_age`
  * 类型：integer
  * 使用场景：当你想使用不超过指定最大时长的缓存数据时 (毫秒) 。

* `min_age`
  * 类型：integer
  * 使用场景：当你只想在缓存数据至少达到该时长时才使用时 (毫秒) 。

* `store_in_cache`
  * 类型：boolean
  * 使用场景：当你想让 Firecrawl 缓存结果时。

* `profile`
  * 类型：包含 `name:` 和可选 `save_changes:` (或 `saveChanges:`) 的关键字列表
  * 使用场景：当你想使用可在多次抓取和交互之间共享的持久化浏览器配置时。

* `zero_data_retention`
  * 类型：boolean
  * 使用场景：当你希望对此次抓取启用零数据保留时。

<div id="interact">
  ## 交互
</div>

<div id="why-use-it">
  ### 为什么使用它
</div>

当页面在开始抓取后需要进行浏览器 actions 或执行代码时，请使用交互。

<div id="preferred-sdk-method">
  ### 推荐的 SDK 方法
</div>

`Firecrawl.interact_with_scrape_browser_session(job_id, params \\ [], opts \\ [])`

<div id="simple-example">
  ### 简单示例
</div>

```elixir
{:ok, res} = Firecrawl.interact_with_scrape_browser_session(
  "<scrapeJobId>",
  code: "console.log(await page.title());",
  language: :node,
  timeout: 60
)
```

<div id="complex-example">
  ### 复杂示例
</div>

```elixir
{:ok, res} = Firecrawl.interact_with_scrape_browser_session(
  "<scrapeJobId>",
  code: "// 在此处使用 Playwright page 方法",
  language: :node,
  timeout: 120
)
```

<div id="parameters">
  ### 参数
</div>

* `job_id`
  * 类型：string
  * 适用场景：你已有抓取任务 ID。

* `code`
  * 类型：string
  * 适用场景：你想在浏览器会话中运行代码。

* `language`
  * 类型：atom 或 string
  * 适用场景：你需要指定特定的运行时环境。
  * 已确认的值：`:python`、`:node`、`:bash` (或其对应的字符串形式)

* `timeout`
  * 类型：integer
  * 适用场景：你需要以秒为单位设置执行超时时间。

* `origin`
  * 类型：string
  * 适用场景：你想为执行遥测指定一个可选的来源标签。

<div id="stop-interactive-session">
  ### 停止交互会话
</div>

`Firecrawl.stop_interactive_scrape_browser_session(job_id, opts \\ [])` 会调用 `DELETE /scrape/{jobId}/interact`，并关闭该抓取任务的浏览器会话。另提供 bang 变体 `stop_interactive_scrape_browser_session!/2`。

```elixir
{:ok, res} = Firecrawl.stop_interactive_scrape_browser_session("<scrapeJobId>")
```

<div id="notes">
  ## 注意事项
</div>

* Elixir 客户端遵循 OpenAPI 生成风格；函数名和参数键名均根据 spec 自动生成。
* 每个公共函数都有一个 bang (`!`) 变体；出错时会抛出异常，而不是返回 `{:error, _}`。
* 此 SDK 仅提供基于代码的交互方式 (`interact_with_scrape_browser_session` 上不支持 `prompt` 参数) 。
