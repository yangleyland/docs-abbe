面向代理的 Firecrawl Node.js 权威来源。与 `firecrawl` **v4.18.2** (`firecrawl/apps/js-sdk/firecrawl`) 及 v2 OpenAPI 规范保持一致。方法名称、参数和类型均与 SDK 的公共 API 一致。

<div id="install">
  ## 安装
</div>

```bash
npm install firecrawl
```

<div id="authenticate">
  ## 身份验证
</div>

```ts
import { Firecrawl } from "firecrawl";

const client = new Firecrawl({
  apiKey: process.env.FIRECRAWL_API_KEY,
  // apiUrl: "https://api.firecrawl.dev" // 可选；未设置时回退到 FIRECRAWL_API_URL 或云端默认值
});
```

<div id="when-to-use-what">
  ## 何时使用哪一种
</div>

* `search`：当你从查询出发、需要先发现目标时使用。
* `scrape`：当你已经有 URL，想获取页面内容时使用。
* `interact`：当页面在 `scrape` 创建会话后需要点击、填写表单或其他浏览器操作时使用。对于多步骤交互流程，优先使用 `interact`，而不是抓取时的 `actions`。
* `support/ask`：当 Firecrawl API 调用失败或返回非预期结果，需要排查原因时使用。
* `support/docs-search`：当你需要查找 Firecrawl 文档时使用。

<div id="search">
  ## Search
</div>

### 使用场景

使用 search 根据查询发现相关页面，然后选择要抓取或交互的 URL。你可以使用 `site:` 将结果限定在某个站点范围内，例如 `site:docs.firecrawl.dev crawl webhooks`。

<div id="preferred-sdk-method">
  ### 首选 SDK 方法
</div>

`client.search(query, options?)` → `Promise<SearchData>`

<div id="simple-example">
  ### 简单示例
</div>

```ts
const results = await client.search("site:docs.firecrawl.dev webhook retries");
```

<div id="complex-example">
  ### 复杂示例
</div>

```ts
const results = await client.search("site:docs.firecrawl.dev crawl webhooks", {
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:m",
  location: "San Francisco,California,United States",
  ignoreInvalidURLs: true,
  timeout: 60000,
  scrapeOptions: {
    formats: [
      "markdown",
      "links",
      { type: "json", prompt: "Extract title and key endpoints." }
    ],
    onlyMainContent: true,
    includeTags: ["main", "article"],
    excludeTags: ["nav", "footer"],
    waitFor: 1000
  }
});
```

<div id="return-value">
  ### 返回值
</div>

`SearchData` 是一个对象，包含可选数组 (每一项要么是轻量级结果对象，要么是当 `scrapeOptions` 对命中结果进行 hydration 后返回的完整 `Document`) ：

* `web`：网页索引命中结果
* `news`：新闻命中结果
* `images`：图片命中结果

**避免走错方向：** `search()` 不会返回 `{ data: [...] }`。不要访问 `result.data`。网页结果在 `result.web` 中，新闻结果在 `result.news` 中，图片结果在 `result.images` 中。

<div id="parameters">
  ### 参数
</div>

* `query`
  * 类型: string
  * 使用场景：需要提供搜索查询时。
  * 说明：使用 `site:example.com` 可将结果限制在某个域名下。

* `options` (可选；第二个参数默认值为 `{}`)
  * 类型: `Omit<SearchRequest, "query">`

* `options.sources`
  * 类型: 来源名称数组或带类型的来源对象数组
  * 使用场景：你想控制搜索哪些来源。
  * 已确认的值：
    * `"web"`: 网页索引结果
    * `"news"`: 新闻结果
    * `"images"`: 图片结果
    * `{ type: "web" | "news" | "images" }`: 带类型的来源对象形式

* `options.categories`
  * 类型: 类别名称数组或带类型的类别对象数组
  * 使用场景：你想按类别筛选结果。
  * 已确认的值：
    * `"github"`: 侧重 GitHub 的结果
    * `"research"`: 研究和学术结果
    * `"pdf"`: 侧重 PDF 的结果
    * `{ type: "github" | "research" | "pdf" }`: 带类型的类别对象形式

* `options.limit`
  * 类型: number
  * 使用场景：你想限制结果数量。

* `options.tbs`
  * 类型: string
  * 使用场景：你需要按时间筛选时 (例如 `qdr:d`, `qdr:w`, `sbd:1,qdr:m`) 。

* `options.location`
  * 类型: string
  * 使用场景：你想获取本地化结果。

* `options.ignoreInvalidURLs`
  * 类型: boolean
  * 使用场景：你想忽略其他端点无法抓取的 URL。

* `options.timeout`
  * 类型: number
  * 使用场景：你需要以毫秒为单位设置请求超时时间。

* `options.scrapeOptions`
  * 类型: `ScrapeOptions`
  * 使用场景：你想抓取每条搜索结果 (字段请参见 Scrape 参数) 。SDK 会执行与 `scrape` 相同的校验 (例如，`formats` 中的普通字符串 `"json"` 会被拒绝)。

<div id="scrape">
  ## 抓取
</div>

### 为什么使用

当你已经有一个 URL，并且希望获取一种或多种 formats 的结构化内容时，请使用 `scrape`。

<div id="preferred-sdk-method">
  ### 首选 SDK 方法
</div>

`client.scrape(url, options?)` → `Promise<Document>`

<div id="simple-example">
  ### 简单示例
</div>

```ts
const doc = await client.scrape("https://docs.firecrawl.dev", {
  formats: ["markdown"]
});
```

<div id="complex-example">
  ### 复杂示例
</div>

```ts
const doc = await client.scrape("https://example.com/pricing", {
  formats: [
    "markdown",
    "links",
    { type: "json", prompt: "Extract plan names and prices." },
    { type: "screenshot", fullPage: true, quality: 80, viewport: { width: 1280, height: 720 } },
    { type: "changeTracking", modes: ["git-diff"], tag: "pricing" },
    { type: "attributes", selectors: [{ selector: "a", attribute: "href" }] }
  ],
  headers: {
    "User-Agent": "FirecrawlDocsBot/1.0"
  },
  onlyMainContent: true,
  waitFor: 1000,
  parsers: [{ type: "pdf", mode: "auto", maxPages: 5 }],
  actions: [
    { type: "click", selector: "#accept" },
    { type: "wait", milliseconds: 750 },
    { type: "scrape" }
  ],
  location: { country: "US", languages: ["en-US"] },
  removeBase64Images: true,
  fastMode: true,
  blockAds: true,
  proxy: "auto",
  maxAge: 86400000,
  minAge: 1,
  storeInCache: true,
  profile: { name: "docs-session", saveChanges: true }
});
```

<div id="parameters">
  ### 参数
</div>

* `url`
  * 类型：string
  * 适用场景：当你想抓取特定页面时。

* `options.formats`
  * Type: 格式字符串或格式对象数组
  * 使用场景：需要多种输出格式时。
  * 纯字符串格式 (`FormatString` 联合类型也包含 `"json"`，但 SDK 会拒绝将 `"json"` 作为纯字符串——请改用下方的 `json` 对象) ：
    * `"markdown"`: markdown 内容
    * `"html"`: 清理后的 HTML
    * `"rawHtml"`: 原始 HTML
    * `"links"`: 页面链接
    * `"images"`: 图片 URL
    * `"screenshot"`: screenshot 输出
    * `"summary"`: 摘要输出
    * `"changeTracking"`: 变更追踪输出 (如需使用 `modes` 等选项，请使用 `{ type: "changeTracking", modes: [...] }`——在类型定义中，该对象的 `modes` 为必填)
    * `"attributes"`: 属性提取 (传入 `selectors` 时，请使用 `{ type: "attributes", selectors: [...] }`)
    * `"branding"`: 品牌资料输出
    * `"audio"`: 音频提取
    * `"video"`: 视频提取
  * 仅支持对象的格式类型 (至少需要包含如下所示的 `type`) ：
    * `{ type: "json", prompt?: string, schema?: JSON schema or Zod schema }`: `prompt` 或 `schema` 至少需要提供一个 (SDK 验证) 。
    * `{ type: "question", question: string }`: 问答式提取。
    * `{ type: "highlights", query: string }`: 相关源文本提取。
    * `{ type: "screenshot", fullPage?, quality?, viewport? }`: 与字符串形式相同的选项，但以对象形式传递。
    * `{ type: "changeTracking", modes: ("git-diff" | "json")[], schema?, prompt?, tag? }`: `modes` 为必填。
    * `{ type: "attributes", selectors: Array<{ selector, attribute }> }`
  * 适用时的共享对象字段：
    * `schema`: 用于 `json` 的 JSON schema 或 Zod schema，或用于 `changeTracking` 的 `json` 模式 (Zod 会由 SDK 转换为 JSON Schema) 。
    * `modes`: 仅用于 `changeTracking`：`"git-diff"` 和/或 `"json"`。
    * `tag`: 可选的变更追踪分支标签。
    * `fullPage`, `quality`, `viewport`: screenshot 选项。

* `options.headers`
  * 类型：键和值均为字符串的记录
  * 使用场景：需要自定义请求头时。

* `options.includeTags`
  * 类型：字符串数组
  * 适用场景：当你只想包含特定的 HTML 标签时。

* `options.excludeTags`
  * 类型：字符串数组
  * 适用场景：当你想排除特定的 HTML 标签时。

* `options.onlyMainContent`
  * 类型：布尔值
  * 使用场景：当你想去除导航栏、页脚及其他模板化内容时。

* `options.timeout`
  * 类型：number
  * 适用场景：需要以毫秒为单位的超时时间时。

* `options.waitFor`
  * 类型：number
  * 使用场景：需要等待页面渲染时使用 (毫秒) 。

* `options.mobile`
  * 类型：布尔值
  * 适用场景：当你需要移动端视口时。

* `options.parsers`
  * 类型：解析器名称数组或解析器对象数组
  * 适用场景：当你需要控制文件解析时 (例如 PDF 解析) 。
  * 已确认的值：
    * `"pdf"`：启用 PDF 解析
    * `{ type: "pdf", mode?: "fast" | "auto" | "ocr", maxPages?: number }`：PDF 解析器选项

* `options.actions`
  * 类型：action 对象数组
  * 使用场景：需要使用轻量级预抓取 actions 时。
  * 已确认的 action 类型：
    * `wait`：必填 `milliseconds` 或 `selector`
    * `screenshot`：`fullPage`、`quality`、`viewport` 为可选
    * `click`：必填 `selector`
    * `write`：必填 `text` (请先点击输入框使其获得焦点)
    * `press`：必填 `key`
    * `scroll`：必填 `direction` (`up` 或 `down`) ，`selector` 可选
    * `scrape`：无额外字段
    * `executeJavascript`：必填 `script`
    * `pdf`：`format` (A0、A1、A2、A3、A4、A5、A6、Letter、Legal、Tabloid、Ledger) 、`landscape`、`scale` 可选

* `options.location`
  * 类型：包含 `country` 和 `languages` 的对象
  * 使用场景：当你需要按地理位置或语言进行抓取时。

* `options.skipTlsVerification`
  * 类型：布尔值
  * 使用场景：需要跳过 TLS 验证时。

* `options.removeBase64Images`
  * 类型：布尔值
  * 使用场景：当你希望从 Markdown 输出中去除 base64 图片时。

* `options.fastMode`
  * 类型：boolean
  * 适用场景：你希望以较低保真度换取更快的抓取速度。

* `options.blockAds`
  * 类型：布尔值
  * 使用场景：当你想启用广告和 Cookie 弹窗封禁时。

* `options.proxy`
  * 类型：string
  * 适用场景：需要控制代理时。
  * 可选值：
    * `"basic"`
    * `"stealth"`
    * `"enhanced"`
    * `"auto"`
    * 自定义代理 URL 字符串

* `options.maxAge`
  * 类型：number
  * 使用场景：当你想获取最长缓存时间不超过指定时长 (毫秒) 的数据时。

* `options.minAge`
  * 类型：number
  * 使用时机：当你只希望返回缓存数据，且其缓存时长至少达到该值 (毫秒) 时使用。

* `options.storeInCache`
  * 类型：布尔值
  * 适用场景：当你希望 Firecrawl 缓存结果时。

* `options.profile`
  * 类型：包含 `name` 和可选 `saveChanges` 的对象
  * 适用场景：当你希望在多次抓取和交互过程中共享同一个持久化浏览器配置文件时。

<div id="interact">
  ## 交互
</div>

<div id="why-use-it">
  ### 为什么要用它
</div>

使用 `interact` 可通过代码或自然语言控制与抓取任务关联的浏览器会话 (通过 `metadata.scrapeId`) 。SDK 要求至少提供 `code` 或 `prompt` 之一。对于不只是抓取前快速调整的流程，优先使用 `interact`，而不是抓取时的 `actions`。

<div id="preferred-sdk-method">
  ### 首选 SDK 方法
</div>

`client.interact(jobId, args)` → `Promise<ScrapeExecuteResponse>`

<div id="simple-example">
  ### 简单示例
</div>

```ts
const doc = await client.scrape("https://example.com", { formats: ["markdown"] });
const jobId = doc.metadata?.scrapeId;
if (!jobId) throw new Error("scrape 响应中缺少 scrapeId");

const result = await client.interact(jobId, {
  prompt: "点击定价选项卡并总结相关方案。"
});
```

<div id="complex-example">
  ### 复杂示例
</div>

```ts
const result = await client.interact("<scrapeJobId>", {
  code: "console.log(await page.title());",
  language: "node",
  timeout: 60
});
```

<div id="parameters">
  ### 参数
</div>

* `jobId`
  * 类型：string
  * 适用场景：你已从 `document.metadata.scrapeId` 获取抓取任务 ID。

* `args.code`
  * 类型：string
  * 适用场景：你想在浏览器会话中运行代码 (例如在 `node` 运行时中使用 Playwright 的 `page`) 。

* `args.prompt`
  * 类型：string
  * 适用场景：你希望浏览器代理按自然语言指令执行。

* `args.code` 或 `args.prompt` 中至少有一个必须非空 (否则 SDK 会抛出异常) 。

* `args.language`
  * 类型：string
  * 适用场景：你需要指定特定的运行时。
  * 可用值：`"python"`、`"node"`、`"bash"`
  * 省略时默认值：`"node"` (由 SDK 在请求体中设置) 。

* `args.timeout`
  * 类型：number
  * 适用场景：你需要设置以秒为单位的执行超时时间。

<div id="return-value-interact">
  ### 返回值 (`interact`)
</div>

`ScrapeExecuteResponse` 与 `BrowserExecuteResponse` 保持一致。已确认的字段包括：

* `success`: 布尔值
* `output`: 字符串 (存在时为汇总输出) 
* `stdout`: 字符串
* `result`: 字符串 (类型定义中与 stdout 并列的别名字段) 
* `stderr`: 字符串
* `exitCode`: 数值
* `killed`: 布尔值
* `liveViewUrl`: 字符串
* `interactiveLiveViewUrl`: 字符串
* `error`: 字符串

<div id="stop-session">
  ### 停止会话
</div>

`client.stopInteraction(jobId)` → `Promise<ScrapeBrowserDeleteResponse>`

* `jobId`：抓取任务 ID (与 `interact` 使用的 ID 相同) 。
* 类型定义中的响应字段包括 `success`、`sessionDurationMs`、`creditsBilled` 和 `error`。

<div id="ask-agentic-debugging">
  ## Ask (智能代理调试)
</div>

<div id="why-use-it">
  ### 为什么要用它
</div>

当 Firecrawl API 调用失败或返回异常结果时，可使用 `ask`。AI 支持代理会诊断问题、提出修复参数，并可选择在实时 API 上验证修复方案。典型延迟：15–30 秒。

<div id="preferred-sdk-method">
  ### 首选 SDK 方法
</div>

Node.js SDK 中暂未提供。请直接使用 `fetch`。

### 基础示例

```ts
const response = await fetch("https://api.firecrawl.dev/v2/support/ask", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.FIRECRAWL_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    question: "my scrape returned empty markdown for https://example.com",
  }),
});
const result = await response.json();
console.log(result.answer);
console.log(result.fixParameters);
```

<div id="complex-example">
  ### 复杂示例
</div>

```ts
const response = await fetch("https://api.firecrawl.dev/v2/support/ask", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.FIRECRAWL_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    question: "crawl returned 3 pages but I expected 50",
    rationale: "user is on their third failed crawl attempt today",
    context: { userPlan: "standard", retryCount: 3 },
  }),
});
const result = await response.json();
```

<div id="agent-retry-pattern">
  ### 代理重试模式
</div>

```ts
import { Firecrawl } from "firecrawl";

const client = new Firecrawl({ apiKey: process.env.FIRECRAWL_API_KEY });

const doc = await client.scrape("https://example.com/pricing", {
  formats: ["markdown"],
});

if (!doc.markdown || doc.markdown.length < 100) {
  const diagResponse = await fetch(
    "https://api.firecrawl.dev/v2/support/ask",
    {
      method: "POST",
      headers: {
        Authorization: `Bearer ${process.env.FIRECRAWL_API_KEY}`,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        question: `scrape returned only ${doc.markdown?.length ?? 0} chars for https://example.com/pricing`,
      }),
    }
  );
  const diagnosis = await diagResponse.json();

  if (diagnosis.fixParameters) {
    const retryDoc = await client.scrape("https://example.com/pricing", {
      formats: ["markdown"],
      ...diagnosis.fixParameters,
    });
  }
}
```

<div id="parameters">
  ### 参数
</div>

* `question`
  * 类型：string (必填，1-8000 个字符) 
  * 适用场景：需要描述问题时。

* `rationale`
  * 类型：string (1-2000 个字符) 
  * 适用场景：你是代表用户发起调用的 AI 代理时。

* `context`
  * 类型：object (任意格式) 
  * 适用场景：想将元数据传入调试 prompt 时。

<div id="docs-search">
  ## 文档搜索
</div>

<div id="why-use-it">
  ### 为什么要用它
</div>

使用 docs-search 查阅 Firecrawl 文档。

<div id="simple-example">
  ### 简单示例
</div>

```ts
const response = await fetch(
  "https://api.firecrawl.dev/v2/support/docs-search",

    method: "POST",
    headers: {
      Authorization: `Bearer ${process.env.FIRECRAWL_API_KEY}`,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      question: "how do I verify webhook signatures?",
    }),
  }
);
const result = await response.json();
console.log(result.answer);
```

<div id="parameters">
  ### 参数
</div>

* `question`
  * 类型：string (必填，1-8000 个字符) 
  * 使用场景：当你需要基于文档的答案时。

<div id="notes">
  ## 说明
</div>

* 已弃用的客户端别名：`scrapeExecute` → `interact`；`stopInteractiveBrowser` 和 `deleteScrapeBrowser` → `stopInteraction`。
* 默认导出的 `Firecrawl` 是 v2 客户端；v1 仍可通过 `client.v1` 使用。
* 传递给 `formats` (用于 `json` 或 `changeTracking`) 的 Zod schema 会被 SDK 转换为 JSON Schema。
* 该软件包在 `engines` 中声明 **Node.js &gt;= 22**。
