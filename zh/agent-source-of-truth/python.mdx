面向代理的 Firecrawl Python 权威来源。根据 SDK 源代码 (`firecrawl-py` / `firecrawl` **4.22.1**) 和 v2 OpenAPI 规范生成。除非另有说明，否则方法名称、参数和返回类型与 `firecrawl/v2/client.py` 中的 v2 客户端一致。

<div id="install">
  ## 安装
</div>

```bash
pip install firecrawl-py
```

<div id="authenticate">
  ## 身份验证
</div>

```py
import os
from firecrawl import Firecrawl

client = Firecrawl(api_key=os.environ.get("FIRECRAWL_API_KEY"))
# client = Firecrawl(api_key="fc-...", api_url="https://api.firecrawl.dev")
```

<div id="when-to-use-what">
  ## 何时该用哪个
</div>

* `search`：当你从查询词开始，需要先发现相关结果时使用。
* `scrape`：当你已经有 URL，并希望获取页面内容时使用。
* `interact`：当页面需要点击、填写表单或在抓取后执行浏览器 actions 时使用。

- `support/ask`：当 Firecrawl API 调用失败或返回非预期结果，且你需要诊断原因时使用。
- `support/docs-search`：当你需要查找 Firecrawl 文档时使用。

<div id="search">
  ## Search
</div>

### 使用场景

使用 search 通过查询发现相关页面，然后选择 URL 进行抓取或交互。你可以使用 `site:` 将结果限定在某个网站范围内，例如 `site:docs.firecrawl.dev crawl webhooks`。

<div id="preferred-sdk-method">
  ### 首选 SDK 方法
</div>

`client.search(query, **options)` → `SearchData` (`firecrawl.v2.types`)

<div id="return-value">
  ### 返回值
</div>

返回一个 `SearchData` 模型，其中包含可选列表：

* `web` — 网页命中 (`SearchResultWeb`，或者在 `scrape_options` 补全内容时为完整的 `Document`)
* `news` — 新闻命中 (`SearchResultNews` 或 `Document`)
* `images` — 图片命中 (`SearchResultImages` 或 `Document`)

如果 API 未返回该 key，对应的 bucket 将为 `None`。

**要避免的错误理解：** `search()` 不会返回 `{ data: [...] }`。不要访问 `result.data`。网页结果在 `result.web` 中，新闻结果在 `result.news` 中，图片结果在 `result.images` 中。

<div id="simple-example">
  ### 简单示例
</div>

```py
results = client.search("site:docs.firecrawl.dev webhook retries")
for item in results.web or []:
    print(getattr(item, "url", None), getattr(item, "title", None))
```

<div id="complex-example">
  ### 复杂示例
</div>

```py
from firecrawl.v2.types import ScrapeOptions

results = client.search(
    "site:docs.firecrawl.dev crawl webhooks",
    sources=["web", "news"],
    categories=["research"],
    limit=10,
    tbs="qdr:m",
    location="San Francisco,California,United States",
    ignore_invalid_urls=True,
    timeout=300000,
    scrape_options=ScrapeOptions(
        formats=[
            "markdown",
            "links",
            {"type": "json", "prompt": "Extract title and key endpoints."},
        ],
        only_main_content=True,
        include_tags=["main", "article"],
        exclude_tags=["nav", "footer"],
        wait_for=1000,
        actions=[
            {"type": "click", "selector": "button#accept"},
            {"type": "wait", "milliseconds": 750},
            {"type": "scrape"},
        ],
    ),
)
```

<div id="parameters">
  ### 参数
</div>

* `query`
  * 类型：str
  * 使用时机：需要提供搜索查询时。
  * 说明：使用 `site:example.com` 可将结果限制在某个域名内。

* `sources`
  * 类型：来源名称列表或 `Source` 对象
  * 使用时机：想控制搜索哪些来源时。
  * 已确认的值：
    * `"web"`：网页索引结果
    * `"news"`：新闻结果
    * `"images"`：图片结果
    * `Source(type="web" | "news" | "images")`：带类型的来源对象形式

* `categories`
  * 类型：类别名称列表或 `Category` 对象
  * 使用时机：想按类别筛选结果时。
  * 已确认的值：
    * `"github"`：偏向 GitHub 的结果
    * `"research"`：研究和学术结果
    * `"pdf"`：偏向 PDF 的结果
    * `Category(type="github" | "research" | "pdf")`：带类型的类别对象形式

* `limit`
  * 类型：int
  * 使用时机：想限制结果数量时。
  * 说明：在 SDK 模型中默认值为 `5`。

* `tbs`
  * 类型：str
  * 使用时机：需要按时间筛选时 (例如 `qdr:d`、`qdr:w`、`sbd:1,qdr:m`) 。

* `location`
  * 类型：str
  * 使用时机：想获取本地化结果时。

* `ignore_invalid_urls`
  * 类型：bool
  * 使用时机：想忽略无法被其他端点抓取的 URL 时。

* `timeout`
  * 类型：int
  * 使用时机：需要设置以毫秒为单位的请求超时时间时。
  * 说明：在 SDK 模型中默认值为 `300000`。

* `scrape_options`
  * 类型：`ScrapeOptions`
  * 使用时机：想抓取每个搜索结果时 (字段请参见 Scrape 参数) 。

<div id="scrape">
  ## 抓取
</div>

### 为什么使用

当你已经有一个 URL，并希望获取一种或多种 formats 的结构化内容时，请使用 抓取。

<div id="preferred-sdk-method">
  ### 首选 SDK 方法
</div>

`client.scrape(url, **options)` → `Document` (`firecrawl.v2.types`)

<div id="simple-example">
  ### 简单示例
</div>

```py
doc = client.scrape("https://docs.firecrawl.dev", formats=["markdown"])
```

<div id="complex-example">
  ### 复杂示例
</div>

```py
doc = client.scrape(
    "https://example.com/pricing",
    formats=[
        "markdown",
        "links",
        {"type": "json", "prompt": "Extract plan names and prices."},
        {"type": "screenshot", "full_page": True, "quality": 80, "viewport": {"width": 1280, "height": 720}},
        {"type": "changeTracking", "modes": ["git-diff"], "tag": "pricing"},
        {"type": "attributes", "selectors": [{"selector": "a", "attribute": "href"}]},
    ],
    headers={"User-Agent": "FirecrawlDocsBot/1.0"},
    only_main_content=True,
    wait_for=1000,
    parsers=[{"type": "pdf", "mode": "auto", "max_pages": 5}],
    actions=[
        {"type": "click", "selector": "#accept"},
        {"type": "wait", "milliseconds": 750},
        {"type": "scrape"},
    ],
    location={"country": "US", "languages": ["en-US"]},
    remove_base64_images=True,
    fast_mode=True,
    block_ads=True,
    proxy="auto",
    max_age=86400000,
    store_in_cache=True,
    profile={"name": "docs-session", "save_changes": True},
)
```

<div id="parameters">
  ### 参数
</div>

* `url`
  * 类型：str
  * 适用场景：当你想抓取某个特定页面时。

* `formats`
  * 类型：格式字符串或格式对象组成的列表
  * 使用场景：需要多种输出格式时。
  * 已确认的格式字符串：
    * `"markdown"`：Markdown 内容
    * `"html"`：清理后的 HTML
    * `"rawHtml"` 或 `"raw_html"`：原始 HTML
    * `"links"`：页面链接
    * `"images"`：图片 URL
    * `"screenshot"`：截图输出
    * `"summary"`：摘要输出
    * `"changeTracking"` 或 `"change_tracking"`：变更追踪输出
    * `"attributes"`：属性提取
    * `"branding"`：品牌画像输出
    * `"audio"`：音频提取
    * `"video"`：视频提取
  * 仅对象可用的格式类型：
    * `{"type": "json", ...}`：JSON 提取。请使用对象，而不是普通字符串 `"json"`。
    * `{"type": "question", "question": "..."}`：问答输出。
    * `{"type": "highlights", "query": "..."}`：相关源文本输出。
  * 格式对象字段：
    * `type`：上述格式字符串之一，或仅对象格式使用的 `"json"`、`"question"` 或 `"highlights"`
    * `question`：用于 `type: "question"`
    * `query`：用于 `type: "highlights"`
    * `prompt`：对于 `type: "json"` 为可选
    * `schema`：用于 `type: "json"` 或变更追踪 JSON 模式的 JSON schema
    * `modes`：用于 `type: "changeTracking"`，取值为 `"git-diff"` 或 `"json"` 的数组
    * `tag`：用于 `type: "changeTracking"` 的变更追踪标签
    * `full_page`, `quality`, `viewport`：用于 `type: "screenshot"` 的截图选项
    * `selectors`：用于 `type: "attributes"` 的 `{selector, attribute}` 数组

* `headers`
  * 类型：str 到 str 的 dict
  * 使用时机：需要自定义请求头时。

* `include_tags`
  * 类型：`str` 列表
  * 适用场景：当你只想包含特定的 HTML 标签时。

* `exclude_tags`
  * 类型：str 列表
  * 适用场景：当你想排除特定的 HTML 标签时。

* `only_main_content`
  * 类型：bool
  * 适用场景：当你想移除导航栏、页脚及其他样板内容时。

* `timeout`
  * 类型：int
  * 使用场景：你需要以毫秒为单位指定超时时间。

* `wait_for`
  * 类型：int
  * 使用场景：需要等待页面渲染完成时使用 (单位：毫秒) 。

* `mobile`
  * 类型：bool
  * 使用场景：当你需要移动端视口时。

* `parsers`
  * 类型：解析器名称列表或解析器对象列表
  * 适用场景：需要控制文件解析时 (例如 PDF 解析) 。
  * 已确认的取值：
    * `"pdf"`：启用 PDF 解析
    * `{ "type": "pdf", "mode": "fast" | "auto" | "ocr", "max_pages": int }`：PDF 解析器选项

* `actions`
  * 类型：action 对象列表
  * 适用场景：需要轻量级的预抓取 actions 时使用。
  * 已确认的 action 类型：
    * `wait`：必填 `milliseconds` 或 `selector`
    * `screenshot`：`full_page`、`quality`、`viewport` 为可选
    * `click`：必填 `selector`
    * `write`：必填 `text` (请先点击以聚焦输入框)
    * `press`：必填 `key`
    * `scroll`：必填 `direction` (`up` 或 `down`) ，`selector` 可选
    * `scrape`：无额外字段
    * `executeJavascript`：必填 `script`
    * `pdf`：`format` (A0、A1、A2、A3、A4、A5、A6、Letter、Legal、Tabloid、Ledger) 、`landscape`、`scale` 可选

* `location`
  * 类型：包含 `country` 和 `languages` 的 dict
  * 使用场景：当你需要根据地理位置或语言进行抓取时。

* `skip_tls_verification`
  * 类型：bool
  * 使用场景：需要跳过 TLS 验证时。

* `remove_base64_images`
  * 类型：bool
  * 使用场景：当你想从 Markdown 输出中去除 base64 图片时。

* `fast_mode`
  * 类型：bool
  * 使用场景：当你希望以降低保真度为代价来换取更快的抓取速度时。

* `block_ads`
  * 类型：bool
  * 适用场景：需要封禁广告和 Cookie 弹窗时。

* `代理`
  * 类型：str
  * 使用场景：需要控制代理时。
  * 确认可选值：`"basic"`、`"stealth"`、`"enhanced"`、`"auto"`

* `max_age`
  * 类型：int
  * 使用场景：当你希望获取缓存数据，且其最长缓存时效不超过指定时长 (毫秒) 时使用。

* `min_age`
  * 类型：int
  * 使用场景：当你只想获取缓存时间至少达到这么久 (毫秒) 的数据时。
  * 说明：仅适用于作为 `scrape_options` 传递给 `client.search` 的 `ScrapeOptions`。`client.scrape` 方法不接受 `min_age`。

* `store_in_cache`
  * 类型：bool
  * 使用场景：当你希望 Firecrawl 缓存结果时。

* `profile`
  * 类型：包含 `name` 以及可选的 `save_changes` 或 `saveChanges` 的字典
  * 使用场景：当你希望在多次抓取和交互之间共享同一个持久化浏览器配置时。

<div id="interact">
  ## 交互
</div>

### 为什么使用

当页面在抓取开始后需要执行浏览器 actions 或运行代码时，请使用 交互。

<div id="preferred-sdk-method">
  ### 首选 SDK 方法
</div>

`client.interact(job_id, code=None, *, prompt=None, language="node", timeout=None)`

`prompt` 是仅限关键字参数：请调用 `client.interact(job_id, prompt="...")` 或 `client.interact(job_id, code="...")`。`code` 和 `prompt` 至少有一个必须为非空 (在 `firecrawl/v2/methods/scrape.py` 中校验) 。

<div id="simple-example">
  ### 简单示例
</div>

```py
doc = client.scrape("https://example.com", formats=["markdown"])
job_id = doc.metadata.scrape_id if doc.metadata else None
if not job_id:
    raise RuntimeError("scrape 响应中缺少 scrape_id")

result = client.interact(job_id, prompt="Click the pricing tab and summarize the plans.")
```

<div id="complex-example">
  ### 复杂示例
</div>

```py
result = client.interact(
    "<scrapeJobId>",
    code="print(await page.title())",
    language="python",
    timeout=60,
)
```

<div id="stop-session">
  ### 停止会话
</div>

`client.stop_interaction(job_id)` 会结束与抓取任务绑定的浏览器会话。返回 `BrowserDeleteResponse` (`success`、可选的 `session_duration_ms`、`credits_billed`、`error`) 。

<div id="parameters">
  ### 参数
</div>

* `job_id`
  * 类型: str
  * 使用场景：当你有来自 `document.metadata.scrape_id` 的抓取任务 ID 时使用。

* `code`
  * 类型: str
  * 使用场景：当你想在浏览器会话中运行代码时使用 (如果已设置 `prompt`，则为可选) 。

* `prompt`
  * 类型: str
  * 使用场景：当你希望浏览器代理按自然语言指令执行时使用 (如果已设置 `code`，则为可选) 。

* `language`
  * 类型: str
  * 使用场景：当你需要指定特定运行时时使用。
  * 已确认的值：`"python"`、`"node"`、`"bash"`
  * 说明：在 SDK 中，默认值为 `"node"`。

* `timeout`
  * 类型: int
  * 使用场景：当你需要以秒为单位设置执行超时时间时使用。

<div id="return-value">
  ### 返回值
</div>

返回 `BrowserExecuteResponse`：`success`、可选的 `live_view_url`、`interactive_live_view_url`、`output`、`stdout`、`result`、`stderr`、`exit_code`、`killed`、`error` (在模型中，API 的 camelCase 会规范化为 snake&#95;case) 。

<div id="ask-agentic-debugging">
  ## Ask (代理调试)
</div>

<div id="why-use-it">
  ### 为什么要使用它
</div>

当 Firecrawl API 调用失败或返回了非预期结果时，请使用 `ask`。AI 支持代理会诊断问题、提出修复参数，并可选择通过实时 API 验证修复效果。典型延迟：15–30 秒。

<div id="preferred-sdk-method">
  ### 首选 SDK 方法
</div>

Python SDK 中暂未提供该方法。请直接使用 `requests` 或 `httpx`。

<div id="simple-example">
  ### 简单示例
</div>

```py
import requests

response = requests.post(
    "https://api.firecrawl.dev/v2/support/ask",
    headers={
        "Authorization": f"Bearer {os.environ['FIRECRAWL_API_KEY']}",
        "Content-Type": "application/json",
    },
    json={"question": "my scrape returned empty markdown for https://example.com"},
)
result = response.json()
print(result["answer"])
print(result["fixParameters"])
```

<div id="complex-example">
  ### 复杂示例
</div>

```py
import requests

response = requests.post(
    "https://api.firecrawl.dev/v2/support/ask",
    headers={
        "Authorization": f"Bearer {os.environ['FIRECRAWL_API_KEY']}",
        "Content-Type": "application/json",
    },
    json={
        "question": "crawl returned 3 pages but I expected 50",
        "rationale": "user is on their third failed crawl attempt today",
        "context": {"userPlan": "standard", "retryCount": 3},
    },
)
result = response.json()
```

<div id="agent-retry-pattern">
  ### 代理重试模式
</div>

```py
from firecrawl import Firecrawl
import requests, os

client = Firecrawl(api_key=os.environ["FIRECRAWL_API_KEY"])

doc = client.scrape("https://example.com/pricing", formats=["markdown"])

if not doc.markdown or len(doc.markdown) < 100:
    diagnosis = requests.post(
        "https://api.firecrawl.dev/v2/support/ask",
        headers={
            "Authorization": f"Bearer {os.environ['FIRECRAWL_API_KEY']}",
            "Content-Type": "application/json",
        },
        json={
            "question": f"scrape returned only {len(doc.markdown or '')} chars for https://example.com/pricing",
        },
    ).json()

    if diagnosis.get("fixParameters"):
        doc = client.scrape(
            "https://example.com/pricing",
            formats=["markdown"],
            **diagnosis["fixParameters"],
        )
```

<div id="parameters">
  ### 参数
</div>

* `question`
  * 类型：str (必填，1-8000 个字符) 
  * 适用场景：需要描述问题时。

* `rationale`
  * 类型：str (1-2000 个字符) 
  * 适用场景：作为代表用户调用的 AI 代理时。

* `context`
  * 类型：dict (自由格式) 
  * 适用场景：希望将元数据传入调试 prompt 时。

<div id="docs-search">
  ## 文档搜索
</div>

<div id="why-use-it">
  ### 为什么要使用它
</div>

使用 docs-search 查询 Firecrawl 文档。

<div id="simple-example">
  ### 简单示例
</div>

```py
import requests

FIRECRAWL_API_KEY = "fc-YOUR_API_KEY"

response = requests.post(
    "https://api.firecrawl.dev/v2/support/docs-search",
    headers={"Authorization": f"Bearer {FIRECRAWL_API_KEY}"},
    json={"question": "how do I verify webhook signatures?"},
)
print(response.json()["answer"])
```

<div id="parameters">
  ### 参数
</div>

* `question`
  * 类型：str (必填，1-8000 个字符) 
  * 适用场景：你需要一个基于文档的答案。

<div id="notes">
  ## 说明
</div>

* 已弃用的别名：`scrape_execute` → `interact`；`stop_interactive_browser` 和 `delete_scrape_browser` → `stop_interaction`。
* 顶层 `Firecrawl` 客户端直接提供 v2 方法；v1 仍位于 `client.v1` 下。
* 随附的 v2 OpenAPI 中，`POST /v2/scrape/{jobId}/interact` 的代码片段可能只记录了 `code`；但 Python SDK 和服务器对此端点都接受 `code` 或 `prompt`。
