面向代理的 Firecrawl Rust 权威来源。根据 SDK 源代码和 v2 OpenAPI 规范生成。

<div id="install">
  ## 安装
</div>

```bash
cargo add firecrawl
```

Crate：crates.io 上的 **`firecrawl`**。当前 SDK 版本为 **2.0.0** (固定版本前，请先在 crates.io 上确认最新发布版本) 。

<div id="authenticate">
  ## 认证
</div>

```rust
use firecrawl::Client;

let client = Client::new("fc-your-api-key")?;
// 自托管示例：
// let client = Client::new_selfhosted("http://localhost:3002", Some("fc-your-api-key"))?;
```

<div id="when-to-use-what">
  ## 何时该用哪一种
</div>

* `search`：当你从查询词开始、需要先发现目标时使用。
* `抓取`：当你已经有 URL，并想获取页面内容时使用。
* `交互`：当页面需要点击、填写表单，或执行抓取后的浏览器 actions 时使用。

- `support/ask`：当 Firecrawl API 调用失败或返回了不符合预期的结果，并且你需要进行诊断时使用。
- `support/docs-search`：当你需要查阅 Firecrawl 文档时使用。

<div id="search">
  ## 搜索
</div>

### 为什么使用它

使用 search 根据查询发现相关页面，然后选择要抓取或交互的 URL。你可以使用 `site:` 将结果限制在某个站点下，例如 `site:docs.firecrawl.dev crawl webhooks`。

<div id="preferred-sdk-method">
  ### 推荐使用的 SDK 方法
</div>

`client.search(query, options)` → `Result<SearchResponse, FirecrawlError>`

<div id="simple-example">
  ### 简单示例
</div>

```rust
use firecrawl::Client;

let results = client
    .search("site:docs.firecrawl.dev webhook retries", None)
    .await?;
```

<div id="complex-example">
  ### 复杂示例
</div>

```rust
use firecrawl::{
    Client, SearchOptions, SearchSource, SearchCategory, ScrapeOptions, Format, JsonOptions,
};

let options = SearchOptions {
    sources: Some(vec![SearchSource::Web, SearchSource::News]),
    categories: Some(vec![SearchCategory::Research]),
    limit: Some(10),
    tbs: Some("qdr:m".to_string()),
    location: Some("San Francisco,California,United States".to_string()),
    ignore_invalid_urls: Some(true),
    timeout: Some(60000),
    scrape_options: Some(ScrapeOptions {
        formats: Some(vec![Format::Markdown, Format::Links, Format::Json]),
        json_options: Some(JsonOptions {
            prompt: Some("Extract title and key endpoints.".to_string()),
            ..Default::default()
        }),
        only_main_content: Some(true),
        ..Default::default()
    }),
    ..Default::default()
};

let results = client
    .search("site:docs.firecrawl.dev crawl webhooks", options)
    .await?;
```

<div id="return-type">
  ### 返回类型
</div>

`client.search(...)` 返回 `Result<SearchResponse, FirecrawlError>`。

* `SearchResponse`
  * `success: bool`
  * `data: SearchData`
  * `warning: Option<String>`
* `SearchData`
  * `web: Option<Vec<SearchResultOrDocument>>` — 每一项要么是 `SearchResultOrDocument::WebResult(SearchResultWeb)` (url、title、description、…) ，要么是 `SearchResultOrDocument::Document(Document)`；当 API 为该条目返回抓取到的内容时，就会出现这种情况 (可通过是否存在 markdown、html、rawHtml 或 metadata 来判断) 。
  * `news: Option<Vec<SearchResultNews>>`
  * `images: Option<Vec<SearchResultImage>>`

<div id="parameters">
  ### 参数
</div>

* `query`
  * 类型：`impl AsRef<str>`
  * 使用场景：需要提供搜索查询时。
  * 说明：使用 `site:example.com` 可将结果限制在某个域名内。

* `options.sources`
  * 类型：`Vec<SearchSource>`
  * 使用场景：想要控制搜索哪些来源时。
  * 确认可用的值：`Web`、`News`、`Images`

* `options.categories`
  * 类型：`Vec<SearchCategory>`
  * 使用场景：想要按类别筛选结果时。
  * 确认可用的值：`Github`、`Research`、`Pdf`

* `options.limit`
  * 类型：u32
  * 使用场景：想要限制结果数量时。

* `options.tbs`
  * 类型：String
  * 使用场景：需要按时间筛选时 (例如 `qdr:d`、`qdr:w`、`sbd:1,qdr:m`) 。

* `options.location`
  * 类型：String
  * 使用场景：想要获取本地化结果时。

* `options.ignore_invalid_urls`
  * 类型：bool
  * 使用场景：想要忽略无法被其他端点抓取的 URL 时。

* `options.timeout`
  * 类型：u32
  * 使用场景：需要设置以毫秒为单位的请求超时时间时。

* `options.scrape_options`
  * 类型：`ScrapeOptions`
  * 使用场景：想要抓取每个搜索结果时 (字段请参见 抓取 参数) 。

* `options.integration`
  * 类型：`Option<String>`
  * 使用场景：需要用于服务端跟踪的 integration 标识符时。
  * 说明：在面向代理的示例中通常应省略，除非你的产品有意设置它。

<div id="scrape">
  ## 抓取
</div>

<div id="why-use-it">
  ### 为什么使用 抓取
</div>

当你已经有 URL，并希望获取一种或多种 formats 的结构化内容时，请使用 抓取。

### 推荐的 SDK 方法

`client.scrape(url, options)` → `Result<Document, FirecrawlError>` (SDK 会直接返回 HTTP 响应中的 `data` 字段) 。

<div id="simple-example">
  ### 简单示例
</div>

```rust
use firecrawl::{Client, ScrapeOptions, Format};

let doc = client
    .scrape("https://docs.firecrawl.dev", ScrapeOptions {
        formats: Some(vec![Format::Markdown]),
        ..Default::default()
    })
    .await?;
```

<div id="complex-example">
  ### 复杂示例
</div>

```rust
use firecrawl::{
    Client, ScrapeOptions, Format, JsonOptions, ScreenshotOptions, ChangeTrackingOptions,
    ChangeTrackingMode, AttributeSelector, Action, ParserConfig, ProxyType,
};

let doc = client
    .scrape("https://example.com/pricing", ScrapeOptions {
        formats: Some(vec![
            Format::Markdown,
            Format::Links,
            Format::Json,
            Format::Screenshot,
            Format::ChangeTracking,
            Format::Attributes,
        ]),
        json_options: Some(JsonOptions {
            prompt: Some("Extract plan names and prices.".to_string()),
            ..Default::default()
        }),
        screenshot_options: Some(ScreenshotOptions {
            full_page: Some(true),
            quality: Some(80),
            ..Default::default()
        }),
        change_tracking_options: Some(ChangeTrackingOptions {
            modes: Some(vec![ChangeTrackingMode::GitDiff]),
            tag: Some("pricing".to_string()),
            ..Default::default()
        }),
        attribute_selectors: Some(vec![AttributeSelector {
            selector: "a".to_string(),
            attribute: "href".to_string(),
        }]),
        parsers: Some(vec![ParserConfig::Pdf {
            parser_type: "pdf".to_string(),
            max_pages: Some(5),
        }]),
        actions: Some(vec![
            Action::Click { selector: "#accept".to_string() },
            Action::Wait { milliseconds: Some(750), selector: None },
            Action::Scrape,
        ]),
        proxy: Some(ProxyType::Auto),
        ..Default::default()
    })
    .await?;
```

<div id="parameters">
  ### 参数
</div>

* `url`
  * 类型：`impl AsRef<str>`
  * 适用场景：想抓取特定页面时。

* `options.formats`
  * 类型：`Vec<Format>`
  * 适用场景：需要多种输出格式时使用。
  * 可用值：`Markdown`, `Html`, `RawHtml`, `Links`, `Images`, `Screenshot`, `Summary`, `ChangeTracking`, `Json`, `Attributes`, `Branding`, `Audio`, `Video`

* `options.headers`
  * 类型：`HashMap<String, String>`
  * 适用场景：需要自定义请求头时。

* `options.include_tags`
  * 类型：`Vec<String>`
  * 适用场景：你只想包含特定的 HTML 标签。

* `options.exclude_tags`
  * 类型：`Vec<String>`
  * 适用场景：当你想排除特定的 HTML 标签时。

* `options.only_main_content`
  * 类型：bool
  * 适用场景：当你想移除导航栏、页脚和其他样板内容时。

* `options.timeout`
  * 类型：u32
  * 适用场景：需要以毫秒为单位的超时时间时。

* `options.wait_for`
  * 类型：u32
  * 使用时机：需要等待页面渲染完成时 (单位：毫秒) 。

* `options.mobile`
  * 类型：bool
  * 适用场景：你希望使用移动端视口。

* `options.parsers`
  * 类型：`Vec<ParserConfig>`
  * 适用场景：你需要控制文件解析方式时。
  * 已确认的取值：
    * `ParserConfig::Simple("pdf".to_string())`
    * `ParserConfig::Pdf { parser_type: "pdf", max_pages: Some(n) }`

* `options.actions`
  * 类型：`Vec<Action>`
  * 适用场景：需要使用轻量级的预抓取 actions。
  * 已确认的 action 类型：
    * `Wait`：必填 `milliseconds` 或 `selector`
    * `Screenshot`：`full_page`、`quality`、`viewport` 为可选
    * `Click`：必填 `selector`
    * `Write`：必填 `text` (请先点击以使输入框获得焦点)
    * `Press`：必填 `key`
    * `Scroll`：必填 `direction` (`Up` 或 `Down`) ，`selector` 可选
    * `Scrape`：无需其他字段
    * `ExecuteJavascript`：必填 `script`
    * `Pdf`：`格式` (A0、A1、A2、A3、A4、A5、A6、Letter、Legal、Tabloid、Ledger) 、`landscape`、`scale` 可选

* `options.location`
  * 类型：`LocationConfig`
  * 适用场景：需要进行基于地理位置或语言的抓取时。
  * 已确认字段：`country`、`languages`

* `options.skip_tls_verification`
  * 类型：bool
  * 使用场景：需要跳过 TLS 验证时。

* `options.remove_base64_images`
  * 类型：bool
  * 适用场景：当你想从 Markdown 输出中移除 base64 图片时。

* `options.fast_mode`
  * 类型：bool
  * 使用场景：当你希望以降低保真度为代价来换取更快的抓取速度时。

* `options.block_ads`
  * 类型：bool
  * 适用场景：需要封禁广告和 Cookie 弹窗时。

* `options.proxy`
  * 类型：`ProxyType`
  * 适用场景：需要控制代理时。
  * 已确认的取值：`Basic`、`Stealth`、`Enhanced`、`Auto`

* `options.max_age`
  * 类型：u32
  * 使用场景：当你希望使用缓存数据，且其最长缓存时间不超过指定时长 (毫秒) 时。

* `options.min_age`
  * 类型：u32
  * 使用时机：当你只希望获取至少已缓存这么久 (毫秒) 的数据时。

* `options.store_in_cache`
  * 类型：bool
  * 使用场景：当你希望 Firecrawl 缓存结果时。

* `options.profile`
  * 类型：`ProfileConfig`
  * 使用场景：你希望在多次抓取和交互过程中共享同一个持久化的浏览器配置。
  * 已确认字段：`name`、`save_changes`

* `options.integration`
  * 类型：`Option<String>`
  * 适用场景：需要用于服务端追踪的集成标识符时。
  * 说明：在面向代理的示例中，除非你的产品有意设置此项，否则请省略。

* `options.json_options`
  * 类型：`JsonOptions`
  * 适用场景：当你需要配置 JSON 提取时。
  * 已确认字段：`schema`、`system_prompt`、`prompt`

* `options.screenshot_options`
  * 类型：`ScreenshotOptions`
  * 使用场景：用于配置截图输出。
  * 已确认字段：`full_page`、`quality`、`viewport`

* `options.change_tracking_options`
  * 类型：`ChangeTrackingOptions`
  * 适用场景：需要配置变更追踪输出时。
  * 已确认字段：`modes` (`GitDiff` 或 `Json`) 、`schema`、`prompt`、`tag`

* `options.attribute_selectors`
  * 类型：`Vec<AttributeSelector>`
  * 使用场景：需要属性提取输出时。
  * 已确认字段：`selector`、`attribute`

<div id="interact">
  ## 交互
</div>

### 为什么使用它

当页面需要在抓取开始后执行浏览器 actions 或运行代码时，请使用交互。

### 推荐的 SDK 方法

`client.interact(job_id, options)`

如需结束浏览器会话，请使用 `client.stop_interaction(job_id)` (对 `/v2/scrape/{jobId}/interact` 发起 HTTP `DELETE` 请求) 。

<div id="simple-example">
  ### 简单示例
</div>

```rust
use firecrawl::{Client, ScrapeExecuteOptions};

let result = client
    .interact(
        "<scrapeJobId>",
        ScrapeExecuteOptions {
            prompt: Some("Click the pricing tab and summarize the plans.".to_string()),
            ..Default::default()
        },
    )
    .await?;
```

<div id="complex-example">
  ### 复杂示例
</div>

```rust
use firecrawl::{Client, ScrapeExecuteOptions, ScrapeExecuteLanguage};

let result = client
    .interact(
        "<scrapeJobId>",
        ScrapeExecuteOptions {
            code: Some("console.log(await page.title());".to_string()),
            language: Some(ScrapeExecuteLanguage::Node),
            timeout: Some(60),
            ..Default::default()
        },
    )
    .await?;
```

<div id="stop-session">
  ### 结束会话
</div>

```rust
let stopped = client.stop_interaction("<scrapeJobId>").await?;
// stopped: ScrapeBrowserDeleteResponse { success, session_duration_ms, credits_billed, error, ... }
```

<div id="parameters">
  ### 参数
</div>

* `job_id`
  * 类型：`impl AsRef<str>`
  * 适用场景：你已有抓取任务 ID。

* `options.code`
  * 类型：`Option<String>`
  * 适用场景：你想在浏览器会话中运行代码。

* `options.prompt`
  * 类型：`Option<String>`
  * 适用场景：你希望浏览器代理按照自然语言指令执行。

* `options.language`
  * 类型：`ScrapeExecuteLanguage`
  * 适用场景：你需要指定特定的运行时。
  * 可选值：`Python`、`Node`、`Bash`
  * 说明：如果省略，SDK 默认使用 `Node`。

* `options.timeout`
  * 类型：u32
  * 适用场景：你需要设置以秒为单位的执行超时时间。

* `options.origin`
  * 类型：`Option<String>`
  * 适用场景：你需要为执行遥测设置一个可选的来源标签。
  * 说明：在面向代理的示例中，除非你的产品有意设置它，否则请省略。

`options.code` 或 `options.prompt` 至少必须有一个为非空；否则，SDK 会在调用 API 之前返回 `FirecrawlError::Misuse`。

<div id="response-types">
  ### 响应类型
</div>

* `interact` → `Result<ScrapeExecuteResponse, FirecrawlError>`
  * `success: bool` 始终存在；`live_view_url`、`interactive_live_view_url`、`output`、`stdout`、`result`、`stderr`、`exit_code`、`killed` 和 `error` 在该结构体中均为 `Option` 字段。
* `stop_interaction` → `Result<ScrapeBrowserDeleteResponse, FirecrawlError>`
  * `success: bool` 始终存在；`session_duration_ms`、`credits_billed` 和 `error` 为可选字段。

v2 OpenAPI 规范当前将交互请求体建模为 `code` 必填；Rust SDK 则通过接受 **`code`** 或 **`prompt`** 之一来匹配服务器行为 (请参见 `scrape.rs` 中的 `ScrapeExecuteOptions`) 。

<div id="notes">
  ## 说明
</div>

* 已弃用别名：`scrape_execute`、`stop_interactive_browser` 和 `delete_scrape_browser` 对应 `interact` 和 `stop_interaction`。
* `ScrapeOptions` 包含专用的 `json_options`、`screenshot_options` 和 `change_tracking_options`，适用于高级 formats。
* `search_and_scrape(query, limit)` 是一个便捷辅助函数：它会使用默认 `ScrapeOptions` 调用 `search`，并返回根据 `data.web` 中 `SearchResultOrDocument::Document` 条目构建的 `Vec<Document>` (请参见 `search.rs`) 。
* v2 SDK 在 crate 根导出所有类型：`use firecrawl::Client` (而不是 `use firecrawl::v2::Client`) 。
* v2 中的错误类型已简化：`CrawlJobFailed(String, CrawlStatus)` → `JobFailed(String)`，`Missuse` → `Misuse`。
