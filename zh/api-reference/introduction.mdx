<div id="base-url">
  ## 基础 URL
</div>

所有请求使用以下基础 URL：

```bash
https://api.firecrawl.dev 
```

<div id="authentication">
  ## 身份验证
</div>

进行身份验证时，需要在请求中添加 Authorization 标头。其值应为 `Bearer fc-123456789`，其中 `fc-123456789` 是你的 API Key。

```bash
Authorization：Bearer fc-123456789
```

​

<div id="response-codes">
  ## 响应代码
</div>

Firecrawl 使用常见的 HTTP 状态码来表示请求结果。

通常，2xx HTTP 状态码表示成功，4xx 表示用户相关错误，5xx 表示基础设施问题。

| Status | Description          |
| ------ | -------------------- |
| 200    | 请求成功。                |
| 400    | 请检查参数是否正确。           |
| 401    | 未提供 API 密钥。          |
| 402    | 需要付款。                |
| 404    | 未找到所请求的资源。           |
| 408    | 请求超时 (例如，页面加载时间过长) 。 |
| 429    | 超出速率限制。              |
| 5xx    | 表示 Firecrawl 服务器错误。  |

有关所有可能的 API 错误的详细说明，请参见“错误代码”部分。

<div id="firecrawl-error-codes">
  ### Firecrawl 错误代码
</div>

当出现 408 或 5xx 错误时，Firecrawl 会提供更具体的错误代码来说明问题。​

| Error Code                           | Status | Description                                                                                                       |
| ------------------------------------ | ------ | ----------------------------------------------------------------------------------------------------------------- |
| `SCRAPE_TIMEOUT`                     | 408    | 页面加载或渲染时间过长。请尝试增大 `timeout` 参数 (最高可达 60000ms) 。                                                                   |
| `SCRAPE_ALL_ENGINES_FAILED`          | 500    | 所有抓取引擎均失败。                                                                                                        |
| `SCRAPE_SSL_ERROR`                   | 500    | 页面 SSL 证书无效。你可以使用 `skipTlsVerification:true` 来跳过此检查。                                                              |
| `SCRAPE_SITE_ERROR`                  | 500    | 不可恢复的站点错误。                                                                                                        |
| `SCRAPE_DNS_RESOLUTION_ERROR`        | 500    | DNS 解析失败。                                                                                                         |
| `SCRAPE_ACTION_ERROR`                | 500    | 执行页面 actions 时出错。                                                                                                 |
| `SCRAPE_PDF_PREFETCH_FAILED`         | 500    | 预取 PDF 失败。                                                                                                        |
| `SCRAPE_PDF_INSUFFICIENT_TIME_ERROR` | 500    | 处理 PDF 的时间不足。                                                                                                     |
| `SCRAPE_PDF_ANTIBOT_ERROR`           | 500    | PDF 被反机器人机制拦截。                                                                                                    |
| `SCRAPE_ZDR_VIOLATION_ERROR`         | 500    | 零数据保留冲突：当 `zeroDataRetention:true`，但其他选项 (如 `screenshot`) 需要临时存储时发生。                                              |
| `SCRAPE_UNSUPPORTED_FILE_ERROR`      | 500    | 不支持的文件类型，或文件大小超过 10MB 限制。                                                                                         |
| `SCRAPE_LOCKDOWN_CACHE_MISS`         | 404    | 已启用 Lockdown 模式，但该 URL 没有可用的缓存数据。请先使用非 lockdown 抓取为缓存预置数据，或禁用 `lockdown`。请参见 [Lockdown Mode](/zh/features/lockdown)。 |
| `UNKNOWN_ERROR`                      | 500    | 通用或意外错误。                                                                                                          |

<div id="429-responses">
  ## 429 响应
</div>

当超出您套餐的限流或并发限制时，API 会返回 429 响应代码。请参见[限流](/zh/rate-limits)了解各套餐的限制。
