通过 MCP，将 Firecrawl 的搜索、抓取、爬取和浏览器工具添加到 [Google Antigravity](https://antigravity.google/)。

<div id="quick-setup">
  ## 快速开始
</div>

<div id="1-get-your-api-key">
  ### 1. 获取您的 API 密钥
</div>

前往 [firecrawl.dev/app](https://www.firecrawl.dev/app) 注册，并复制您的 API 密钥。

<div id="2-add-firecrawl-to-antigravity">
  ### 2. 将 Firecrawl 添加到 Antigravity
</div>

打开 Antigravity 的设置 (`Cmd/Ctrl + ,`) ，搜索 **MCP Servers**，然后按以下内容添加一个新服务器：

```json
{
  "mcpServers": {
    "firecrawl": {
      "command": "npx",
      "args": ["-y", "firecrawl-mcp"],
      "env": {
        "FIRECRAWL_API_KEY": "fc-YOUR-API-KEY"
      }
    }
  }
}
```

将 `fc-YOUR-API-KEY` 替换为你的 Firecrawl API 密钥。

<div id="3-reload-antigravity">
  ### 3. 重新加载 Antigravity
</div>

重新加载窗口 (`Cmd/Ctrl + Shift + P` → `Reload Window`) 。现在，代理已可使用 Firecrawl 的网页工具。

<div id="quick-demo">
  ## 快速演示
</div>

在 Antigravity 代理的聊天中：

```
Search the web for "Vercel AI SDK v5 release notes" and summarize.
```

```
Scrape https://docs.firecrawl.dev/ai-onboarding and list every linked guide.
```

```
Crawl https://example.com and extract every page title.
```

Antigravity 会自动通过 Firecrawl MCP 转发这些工具调用。

<div id="remote-hosted-url-no-nodejs-required">
  ## Remote Hosted URL (无需 Node.js)
</div>

```json
{
  "mcpServers": {
    "firecrawl": {
      "url": "https://mcp.firecrawl.dev/fc-YOUR-API-KEY/v2/mcp"
    }
  }
}
```

<div id="troubleshooting">
  ## 故障排查
</div>

* **服务器显示为“failed”** — 检查 MCP 输出面板中的 stderr。大多数失败原因是缺少 API 密钥，或 `npx` 不在 `PATH` 环境变量中。
* **工具未被调用** — 在你的第一个 prompt 中明确提到 Firecrawl (例如“使用 Firecrawl 进行抓取……”) ，这样代理就会选择正确的工具。
