使用 Firecrawl MCP 为 Cursor 接入网页抓取与搜索功能。

<div id="quick-setup">
  ## 快速安装
</div>

<div id="1-get-your-api-key">
  ### 1. 获取 API 密钥
</div>

在 [firecrawl.dev/app](https://firecrawl.dev/app) 注册并复制你的 API 密钥。

<div id="2-add-to-cursor">
  ### 2. 添加到 Cursor
</div>

打开设置（`Cmd+,`），搜索 “MCP”，并添加：

```json
{
  "mcpServers": {
    "firecrawl": {
      "command": "npx",
      "args": ["-y", "firecrawl-mcp"],
      "env": {
        "FIRECRAWL_API_KEY": "your_api_key_here"
      }
    }
  }
}
```

将 `your_api_key_here` 替换为你的实际 Firecrawl API 密钥。

<div id="3-restart-cursor">
  ### 3. 重启 Cursor
</div>

完成！现在你可以在 Cursor 中搜索并爬取网页了。

<div id="windows-troubleshooting">
  ## Windows 故障排查
</div>

如果你在 Windows 上看到 `spawn npx ENOENT` 或 “No server info found” 错误，说明 Cursor 在你的 PATH 中找不到 `npx`。请尝试以下任一解决方案：

**选项 A：使用 `npx.cmd` 的完整路径**

在命令提示符中运行 `where npx` 获取完整路径，然后更新你的配置：

```json
{
  "mcpServers": {
    "firecrawl": {
      "command": "C:\\Program Files\\nodejs\\npx.cmd",
      "args": ["-y", "firecrawl-mcp"],
      "env": {
        "FIRECRAWL_API_KEY": "your_api_key_here"
      }
    }
  }
}
```

将 `command` 路径替换为 `where npx` 命令的输出。

**选项 B：使用远程托管的 URL（无需 Node.js）**

```json
{
  "mcpServers": {
    "firecrawl": {
      "url": "https://mcp.firecrawl.dev/YOUR-API-KEY/v2/mcp"
    }
  }
}
```

将 `YOUR-API-KEY` 替换为你的 Firecrawl API 密钥。
