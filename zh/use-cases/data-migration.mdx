从任何网站提取内容、结构和元数据，并将其转换后导入新平台。Firecrawl 负责抓取工作，让你可以专注于将数据映射到目标系统。

  ## 从模板开始
</div>

<Card title="Firecrawl Migrator" icon="github" href="https://github.com/firecrawl/firecrawl-migrator">
  高效在不同平台与系统之间迁移数据
</Card>

<div id="how-it-works">
  ## 工作原理
</div>

让 Firecrawl 对你的源网站进行爬取，获取每个页面并返回整洁、结构化的数据。然后，你可以转换输出，使其符合目标平台的 schema，并通过该平台的 API 或批量导入工具将其导入。

<div id="what-you-can-migrate">
  ## 可迁移内容
</div>

* **内容**：页面、博文、文章、媒体文件、元数据
* **结构**：层级结构、分类、标签、分类体系
* **用户**：公开可访问的用户资料及相关数据
* **设置**：配置、自定义字段、工作流程
* **电商**：商品、目录、库存、订单

<div id="common-migration-use-cases">
  ## 常见迁移用例
</div>

用户使用 Firecrawl 构建迁移工具，从各种平台提取数据：

<CardGroup cols={2}>
  <Card title="CMS 内容提取">
    从 WordPress、Drupal 和 Joomla 网站或自定义 CMS 平台提取内容。保留内容结构和元数据，然后导出并导入到 Contentful、Strapi 或 Sanity 等新系统中。
  </Card>

  <Card title="电商数据提取">
    从 Magento 和 WooCommerce 商店提取产品目录，包括库存、价格、描述和规格参数。将数据整理后导入 Shopify、BigCommerce 或其他平台。
  </Card>
</CardGroup>

<div id="faqs">
  ## 常见问题
</div>

<AccordionGroup>
  <Accordion title="How do you handle large-scale migrations?">
    我们的基础设施会自动扩容以应对大规模迁移。我们支持带有批处理和并行提取的增量处理，让您将数百万个页面拆分为可管理的批次，并配合进度跟踪完成迁移。
  </Accordion>

  <Accordion title="Can I preserve SEO value during migration?">
    可以！提取所有 SEO 元数据 (包括 URL、标题、描述) ，并实施正确的重定向。我们会在迁移过程中帮助维持您的搜索排名。
  </Accordion>

  <Accordion title="What about media files and attachments?">
    Firecrawl 可以提取并编目所有媒体文件。您可以下载后重新上传到新平台，或者在保留相同 CDN 的情况下直接引用。
  </Accordion>

  <Accordion title="How do I validate the migration?">
    我们提供详细的提取报告并支持对比工具。您可以核验内容完整性、检查失效链接，并校验数据一致性。
  </Accordion>

  <Accordion title="Can I migrate user-generated content and comments?">
    可以，您可以提取公开可见的用户生成内容，包括评论、评价和论坛帖子。私有用户数据需要相应的身份验证与权限。
  </Accordion>
</AccordionGroup>
